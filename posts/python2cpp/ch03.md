# 常用容器（下）：`tuple`、`set`、`optional`、`variant`

接上篇，我们把 ch01 容器对照表里剩下的几个补上：元组、集合、可选值、联合体。这些容器在 Python 里用起来很自然，但在 C++ 里各自有一些需要习惯的地方。

---

## 快速对照表

| Python | C++ | 关键区别 |
|--------|-----|---------|
| `tuple` | `std::tuple` / `std::pair` | 编译期定死大小和类型，可以存引用 |
| `set` | `std::unordered_set` / `std::set` | 哈希表 vs 红黑树，排序行为不同 |
| `None` | `std::optional` | 显式表达"可能有值，也可能没有" |
| `Union` / `Optional Type` | `std::variant` | 类型安全的联合体，不能随便访问 |

---

## `tuple`：元组的对应

Python 的 `tuple` 很灵活，长度和类型都可以不同：

```python
t = (1, "hello", 3.14)
print(t[0])    # 1
print(t[1])    # "hello"
```

C++ 的 `std::tuple` 也类似，但类型和大小在编译期就定死了：

```cpp
#include <tuple>
#include <iostream>

std::tuple<int, std::string, double> t = {1, "hello", 3.14};

// 取值方式一：std::get<索引>
std::cout << std::get<0>(t);  // 1
std::cout << std::get<1>(t);  // "hello"

// 取值方式二：std::get<类型>（如果类型不重复）
std::cout << std::get<int>(t);  // 1
```

### `std::pair`：两元素特化

如果元组只有两个元素，用 `std::pair` 更简洁：

```cpp
#include <utility>

std::pair<int, std::string> p = {1, "hello"};
std::cout << p.first;   // 1
std::cout << p.second;  // "hello"
```

### 结构化绑定（C++17）

C++17 开始可以像 Python 一样拆包：

```cpp
auto [x, y, z] = t;  // 类似 Python 的 x, y, z = t
std::cout << x << y << z;

// pair 也可以
auto [key, value] = p;
```

对比 Python：

```python
x, y, z = (1, "hello", 3.14)     # Python 拆包
x, y = (1, "hello")               # Python 拆 pair
```

## `set`：集合

Python 的 `set` 默认就是哈希集合，C++ 对应的是 `std::unordered_set`。

```python
s = {3, 1, 2}
s.add(4)
print(1 in s)       # True
print(len(s))       # 4
```

```cpp
#include <unordered_set>

std::unordered_set<int> s = {3, 1, 2};
s.insert(4);
std::cout << s.count(1);      // 1（类似 Python 的 1 in s）
std::cout << s.size();        // 4
```

### 顺序问题

| Python `set` | C++ `std::unordered_set` | C++ `std::set` |
|-------------|--------------------------|----------------|
| 无顺序保证（3.7 前也是） | 无顺序，取决于哈希 | 按键升序排列 |

```cpp
std::unordered_set<int> us = {3, 1, 2, 4};
for (int x : us) {
    std::cout << x << " ";   // 顺序不确定，可能 4 2 1 3
}

std::set<int> os = {3, 1, 2, 4};
for (int x : os) {
    std::cout << x << " ";   // 1 2 3 4，永远有序
}
```

如果你不在乎顺序，用 `std::unordered_set`（更快）；如果需要**按大小排序**或**范围查询**（比如找大于 2 的元素），用 `std::set`。

### 集合运算

Python 的 `&`、`|`、`-` 在 C++ 里有对应的函数：

```python
a = {1, 2, 3}
b = {3, 4, 5}

print(a & b)   # {3}              交集
print(a | b)   # {1, 2, 3, 4, 5}  并集
print(a - b)   # {1, 2}           差集
```

```cpp
std::unordered_set<int> a = {1, 2, 3};
std::unordered_set<int> b = {3, 4, 5};

// 交集
std::unordered_set<int> inter;
for (int x : a) {
    if (b.count(x)) inter.insert(x);
}

// 并集
std::unordered_set<int> uni = a;
uni.insert(b.begin(), b.end());

// 差集
std::unordered_set<int> diff;
for (int x : a) {
    if (!b.count(x)) diff.insert(x);
}
```

> 没有 Python 那样的运算符糖，都要手动循环。如果集合运算很频繁，可以用 `std::set`（有序）配合 `<algorithm>` 里的 `std::set_intersection` 等函数。

## `None` / `Optional`：可空值的类型安全写法

Python 里任何变量都可以是 `None`，C++ 用 `std::optional<T>` 显式表示"可能有值，也可能没有"。

```python
def find(d, key):
    return d.get(key)  # 不存在返回 None

v = find({"a": 1}, "b")
if v is not None:
    print(v)
```

```cpp
#include <optional>

{% raw %}
std::optional<int> find(const std::map<std::string, int>& m, const std::string& key) {
    auto it = m.find(key);
    if (it != m.end()) return it->second;
    return std::nullopt;  // 相当于返回 None
}

int main() {
    std::map<std::string, int> m = {{"a", 1}};
{% endraw %}
    auto v = find(m, "b");

    if (v) {                     // 检查有没有值
        std::cout << *v;         // 解引用获取值
    } else {
        std::cout << "not found";
    }
}
```

比返回 `-1` 或 `nullptr` 表示"没有"要安全得多——不会混淆"真值"和"空标记"。

### 常用操作

```cpp
std::optional<int> v = 42;       // 有值
std::optional<int> e = std::nullopt;  // 空

v.value()      // 获取值（没有则抛 std::bad_optional_access）
v.value_or(0)  // 有值返回值，没有返回 0
v.has_value()  // 等同于 if (v)
```

## `Union` / `Optional Type`：类型安全的联合体

Python 的 `Union[int, str]` 允许一个变量在运行时是 int 或 str。C++ 的对应是 `std::variant`。

```python
from typing import Union

v: Union[int, str] = 42
v = "hello"      # 可以随时切换类型
```

```cpp
#include <variant>
#include <iostream>

std::variant<int, std::string, double> v = 42;     // 当前是 int
v = "hello";                                         // 切换成 string

// 访问方式：std::visit + 函数对象
std::visit([](auto&& val) {
    std::cout << val << "\n";
}, v);

// 或者按类型检查
if (std::holds_alternative<int>(v)) {
    std::cout << std::get<int>(v);
} else if (std::holds_alternative<std::string>(v)) {
    std::cout << std::get<std::string>(v);
}
```

### 和 `Union` 的区别

Python 的 `Union` 完全是动态的，一个变量可以塞任何类型。C++ 的 `std::variant` 是**编译期枚举了所有可能类型**，你只能在预设的类型集合里切换：

```cpp
std::variant<int, std::string> v;
v = 42;               // OK
v = "hello";          // OK
// v = 3.14;          // 编译错误！double 不在可选类型里
```

这样做的好处是**类型安全**：你永远知道 variant 里可能是什么类型，不需要运行时猜。

---

## 总结：这四种容器适合什么场景

| 容器 | 替代 Python 的 | 一句话记 |
|------|---------------|----------|
| `std::tuple` / `pair` | `tuple` | 编译期定死的异构集合 |
| `std::unordered_set` / `set` | `set` | 哈希快但无序，红黑树慢但有顺序 |
| `std::optional` | `None` 返回值 | 显式表示"可能有值"，比魔法数安全 |
| `std::variant` | `Union` 类型 | 编译期枚举所有可能，类型安全 |
