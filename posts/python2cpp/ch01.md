# 类型系统

## 灵活的动态类型

python 一个变量可以在不同的类型的内容中间随意切换
```python
x = 42        # x 现在指向一个 int 对象
x = "hello"   # 同一个 x，现在指向 str 对象
```
c++ 一个变量的类型是明确而不可改变的
这是为什么呢？这是静态类型系统和动态类型系统的区别。
我们可以假设x是一个车牌。
在py的世界观里，这个车牌很自由，他可以是挂在"油车"上的，也可以是挂在"新能源汽车"上的。python语法并不会限制他。（当然这种操作在很多python的语言规范中都被标记为不好的行为了）。
但是在c++的世界观里，他就好像有一个严格的车辆类型系统，编译器就像交警一样，严格仔细的限制，"油车"的车牌就只能挂在"油车"上，其他各种类型的车也同理。



## 值语义 vs 引用语义

python 默认都是引用，所以要小心修改同一个数据
```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)  # [1, 2, 3, 4] —— a 也变了！
```
就像两个人共用同一把伞，一个人把伞弄坏了，另一个人也没得用了。

c++ 默认是拷贝，各玩各的
```cpp
std::vector<int> a = {1, 2, 3};
std::vector<int> b = a;  // 拷贝！a 和 b 是两个独立的 vector
b.push_back(4);
// a 仍然是 {1, 2, 3}
```
就像复印了一份文件，你改你的复印件，不影响我的原件。

如果你想共用，得显式说：
```cpp
std::vector<int>& b = a;  // 引用，像 python 的变量绑定，共用一把伞
std::vector<int>* b = &a; // 指针，我记了你家的地址，按地址找过去
```


## 指针与引用

python 里变量本质上都是指向对象的指针，只不过包装得很好，你看不见。
c++ 把这件事拆开给你看了，让你自己选：

| 方式 | 生活比喻 | 特点 |
|------|---------|------|
| `T obj` | 买了一份原件 | 自己的，随便改不影响别人 |
| `T* ptr` | 记了对方门牌号 | 可能空（查无此人），也可以换个人指 |
| `T& ref` | 起了个别名/外号 | 必须绑定一个人，一辈子不能换，也不能空 |

```cpp
void modify(int x) { x = 10; }      // 传值：改的是复印件，外面不变
void modify(int& x) { x = 10; }     // 传引用：改原件，外面跟着变
void modify(int* x) { *x = 10; }   // 传指针：按地址找过去改

int a = 5;
modify(a);       // 走传引用版本，a 变成 10
modify(&a);      // 走传指针版本，得把地址给他
```

简单记：想让函数改外面的变量，加 `&`；不想被改，加 `const`。


## const：只读契约

python 没有真正的常量，你想改总能改（虽然可以约定俗成地用全大写表示"别改"）。
c++ 的 `const` 是编译器级别的"合同"，签了就不能改，编译器会盯着你的。

```cpp
const int max_size = 100;           // 这数定了，别想改
const std::vector<int> v = ...;     // 只读容器，不能 push_back

void print(const std::string& s);   // 我承诺只看不动，编译器作证
```

记忆口诀：`const` 在 `*` 左边，管的是"指向的内容"（房子不能装修）；在 `*` 右边，管的是"地址本身"（不能换房子租）。


## 类型推导：编译器帮你猜

c++ 也能偷懒，让编译器猜类型：
```cpp
auto i = 42;           // 猜是 int
auto& r = i;           // 猜是 int&，保留引用得显式加 &
auto v = {1, 2, 3};    // std::initializer_list<int>
```

但要注意，`auto` 会"脱掉"引用和顶层 const。如果你希望完全保留原类型，用 `decltype(auto)`（C++14）：
```cpp
int x = 42;
int& rx = x;
auto a = rx;              // int（引用被剥掉了）
decltype(auto) b = rx;    // int&（完全保留原类型）
```

模板就像编译期的鸭子类型：
```cpp
template<typename T>
T add(T a, T b) { return a + b; }

add(1, 2);        // OK，推导为 int
add(1.0, 2.0);    // OK，推导为 double
// add("hello", "world");  // 编译错误：字符串没法 +
```
和 python 的泛型很像，只不过错在编译时就报，不用等到运行时才翻车。


## 容器：从 list/dict 到 vector/map

| Python | C++ | 区别 |
|--------|-----|------|
| `list` | `std::vector` | 数组底子，内存连续，O(1) 随机访问 |
| `dict` | `std::unordered_map` | 哈希表，键需要能算 hash |
| `tuple` | `std::tuple` / `std::pair` | 编译期定死大小，可存引用 |
| `set` | `std::unordered_set` | 同上 |
| `None` | `std::nullopt` / `std::optional` | 显式表达"可能没有" |
| `Union` | `std::variant` | 类型安全的联合体，访问需 `std::visit` |

`optional` 让"可能没有"这件事有了类型保障：
```cpp
std::optional<int> maybe = std::nullopt;  // 类似 Python 的 None
maybe = 42;

if (maybe) {              // 显式检查存在性
    std::cout << *maybe;  // 解引用获取值
}
```
比返回 `-1` 或 `nullptr` 表示失败要安全得多。


## 类与对象

`struct` 默认公开，像 python 的 dataclass，简简单单存数据；`class` 默认私有，封装得好，外人不能随便碰。

最不一样的是 **RAII**（资源获取即初始化）——c++ 的对象出生就干活，死了自动收拾烂摊子：
```cpp
class FileHandle {
    FILE* fp;
public:
    FileHandle(const char* path) { fp = fopen(path, "r"); }
    ~FileHandle() { if (fp) fclose(fp); }  // 析构时自动释放
};

{
    FileHandle f("data.txt");  // 构造，打开文件
}  // 出了这个作用域，自动调用析构函数，文件关闭
```
python 的 `with` 语句就是从这个思想借鉴来的，但 c++ 是语言级别的保证，不需要显式写 `with`。


## 内存模型

python 有 GC（垃圾回收）兜底，内存管理基本不用你操心。
c++ 你得自己负责，但现代 c++ 给了智能指针这个"安全带"：

```cpp
// 独占所有权，离开作用域自动释放，不能复制，只能转移
std::unique_ptr<int> p = std::make_unique<int>(42);

// 共享所有权，引用计数归零时释放
std::shared_ptr<int> p2 = std::make_shared<int>(42);
```

除非写底层库，否则别轻易用 `new`/`delete`，那是老黄历了。智能指针相当于把 python 的自动内存管理"显式化"了，你可以控制什么时候释放。


## 一句话总结

c++ 的类型系统核心就三条：
1. **默认拷贝**：变量赋值是复制，除非你用 `&` 或 `*`
2. **编译期契约**：类型错在编译时暴露，不是生产环境翻车
3. **生命周期即资源**：对象出生和死亡的时刻，就是你管理资源的时刻
