# 异常处理：try-catch 的定位信息去哪了？

写 Python 时，`try-catch` 捕获异常后随手一个 `traceback.print_exc()` 就能看到完整的调用堆栈，精准定位问题。但在写 C++ 的 `try-catch` 时，你可能会发现：**`catch` 块里怎么什么都没有？** 除了 `e.what()` 返回的一行字符串，函数名、行号、调用链，一概欠奉。

这不是 C++ 的"缺陷"，而是两种语言在设计哲学上的根本分歧。

---

## 核心哲学对比

| 维度 | **Python** | **C++** |
| :--- | :--- | :--- |
| **设计哲学** | **内省优先**：虚拟机天然维护帧栈，便于调试。 | **零开销原则**：不为"可能不发生"的事付出性能代价，堆栈记录需手动开启。 |
| **异常对象** | 默认携带 `__traceback__`，自带完整堆栈链。 | `std::exception` 极其轻量，**完全不携带**任何堆栈信息。 |
| **获取方式** | 标准库 `traceback` 模块直接打印。 | 标准库无此功能，必须借助外部手段（调试器、平台 API、第三方库或编码埋点）。 |
| **性能影响** | 即使不抛出，虚拟机也维护帧栈，有固定开销。 | 不抛出时零开销；抛出时依赖 `setjmp/longjmp` 或零成本（`SjLj`/`Itanium` ABI），但不记录堆栈。 |

### 直观对比

```python
# Python：自带导航地图
def inner():
    raise ValueError("出错了")

def outer():
    inner()

try:
    outer()
except ValueError as e:
    import traceback
    traceback.print_exc()
    # 输出：
    # Traceback (most recent call last):
    #   File "demo.py", line 10, in <module>
    #     outer()
    #   File "demo.py", line 8, in outer
    #     inner()
    #   File "demo.py", line 5, in inner
    #     raise ValueError("出错了")
    # ValueError: 出错了
```

```cpp
#include <iostream>
#include <stdexcept>

void inner() {
    throw std::runtime_error("出错了");
}

void outer() {
    inner();
}

int main() {
    try {
        outer();
    } catch (const std::exception& e) {
        std::cout << e.what() << "\n";
        // 只输出：出错了
        // 谁调了谁？完全不知道
    }
}
```

现象很清晰：Python 的异常自带"导航地图"，捕获即得；C++ 的异常是一张"白纸"，你得自己画地图。

---

## 入门方案：给异常加上坐标

既然 C++ 标准库不提供堆栈信息，最简单的出路就是：**在抛出的时候就把位置信息刻在异常对象里**。

利用 C++ 的 `__FILE__`、`__LINE__` 预定义宏作为默认参数：

```cpp
#include <iostream>
#include <stdexcept>
#include <string>

// 自定义异常：构造时自动记录文件名和行号
class traced_exception : public std::runtime_error {
public:
    traced_exception(
        const std::string& msg,
        const char* file = __FILE__,
        int line = __LINE__
    ) : std::runtime_error(msg + " [" + file + ":" + std::to_string(line) + "]") {}
};

void inner() {
    throw traced_exception("出错了");  // 自动记录 inner() 所在位置
}

void outer() {
    inner();
}

int main() {
    try {
        outer();
    } catch (const std::exception& e) {
        std::cout << e.what() << "\n";
        // 输出：出错了 [demo.cpp:17]  ← 直接看到抛出点行号
    }
}
```

这基本够用了：`catch` 里调用 `e.what()` 就能看到抛出点的文件名和行号，不需要任何额外工具。
