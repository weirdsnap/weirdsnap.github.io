# 预备章：5 分钟，让你的第一个 C++ 程序跑起来

## 场景

你刚拿到一台新电脑（或者新分配的虚拟机）。Leader 说："先把你本地环境搭起来，跑通一个 Hello World，我们再开始看代码。"

你打开终端，陷入了沉默。Python 环境里你就装一个解释器，配个 VS Code 就能写。C++ 呢？编译器？CMake？头文件路径？怎么好像要装一堆东西？

## 对话

**A**：B 哥，搭 C++ 环境要装什么？Python 我就 `apt install python3` 完事，C++ 好像很复杂？

**B**：就三样东西：**编译器**（把代码翻译成机器能跑的指令）、**CMake**（告诉编译器怎么组织文件）、**一个编辑器**（你继续用 VS Code 就行）。我给你一个"一键仪式"，你先跑通，后面再懂每一步在干什么。

**A**：不要先跟我讲原理，直接告诉我复制粘贴什么。

**B**：好。先看你是什么系统。

### Linux（Ubuntu/Debian/WSL2）

```bash
# 1. 编译器 + 基础工具
sudo apt update
sudo apt install build-essential cmake

# 验证
g++ --version    # 应该显示 9.0 或更高
cmake --version  # 应该显示 3.10 或更高
```

### macOS

```bash
# 1. 编译器（Xcode 命令行工具）
xcode-select --install

# 2. CMake
brew install cmake

# 验证
clang++ --version
cmake --version
```

### Windows

**B**：Windows 原生 C++ 环境比较麻烦。建议直接装 **WSL2**（Windows 内置的 Linux 子系统），然后按上面的 Linux 步骤走。如果你必须用 Windows 原生，装 **Visual Studio Community**（勾选"使用 C++ 的桌面开发"），但后续章节的命令行指令都是 Linux 风格的。

**A**：我的是 Ubuntu。装完了，然后呢？

**B**：创建一个文件夹，里面放两个文件。不要问为什么，先复制。

**文件 1：`CMakeLists.txt`**

```cmake
cmake_minimum_required(VERSION 3.10)
project(hello)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_executable(hello main.cpp)
```

**文件 2：`main.cpp`**

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}
```

**A**：`CMakeLists.txt` 是干什么用的？看起来比 `requirements.txt` 复杂多了。

**B**：现在不要深究。把它当作 C++ 项目的"启动咒语"。只要你的项目里有 `.cpp` 文件，就需要一个 `CMakeLists.txt` 告诉系统："这里有代码，请帮我编译。"

**A**：行，我建好了。怎么跑？

**B**：在文件夹里执行这三步：

```bash
# 第一步：配置（告诉 CMake 怎么生成编译指令）
cmake -B build -S .

# 第二步：编译（真正生成可执行文件）
cmake --build build

# 第三步：运行
./build/hello
```

你应该看到屏幕上输出：

```
Hello, World!
```

**A**：确实跑出来了。但这比 Python 麻烦啊，Python 就 `python main.py` 一步。

**B**：对，C++ 是"先翻译，再运行"。记住这个**三步仪式**：

| 步骤 | 命令 | 类比 |
|------|------|------|
| 配置 | `cmake -B build -S .` | 告诉系统"我要编译这个项目" |
| 编译 | `cmake --build build` | 真正把代码翻译成机器码 |
| 运行 | `./build/hello` | 执行生成的二进制文件 |

**A**：那个 `build` 文件夹是什么？

**B**：编译产物（二进制文件、中间文件）的临时目录。相当于 Python 的 `__pycache__`，但更大更乱。**不要把它提交到 Git**。

**A**：我修改了 `main.cpp` 里的字符串，想再看效果，要重新执行哪几步？

**B**：只需要后两步：

```bash
cmake --build build
./build/hello
```

如果改了 `CMakeLists.txt`（比如加了新文件），才需要重新执行第一步。

## ⚠️ 负向标记（环境搭建版）

1. **不要现在去搜"CMake 完整教程"**  
   网上的 CMake 教程动辄几十页，会把你淹死。先记住上面的 5 行 `CMakeLists.txt`，它能撑到第 6 章（项目实战）之前。

2. **如果 `cmake -B build -S .` 报错**  
   先检查是不是在当前目录（有 `CMakeLists.txt` 的那个目录）执行的。如果报错说 "No CMakeLists.txt found"，就是你进错文件夹了。

3. **`build` 目录不要进 Git**  
   在 `.gitignore` 里加一行 `build/`。这是编译产物，就像 Python 的 `__pycache__`。

4. **如果编译报错说 `iostream` 找不到**  
   说明你编译器没装好（`build-essential` 没装全），或者 CMake 没找对编译器。先执行 `g++ --version` 确认。

5. **不要手动创建 `build` 目录里的文件**  
   `build` 目录里的所有东西都是自动生成的，你不需要也不应该手动修改。

## 最小项目模板（保存备用）

后续每一章的代码练习，都可以套用这个结构：

```
my_project/
├── CMakeLists.txt      # 启动咒语（复制粘贴，几乎不用改）
├── .gitignore          # 里面写 build/
└── main.cpp            # 你的代码
```

**`.gitignore` 内容**：
```
build/
```

**`CMakeLists.txt` 内容**（撑到第 6 章之前）：
```cmake
cmake_minimum_required(VERSION 3.10)
project(my_project)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_executable(my_app main.cpp)
```
