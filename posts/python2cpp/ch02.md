# 常用容器：`string`、`vector`、`map`

Python 和 C++ 里面我们都会用到一些很常用的数据结构来作为存储的容器。
从简单的字符串存储一组字符，到数组，列表，字典等复杂的结构。
那么两者之间，肯定有一些差异，我们就来看一下，相同的目标，应该用什么手段在 cpp 中类似地实现呢？两者的底层差异又会引入哪些不一样的细节需要注意。

---

## 快速对照表

| 目标 | Python | C++ |
|--------|--------|-----|
| 拼接两个字符串 | `s1 + s2` | `s1 + s2`（至少一边是 `std::string`） |
| 字符串追加内容 | `s += "abc"` | `s += "abc"` 或 `s.append("abc")` |
| 字符串加一个字 | `s += 'x'` | `s.push_back('x')` |
| 取子串 | `s[0:5]` | `s.substr(0, 5)` |
| 列表/数组尾部加一个 | `lst.append(x)` | `v.push_back(x)` |
| 列表/数组尾部加多个 | `lst.extend([x, y])` | `v.insert(v.end(), {x, y})` |
| 弹出最后一个 | `lst.pop()` | `v.pop_back()` |
| 插入到中间 | `lst.insert(i, x)` | `v.insert(v.begin() + i, x)` |
| 字典/映射设值 | `d[k] = v` | `m[k] = v` |
| 判断键是否存在 | `k in d` | `m.count(k)` 或 `m.find(k) != end` |



## `str` 的操作，在 C++ 里怎么写

### 拼接

```python
s = "hello"
s = s + " world"
print(s)        # "hello world"
```

```cpp
std::string s = "hello";
s = s + " world";
std::cout << s; // "hello world"
```

注意：C++ 里 `"hello" + " world"` 会编译报错，因为两边都是 `const char*`，相加不是字符串拼接。至少一边要是 `std::string`：

```cpp
std::string s = std::string("hello") + " world";
// 或
std::string s = "hello";
s += " world";
```

### 追加

```python
s = "hello"
s += " world"
```

```cpp
std::string s = "hello";
s += " world";              // 和 Python 最像
s.append(" world");         // 显式调用 append
s.push_back('!');           // 只加一个字符
```

### 取子串

```python
s = "hello world"
print(s[0:5])      # "hello"   从 0 开始，到 5 之前结束
print(s[6:11])     # "world"
```

```cpp
std::string s = "hello world";
std::cout << s.substr(0, 5);   // "hello"   从 0 开始，取 5 个字符
std::cout << s.substr(6, 5);   // "world"   从 6 开始，取 5 个字符
std::cout << s.substr(6);      // "world"   从 6 开始，取到末尾
```

**关键区别**：Python 切片是 `[start, end)`，C++ `substr` 是 `(start, length)`。

### 查找与长度

```python
s = "hello world"
print(len(s))              # 11
print(s.find("world"))     # 6
print("world" in s)        # True
```

```cpp
std::string s = "hello world";
std::cout << s.size();                 // 11（也可以用 s.length()）
std::cout << s.find("world");          // 6
std::cout << (s.find("world") != std::string::npos);  // true
```

### 修改某个字符

```python
s = "hello"
# s[0] = "H"  不行，str 不可变，只能新建
s = "H" + s[1:]
print(s)       # "Hello"
```

```cpp
std::string s = "hello";
s[0] = 'H';    // 可以直接改
std::cout << s; // "Hello"
```

## `list` 的操作，在 C++ 里怎么写

### 常用操作

```python
v = [1, 2, 3]
v.append(4)             # [1, 2, 3, 4]
v.extend([5, 6])        # [1, 2, 3, 4, 5, 6]
v.insert(0, 0)          # [0, 1, 2, 3, 4, 5, 6]
v.pop()                 # [0, 1, 2, 3, 4, 5]
print(len(v))           # 6
```

```cpp
std::vector<int> v = {1, 2, 3};
v.push_back(4);                         // {1, 2, 3, 4}
v.insert(v.end(), {5, 6});              // {1, 2, 3, 4, 5, 6}
v.insert(v.begin(), 0);                 // {0, 1, 2, 3, 4, 5, 6}
v.pop_back();                           // {0, 1, 2, 3, 4, 5}
std::cout << v.size();                  // 6
```

### 索引访问：一个容易炸的地方

```python
v = [1, 2, 3]
print(v[100])    # IndexError
```

```cpp
std::vector<int> v = {1, 2, 3};
std::cout << v[100];   // 不一定会报错，但访问了非法内存，可能崩溃
```

C++ 的 `[]` 不检查边界。想要安全，用 `at()`：

```cpp
std::cout << v.at(100);   // 抛出 std::out_of_range
```

### 初始化

```python
zeros = [0] * 10
```

```cpp
std::vector<int> zeros(10, 0);
```

### 切片

```python
sub = lst[1:4]
```

C++ 没有 `[]` 切片，需要用迭代器区间：

```cpp
std::vector<int> sub(v.begin() + 1, v.begin() + 4);
```

### 列表推导式

```python
squares = [x * x for x in lst if x % 2 == 0]
```

C++ 没有等价语法，用循环最直观：

```cpp
std::vector<int> squares;
for (int x : v) {
    if (x % 2 == 0) squares.push_back(x * x);
}
```

## `dict` 的操作，在 C++ 里怎么写

Python 的 `dict` 是哈希表。C++ 里对应的是 `std::unordered_map`（哈希表）和 `std::map`（红黑树，会自动排序）。

如果只是作为 Python `dict` 的平替，用 `std::unordered_map`。

### 增删查改

```python
d = {}
d["a"] = 1
print(d["a"])           # 1
print("b" in d)         # False
print(d.get("b", 0))    # 0
```

```cpp
std::unordered_map<std::string, int> m;
m["a"] = 1;
std::cout << m["a"];          // 1
std::cout << m.count("b");    // 0

// 访问不存在的键会默认插入一个值，通常不是你想干的
// std::cout << m["b"];       // 输出 0，但 map 里多了 {"b": 0}

// 安全写法
auto it = m.find("b");
if (it != m.end()) {
    std::cout << it->second;
}
```

### 遍历

```python
for k, v in d.items():
    print(k, v)
```

```cpp
for (const auto& [k, v] : m) {   // C++17
    std::cout << k << " " << v << "\n";
}
```

### 顺序问题

```python
d = {"b": 2, "a": 1, "c": 3}
print(list(d.keys()))   # Python 3.7+ 保持插入顺序：['b', 'a', 'c']
```

```cpp
std::unordered_map<std::string, int> um = {{"b", 2}, {"a", 1}, {"c", 3}};
for (const auto& [k, v] : um) {
    std::cout << k << " ";   // 顺序不确定
}

std::map<std::string, int> tm = {{"b", 2}, {"a", 1}, {"c", 3}};
for (const auto& [k, v] : tm) {
    std::cout << k << " ";   // 按键排序：a b c
}
```

## 底层差异（了解即可）

前面只说了“怎么写”。如果你好奇背后有什么区别，记住这几点：

1. **Python `str` 不可变，C++ `std::string` 可变**
   - Python 的 `s += "x"` 会悄悄创建一个新字符串。
   - C++ 的 `s += "x"` 通常是在原字符串后面追加，可能复用已有内存。

2. **C++ `[]` 访问不检查边界**
   - Python 越界会抛异常；C++ 越界是未定义行为，可能崩溃、也可能恰好读出垃圾。
   - 要安全用 `at()`，要速度用 `[]`。

3. **C++ `m[key]` 会自动插入默认值**
   - Python `d[key]` 不存在会抛 `KeyError`。
   - C++ `m[key]` 不存在会插入一个默认值（比如 int 是 0），然后返回它。

4. **C++ 容器通常要提前声明元素类型**
   - Python：`lst = []`，里面随便装什么。
   - C++：`std::vector<int> v`，只能装 `int`。
