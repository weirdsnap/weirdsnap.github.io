# 常用容器（上）：队列、栈、堆、数组、字符串

Python 程序员遇到一个新场景时，想的是"我需要什么数据结构"。C++ 的标准库正好提供了**按数据结构命名的容器适配器**，比 Python 的通用 `list` 更明确。

---

## 队列（先进先出）

**抽象理解**：排队打饭，先排的先打到。尾部添加、头部取出。

| 操作 | Python | C++ |
|------|--------|-----|
| 创建 | `q = collections.deque()` | `std::queue<int> q;` |
| 入队 | `q.append(x)` | `q.push(x)` |
| 出队 | `q.popleft()` | `q.pop()`（不返回元素） |
| 看队头 | `q[0]` | `q.front()` |
| 看队尾 | `q[-1]` | `q.back()` |
| 是否空 | `len(q) == 0` | `q.empty()` |
| 大小 | `len(q)` | `q.size()` |

```python
from collections import deque
q = deque()
q.append("a")      # 入队
q.append("b")
x = q.popleft()    # 出队，x = "a"
print(q[0])        # 看队头："b"
```

```cpp
#include <queue>

std::queue<std::string> q;
q.push("a");           // 入队
q.push("b");
std::string x = q.front();  // 取出（看队头）
q.pop();                     // 出队（删除）
// 现在 x = "a"，队头是 "b"
```

**踩坑点**：

- C++ `q.pop()` **不返回被弹出的元素**。必须先 `front()` 取值，再 `pop()` 删除。这是 C++ 容器适配器的一贯风格——`pop` 只负责销毁，不管存储。
- Python `list` 做队列用 `pop(0)` 是 O(n) 的，应该用 `collections.deque`。C++ `std::queue` 底层默认也是 deque，所以头尾操作都是 O(1)。
- 队列的底层容器可以换：`std::queue<int, std::list<int>>` 会改用 list 做底层，但一般不需要动。

## 栈（后进先出）

**抽象理解**：往桶里叠盘子，后放进去的先拿出来。

| 操作 | Python | C++ |
|------|--------|-----|
| 创建 | 用 `list` 模拟 | `std::stack<int> s;` |
| 压栈 | `s.append(x)` | `s.push(x)` |
| 弹栈 | `s.pop()` | `s.pop()`（不返回元素） |
| 看栈顶 | `s[-1]` | `s.top()` |
| 是否空 | `len(s) == 0` | `s.empty()` |

```python
s = []
s.append(1)      # 压栈
s.append(2)
x = s.pop()      # 弹栈，x = 2
print(s[-1])     # 看栈顶：1
```

```cpp
#include <stack>

std::stack<int> s;
s.push(1);           // 压栈
s.push(2);
int x = s.top();     // 取栈顶
s.pop();             // 弹栈
// 现在 x = 2，栈顶是 1
```

**踩坑点**：

- 和 `queue` 一样，`pop()` **不返回值**，必须先 `top()` 再 `pop()`。
- Python 没有专门的栈类型，用 `list` 模拟（`append` + `pop`）已经 O(1)。C++ `std::stack` 默认底层也是 deque，同样 O(1)。
- 如果你只是在尾部操作，Python `list` 和 `std::vector` 也够了；但用 `std::stack` 语义更清晰，别人一看就知道你在用栈。

## 堆 / 优先队列

**抽象理解**：每次都能取出最大（或最小）的元素，像医院叫号——病情最重的先处理。

| 操作 | Python | C++ |
|------|--------|-----|
| 导入 | `import heapq` | `#include <queue>` |
| 创建 | `h = []` | `std::priority_queue<int> pq;` |
| 插入 | `heapq.heappush(h, x)` | `pq.push(x)` |
| 取出最小 | `heapq.heappop(h)` | `pq.top()` + `pq.pop()` |
| 看最小 | `h[0]` | `pq.top()` |
| 大小 | `len(h)` | `pq.size()` |

```python
import heapq
h = []
heapq.heappush(h, 3)
heapq.heappush(h, 1)
heapq.heappush(h, 2)
x = heapq.heappop(h)   # x = 1（最小）
```

{% raw %}
```cpp
#include <queue>

std::priority_queue<int> pq;
pq.push(3);
pq.push(1);
pq.push(2);
int x = pq.top();   // x = 3（最大）
pq.pop();
```
{% endraw %}

**默认行为差异**：

- Python `heapq` 默认是**最小堆**，`heappop` 返回最小的元素
- C++ `std::priority_queue` 默认是**最大堆**，`top()` 返回最大的元素

要改成最小堆：

```cpp
// 方式一：用 greater
std::priority_queue<int, std::vector<int>, std::greater<int>> min_pq;

// 方式二：存负数（取的时候再转回来）
std::priority_queue<int> pq;
pq.push(-3);   // 相当于插入了 3，但取出来的是 -3，转正即可
```

**踩坑点**：

- C++ `pop()` 仍不返回值，先 `top()` 再 `pop()`。
- 默认最大堆，和 Python 习惯相反，容易踩坑。
- C++ `priority_queue` 不支持遍历内部元素，也不支持"删除任意元素"——如果需要，通常用 `multiset` 或手动维护。

### 自定义优先级

```python
# Python：存 (优先级, 值) 元组
heapq.heappush(h, (3, "低"))
heapq.heappush(h, (1, "高"))
```

```cpp
// C++：可以自定义比较函数
struct Task {
    int priority;
    std::string name;
};

auto cmp = [](const Task& a, const Task& b) {
    return a.priority > b.priority;  // 小顶堆
};
std::priority_queue<Task, std::vector<Task>, decltype(cmp)> pq(cmp);
```

## 动态数组（日常"列表"）

**抽象理解**：有序的列表，通过索引访问，能动态增长。

| 操作 | Python | C++ |
|------|--------|-----|
| 尾部添加一个 | `lst.append(x)` | `v.push_back(x)` |
| 尾部添加多个 | `lst.extend([x, y])` | `v.insert(v.end(), {x, y})` |
| 弹出最后一个 | `lst.pop()` | `v.pop_back()` |
| 插入到中间 | `lst.insert(i, x)` | `v.insert(v.begin() + i, x)` |
| 删除某个 | `lst.remove(x)` | `v.erase(iterator)` |
| 按索引访问 | `lst[i]` | `v[i]` |
| 安全访问 | 自动检查 | `v.at(i)`，越界抛异常 |
| 大小 | `len(lst)` | `v.size()` |
| 排序 | `lst.sort()` / `sorted(lst)` | `std::sort(v.begin(), v.end())` |
| 反转 | `lst.reverse()` / `lst[::-1]` | `std::reverse(v.begin(), v.end())` |
| 找元素 | `x in lst` | `std::find(v.begin(), v.end(), x) != end` |

```python
v = [1, 2, 3]
v.append(4)                    # [1, 2, 3, 4]
v.insert(0, 0)                 # [0, 1, 2, 3, 4]
x = v.pop()                    # x = 4，v = [0, 1, 2, 3]
v.sort()                       # 原地升序
rev = v[::-1]                  # 新反转列表
```

```cpp
std::vector<int> v = {1, 2, 3};
v.push_back(4);                           // {1, 2, 3, 4}
v.insert(v.begin(), 0);                   // {0, 1, 2, 3, 4}
int x = v.back();                         // x = 4
v.pop_back();                             // {0, 1, 2, 3}
std::sort(v.begin(), v.end());            // 升序
std::vector<int> rev(v.rbegin(), v.rend()); // 新反转 vector
```

**踩坑点**：

- `v[i]` 越界不会报错，是未定义行为（可能崩溃、可能读出垃圾）。用 `v.at(i)` 安全。
- `v.insert(v.begin(), x)` 是 O(n) 的，频繁在头部插入请用 `std::deque`。
- C++ 没有列表推导式，需要用循环显式生成。
- 排序、查找、反转都需要 `<algorithm>` 头文件和迭代器，不像 Python 那样直接是容器的方法。

## 字符串

| 操作 | Python | C++ |
|------|--------|-----|
| 拼接 | `s1 + s2` | `s1 + s2`（至少一边是 `std::string`） |
| 追加 | `s += "x"` | `s += "x"` 或 `s.append("x")` |
| 取子串 | `s[0:5]` | `s.substr(0, 5)`（位置，长度） |
| 长度 | `len(s)` | `s.size()` 或 `s.length()` |
| 查找 | `"world" in s` | `s.find("world") != std::string::npos` |
| 修改字符 | ❌ 不可变 | ✅ `s[i] = 'H'` 直接改 |
| 遍历 | `for c in s:` | `for (char c : s)` |

**踩坑点**：

- Python `"hello" + "world"` 可以，C++ 报错，因为两边都是 `const char*`。至少一边是 `std::string`。
- C++ `substr` 参数是 `(起始位置, 长度)`，不是 Python 的 `[start:end]`。
- Python `str` 不可变，C++ `std::string` 可变，可以直接改某个字符。

## 快速对照：按场景选容器

| 你需要 | Python | C++（头文件） | 特点 |
|--------|--------|-------------|------|
| 队列 | `collections.deque` | `std::queue` (`<queue>`) | 先进先出，`pop` 不返回值 |
| 栈 | `list` 模拟 | `std::stack` (`<stack>`) | 后进先出，`pop` 不返回值 |
| 优先队列 | `heapq` | `std::priority_queue` (`<queue>`) | 默认最大堆 vs Python 最小堆 |
| 动态数组 | `list` | `std::vector` (`<vector>`) | 通用，索引用 `at()` 更安全 |
| 字符串 | `str` | `std::string` (`<string>`) | C++ 可变，Python 不可变 |
