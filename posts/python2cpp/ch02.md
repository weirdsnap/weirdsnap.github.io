# 常用容器：`string`、`vector`、`map`

Python 和 C++ 都有用来装字符串、列表、键值对的工具，名字和用法看起来很像，但细节上一不留神就会踩坑。这一章把三兄弟拉出来逐个对比。

---

## `str` vs `std::string`

### 最大的区别：可变还是不可变

```python
s = "hello"
s[0] = "H"        # ❌ TypeError: 'str' object does not support item assignment
```

```cpp
std::string s = "hello";
s[0] = 'H';       // ✅ 直接改，s 变成 "Hello"
```

Python 的 `str` 是**不可变**的，想改只能生成新的字符串；C++ 的 `std::string` 是**可变**的，可以原地修改。

### 拼接

```python
s = "hello" + " " + "world"   # 生成新字符串
s += "!"                      # 也是生成新字符串（因为 str 不可变）
```

```cpp
std::string s = "hello" + std::string(" ") + "world";
s += "!";                     // 原地追加，通常更高效
```

注意：C++ 里 `"hello" + " "` 不能直接写，因为两边都是 `const char*`，相加是指针运算，会编译报错。至少一边要转成 `std::string`。

### 追加元素

| 操作 | Python | C++ |
|------|--------|-----|
| 尾部加一个字符 | 无（str 不可变） | `s.push_back('x')` |
| 尾部加一段 | `s += "abc"` | `s.append("abc")` / `s += "abc"` |
| 插入中间 | 无直接方法 | `s.insert(pos, "abc")` |

### 切片 / 子串

这是最容易混淆的地方：

```python
s = "hello world"
print(s[0:5])      # "hello"   左闭右开 [0, 5)
print(s[6:])       # "world"
print(s[-5:])      # "world"
```

```cpp
std::string s = "hello world";
std::cout << s.substr(0, 5);   // "hello"   参数是 (起始位置, 长度)
std::cout << s.substr(6);      // "world"   只有起始位置，取到末尾
```

Python 切片是 `[start, end)`，C++ `substr` 是 `(start, length)`。一个是“到哪里”，一个是“取多长”。

### 查找与长度

```python
s = "hello world"
len(s)                    # 11
s.find("world")           # 6，找不到返回 -1
"world" in s              # True
```

```cpp
std::string s = "hello world";
s.size();                 // 11，也可以用 s.length()
s.find("world");          // 6，找不到返回 std::string::npos
s.find("world") != std::string::npos;  // true
```

### 小结

| 特性 | Python `str` | C++ `std::string` |
|------|--------------|-------------------|
| 可变 | ❌ 不可变 | ✅ 可变 |
| 拼接 | `+` / `+=` | `+`（注意类型）/ `+=` / `append` |
| 切片 | `s[a:b]` | `s.substr(pos, len)` |
| 长度 | `len(s)` | `s.size()` / `s.length()` |
| 查找 | `s.find()` / `in` | `s.find()`，无结果返回 `npos` |

## `list` vs `std::vector`

Python 的 `list` 和 C++ 的 `std::vector` 都是**动态数组**，但 C++ 对边界更严格。

### 增删查改

```python
lst = [1, 2, 3]
lst.append(4)        # 尾部加一个
lst.extend([5, 6])   # 尾部加一堆
lst.pop()            # 尾部弹出
lst.insert(0, 0)     # 中间插入
len(lst)             # 长度
```

```cpp
std::vector<int> v = {1, 2, 3};
v.push_back(4);                    // 尾部加一个
v.insert(v.end(), {5, 6});         // 尾部加一堆（C++11）
v.pop_back();                      // 尾部弹出
v.insert(v.begin(), 0);            // 中间插入
v.size();                          // 长度
```

### 索引访问：一个很凶的区别

```python
lst = [1, 2, 3]
print(lst[100])    # ❌ IndexError: list index out of range
```

```cpp
std::vector<int> v = {1, 2, 3};
std::cout << v[100];   // ⚠️ 不会报错，但访问越界内存，未定义行为（可能崩溃、可能读出垃圾）
```

C++ 的 `operator[]` 不检查边界。想要安全访问，用 `at()`：

```cpp
std::cout << v.at(100);   // ✅ 抛出 std::out_of_range 异常
```

### 初始化

```python
zeros = [0] * 10
```

```cpp
std::vector<int> zeros(10, 0);   // 10 个 0
```

### 切片

```python
sub = lst[1:4]
```

C++ 没有内置切片，需要手动拷贝：

```cpp
std::vector<int> sub(v.begin() + 1, v.begin() + 4);
```

### 列表推导式

```python
squares = [x * x for x in lst if x % 2 == 0]
```

C++ 没有直接的列表推导式，通常用循环或 `std::transform` + `std::copy_if`：

```cpp
std::vector<int> squares;
for (int x : v) {
    if (x % 2 == 0) squares.push_back(x * x);
}
```

### 小结

| 特性 | Python `list` | C++ `std::vector` |
|------|---------------|-------------------|
| 尾部加元素 | `append(x)` | `push_back(x)` |
| 尾部加多个 | `extend([...])` | `insert(end, {...})` |
| 弹出尾部 | `pop()` | `pop_back()` |
| 越界访问 | 抛 `IndexError` | 未定义行为；安全版用 `at()` |
| 切片 | `lst[a:b]` | 用迭代器区间构造 |
| 推导式 | `[x for x in lst]` | 无直接语法，用循环或算法 |

## `dict` vs `std::map` / `std::unordered_map`

Python 的 `dict` 本质上是**哈希表**。C++ 有两个对应物：

- `std::map`：基于红黑树，按键**排序**，查找 O(log n)
- `std::unordered_map`：基于哈希表，**不排序**，平均查找 O(1)

如果只是做 Python `dict` 的平替，一般选 `std::unordered_map`。

### 增删查改

```python
d = {}
d["a"] = 1
print(d["a"])       # 1
print("b" in d)     # False
print(d.get("b", 0)) # 0，不存在的键给默认值
```

```cpp
std::unordered_map<std::string, int> m;
m["a"] = 1;
std::cout << m["a"];     // 1
std::cout << m.count("b"); // 0，不存在

// 访问不存在的键会默认构造一个值
std::cout << m["b"];     // 输出 0，但 map 里会多出一个 {"b": 0}

// 安全访问：
auto it = m.find("b");
if (it != m.end()) std::cout << it->second;
```

### `[]` 访问的陷阱

Python 的 `d[key]` 不存在时会抛 `KeyError`；C++ 的 `m[key]` 不存在时会**自动插入一个默认值**。这是初学时最容易踩的坑之一。

### 遍历

```python
for k, v in d.items():
    print(k, v)
```

```cpp
for (const auto& [k, v] : m) {   // C++17 结构化绑定
    std::cout << k << " " << v << "\n";
}
```

### 顺序问题

```python
d = {"b": 2, "a": 1, "c": 3}
print(list(d.keys()))   # Python 3.7+：['b', 'a', 'c']，保持插入顺序
```

```cpp
std::unordered_map<std::string, int> um = {{"b", 2}, {"a", 1}, {"c", 3}};
for (const auto& [k, v] : um) {
    std::cout << k << " ";   // 输出顺序不确定
}

std::map<std::string, int> tm = {{"b", 2}, {"a", 1}, {"c", 3}};
for (const auto& [k, v] : tm) {
    std::cout << k << " ";   // 按 key 排序：a b c
}
```

### 小结

| 特性 | Python `dict` | C++ `std::unordered_map` | C++ `std::map` |
|------|---------------|--------------------------|----------------|
| 底层 | 哈希表 | 哈希表 | 红黑树 |
| 查找 | O(1) 平均 | O(1) 平均 | O(log n) |
| 顺序 | 插入顺序（3.7+） | 无序 | 按键排序 |
| 访问不存在键 | 抛 `KeyError` | 自动插入默认值 | 自动插入默认值 |
| 安全查找 | `d.get(k, default)` | `m.find(k)` / `m.count(k)` | `m.find(k)` / `m.count(k)` |
| 遍历 | `d.items()` | 范围 for + 结构化绑定 | 范围 for + 结构化绑定 |

## 一句话记忆

- **Python 容器像便利贴**：写法灵活、运行时会帮你检查、错了抛异常。
- **C++ 容器像工具箱**：快、可控，但越界和未初始化都可能炸，需要你自己显式说“我要安全的”或“我知道自己在做什么”。
