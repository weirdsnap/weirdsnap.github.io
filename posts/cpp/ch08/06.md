# 异常安全三等级

C++ 异常安全（Exception Safety）是衡量代码在抛出异常时行为正确性的标准。Herb Sutter 提出了三个层级：

| 等级 | 保证 | 通俗解释 |
|------|------|---------|
| **基本保证（Basic）** | 不泄漏资源，对象处于有效但不确定状态 | 程序还能继续运行 |
| **强保证（Strong）** | 操作要么成功，要么回滚到操作前的状态 | 原子提交或全部回滚 |
| **不抛保证（Noexcept）** | 保证不抛出异常 | 绝不出错 |

---

## 一、不抛保证（Noexcept）

函数保证永不抛异常，用 `noexcept` 声明：

```cpp
int add(int a, int b) noexcept {
    return a + b;            // 不可能抛异常
}

void safe_swap(T& a, T& b) noexcept {
    // swap 是 noexcept 的核心要求之一
}
```

**这是最高等级**。如果一个函数不满足 `noexcept`，编译器也不会报错——但标准库（如 `std::vector` 的移动操作）会基于它做决策。

---

## 二、强保证（Strong）

操作要么完全成功，要么回到调用前的状态，没有副作用。

```cpp
class Stack {
    std::vector<int> data_;
    size_t size_ = 0;
public:
    void push(int val) {
        data_.push_back(val);   // 如果这里抛异常（内存不足），size_ 不会增加
        ++size_;                 // 只有 push_back 成功后才会修改 size_
    }
};
```

**copy-and-swap 是实现强保证的经典手法**：

```cpp
void push(const T& val) {
    auto tmp = data_;           // 在副本上操作
    tmp.push_back(val);
    data_.swap(tmp);            // 交换是 noexcept，至此原数据才被修改
}
```

---

## 三、基本保证（Basic）

不泄漏资源，但对象状态可能会被部分修改：

```cpp
void append(const std::string& s) {
    size_t old_len = str_.size();
    str_.append(s);
    if (str_.size() < old_len + s.size()) {
        // 发生截断，抛出异常，但 str_ 仍有有效内容
        throw std::overflow_error("truncated");
    }
}
```

**这是大多数代码的最低要求**。连基本保证都没有的函数在异常面前是不安全的。

---

## 四、如何编写异常安全代码

### 第一条：RAII

RAII 自动满足基本保证中的"不泄漏资源"：

```cpp
void bad() {
    int* p = new int[100];
    // 如果这里抛异常，p 泄漏！
    delete[] p;
}

void good() {
    auto p = std::make_unique<int[]>(100);
    // RAII 自动释放，异常安全
}
```

### 第二条：swap 必须是 noexcept

几乎所有的强保证实现都依赖 `swap` 不抛异常。C++ 标准库要求 `std::swap` 是 `noexcept` 的。

### 第三条：遵循"在副本上操作，成功再交换"

```cpp
// ✅ 强保证
T operator=(T other) noexcept {  // copy-and-swap
    swap(other);
    return *this;
}
```

## 五、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `noexcept` 只是优化提示 | ✅ 它也是接口契约，标准库会据此决定是否使用移动操作 |
| ❌ 强保证对所有操作都适用 | ✅ 强保证成本高（需要副本），对性能敏感代码可能只给基本保证 |
| ❌ `try-catch` 包围整个函数就是强保证 | ✅ 如果中途已有副作用发生，catch 后可能无法回滚 |
| ❌ 析构函数可以不写 `noexcept` | ✅ 析构函数默认 `noexcept`，如果真抛异常会 `std::terminate` |

---

## 六、异常安全等级判断表

| 问题 | 不抛保证 | 强保证 | 基本保证 |
|------|---------|--------|---------|
| 是否泄漏资源 | ❌ 绝不 | ❌ 绝不 | ❌ 绝不 |
| 对象是否可用 | ✅ 始终有效 | ✅ 始终有效 | ✅ 有效但可能不确定 |
| 是否回滚到操作前状态 | ✅ 是 | ✅ 是 | ❌ 不一定 |
| 是否抛出异常 | ❌ 绝不 | ✅ 可能丢 | ✅ 可能丢 |

---

## 七、一句话总结

> 异常安全三段论：用 RAII 拿到**基本保证**，用 copy-and-swap 拿到**强保证**，用 `noexcept` 声明**不抛保证**。
