# return 语句析构异常与返回对象销毁

C++23 规定：在 `return` 语句触发局部变量销毁期间若抛出异常，已构造的返回对象也必须在栈展开中被销毁。

**场景示例：**
```cpp
A f() {
    try {
        A a('a');
        Y y;  // ~Y() noexcept(false)，抛出异常
        A b('b');
        return {'c'};  // 返回对象构造完成
    } catch (...) {}
    return {'d'};
}
// 标准保证输出：bcad
// GCC 实际：bacd；Clang/MSVC 实际：bad
```

**常见误区**

- ❌ 返回对象只在函数正常结束时销毁 — 错误。标准明确要求异常路径中返回对象也要被销毁。
- ❌ 所有编译器都遵循标准输出 bcad — 错误。GCC 输出 `bacd`，Clang/MSVC 输出 `bad`。

**延伸点**：此规则揭示了返回值生命周期的边界情况：即使 RVO/NRVO 将返回对象构造在调用者栈帧，若 `return` 语句触发局部变量销毁时异常抛出，返回对象仍被视为"已构造的自动对象"而需销毁。
