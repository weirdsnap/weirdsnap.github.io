# 析构函数抛出异常与栈展开

析构函数默认隐式 `noexcept(true)`，若在栈展开期间抛出异常且未被捕获，或因违反 `noexcept` 契约，均触发 `std::terminate`。

| 场景 | 结果 |
|------|------|
| 默认 `noexcept` 析构抛异常 | 立即 `terminate`（违反契约） |
| 栈展开期间析构抛异常 | 立即 `terminate`（双异常规则） |
| 正常调用，显式 `noexcept(false)` 析构抛异常 | 异常传播（极危险，不推荐） |

**原理解析 + 常见误区**

- ❌ 析构函数可以抛异常，外面 catch 住就行 — 错误。默认 `noexcept` 的析构函数抛出异常不会进入任何 catch 块。
- ❌ `terminate` 前会完成其他对象的析构 — 错误。`std::terminate` 是立即终止，不保证调用其他析构函数。

**延伸点**：析构函数中绝对禁止抛出异常；若内部操作可能抛，应在析构函数内用 `try-catch` 就地吞掉异常并记录日志。
