# `std::string` 构造函数的隐藏重载

`std::string` 的构造函数有多个重载，参数不同行为完全不同。一个最常见的陷阱是 `size_t` 参数的含义——当它来自 C 字符串还是 `std::string` 时，语义完全相反。

---

## 一、核心陷阱

```cpp
#include <string>
#include <iostream>

int main() {
  using namespace std::string_literals;
  std::string s1("hello world", 5);     // 取前 5 个字符
  std::string s2("hello world"s, 5);    // 从第 5 个开始到末尾

  std::cout << s1 << s2;  // 输出 "hello world"
}
```

`s1` 和 `s2` 一个 `"hello"`，一个 `" world"`，拼起来正好是 `"hello world"`。但同样的 `5`，含义完全不同。

---

## 二、为什么行为不同？

关键在于第一个参数的类型不同，触发了不同的构造函数重载：

### `s1`：`string(const char*, size_t n)`

```cpp
std::string s1("hello world", 5);
// 匹配构造函数：string(const char* s, size_t count)
// 取 s 的前 count 个字符 → "hello"
```

### `s2`：`string(const string&, size_t pos)`

```cpp
std::string s2("hello world"s, 5);
// 第一个参数是 std::string（因为有 s 后缀），
// 匹配构造函数：string(const string& str, size_t pos, size_t count = npos)
// 从位置 pos 开始到末尾 → " world"
```

| 调用 | 参数类型 | 匹配的构造函数 | 5 的含义 | 结果 |
|------|---------|---------------|---------|------|
| `string("hello world", 5)` | `const char*` + `size_t` | `string(const char*, size_t count)` | 取前 5 个字符 | `"hello"` |
| `string("hello world"s, 5)` | `string` + `size_t` | `string(const string&, size_t pos)` | 从位置 5 开始 | `" world"` |

---

## 三、`s` 字面量后缀

```cpp
using namespace std::string_literals;

auto s1 = "hello";          // const char[6]
auto s2 = "hello"s;         // std::string
auto s3 = "hello"sv;        // std::string_view（C++17）
```

`s` 后缀定义在 `std::string_literals` 命名空间中，它将 C 字符串字面量转换为 `std::string` 类型，从而让重载决议匹配不同的构造函数。

---

## 四、`std::string` 的构造函数家族

```cpp
// 1. 默认构造
std::string s;                  // 空字符串

// 2. C 字符串构造
std::string s("hello");         // const char* → string

// 3. 取前 n 个字符
std::string s("hello world", 5);  // "hello"

// 4. 从 string 子串构造
std::string s(str, 5);          // 从位置 5 到末尾
std::string s(str, 5, 3);       // 从位置 5 取 3 个字符

// 5. 填充
std::string s(5, 'a');          // "aaaaa"

// 6. 迭代器区间
std::string s(begin, end);      // 从迭代器区间

// 7. 拷贝/移动
std::string s2(s1);             // 拷贝构造
std::string s2(std::move(s1));  // 移动构造
```

---

## 五、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `string("hello", 5)` 从位置 5 开始 | ✅ 取前 5 个字符 |
| ❌ `string(str, 5)` 取前 5 个字符 | ✅ 从位置 5 开始到末尾 |
| ❌ 不加 `s` 后缀和加 `s` 后缀没区别 | ✅ 类型不同（`const char*` vs `string`），可能导致不同构造函数重载 |
| ❌ `string("hello"s, 5)` 是取前 5 个 | ✅ 是从位置 5 开始取到末尾 |

---

## 六、一句话总结

> `std::string("hello world", 5)` 取前 5 个字符，`std::string("hello world"s, 5)` 从位置 5 取到末尾——同样的 `5`，语义取决于第一个参数是 C 字符串还是 `std::string`。
