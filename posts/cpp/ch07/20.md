# 重置与复用：`str()` 与 `clear()` 的分工

`str("")` 替换底层字符串内容，但**不重置错误状态位**；`clear()` 重置错误状态位，但**不碰内容**。两者正交，复用流对象时必须**先后调用**。

| 操作 | 影响内容 | 影响状态位 | 影响 capacity | 适用场景 |
|------|---------|-----------|--------------|---------|
| `ss.str("")` | 清空为 "" | ❌ 不碰 | 可能保留 | 清空内容，但状态仍保留 |
| `ss.clear()` | ❌ 不碰 | 全重置为 good | 不碰 | 错误恢复，继续读旧内容 |
| `ss.str("new")` | 替换为新串 | ❌ 不碰 | 可能保留 | 注入新输入 |
| `ss.str(""); ss.clear();` | 清空 + 重置 | 重置 | 可能保留 | **完全复用** |

```cpp
#include <sstream>
#include <iostream>

int main() {
    std::stringstream ss;
    ss << "abc";
    ss.setstate(std::ios::failbit);     // 模拟错误
    
    // 错误：只清内容，状态仍是 fail
    ss.str("123");
    int x;
    ss >> x;            // ❌ 仍失败，因为 failbit 还在
    
    // 正确复用流程
    ss.str("");         // 1. 清空内容
    ss.clear();         // 2. 重置状态位（关键！）
    ss << "456";
    ss >> x;            // ✅ x=456
    std::cout << x << "\n";
}
```

**原理解析 + 常见误区**

- ❌ `str("")` 会自动 `clear()` — 错误。只清内容，错误位还在。
- ❌ `clear()` 会清空内容 — 错误。只重置状态位。
- ❌ 复用流对象只需要 `str("")` — 错误。必须先 `str("")` 再 `clear()`。

**延伸点**：`ostringstream` 的 `str()` 返回底层字符串的**拷贝**；`str("")` 可能不释放已分配内存（取决于 libstdc++/libc++ 实现）。若在高频循环中复用，可配合 `std::string::reserve` 预分配，或改用 `std::vector<char>` + `std::to_chars` 避免流开销。
