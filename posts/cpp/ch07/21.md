# 定位操作：seekp / seekg 与读写指针

`stringstream` 底层共享同一 `stringbuf`，但维护**独立的 put（写）指针和 get（读）指针**。`seekp` 移动写指针，`seekg` 移动读指针；文件流中两者物理分离，但字符串流中操作同一内存。

| 函数 | 操作对象 | 方向 | 适用流 | 返回值 |
|------|---------|------|--------|--------|
| `tellg()` | 读指针 | 查询 | 所有输入流 | `streampos` |
| `seekg(pos)` | 读指针 | 绝对移动 | 所有输入流 | `*this` |
| `seekg(off, dir)` | 读指针 | 相对移动 | 所有输入流 | `*this` |
| `tellp()` | 写指针 | 查询 | 所有输出流 | `streampos` |
| `seekp(pos)` | 写指针 | 绝对移动 | 所有输出流 | `*this` |
| `seekp(off, dir)` | 写指针 | 相对移动 | 所有输出流 | `*this` |

```cpp
#include <sstream>
#include <iostream>

int main() {
    std::stringstream ss;
    ss << "ABCDEF";          // 写指针在 6
    
    ss.seekg(2);             // 读指针移到 'C'
    char c;
    ss >> c;                 // c = 'C'
    
    ss.seekp(1);             // 写指针移到 'B'
    ss << 'X';               // 内容变为 "AXCDEF"
    
    // 查询位置
    std::cout << "read pos: " << ss.tellg() << "\n";
    std::cout << "write pos: " << ss.tellp() << "\n";
    std::cout << "content: " << ss.str() << "\n";
}
```

**原理解析 + 常见误区**

- ❌ `seekp` 和 `seekg` 作用相同 — 错误。一个移动写指针，一个移动读指针。
- ❌ `stringstream` 读写指针是同一个 — 错误。底层共享 `stringbuf`，但读写指针独立。
- ❌ 在 `istringstream` 上用 `seekp` — 错误。`istringstream` 只继承输入流，没有 `seekp`。

**延伸点**：`stringstream` 的 `seekp` 在末尾之后写入时，中间空白区域由 `char()`（通常是 `\0`）填充。若先读后写，需确保 `seekp` 位置正确，否则可能覆盖已有数据。对于 `istringstream`，`seekg` 是主要操作，`seekp` 不可用（继承限制）。
