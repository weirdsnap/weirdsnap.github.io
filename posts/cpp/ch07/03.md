# cout / cerr / clog 缓冲策略差异

`cout` 缓冲输出到 `stdout`；`cerr` **无缓冲**直接输出到 `stderr`；`clog` 缓冲输出到 `stderr`。紧急错误用 `cerr`，日志用 `clog`。

| 对象 | 绑定 FD | 缓冲策略 | 刷新时机 | 典型用途 |
|------|--------|---------|---------|---------|
| `std::cout` | `stdout` | 全缓冲/行缓冲 | 程序结束 / 缓冲区满 / 手动 flush | 正常输出 |
| `std::cerr` | `stderr` | **无缓冲** (`unitbuf`) | 立即 | 紧急错误、调试 |
| `std::clog` | `stderr` | 全缓冲/行缓冲 | 同 `cout` | 日志记录 |

```cpp
#include <iostream>

int main() {
    std::cout << "cout start";
    std::cerr << "cerr interrupt";  // 立即出现，即使 cout 未换行
    std::clog << "clog log";        // 可能等缓冲区满或程序结束才出现
    std::cout << " ...cout end\n";  // 行缓冲在此刷新
}
```

**常见误区**

- ❌ `cerr` 和 `clog` 完全一样只是名字不同 — 错误。`cerr` 无缓冲立即输出，`clog` 有缓冲。
- ❌ 用 `cout` 输出错误信息也可以 — 可以运行，但 `stdout` 可能被重定向到文件，而错误信息通常需要保证到达用户。

**延伸点：** `std::cout` 默认与 `std::cin` 通过 `tie()` 绑定，每次读 `cin` 前会自动刷新 `cout`。可用 `std::cout.tie(nullptr)` 解除以提升性能，但需手动管理输出顺序。
