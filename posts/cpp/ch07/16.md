# 输出运算符与逻辑运算符的优先级陷阱

C++ 中 `<<`（输出/左移）、`>>`（输入/右移）、`&&`、`||` 的优先级**低于**比较和算术运算符，却**高于**赋值运算符。在 `std::cout` 链式输出和条件表达式中很容易因省略括号写出错误代码。

| 运算符组 | 优先级（高→低） | 结合性 |
|---------|----------------|--------|
| 算术 | `* / %` > `+ -` | 左结合 |
| 移位 | `<<` `>>` | 左结合 |
| 比较 | `<` `<=` `>` `>=` `==` `!=` | 左结合 |
| 按位 | `&` > `^` > `\|` | 左结合 |
| 逻辑 | `&&` > `\|\|` | 左结合 |
| 赋值 | `=` `+=` 等 | 右结合 |

**关键记忆**：`<<` 的优先级**低于** `<` `+`，但**高于** `&&` `\|\|` 和 `=`。

```cpp
#include <iostream>

int main() {
    int a = 1, b = 2, c = 3, d = 4;

    // 陷阱 1：cout << a < b
    // 解析为 (std::cout << a) < b，编译错误：ostream 不能和 int 比较
    // std::cout << a < b;   // ERROR
    std::cout << (a < b) << "\n";  // OK，输出 1

    // 陷阱 2：混合逻辑与比较
    // 虽然这里结果碰巧一样，但不要依赖记忆
    if (a < b && c < d) { }        // OK：(<) 高于 (&&)
    if (a && b || c && d) { }      // OK：等价于 (a && b) || (c && d)

    // 陷阱 3：移位与算术
    int x = 1 + 2 << 3;   // 解析为 (1 + 2) << 3 = 24
    std::cout << x << "\n";

    // 陷阱 4：赋值与输出链
    int y;
    // std::cout << y = 42;  // ERROR：= 优先级低于 <<，解析为 (std::cout << y) = 42
    std::cout << (y = 42) << "\n";  // OK
}
```

**原理解析 + 常见误区**

- ❌ `<<` 和 `+` 优先级相同 — 错误。`+` 优先级高于 `<<`。
- ❌ `&&` 优先级高于 `\|\|`，所以 `a && b || c` 需要括号 — 错误。`&&` 确实高于 `\|\|`，`a && b || c` 等价于 `(a && b) || c`。
- ❌ `std::cout << a < b` 会输出比较结果 — 错误。它解析为 `(std::cout << a) < b`，通常编译失败。

**延伸点**：C++ 运算符优先级表有十几级，不要依赖记忆。工程实践中，只要表达式稍微复杂，就加括号——可读性和正确性远比省两个字符重要。特别警惕 `cout` 配合比较/算术/逻辑运算符的场景。
