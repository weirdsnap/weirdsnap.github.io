# std::endl 与 '\n' 的本质差异

`endl` 是**换行 + 强制刷新缓冲区**；`'\n'` 仅换行，刷新由缓冲区策略决定。高频输出中滥用 `endl` 性能损耗严重。

| 操作 | 功能 | 刷新缓冲区 | 性能影响 | 适用场景 |
|------|------|-----------|---------|---------|
| `'\n'` | 换行 | 否（由策略决定） | 低 | 大量文本输出、文件写入 |
| `std::endl` | 换行 | **强制 flush** | 高（系统调用开销） | 需要立即落盘、交互式显示 |
| `std::flush` | 无 | 强制 flush | 高 | 仅刷新不换行 |

```cpp
#include <iostream>
#include <fstream>

int main() {
    std::ofstream f("test.txt");
    
    // 慢：10万次系统调用
    for (int i = 0; i < 100000; ++i) {
        f << i << std::endl;  
    }
    
    // 快：缓冲区满才写入，系统调用极少
    for (int i = 0; i < 100000; ++i) {
        f << i << '\n';  
    }
}
```

**常见误区**

- ❌ `endl` 和 `'\n'` 完全等价 — 错误。`endl` 会触发 flush，频繁系统调用导致性能瓶颈。
- ❌ 文件输出必须用 `endl` 才能写入磁盘 — 错误。文件流会在关闭或缓冲区满时自动刷新，显式 flush 只在需要立即落盘时使用。

**延伸点：** `std::ios::sync_with_stdio(false)` 可解除 C++ 流与 C 流的同步，提升性能但禁止混用 `cout` 和 `printf`。竞赛编程常见技巧。
