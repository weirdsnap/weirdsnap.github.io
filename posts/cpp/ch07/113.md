# 宽字符流 `wstringstream` 的平台差异

`std::wstringstream` 使用 `wchar_t`，但 `wchar_t` 编码**平台相关**：Windows 上为 16 位（UTF-16），大多数 Linux 上为 32 位（UTF-32）。不能与 `stringstream` 直接混用，构造时类型不匹配直接编译失败。

| 特性 | `std::stringstream` | `std::wstringstream` |
|------|---------------------|----------------------|
| 字符类型 | `char` | `wchar_t` |
| Windows 编码 | ANSI/UTF-8（取决于代码页） | UTF-16 |
| Linux 编码 | UTF-8 | UTF-32 |
| 互操作性 | 与 `std::string` 直接交互 | 与 `std::wstring` 交互 |
| 混用可能性 | ❌ 不可直接混用 | ❌ 不可直接混用 |

```cpp
#include <sstream>
#include <string>

int main() {
    std::wstringstream wss;
    wss << L"宽字符: " << 123;
    std::wstring w = wss.str();
    
    // ❌ 编译错误：不能从 wstringstream 提取 std::string
    // std::string s = wss.str();
    
    // 转换需显式编码（平台相关，非标准）
    // Windows: WideCharToMultiByte
    // Linux: wcstombs / iconv
}
```

**原理解析 + 常见误区**

- ❌ `wchar_t` 在所有平台都是 UTF-16 — 错误。Windows 是 UTF-16，Linux 通常是 UTF-32。
- ❌ `stringstream` 和 `wstringstream` 可以直接赋值 — 错误。字符类型不同，编译失败。
- ❌ 用 `wstringstream` 处理 Unicode 一定正确 — 错误。编码和平台相关，转换复杂。

**延伸点**：C++11 的 `std::codecvt` 已弃用（C++17 标记，C++26 可能移除）。现代跨平台方案：Windows 用 `MultiByteToWideChar`/`WideCharToMultiByte`，Linux 直接用 UTF-8 `char` 流。C++20 的 `std::format` 和 C++23 的 `std::print` 对宽字符支持仍有限，新项目建议统一使用 UTF-8 `char` 流。
