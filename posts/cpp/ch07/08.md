# `write()` 系统调用与信号安全

POSIX `write()` 绕过 C/C++ 标准库缓冲直接操作内核，是**异步信号安全（async-signal-safe）**的，可在信号处理函数中合法调用，而 `printf`/`cout` 不行。

| 函数 | 缓冲层 | 信号安全 | 可重入 | 适用场景 |
|------|--------|---------|--------|---------|
| `write()` | 无（内核） | ✅ | ✅ | 信号处理、底层控制 |
| `printf()` | C 库 | ❌ | ❌ | 普通输出 |
| `std::cout` | C++ 流 | ❌ | ❌ | 普通输出 |
| `std::cerr` | C++ 流（无缓冲） | ❌ | ❌ | 紧急输出（非信号） |

```cpp
#include <unistd.h>
#include <signal.h>
#include <cstring>

void handler(int) {
    const char msg[] = "SIGINT caught\n";
    // 信号安全：唯一可安全使用的输出方式
    write(STDERR_FILENO, msg, sizeof(msg) - 1);  
}

int main() {
    struct sigaction sa;
    std::memset(&sa, 0, sizeof(sa));
    sa.sa_handler = handler;
    sigaction(SIGINT, &sa, nullptr);
    while (true) {}
}
```

**原理解析 + 常见误区**

- ❌ `std::cerr` 无缓冲所以信号安全 — 错误。虽然无缓冲，但仍使用 C++ 流内部锁，信号处理中调用可能死锁。
- ❌ `write()` 比 `printf` 慢 — 错误。`write()` 无缓冲直接进内核，单次调用更快，但频繁小写入效率低。

**延伸点：** `man 7 signal` 列出所有 async-signal-safe 函数。除 `write` 外，`_exit`、`kill`、`sigaction` 等也是安全的。信号处理中绝对禁止调用 `malloc`、`printf`、`cout` 等，可能引发死锁或堆损坏。
