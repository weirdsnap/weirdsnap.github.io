# 常见的未定义行为（UB）

| UB 场景 | 典型示例 | 常见后果 |
|---------|---------|---------|
| Use-after-free | `delete p; *p;` | 堆损坏、信息泄漏 |
| Double-free | `delete p; delete p;` | 堆管理器元数据破坏 |
| 缓冲区溢出 | `arr[n]` 其中 `n >= size` | 栈/堆金丝雀破坏 |
| 未初始化使用 | `int x; cout << x;` | 垃圾值、信息泄漏 |
| `this == nullptr` 调用成员 | `((T*)nullptr)->f();` | 若 `f` 未访问成员可能"正常"，但标准定义为 UB |

**原理解析 + 常见误区**

- ❌ UB 一定会崩溃 — 错误。UB 最常见的表现是"看似正常"，导致 bug 隐蔽且难以复现。
- ❌ 调试模式能捕获所有 UB — 错误。use-after-free 和未初始化读取在调试模式下同样可能静默通过。

**延伸点**：AddressSanitizer (ASan) 和 MemorySanitizer (MSan) 是检测 UB 的主要工具，但只能检测运行时触发的路径。
