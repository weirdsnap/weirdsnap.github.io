# 格式化操纵器的持久性陷阱

多数操纵器（`hex`、`fixed`、`boolalpha`）一旦设置即**持久生效**，作用于后续所有 IO；唯独 `setw` 是**一次性**的，仅影响下一次输出。混用持久与一次性操纵器是常见 bug 来源。

| 操纵器 | 作用 | 持久性 | 头文件 | 常见误用 |
|--------|------|--------|--------|---------|
| `std::hex` / `std::dec` | 改变整数进制 | ✅ 持久 | `<ios>` | 忘记恢复导致后续输出全为十六进制 |
| `std::fixed` / `scientific` | 浮点格式 | ✅ 持久 | `<ios>` | 与 `std::setprecision` 组合后全局生效 |
| `std::boolalpha` | bool 输出 true/false | ✅ 持久 | `<ios>` | 持久生效，想用 1/0 时需 `noboolalpha` |
| `std::setw(n)` | 设置字段宽度 | ❌ **仅下次** | `<iomanip>` | 以为会持续对齐，实际只影响一次 |
| `std::setfill(c)` | 填充字符 | ✅ 持久 | `<iomanip>` | 常与 setw 搭配，但会持久影响 |
| `std::left` / `right` | 对齐方向 | ✅ 持久 | `<iomanip>` | 影响后续所有输出 |

```cpp
#include <sstream>
#include <iomanip>
#include <iostream>

int main() {
    std::ostringstream oss;
    
    oss << std::hex << 255 << " ";    // 持久：ff
    oss << 256 << "\n";                // 仍是 hex：100
    
    oss << std::setw(10) << std::setfill('*') << std::left << 42;
    // setw 仅影响 42 这一次；setfill 和 left 持久
    oss << 99 << "\n";                  // 输出: 99（无宽度，因 setw 已失效）
    
    std::cout << oss.str();
    // 输出: ff 100
    //       42********99
}
```

**原理解析 + 常见误区**

- ❌ `setw` 设置后一直有效 — 错误。只影响下一次输出。
- ❌ 持久操纵器会自动恢复 — 错误。`hex`、`fixed`、`boolalpha` 等一旦设置就一直有效。
- ❌ 临时改格式后不用恢复 — 错误。可能污染后续输出或调用者的流状态。

**延伸点**：若需在函数内临时改变格式，建议用 RAII 守卫保存并恢复流状态：`fmtflags old = ss.flags(); ss.flags(old);` 或 `std::ios_base::fmtflags old = ss.setf(std::ios::hex); ss.unsetf(std::ios::hex);`。C++20 的 `std::format` 彻底避免此类状态污染问题。
