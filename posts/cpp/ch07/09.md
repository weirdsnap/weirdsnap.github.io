# std::ostringstream 内存拼接

`ostringstream` 在内存中灵活拼接字符串，支持任意类型的 `<<` 重载，但每次操作涉及动态分配和虚函数调用，**性能显著低于** `to_chars` 或 `string::append`。

| 方案 | 灵活性 | 动态分配 | 性能 | 典型场景 |
|------|--------|---------|------|---------|
| `ostringstream` | 最高（任意类型） | 多次 | 低 | 构造复杂日志、调试信息 |
| `string::append` | 中（需手动转字符串） | 可控 | 中 | 已知类型的简单拼接 |
| `to_chars` | 低（仅数字） | **无** | 最高 | 热路径数值格式化 |
| `format_to` | 高 | 可控 | 高 | 现代替代 ostringstream |

```cpp
#include <sstream>
#include <string>
#include <iostream>

int main() {
    std::ostringstream oss;
    oss << "Error code: " << 404 << ", Path: " << "/api" << ", Retry: " << true;
    
    std::string msg = oss.str();  // 一次性获取，减少拷贝
    std::cout << msg << "\n";
    
    // 复用：清空内容但保留容量
    oss.str("");  
    oss.clear();  // 清除错误状态位
    oss << "Next log";
}
```

**常见误区**

- ❌ `ostringstream` 复用时 `clear()` 清空内容 — 错误。`clear()` 只清除错误状态位，清空内容用 `str("")`。
- ❌ `ostringstream` 适合高频热路径 — 错误。多次动态分配和虚函数调用使其性能低下，热路径应选 `to_chars` 或 `format_to`。

**延伸点：** C++20 起 `ostringstream` 支持移动语义，`std::move(oss).str()` 可避免深拷贝。若仅需字符串拼接而非格式化，考虑 `std::string::reserve` + `append` 或 `std::format_to`。
