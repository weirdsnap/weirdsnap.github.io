# perror 仅限系统错误

`perror` 读取全局 `errno` 并输出到 `stderr`。它**只能描述系统调用失败**（如 `open`/`malloc` 失败），不能用于业务逻辑错误码。

| 函数 | 输入 | 输出目标 | 适用场景 |
|------|------|---------|---------|
| `perror("msg")` | 隐式读取 `errno` | `stderr` | 系统调用失败 |
| `strerror(errno)` | 显式传入错误码 | 返回字符串 | 需要自定义格式 |
| `std::error_code` | 平台抽象错误码 | 任意 | 现代 C++ 跨平台错误处理 |

```cpp
#include <cstdio>
#include <cerrno>
#include <cstring>

int main() {
    FILE* f = std::fopen("not_exist.txt", "r");
    if (!f) {
        perror("fopen");              // 输出: fopen: No such file or directory
        // 等价但更灵活：
        std::fprintf(stderr, "error: %s\n", std::strerror(errno));
    }
}
```

**常见误区**

- ❌ `perror` 适用于所有错误场景 — 错误。只能用于系统调用失败，业务错误码需要自定义错误机制。
- ❌ `errno` 不是线程安全的 — 不完全。现代 libc 的 `errno` 通常是线程局部存储，但依赖实现，不可移植。

**延伸点：** C++11 起推荐使用 `<system_error>` 的 `std::error_code` 和 `std::error_category`，支持跨平台错误码映射和自定义错误类别。
