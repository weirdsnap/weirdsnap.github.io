# 输出方案选择矩阵

没有绝对最好的输出工具，只有最适合当前场景的选择。核心权衡维度是：**类型安全、性能、缓冲控制、信号安全**。

| 场景 | 推荐方案 | 核心理由 |
|------|---------|---------|
| C++ 日常输出 | `std::cout` / `std::println` | 类型安全、可扩展、现代语法 |
| 紧急错误 / 调试 | `std::cerr` / `fprintf(stderr)` | 无缓冲，立即显示，不干扰 stdout |
| 系统调用失败 | `perror` / `strerror(errno)` | 自动翻译 `errno`，标准惯例 |
| 高性能数值输出 | `std::to_chars` | 零分配、零异常、极快 |
| 格式化字符串 | `std::format` / `std::println` | 编译期类型检查，语法现代 |
| 写日志文件 | `std::ofstream` / `std::clog` | 可持久化，缓冲减少 IO 次数 |
| 内存中拼接 | `std::ostringstream` | 灵活，但避免热路径 |
| 信号处理 / 底层 | `write()` | 异步信号安全，无锁无缓冲 |

```cpp
#include <cerrno>
#include <cstdio>
#include <cstring>
#include <fstream>
#include <iostream>
#include <format>
#include <charconv>
#include <array>
#include <unistd.h>

int main() {
    // 1. 系统错误：perror
    FILE* f = std::fopen("none.txt", "r");
    if (!f) perror("fopen");

    // 2. 紧急错误：cerr
    std::cerr << "Immediate error\n";

    // 3. 日常输出：format/println
#if __cplusplus >= 202302L
    std::println("Modern: errno = {}", errno);
#else
    std::cout << std::format("Legacy: errno = {}\n", errno);
#endif

    // 4. 高性能数值
    std::array<char, 20> buf;
    auto [p, ec] = std::to_chars(buf.data(), buf.data() + buf.size(), 12345);
    std::string_view sv(buf.data(), p - buf.data());

    // 5. 日志文件
    std::ofstream log("app.log");
    log << sv << "\n";

    // 6. 底层信号安全输出
    const char msg[] = "Raw write\n";
    write(STDERR_FILENO, msg, sizeof(msg) - 1);
}
```

**常见误区**

- ❌ 只用一种输出方案就够了 — 错误。不同场景需要不同工具，日常用 `cout`、信号处理必须用 `write`。
- ❌ `ostringstream` 适合所有字符串构造场景 — 错误。热路径下频繁动态分配性能很差，应选 `format_to` 或预分配 `string`。

**延伸点：** 生产级日志通常不直接用标准库，而是使用 **spdlog**（基于 `fmt`/`std::format`）或 **glog**。它们提供日志级别、异步写入、轮转、背压控制等企业级特性，底层仍然遵循「缓冲 + 批量写入」的性能原则。
