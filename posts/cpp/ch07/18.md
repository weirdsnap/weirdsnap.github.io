# operator>> 提取陷阱

`>>` 以**空白字符**（空格、`\t`、`\n`）为分隔符；提取失败时设置 `failbit`，到达末尾设置 `eofbit`（通常连带 `failbit`）。提取到 `std::string` 不会跨行，提取到数值时前导非数字字符直接失败。

| 目标类型 | 输入样例 | 提取结果 | 流状态 | 剩余内容 |
|----------|---------|---------|--------|---------|
| `int` | `"123abc"` | `123` | `good` | `"abc"` |
| `int` | `"abc123"` | 未修改 | `fail` | 原样（`abc123`） |
| `int` | `"9999999999"` | 未定义/未修改 | `fail` | 原样（C++11 起溢出设 fail） |
| `std::string` | `"hello world"` | `"hello"` | `good` | `" world"` |
| `double` | `"3.14.15"` | `3.14` | `good` | `".15"` |

```cpp
#include <sstream>
#include <iostream>

int main() {
    std::istringstream iss("42 abc 3.14");
    
    int a, b;
    iss >> a;           // ✅ a=42
    iss >> b;           // ❌ b 未初始化，failbit 被设置
    // 后续所有 >> 自动失败，除非先 clear()
    
    if (iss.fail()) {
        std::cout << "提取失败，b 值未定义: " << b << "\n";
        iss.clear();                    // 清除错误
        iss.ignore(100, ' ');           // 跳过 "abc" 到下一个空白
        double d;
        iss >> d;                       // ✅ d=3.14
    }
}
```

**原理解析 + 常见误区**

- ❌ `>>` 失败后会自动恢复 — 错误。`failbit` 会阻止后续所有 `>>`，必须手动 `clear()`。
- ❌ `>>` 提取字符串可以跨行 — 错误。以空白分隔，换行也是空白。
- ❌ 混用 `>>` 和 `getline` 不需要处理残留 — 错误。`>>` 留下的换行会导致 `getline` 读到空行。

**延伸点**：默认 `skipws` 标志会跳过前导空白，但提取后**分隔符留在流中**。混用 `>>` 和 `std::getline` 时，`>>` 留下的换行符会导致 `getline` 读到空行——需之间插入 `iss.ignore()` 或 `std::getline(iss, dummy)` 消耗残余换行。
