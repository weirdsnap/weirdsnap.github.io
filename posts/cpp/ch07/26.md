# 字符串分割与 >> / getline 混用陷阱

`>>` 以空白分隔且不消耗换行；`std::getline` 以指定分隔符分割且**消耗分隔符**。两者混用时，`>>` 留下的换行会导致 `getline` 读到空行。

| 方法 | 分隔符 | 是否消耗分隔符 | 连续分隔符行为 | 换行符处理 |
|------|--------|--------------|--------------|-----------|
| `iss >> token` | 任意空白 | 是 | 合并（跳过） | 作为空白跳过 |
| `getline(iss, token, '\n')` | `\n` | 是 | 产生空字符串 | 精确按行 |
| `getline(iss, token, ',')` | `,` | 是 | 产生空字符串 | 作为普通字符 |

```cpp
#include <sstream>
#include <string>
#include <iostream>

int main() {
    std::istringstream iss("42\nJohn Doe\n3.14");
    
    int id;
    std::string name;
    double score;
    
    iss >> id;                    // 读到 42，换行符 \n 留在流中
    // iss >> std::ws;            // 若不添加，getline 会读到空行！
    std::getline(iss, name);      // ❌ 先读到 ""（空行）
    
    // 正确做法：
    std::istringstream iss2("42\nJohn Doe\n3.14");
    iss2 >> id;
    iss2.ignore(1, '\n');         // 或 iss2 >> std::ws; 消耗换行
    std::getline(iss2, name);     // ✅ name = "John Doe"
    iss2 >> score;                // ✅ score = 3.14
}
```

**原理解析 + 常见误区**

- ❌ `>>` 会消耗换行符 — 错误。`>>` 跳过前导空白，但分隔符留在流中。
- ❌ `getline` 不会跳过空行 — 正确理解。连续换行会产生空字符串。
- ❌ 用 `stringstream` 手动解析 CSV 很方便 — 不建议。复杂格式应用专用库。

**延伸点**：分割 CSV 等复杂格式时，不要用 `stringstream` 手动解析，应使用专用库（如 `fast-cpp-csv-parser`）或正则表达式。`std::getline` 对 `std::string` 的扩展是分配未知的——若行长度差异大，可先用 `std::string::reserve` 预分配平均长度，减少重新分配次数。
