# `stringstream` 家族与继承体系

`std::stringstream` 是双向流，但底层读写指针独立；实际使用中应优先区分 `std::ostringstream`（只写）和 `std::istringstream`（只读），避免读写指针互相干扰的心智负担。

| 类型 | 继承自 | 模式 | 典型用途 | 缓冲区共享 |
|------|--------|------|---------|-----------|
| `istringstream` | `basic_istream` | 只读 | 解析字符串 | 引用内部字符串 |
| `ostringstream` | `basic_ostream` | 只写 | 拼接字符串 | 内部字符串增长 |
| `stringstream` | `basic_iostream` | 双向 | 同时读写 | 同一 `stringbuf`，但读写指针分离 |

```cpp
#include <sstream>
#include <iostream>

int main() {
    // 只读解析
    std::istringstream iss("42 3.14 hello");
    int i; double d; std::string s;
    iss >> i >> d >> s;  // 按空白分隔提取
    
    // 只写拼接
    std::ostringstream oss;
    oss << "Error code: " << 404 << ", path: /api";
    std::string msg = oss.str();
    
    // 双向：需谨慎管理读写位置
    std::stringstream ss;
    ss << "data";
    ss.seekg(0);           // 读指针回到开头
    std::string r;
    ss >> r;               // 读到 "data"
}
```

**原理解析 + 常见误区**

- ❌ `stringstream` 适合频繁同时读写 — 错误。读写指针独立管理容易出错，应优先用 `ostringstream` / `istringstream`。
- ❌ `str()` 返回底层字符串引用 — 错误。`str()` 返回的是**拷贝**。
- ✅ `ostringstream` 只写，`istringstream` 只读，职责单一更清晰。

**延伸点**：`str()` 获取副本，`str("")` 清空内容但底层 `std::string` 可能保留 `capacity`（不保证释放内存）。频繁创建/销毁短生命周期的 `stringstream` 时，考虑在循环外复用或换用 `std::string::append` + `to_chars`。
