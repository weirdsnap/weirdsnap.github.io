# `std::to_chars` 零分配高性能

`<charconv>` 提供数字到字符串的**零动态分配、零异常**转换，直接写入用户提供的缓冲区，性能远超 `sprintf` 和 `ostringstream`。

| 方案 | 动态分配 | 异常 | 性能 | 头文件 |
|------|---------|------|------|--------|
| `std::to_chars` | **无** | **无** | 最快 | `<charconv>` |
| `sprintf` | 无（若缓冲区固定） | 无 | 中等 | `<cstdio>` |
| `std::ostringstream` | 有 | 可能 | 最慢 | `<sstream>` |
| `std::format` | 可能有 | 可能有 | 快 | `<format>` |

```cpp
#include <charconv>
#include <array>
#include <string_view>
#include <iostream>

int main() {
    std::array<char, 20> buf;
    int value = 12345;
    
    auto [ptr, ec] = std::to_chars(buf.data(), buf.data() + buf.size(), value);
    if (ec == std::errc()) {
        std::string_view sv(buf.data(), ptr - buf.data());
        std::cout << sv << "\n";  // "12345"
    }
}
```

**原理解析 + 常见误区**

- ❌ `to_chars` 只能转换整数 — 错误。C++17 后逐步完善浮点数支持，C++26 将全面覆盖。
- ❌ `to_chars` 返回 `std::string` — 错误。返回的是写入末尾指针 + 错误码，需自行管理缓冲区。

**延伸点：** C++17 的 `to_chars` 最初仅支持整数；C++17 后通过 DR 和 C++26 逐步完善浮点数支持。对应逆向操作是 `std::from_chars`，同样零分配解析字符串到数字。
