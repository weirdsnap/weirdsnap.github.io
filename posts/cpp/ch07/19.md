# 流状态位（eof/fail/bad）与检测逻辑

`goodbit` 实际值为 0（无错误位）。`fail()` 返回 true 当 `failbit` 或 `badbit` 被设置。`while (iss >> x)` 能终止循环，是因为流对象在布尔上下文中转换为 `!fail()`，而非单纯检测 `eof`。

| 状态位 | 触发条件 | 检测函数 | 清除方式 | 影响后续 IO |
|--------|---------|---------|---------|------------|
| `eofbit` | 到达输入末尾 | `eof()` | `clear()` | 本身不阻止，但通常伴随 fail |
| `failbit` | 逻辑错误（类型不匹配） | `fail()` | `clear()` | **阻止后续所有提取** |
| `badbit` | 严重错误（内存不足） | `bad()` | `clear()` | 阻止后续所有 IO |
| `goodbit` | 无错误（值为 0） | `good()` | N/A | 正常操作 |

```cpp
#include <sstream>
#include <iostream>

int main() {
    std::istringstream iss("123");
    int a, b;
    
    iss >> a;           // a=123, eofbit 被设置（因为读到末尾）
    std::cout << "eof=" << iss.eof() << ", fail=" << iss.fail() << "\n";
    // 输出: eof=1, fail=0（此时 !fail 仍为 true）
    
    iss >> b;           // 因 eof 无法提取，failbit 也被设置
    std::cout << "eof=" << iss.eof() << ", fail=" << iss.fail() << "\n";
    // 输出: eof=1, fail=1
    
    // 经典循环：读到失败时停止
    std::istringstream nums("1 2 3");
    int x;
    while (nums >> x) {  // 等价于 while (!nums.fail())
        std::cout << x << " ";
    }
}
```

**原理解析 + 常见误区**

- ❌ `eof()` 为 true 就停止读取 — 错误。最后一次成功读取时 `eof()` 可能已经是 true，但 `fail()` 仍为 false。
- ❌ `while (!iss.eof())` 是正确循环条件 — 错误。应在 `>>` 失败时停止，即 `while (iss >> x)`。
- ❌ `failbit` 和 `badbit` 一样严重 — 错误。`failbit` 通常是可恢复的逻辑错误，`badbit` 是致命错误。

**延伸点**：`operator bool()`（C++11 起显式）等价于 `!fail()`，因此 `if (iss >> x)` 在提取失败或到达 EOF 时均为 false。如果 `badbit` 被设置，流进入不可恢复状态，通常需要检查 `bad()` 以区分「可重试的逻辑错误」和「致命错误」。
