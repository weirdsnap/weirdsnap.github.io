# C++23 spanstream：零拷贝解析

C++23 引入 `<spanstream>`，提供 `std::ispanstream`、`std::ospanstream`、`std::spanstream`。基于 `std::span<char>`，**不拥有内存、零拷贝、零动态分配**，适合解析已有缓冲区。

| 特性 | `std::istringstream` | `std::ispanstream` (C++23) |
|------|---------------------|---------------------------|
| 内存所有权 | 拷贝输入，内部持有 | 引用外部 `span`，不拥有 |
| 动态分配 | 构造时分配 | **无** |
| 缓冲区修改 | 内部私有 | 可写（若 span 非 const） |
| 生命周期要求 | 独立 | 外部缓冲区必须存活 |
| 性能 | 中等 | 极高 |

```cpp
#include <spanstream>  // C++23
#include <span>
#include <iostream>

int main() {
    char buffer[] = "42 3.14 hello";
    std::span<char> sp(buffer);
    
    std::ispanstream iss(sp);  // 零拷贝引用 buffer
    int i; double d; std::string s;
    iss >> i >> d >> s;
    
    std::cout << i << " " << d << " " << s << "\n";
    // 输出: 42 3.14 hello
    // buffer 内容未被修改，iss 仅持有视图
}
```

**常见误区**

- ❌ `spanstream` 会拷贝输入 — 错误。它只持有 `span`，是视图。
- ❌ `spanstream` 的 `str()` 可用 — 错误。`str()` 被删除，因为不拥有内存。
- ❌ 外部缓冲区释放后仍可用 `spanstream` — 错误。必须保证缓冲区存活。

**延伸点**：`spanstream` 的 `str()` 被删除（无法获取字符串拷贝，因为不拥有内存），需通过外部 `span` 访问底层数据。这是「视图型流」的设计哲学：解析网络包、日志行切片时，避免 `std::string` 拷贝，直接操作固定缓冲区。目前 GCC 14+、Clang 18+ 开始支持。
