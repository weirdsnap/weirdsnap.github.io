# printf 家族类型不安全

`printf` 的格式字符串与后续实参类型不匹配时，不会编译报错，而是**运行时未定义行为**（UB）。这是 C 风格 IO 的核心缺陷。

| 特性 | `printf` | `std::format` (C++20) |
|------|----------|----------------------|
| 类型检查 | 运行期（无） | 编译期 |
| 不匹配后果 | UB / 崩溃 / 信息泄露 | 编译失败 |
| 性能 | 较快 | 快 |
| 扩展性 | 有限 | 可自定义类型格式化 |

```cpp
#include <cstdio>

int main() {
    // UB：%d 对应 double，栈读取错误
    std::printf("%d\n", 3.14);  
    
    // 正确
    std::printf("%f\n", 3.14);  
    std::printf("%s\n", "hello"); // %s 对应 char*，传 std::string 也会 UB
}
```

**原理解析 + 常见误区**

- ❌ GCC/Clang 的 `-Wformat` 能完全避免 `printf` 类型错误 — 错误。只能检查部分编译期可见的情况，无法处理跨翻译单元或动态格式字符串。
- ❌ `printf` 比 `std::cout` 快很多所以应该优先用 — 片面。现代编译器下差距不大，且类型安全带来的可维护性远高于微优化。

**延伸点：** C++20 的 `std::format` 彻底解决这个问题，提供编译期类型检查且性能不劣于 `printf`。
