# 九、编译期元编程与深层机制（99–101）

---

### 99. 拷贝并交换惯用法（Copy-and-Swap）

通过值传递隐式完成拷贝，利用临时对象析构自动释放旧资源，天然保证自赋值安全和强异常安全。

**场景示例：**
```cpp
T& operator=(T other) {  // 按值传递，触发拷贝构造
    swap(other);          // 与临时对象交换资源
    return *this;         // 临时对象析构时释放旧资源
}
```

**延伸点**：这是现代 C++ 实现赋值运算符的最佳实践，无需写析构逻辑，且消除了自赋值检查的需求。

### 100. placement new 与显式析构

在已分配的内存上构造对象（不分配新内存），常用于内存池、容器（如 `std::vector`）。需手动调用析构函数，且不能对该指针使用 `delete`。

**延伸点**：`std::vector` 的 `push_back` / `emplace_back` 内部使用 placement new 在已分配的容量空间中构造对象，配合 `std::uninitialized_copy` 等算法实现异常安全的批量构造。

---

### 101. `std::move_only_function` / `function_ref`

C++23 `std::move_only_function` 支持移动语义避免拷贝分配；`llvm::function_ref`（非标准）是不拥有可调用对象的零拷贝视图。

**延伸点**：若对延迟极度敏感，可改用 `std::move_only_function` 或 `llvm::function_ref`，前者支持移动语义，后者仅作为临时引用，均比 `std::function` 更轻量。
