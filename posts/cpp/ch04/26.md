# 部分显式指定模板参数与隐式转换

函数模板调用时，可以显式指定部分模板参数，剩余的仍由实参自动推导。

## 规则

根据 `[temp.arg.explicit]¶1` 和 `¶4`：

- 显式指定的模板参数从最左边开始，依次绑定
- 尾部未指定的模板参数仍由实参自动推导
- 显式指定的参数**不参与**模板实参推导

```cpp
template<typename T, typename U>
void f(T, U) {
    std::cout << std::is_same_v<T, U>;
}

int i = 0;
double d = 0.0;

f(i, d);          // T=int, U=double → 0
f<int>(i, d);     // T=int（显式指定）, U=double（推导）→ 0
f<double>(i, d);  // T=double（显式指定）, U=double（推导）→ 1
```

**关键点**：`f<double>(i, d)` 中显式指定的 `T=double` 不参与推导，因此 `int i` 可以隐式转换为 `double`——`[temp.arg.explicit]¶7` 规定，如果函数参数类型不包含参与推导的模板参数，则允许隐式转换。

## 与隐式转换的关系

模板推导阶段通常**不允许**隐式转换，以保证类型安全：

```cpp
template<typename T>
void g(T);

g(42);    // T=int，精确匹配
g(3.14);  // T=double，精确匹配
```

但当某个模板参数被显式指定后，它不再参与推导，该参数对应的函数参数类型已经固定，此时实参可以隐式转换到该类型：

```cpp
template<typename T, typename U>
void h(T, U);

h<double>(42, 3.14);  // T=double（显式指定）, U=double（推导）
// 42 从 int 隐式转换为 double，允许！
```

## 常见误区

- ❌ `f<int>(i, d)` 中 `int` 和 `double` 不匹配应该编译错误 — 错误。模板参数可以部分指定，剩下由推导补齐，类型不同只影响模板体中的行为，不导致编译错误。
- ❌ 显式指定模板参数后，实参类型必须精确匹配 — 错误。`[temp.arg.explicit]¶7` 明确指出，若参数类型不含参与推导的模板参数，允许隐式转换。
- ❌ 隐式转换在模板里永远禁止 — 错误。模板推导阶段禁止隐式转换，但显式指定参数后推导不再涉及该参数，隐式转换恢复正常。

## 延伸点

- `f<double>(i, d)` 中 `const int i` 向 `double` 的转换涉及两次类型转换（`const int → int → double`），这在隐式转换链中是允许的（标准转换序列可包含多个步骤）。
- 类模板没有部分指定推导的等价机制——CTAD 要么全推要么全指定（C++17 起），不存在只指定部分模板参数的写法。
- `[temp.arg.explicit]¶7` 的 Note 3 明确说明：模板参数不参与推导的标志就是它被显式指定了。
