# 函数入参推导保留顶层 `const`

按值推导时顶层 `const` 被剥离，但按引用或指针传递时，为维持 const 正确性，编译器会将顶层 `const` 保留在模板参数推导结果中。

**原理解析 + 常见误区**

- ❌ 按引用传递时 const 也会像按值一样被剥离 — 错误。`T&` 绑定 `const int` 时，`T` 必须推导为 `const int`。
- ❌ `const T&` 推导时 `T` 会包含 const — 错误。`const` 是参数类型的一部分，推导时 `T` 本身为 `int`。

**延伸点**：万能引用 `T&&` 的推导：传入 `const int` 左值时，`T` 推导为 `const int&`（经引用折叠），这是万能引用能同时接受左值和右值的关键机制。
