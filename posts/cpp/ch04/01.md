# 模板参数推导（int vs const int）

函数模板按值传递时，实参的顶层 `const`（如 `const int`）会被推导忽略，`T` 推导为 `int`；按引用或指针传递时，顶层 `const` 被保留。

| 参数声明 | 传入 `const int` | `T` 推导结果 |
|----------|-----------------|-------------|
| `T`（按值） | `const int` | `int`（顶层 const 剥离） |
| `T&`（引用） | `const int` | `const int`（保留） |
| `T*`（指针） | `const int*` | `const int`（保留） |

**原理解析 + 常见误区**

- ❌ `const int` 和 `int` 在模板中推导总是不同 — 错误。按值传递（`T`）时推导结果完全相同。
- ❌ `const T&` 推导 `T` 会包含 const — 错误。`const` 已在参数类型中显式写出，`T` 本身为 `int`。

**延伸点**：数组退化的不对称性：按值传递数组时 `T` 推导为指针（`int*`），按引用传递 `T&` 时保留数组类型及大小（`int (&)[5]`）。
