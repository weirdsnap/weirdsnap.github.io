# 模板类基类函数调用

模板派生类中直接调用非限定名称 `func()` 时，依赖基类 `Base<T>` 的成员不可见；全局同名函数优先匹配。

**场景示例：**
```cpp
template<typename T>
class Derived : public Base<T> {
public:
    void call() {
        func();              // ❌ 调用全局 func（基类版本不可见）
        this->func();        // ✅ 依赖名称，实例化时查找基类
        Base<T>::func();     // ✅ 显式限定基类
    }
};
```

**原理解析 + 常见误区**

- ❌ 模板继承后基类成员自动可见 — 错误。模板基类是依赖类型，其成员被第一阶段查找排除。
- ❌ 加 `this->` 只是风格问题 — 错误。`this->` 使 `func` 成为依赖名称，触发第二阶段查找。

**延伸点**：CRTP 中应始终使用 `this->` 前缀调用基类函数，以确保两阶段查找正确工作。
