# CRTP：奇异递归模板模式

派生类将自己作为模板参数传给基类 `Base<Derived>`，基类通过 `static_cast<Derived*>(this)` 调用派生类方法，实现**编译期静态多态**，零运行时开销。

| 特性 | CRTP（静态多态） | 虚函数（动态多态） |
|------|------------------|-------------------|
| 派发时机 | 编译期 | 运行时 |
| 开销 | 零（直接内联） | vtable 查找 + 间接跳转 |
| 对象大小 | 不增加 | 增加 vptr |
| 关系要求 | 继承 `Base<Derived>` | 继承虚基类 |

```cpp
template <typename Derived>
class Base {
public:
    void interface() {
        static_cast<Derived*>(this)->implementation();  // 编译期派发
    }
};

class Derived : public Base<Derived> {
public:
    void implementation() { /* 实际逻辑 */ }
};
```

**原理解析 + 常见误区**

- ✅ `static_cast<Derived*>(this)` 在 `Derived : public Base<Derived>` 时是良定义的。
- ❌ CRTP 是运行时多态 — 错误。没有 vtable，所有调用在编译期解析。
- ❌ `class Evil : public Base<Other>` 也能工作 — 错误。错误指定模板参数会导致未定义行为。
- ❌ 这是真正的递归定义 — 错误。模板惰性实例化使编译器能正确处理前向引用。

**延伸点**：CRTP 典型应用包括 `Comparable` 运算符注入、对象计数器、表达式模板、编译期策略模式。C++20 Concepts 可替代纯接口约束，但 CRTP 仍适合需要共享状态或默认实现的场景；C++23 `Deducing this` 进一步简化了部分静态多态写法。
