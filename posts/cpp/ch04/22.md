# `explicit` 转换运算符

`explicit operator T()` 阻止隐式转换，仅允许通过 `static_cast<T>(obj)`、`T(obj)` 或 `obj.operator T()` 显式调用；`explicit operator bool` 在条件语境中例外允许隐式使用。

| 语法 | 是否调用 `explicit operator T()` |
|------|----------------------------------|
| `static_cast<T>(obj)` | ✅ |
| `T(obj)` / `(T)obj` | ✅ |
| `obj.operator T()` | ✅ |
| `T t = obj;` | ❌ |
| `void f(T); f(obj);` | ❌ |

**原理解析 + 常见误区**

- ❌ `explicit` 阻止所有转换 — 错误。仅阻止隐式转换；显式语法和条件 `bool` 仍可用。
- ❌ `T(obj)` 是隐式转换 — 错误。函数风格转换属于显式类型转换。

**延伸点**：`explicit operator bool` 取代 Safe Bool Idiom，被所有标准库类型（智能指针、流对象、可选值）采用。
