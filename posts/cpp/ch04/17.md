# 可变参数模板

可变参数模板让函数或类能接受任意个数、任意类型的模板参数。C++11 通过递归展开参数包实现，C++17 的折叠表达式则提供了一种更直接的写法。

---

## 基本语法

```cpp
template<typename... Ts>
void foo(Ts... args);
```

- `typename... Ts` 叫**模板参数包**
- `Ts... args` 叫**函数参数包**
- `args...` 叫**包展开**

参数包不能直接像数组一样用下标访问，比如 `args[0]` 或 `Ts[1]` 都是不允许的。

---

## C++11：递归展开

最常见的写法是用递归。下面是一个求和函数：

```cpp
// 终止条件：只剩一个参数
template<typename T>
T sum(T arg) {
    return arg;
}

// 递归：处理第一个参数，剩下的交给下一次调用
template<typename T, typename... Args>
T sum(T arg, Args... args) {
    return arg + sum<T>(args...);
}
```

调用过程类似这样：

```cpp
sum(0.5, 1, 0.5, 1)
// 展开为：0.5 + sum<double>(1, 0.5, 1)
//          0.5 + (1 + sum<double>(0.5, 1))
//          0.5 + (1 + (0.5 + sum<double>(1)))
//          0.5 + (1 + (0.5 + 1))
// 结果：3
```

注意这里的 `sum<T>(args...)` 显式指定了模板参数 `T`。这会带来一个很容易踩的坑。

---

## 一个经典陷阱：递归调用中 T 只推导一次

看下面这段代码：

```cpp
#include <iostream>

template<typename T>
T sum(T arg) {
    return arg;
}

template<typename T, typename... Args>
T sum(T arg, Args... args) {
    return arg + sum<T>(args...);
}

int main() {
    auto n1 = sum(0.5, 1, 0.5, 1);
    auto n2 = sum(1, 0.5, 1, 0.5);
    std::cout << n1 << " " << n2 << "\n";
}
```

输出是：

```
3 2
```

为什么 `n2` 不是 3？

**第一次调用 `sum` 时，T 根据第一个实参推导。**

- `sum(0.5, 1, 0.5, 1)`：第一个参数是 `double`，所以 `T = double`
- `sum(1, 0.5, 1, 0.5)`：第一个参数是 `int`，所以 `T = int`

**但是递归调用时写的是 `sum<T>(args...)`，T 被固定住了。**

所以 `n2` 的展开过程是：

```cpp
sum(1, 0.5, 1, 0.5)
// T = int
// 1 + sum<int>(0.5, 1, 0.5)
// 1 + (0 + sum<int>(1, 0.5))   // 0.5 传给 int 参数，小数部分被截断
// 1 + (0 + (1 + sum<int>(0.5)))
// 1 + (0 + (1 + 0))
// 结果：2
```

这里 `0.5` 作为 `double` 传给 `int` 形参时，按照 C++ 标准的浮点到整数转换规则，**小数部分被直接丢弃**，不是四舍五入。

> A prvalue of a floating point type can be converted to a prvalue of an integer type. The conversion truncates; that is, the fractional part is discarded.
>
> — §[conv.fpint]¶1, C++23

这个例子提醒我们：在递归展开的可变参数模板里，如果显式固定了某个类型，后续所有参数都会被强制转成这个类型。

## C++17：折叠表达式

C++17 引入了折叠表达式，很多场景下不再需要递归：

```cpp
template<typename... Args>
auto sum(Args... args) {
    return (args + ...);  // 一元右折叠
}
```

这个版本每次加法的类型由当前两个操作数共同决定，不会发生上面那种“强制固定为 int”的问题：

```cpp
sum(1, 0.5, 1, 0.5);  // 结果是 3.0，类型是 double
```

但折叠表达式也有局限：如果需要复杂的编译期分支，或者参数类型必须统一，递归仍然有用。

---

## 递归终止的写法

C++11 递归展开必须提供终止条件，通常是零参数或单参数的重载。

```cpp
// 终止条件
template<typename T>
void print(T value) {
    std::cout << value << "\n";
}

// 递归
template<typename T, typename... Rest>
void print(T value, Rest... rest) {
    std::cout << value << " ";
    print(rest...);
}
```

C++17 可以用 `if constexpr` 在单个函数里处理终止：

```cpp
template<typename T, typename... Rest>
void print(T value, Rest... rest) {
    std::cout << value;
    if constexpr (sizeof...(rest) > 0) {
        std::cout << " ";
        print(rest...);
    } else {
        std::cout << "\n";
    }
}
```

## 常见误区

1. **参数包可以像数组一样索引访问**
   - 错误。C++ 不支持 `args[0]` 或 `Ts[1]`。

2. **可变参数函数模板不需要终止条件**
   - 错误。C++11 递归展开必须提供终止重载。

3. **递归调用中模板参数每次都会重新推导**
   - 错误。如果写成 `func<T>(args...)`，T 就被固定了。

4. **浮点数传给整数参数会四舍五入**
   - 错误。小数部分直接截断。

---

## 一句话总结

可变参数模板递归展开时，模板参数只在第一次调用时推导；一旦显式传给递归调用，后续所有参数都会按这个类型处理。设计求和、打印这类函数时，要特别注意类型统一的问题。

> 相关阅读：`if constexpr` 虽然方便，但丢弃分支里有不依赖模板参数的非法语句时会导致 IFNDR，详见 [ch04/25](../25.md)。
