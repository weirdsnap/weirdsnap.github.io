# IFNDR：只有空包才合法的变参模板

```cpp
template<typename... Ts>
struct X {
    X(Ts... args) : Var(0, args...) {}
    int Var;
};

int main() {
    X<> x;   // OK？还是 UB？
}
```

这段代码看起来没问题——`X<>` 展开为 `X() : Var(0) {}`，合法。但如果带参数实例化 `X<int>(1)`，展开为 `X(int arg1) : Var(0, arg1) {}`，试图用两个参数初始化 `int`，非法。

问题在于：**这个模板只有空包一种合法用法**。标准 `[temp.res.general]¶6.3` 规定这种情况是 **IFNDR（Ill-Formed, No Diagnostic Required）**。

---

## 一、什么是 IFNDR

IFNDR 指代码在标准上是非法的，但**编译器不需要诊断**——可以不报错。程序的行为是未定义的，可能编译通过、正常运行、崩溃，也可能在换编译器版本后出错。

```cpp
// GCC 13 和 Clang 16 上，X<> x; 编译通过，运行无问题
// 但标准上说这是 IFNDR，编译器没有被要求报错
```

---

## 二、IFNDR 的常见触发场景

| 场景 | 示例 | 相关标准规则 |
|------|------|-------------|
| 变参模板仅空包合法 | `template<typename... Ts> struct X { ... };` 只有 `X<>` 能用 | `[temp.res.general]¶6.3` |
| `if constexpr` 丢弃分支有不依赖模板参数的非法语句 | 见 ch04/25 | `[temp.res.general]¶6.4` |
| 模板定义时即有不依赖模板参数的非法语法 | 类模板成员中使用了不存在的类型 | `[temp.res.general]¶6` |

---

## 三、如何避免

```cpp
// ❌ IFNDR：只有空包合法
template<typename... Ts>
struct X {
    X(Ts... args) : Var(0, args...) {}
    int Var;
};

// ✅ 正确做法：直接写非模板特化
struct X {
    X() : Var(0) {}
    int Var;
};

// ✅ 或用概念约束阻止非法实例化
template<typename... Ts>
    requires (sizeof...(Ts) == 0)
struct X {
    X(Ts... args) : Var(0, args...) {}
    int Var;
};
```

---

## 四、常见误区

| 误区 | 正解 |
|------|------|
| ❌ 编译通过 = 代码合法 | ✅ IFNDR 不要求编译器报错，通过不代表合法 |
| ❌ 空包合法说明模板定义没问题 | ✅ 模板定义本身有问题——它设计成了一个只有空包才能用的陷阱 |
| ❌ IFNDR 的程序一定会崩溃 | ✅ 可能完全正常运行，这正是它最危险的地方 |
| ❌ IFNDR 只影响变参模板 | ✅ `if constexpr` 场景也会触发 |
