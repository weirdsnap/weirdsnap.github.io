# ADL（参数依赖查找 / Koenig 查找）

ADL（Argument-Dependent Lookup），也叫 Koenig 查找，是 C++ 名称查找的一条核心规则——**如果函数调用的实参类型在某个命名空间内，那么该命名空间也会被纳入查找范围**。

---

## 一、一个经典的 ADL 面试题

```cpp
#include <iostream>

namespace x {
  class C {};
  void f(const C& i) {
    std::cout << "1";
  }
}

namespace y {
  void f(const x::C& i) {
    std::cout << "2";
  }
}

int main() {
  f(x::C());   // 输出什么？
}
```

**输出是：`1`**

## 为什么？

`f(x::C())` 是一个**非限定函数调用**（前面没有 `x::` 或 `y::`），正常情况下列表里找不到 `f` 应该编译失败。但：

1. 实参 `x::C()` 的类型是 `x::C`
2. ADL 将**实参类型所在的命名空间**加入查找范围 → 加入 `namespace x`
3. 在 `namespace x` 中找到 `x::f`
4. 调用 `x::f`，输出 `1`

`y::f` 虽然参数类型匹配，但 `y` 不是 `x::C` 的关联命名空间，因此不会被 ADL 发现。

---

## 二、ADL 的标准规则

C++23 标准 `[basic.lookup.argdep]¶1`：

> 当函数调用中的后缀表达式是一个**非限定标识（unqualified-id）**，并且普通非限定查找找不到任何声明时，查找还会包括实参类型关联命名空间中的 ADL 查找。

### 关联实体包括哪些？

`[basic.lookup.argdep]¶3`：对于类类型 `T`，其关联实体包括：
- 类本身
- 类所在的直接外围命名空间（以及其 inline 命名空间）
- 类的直接和间接基类
- 如果类有成员类型，那些成员类型的关联命名空间

## 三、ADL 的日常影响

你不是每天都会显式用到 ADL，但它在很多地方**静默工作**：

### 1. `std::cout << "hello"` 能工作就是因为 ADL

```cpp
std::cout << "hello";   // 等价于 operator<<(std::cout, "hello")
```

`operator<<` 是一个函数调用，ADL 在 `std::cout` 的类型 `std::basic_ostream` 所在的 `std` 命名空间中找到了 `operator<<` 的重载。如果不加 ADL，你必须写 `std::operator<<(std::cout, "hello")`。

### 2. `std::swap(obj1, obj2)` 的常见写法

```cpp
using std::swap;   // 把 std::swap 引入当前作用域
swap(a, b);        // 如果 a/b 有自己的 swap（ADL），优先选；否则 fallback 到 std::swap
```

### 3. 范围 for 循环中的 `begin` / `end`

```cpp
for (auto x : container) { ... }
// 等价于调用 begin(container) 和 end(container)，ADL 自动查找
```

## 四、常见误区

| 误区 | 正解 |
|------|------|
| ❌ ADL 只查找函数所在的命名空间 | ✅ 它查找**实参类型**的关联命名空间，而不是函数本身的命名空间 |
| ❌ 非限定调用找不到就不会编译 | ✅ ADL 会在普通查找失败后**追加**搜索关联命名空间 |
| ❌ ADL 总是主动参与查找 | ✅ 只有普通非限定查找**找不到任何声明**时，ADL 才会启用（如果找到则不启动） |
| ❌ 所有命名空间都会被搜一遍 | ✅ 只搜**实参类型关联的那些命名空间**，不是所有 |
| ❌ ADL 是 C++ 独有的奇怪特性 | ✅ 类似机制在 Rust（method lookup）、D、Nim 等语言中也存在 |

---

## 五、延伸点

- **ADL 的潜在陷阱**：如果普通查找找到的声明和 ADL 找到的声明有歧义，可能导致编译错误。常见例子是标准库的 `std::get` 和自定义的 `get` 冲突。
- **ADL 可以通过限定调用绕过**：`::f(x::C())`（全局作用域限定）或 `y::f(x::C())` 会跳过 ADL，只搜指定命名空间。
- C++20 的 Concepts 与 ADL 交互时，约束检查也在关联命名空间中进行。
