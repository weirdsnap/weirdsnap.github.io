# 返回局部/临时变量的指针/引用

局部自动存储期对象在函数返回时销毁，返回其指针或引用将导致悬垂，访问即 UB。

**场景示例：**
```cpp
int* bad_ptr() {
    int local = 42;
    return &local;  // 悬垂指针
}
```

**原理解析 + 常见误区**

- ❌ 悬垂引用一定会崩溃 — 错误。可能暂时"正常"运行，是最危险的 bug。
- ❌ 按值返回 `std::string` 有拷贝开销 — 错误。C++17 起 guaranteed copy elision，且优先移动语义。

**延伸点**：Meyers' Singleton 利用静态局部变量返回引用，是线程安全单例的标准做法（C++11 起初始化线程安全）。
