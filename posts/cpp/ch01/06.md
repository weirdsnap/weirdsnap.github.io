# `auto` / `decltype` / `decltype(auto)`

C++ 的类型推导有三种机制：`auto`（模板按值规则）、`decltype`（精确保留）、`decltype(auto)`（合二为一）。理解它们的区别是掌握现代 C++ 的基础。

---

## 一、`auto`：模板按值推导（退化）

`auto` 遵循**模板按值传递的推导规则**：剥离顶层 `const`/`volatile` 和引用，数组与函数退化为指针。

```cpp
const int ci = 42;
const int& cri = ci;
int arr[5] = {};
int foo(double);

auto a1 = ci;     // int（顶层 const 剥离）
auto a2 = cri;    // int（引用剥离）
auto a3 = arr;    // int*（数组退化）
auto a4 = foo;    // int(*)(double)（函数退化）
auto a5 = {1, 2}; // std::initializer_list<int>（花括号特殊规则）
```

| 表达式 | `auto` 推导结果 |
|--------|----------------|
| `const int` | `int` |
| `int&` | `int` |
| `const int&` | `int` |
| `int[5]` | `int*` |
| `int f(double)` | `int(*)(double)` |
| `{1, 2}` | `std::initializer_list<int>` |

---

## 二、`decltype`：精确保留

`decltype(expr)` 的规则取决于 `expr` 的语法形式：

| 情况 | 推导结果 | 示例 |
|------|---------|------|
| 无括号的标识符/成员访问 | 声明时的类型 | `decltype(i)` → `int` |
| 其他表达式，且是**左值** | `T&` | `decltype((i))` → `int&` |
| 其他表达式，且是**将亡值** | `T&&` | `decltype(std::move(i))` → `int&&` |
| 其他表达式，且是**纯右值** | `T` | `decltype(42)` → `int` |

```cpp
int i = 42;
int& r = i;
const int ci = 42;

decltype(i)     a = i;   // int（标识符）
decltype((i))   b = i;   // int&（表达式语境，i 是左值）
decltype(r)     c = i;   // int&（引用保留）
decltype(ci)    d = 42;  // const int（cv 保留）
decltype((ci))  e = ci;  // const int&（左值 + const）
```

**关键**：`decltype((i))` 的括号让 `i` 变成"表达式"，推导为 `int&`（左值引用），这是最常见的陷阱。

## 三、对比表格

| 维度 | `auto` | `decltype` | `decltype(auto)` |
|------|--------|------------|------------------|
| 引用性 | ❌ 去除 | ✅ 保留 | ✅ 保留 |
| 顶层 cv 限定 | ❌ 去除 | ✅ 保留 | ✅ 保留 |
| 数组/函数 | 退化为指针 | ✅ 保留原始类型 | ✅ 保留原始类型 |
| 花括号列表 | `initializer_list` | ❌ 不适用 | ❌ 不适用 |
| 推导规则 | 模板按值 | 精确类型 | 同 `decltype` |
| 引入版本 | C++11 | C++11 | C++14 |

```cpp
const int arr[3] = {1, 2, 3};

auto a = arr;           // const int*（退化 + const 剥离 → int*）
decltype(arr) d = {};   // const int[3]（精确保留）
decltype(auto) da = arr; // const int[3]（同 decltype）
```

## 四、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `auto` 保留引用 — ✅ `auto a = ref;` 中 `a` 是独立副本 |
| ❌ 表达式总是推导为左值引用 — ✅ 左值→`T&`，将亡值→`T&&`，纯右值→`T` |
| ❌ `decltype(auto)` 只是 `auto` 的增强版 — ✅ 二者推导规则完全不同 |
| ❌ `decltype(auto) x = (local);` 是安全的 — ⚠️ 括号使 `(local)` 成为左值表达式，推导为左值引用，可能导致悬垂 |
| ❌ `auto& b = x;` 能绑定 `const int` — ✅ `auto` 推导为 `int`，`b` 变成 `int&`，无法绑定 `const int` |

---

## 五、延伸点

- `decltype(auto)` 是完美转发返回值的核心工具：
  ```cpp
  template<typename F, typename... Args>
  decltype(auto) invoke(F&& f, Args&&... args) {
      return std::forward<F>(f)(std::forward<Args>(args)...);
  }
  ```
- 警惕 `return (local);` 的括号陷阱——正确写法是 `return local;`，括号会导致 `decltype(auto)` 推导为引用。
- `auto&&` 不是右值引用，而是**转发引用**（万能引用），配合 `decltype(auto)` 实现完美转发。
