# 无符号整型回绕

无符号整数算术运算按模 2^N 回绕，不引发异常；有符号整数溢出才是未定义行为（UB）。

| 类型 | 溢出行为 | 标准规定 |
|------|----------|----------|
| 无符号整数 | 模 2^N 回绕 | 定义良好 |
| 有符号整数 | 溢出 | 未定义行为（UB） |

**原理解析 + 常见误区**

- ❌ "有符号溢出也会回绕" — 错误。有符号溢出是 UB，编译器可基于"不会溢出"假设做激进优化。
- ❌ "无符号回绕总是坏事" — 错误。环形缓冲区索引利用回绕是安全且高效的做法。
- ✅ 向下递减循环 `for (unsigned i=10; i>=0; --i)` 会因回绕导致无限循环。

**延伸点**：编译器可基于有符号整数不会溢出的假设删除"不可能到达"的代码路径。例如：

```cpp
bool check_overflow(int x) {
    if (x + 1 < x) {   // 程序员想检查加法是否溢出
        return true;   // 编译器认为有符号加法不会溢出，
                       // 所以 x + 1 < x 永远为假，整段被优化删除
    }
    return false;
}
```

在开启优化的编译器（如 GCC/Clang `-O2`）下，`x + 1 < x` 会被判定为恒假，函数直接返回 `false`。这意味着你写的安全检查实际上不会生效，是安全代码审计中非常隐蔽的陷阱。需要检查溢出时应使用无符号类型、范围更大的类型，或编译器内置函数如 `__builtin_add_overflow`。
> 相关阅读：表达式中的 `unsigned short` 会先经过[整数提升](../28.md)变成 `int` 或 `unsigned int`，并不总是以 `unsigned short` 的宽度运算。
