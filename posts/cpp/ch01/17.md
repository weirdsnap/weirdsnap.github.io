# Lambda 捕获：值、引用、`this` 和陷阱

Lambda 是 C++ 里一种匿名函数对象。写法上像函数，但本质上是一个编译器生成的类对象，它会把你写在方括号里的变量打包成成员变量。这一章重点讲捕获的各种方式和容易踩的坑。

---

## Lambda 的基本形式

```cpp
[捕获列表](参数列表) -> 返回类型 { 函数体 }
```

比如：

```cpp
int x = 10;
auto f = [x](int y) { return x + y; };
std::cout << f(5);  // 15
```

这里的 `[x]` 就是捕获列表，表示 lambda 内部可以使用外部变量 `x`。

---

## 值捕获：复制一份进去

### 基本行为

```cpp
int x = 10;
auto f = [x]() { return x; };
x = 20;
std::cout << f();  // 10，不是 20
```

值捕获相当于把 `x` 复制了一份到 lambda 生成的闭包对象里。外面 `x` 再怎么改，lambda 手里的副本不变。

### 默认不能修改副本

```cpp
int x = 10;
auto f = [x]() {
    x = 20;  // ❌ 编译错误
};
```

默认情况下，lambda 的 `operator()` 是 `const` 成员函数，按值捕获的成员变量在函数里表现得像 `const`。这和类的普通成员函数规则一样。

### 用 mutable 解除 const

```cpp
int x = 10;
auto f = [x]() mutable {
    x = 20;  // ✅ 修改的是闭包内部副本
    return x;
};

std::cout << f();  // 20
std::cout << x;    // 10，外部 x 没变
```

`mutable` 只是允许修改闭包内部副本，不影响外部原变量。

### 按值捕获指针：指针本身是 const，指向的内容不是

```cpp
int* p = new int(10);
auto f = [p]() {
    // p = nullptr;  // ❌ p 是 const，不能改指向
    *p = 20;         // ✅ 但 p 指向的内容可以改
};
```

这是值捕获指针最容易混淆的地方。复制的是指针变量本身，不是它指向的内存。所以指针的指向不能变，但指向的数据可以变。

---

## 引用捕获：用的是外面的变量本身

### 基本行为

```cpp
int x = 10;
auto f = [&x]() {
    x = 20;  // ✅ 修改的就是外部 x
};
f();
std::cout << x;  // 20
```

引用捕获不复制变量，lambda 内部用的是外部变量的别名。修改 lambda 里的 `x`，外部也会变。

### 最大陷阱：悬垂引用

```cpp
auto make_lambda() {
    int x = 10;
    return [&x]() { return x; };  // ❌ x 是局部变量，函数返回后就销毁了
}

auto f = make_lambda();
std::cout << f();  // 未定义行为，可能崩溃或读出垃圾
```

按引用捕获局部变量，然后让 lambda 逃出变量的作用域，是 lambda 相关 bug 里最常见的一种。值捕获就不会有这个风险：

```cpp
auto make_lambda() {
    int x = 10;
    return [x]() { return x; };  // ✅ 闭包里保存了 x 的副本
}
```

## 隐式捕获：`= ` 和 `&`

如果不想逐个写变量，可以用隐式捕获：

```cpp
int a = 1, b = 2;

auto f1 = [=]() { return a + b; };   // 所有外部变量按值捕获
auto f2 = [&]() { a++; b++; };       // 所有外部变量按引用捕获
```

也可以混合使用：

```cpp
int a = 1, b = 2;
auto f = [a, &b]() {  // a 按值，b 按引用
    // a = 10;  // ❌
    b = 20;     // ✅
};
```

隐式捕获省事，但容易不小心捕获太多东西。比如 `[=]` 会复制所有可见局部变量，如果其中有大对象，闭包会变重。

## 捕获 `this`

### 成员函数里的 lambda 默认捕获 `this`

```cpp
class Foo {
public:
    int value = 42;

    void print() {
        auto f = [this]() {
            std::cout << value << "\n";  // 等价于 this->value
        };
        f();
    }
};
```

在成员函数里写 lambda，如果要访问成员变量，必须捕获 `this`。捕获 `this` 后，lambda 持有的是对象的指针。

### 捕获 this 的陷阱

```cpp
auto Foo::get_lambda() {
    return [this]() { return value; };  // 持有 this 指针
}

int main() {
    auto f = Foo().get_lambda();  // Foo 临时对象马上销毁
    std::cout << f();             // ❌ this 指向已经销毁的对象
}
```

这和按引用捕获局部变量的风险一样：lambda 逃出对象作用域后，`this` 变成悬垂指针。

### C++17：用 *this 按值捕获整个对象

```cpp
class Foo {
public:
    int value = 42;

    auto get_lambda() {
        return [*this]() { return value; };  // ✅ 复制整个对象到闭包
    }
};
```

C++17 之前只能 `[this]`，捕获指针。C++17 引入 `[*this]`，可以把对象本身复制进闭包，避免悬垂问题。代价是多一次拷贝。

## 全局变量：不需要捕获，也不能捕获

```cpp
int a = 1;

int main() {
    auto f = [](int b) { return a + b; };
    std::cout << f(4);  // 5
}
```

全局变量不需要出现在捕获列表里，lambda 内部可以直接使用。而且按照 C++ 标准，**只有局部变量才能被捕获**。全局变量根本不能出现在捕获列表中：

> The body of a lambda-expression may refer to local entities of enclosing block scopes by capturing those entities.（只有封闭块作用域中的局部实体才能被捕获）
>
> — §[expr.prim.lambda.capture]¶1, C++23

所以上面代码里的 `a` 不是被捕获的变量，而是直接引用全局变量 `a`。如果你写 `[a]` 试图捕获全局变量，会编译报错。

```cpp
int a = 1;

int main() {
    auto f = [a](int b) { return a + b; };  // ❌ 编译错误：a 不是局部变量
}
```

### 需要注意什么

- 多线程环境下修改全局变量要考虑同步
- lambda 里引用的全局变量就是它本身，不是副本。如果 lambda 异步执行时全局变量被修改，结果也会变
- 静态局部变量和全局变量一样，不需要也不能捕获

---

## 捕获规则快速对照

| 捕获方式 | 写法 | 是否复制 | 能否修改外部变量 | 主要风险 |
|----------|------|----------|------------------|----------|
| 值捕获 | `[x]` | 是 | 否（mutable 只能改副本） | 闭包体积大 |
| 引用捕获 | `[&x]` | 否 | 是 | 悬垂引用 |
| 隐式值捕获 | `[=]` | 是 | 否 | 捕获过多、闭包大 |
| 隐式引用捕获 | `[&]` | 否 | 是 | 容易意外修改外部变量 |
| 捕获 this | `[this]` | 否（存指针） | 能改成员 | this 悬垂 |
| 按值捕获对象 | `[*this]` | 是 | 否（改的是副本） | 拷贝开销 |

---

## 几个常见误区

1. **按值捕获的变量就是 const**
   - 不精确。是闭包对象的成员变量在 const 成员函数中表现为 const。加 `mutable` 可以改副本。

2. **mutable 影响外部原变量**
   - 错误。`mutable` 只影响闭包内部副本。

3. **按值捕获指针就安全了**
   - 错误。复制的是指针变量，不是指向的内存。如果指向的对象销毁了，lambda 里解引用还是 UB。

4. **`[=]` 最安全**
   - 不一定。它复制所有可见局部变量，可能复制大对象；而且如果 lambda 异步执行，副本是最新的快照，不一定符合预期。

5. **成员函数里的 lambda 自动能访问成员变量**
   - 错误。必须显式捕获 `this` 或 `*this`。

---

## 一句话建议

- 局部变量要逃出作用域 → 用值捕获
- 临时/一次性使用，想改外部变量 → 用引用捕获
- 成员函数里想长期持有 → 用 `[*this]`（C++17）
- 不确定该用什么 → 先写值捕获，发现需要改外部或对象太大再调整
