# 声明 vs 定义 vs 初始化

声明（declaration）引入名字和类型，定义（definition）分配存储并提供实现，初始化（initialization）赋予对象初始值。三者是 C++ 分离编译模型的基础。

---

## 一、核心区别

| 概念 | 做了什么事 | 能否出现多次 |
|------|-----------|-------------|
| **声明** | 告诉编译器"有这么个东西" | ✅ 可以多次 |
| **定义** | 分配存储/提供实现 | ❌ 只能一次（ODR） |
| **初始化** | 给定义的对象赋予初始值 | ✅ 定义时一次性 |

在 C++ 中，**大多数时候声明和定义是合一的**——你写一个变量或函数，它既是声明也是定义：

```cpp
int a;               // 声明 + 定义：告诉编译器"有一个 int 叫 a"，同时分配存储
int b = 42;          // 声明 + 定义 + 初始化：分配存储并给初值
void foo() {}        // 声明 + 定义：提供了函数体
```

纯声明（只引入名字，不分配存储）是一个较少见的特殊情况，用 `extern` 标记：

```cpp
extern int x;        // 纯声明：告诉编译器 x 存在，但存储在另一个文件中
                     // 没有分配存储，没有初始化
```

## 二、声明 vs 定义

### 变量

```cpp
extern int a;        // 声明（不分配存储）
int a;               // 定义（分配存储，零初始化）
int a = 5;           // 定义 + 初始化

extern int a = 5;    // ❌ 语法错误：extern + 初始化 = 定义
```

### 函数

```cpp
int foo(int x);      // 声明（函数原型）
int foo(int x) {     // 定义（函数体）
    return x * 2;
}
```

### 类

```cpp
class Widget;        // 前向声明（可用于指针/引用，不能定义对象）
class Widget {       // 定义（完整类型定义）
    int x;
};
```

## 三、ODR（One Definition Rule）

C++ 最核心的规则之一：**每个变量、函数、类在整个程序中只能有一个定义**。

```cpp
// a.cpp
int counter = 0;     // 定义

// b.cpp
int counter = 0;     // ❌ 链接错误：重复定义
```

### ODR 的例外

| 实体 | 规则 | 说明 |
|------|------|------|
| `inline` 变量/函数 | 可出现在多个翻译单元 | 必须完全相同定义 |
| 类/模板/内联函数 | 可出现在多个翻译单元 | 必须完全相同定义 |
| `static` 全局变量 | 每个翻译单元可以有自己的定义 | 文件作用域 |

```cpp
// header.h
inline int version = 2;        // inline 变量，跨翻译单元共享
static int local_counter = 0;  // static，每个 cpp 有自己的副本

// a.cpp
#include "header.h"
// version 是同一个，local_counter 是 a.cpp 自己的

// b.cpp
#include "header.h"
// version 是同一个，local_counter 是 b.cpp 自己的
```

## 四、`extern` 关键字

`extern` 告诉编译器"这个变量/函数在其他地方定义了"：

```cpp
// a.cpp
int global = 42;

// b.cpp
extern int global;   // 声明引用 a.cpp 中的 global
void func() {
    global = 100;    // 修改 a.cpp 中的 global
}
```

### `extern "C"`：C 与 C++ 的链接约定

C++ 支持函数重载（name mangling），C 不支持。`extern "C"` 告诉编译器用 C 的链接方式：

```cpp
extern "C" {
    void c_function(int x);     // C 链接，不 mangling
    int c_variable;
}

// 包含 C 标准头文件时通常这样写：
extern "C" {
    #include <stdio.h>          // printf 等函数以 C 方式链接
}
```

### `extern "C"` 下不能用的 C++ 特性

```cpp
extern "C" {
    void foo(int);
    void foo(double);   // ❌ 编译错误：C 链接不允许重载

    void bar(int&);     // ❌ 编译错误：C 不支持引用参数

    void baz(int = 0);  // ❌ 编译错误：C 不支持默认参数

    struct S { void f(); };  // ✅ 结构体定义允许
    // void S::f() {}        // ❌ 成员函数不能有 C 链接
}
```

### 函数声明 vs 定义的链接差异

`extern "C"` 只影响函数的**链接约定（calling convention + name mangling）**，不影响函数的参数类型。一个 C 链接的函数在 C 编译器看来参数类型仍然是 C++ 的类型系统，所以：

```cpp
extern "C" void process(const int& x);  // ❌ C 没有引用，但 C++ 编译器仍会解释参数类型
// 实际效果：C++ 编译器仍然理解 const int&，但用 C 的方式暴露给链接器
// 这导致 C 代码无法直接调用这个函数，因为 C 不知道引用是什么
```

正确的 C/C++ 互操作接口应该只用 C 兼容的类型：

```cpp
extern "C" {
    void process_int(int* x);     // ✅ 用指针代替引用
    void process_data(const char* data, size_t len);  // ✅ C 兼容
}
```

`extern "C++"` 则显式指定 C++ 链接（默认就是 C++，通常不需要写）。

## 五、初始化的三种形式

```cpp
int a;              // 默认初始化（静态变量零初始，局部变量不确定）
int b = 42;         // 拷贝初始化
int c(42);          // 直接初始化
int d{42};          // 列表初始化（C++11，推荐）
```

初始化和赋值的区别：

```cpp
int x = 42;         // 初始化：构造时确定值
x = 99;             // 赋值：构造完成后修改值
```

详见 [ch01/31 Most Vexing Parse](../31.md) 中对默认初始化 vs 值初始化的讨论。

## 六、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `extern int x;` 是变量定义 | ✅ 只是声明，不分配存储，除非同时有初始化器 |
| ❌ 头文件中定义变量是安全的 | ✅ 非 inline 变量在头文件中定义会导致多翻译单元链接时重复定义 |
| ❌ `static int x;` 在头文件中是安全的 | ✅ 每个包含该头文件的 cpp 会得到独立副本，不会链接冲突 |
| ❌ `extern "C"` 块内可以写任意 C++ 代码 | ✅ 不能出现重载、模板等依赖 name mangling 的特性 |
| ❌ `int x = 42;` 是赋值 | ✅ 这是初始化（拷贝初始化），不是赋值 |

---

## 七、一句话总结

> 声明引入名字（`extern int x;`），定义分配存储（`int x;`），初始化给初值（`int x = 42;`）。ODR 规定每个实体只能有一个定义，`extern "C"` 告诉编译器采用 C 的链接约定。
