# 声明 vs 定义 vs 初始化

声明（declaration）引入名字和类型，定义（definition）分配存储并提供实现，初始化（initialization）赋予对象初始值。这三者是理解 C++ 编译模型的基础。

---

## 一、核心区别

| 概念 | 做的事 | 能否出现多次 |
|------|-------|-------------|
| **声明** | 告诉编译器"有这么个东西" | ✅ 可以多次 |
| **定义** | 分配存储 / 提供实现 | ❌ 只能一次（ODR） |
| **初始化** | 给定义的对象赋予初始值 | ✅ 定义时一次性 |

---

## 二、声明和定义的关系

### 默认情况：声明即定义

在 C++ 中，**大多数时候声明和定义是合一的**——你写一个变量或函数，它既是声明也是定义：

```cpp
int a;               // 声明 + 定义：告诉编译器"有一个 int 叫 a"，同时分配存储
int b = 42;          // 声明 + 定义 + 初始化：分配存储并给初值
void foo() {}        // 声明 + 定义：提供了函数体
```

### 纯声明：只引入名字，不分配存储

纯声明用 `extern` 标记，告诉编译器"这个名字在其他地方定义了"：

```cpp
extern int x;        // 纯声明：告诉编译器 x 存在，但不分配存储
```

`extern` 是较少见的特殊情况——日常写代码时，大部分变量定义同时也是声明。

### 为什么需要区分两者？

C++ 采用"单独编译 + 链接"模型：

```
a.cpp → 编译器 → a.o ─┐
                       ├─→ 链接器 → 可执行文件
b.cpp → 编译器 → b.o ─┘
```

编译器编译 a.cpp 时不知道 b.cpp 里有什么。如果 a.cpp 要用 b.cpp 定义的变量，就需要先告诉编译器"这个变量存在，类型是 int"——这就是**纯声明**的作用：

```cpp
// a.cpp
extern int global;   // 声明：编译器知道 global 存在，但不分配存储
int main() {
    return global;    // 链接器会去 b.o 里找 global 的定义
}

// b.cpp
int global = 42;     // 定义：实际分配存储
```

## 三、声明 vs 定义的具体规则

### 变量

```cpp
extern int a;        // 纯声明（不分配存储）
int a;               // 声明 + 定义（分配存储）
int a = 5;           // 声明 + 定义 + 初始化

extern int a = 5;    // ❌ 语法错误：extern + 初始化 = 定义
                     //（有初始化器就意味着要分配存储，与 extern 矛盾）
```

### 函数

```cpp
int foo(int x);      // 声明（函数原型）
int foo(int x) {     // 定义（函数体）
    return x * 2;
}
```

函数声明让编译器在没有看到函数体时就**知道怎么调用它**：

```cpp
void bar();          // 声明
bar();               // ✅ 即使 bar 定义在后面，这里也能正确调用
void bar() {}        // 定义
```

### 类

```cpp
class Widget;        // 前向声明（可用于指针/引用，不可定义对象）
class Widget {       // 定义（完整类型定义）
    int x;
};
```

## 四、初始化

定义之后不一定要马上初始化，但**不初始化就访问是未定义行为**：

```cpp
int x;          // 定义（静态变量自动零初始，局部变量值不确定）
int y = 42;     // 定义 + 初始化
int z{42};      // 定义 + 列表初始化
```

### 初始化 vs 赋值

关键在于**构造的时间点**：

```cpp
int x = 42;     // 初始化：对象构造时确定值
x = 99;         // 赋值：对象构造完成后修改值
```

对于类类型，区别尤其明显：

```cpp
struct S {
    S() { std::cout << "构造\n"; }
    S& operator=(int) { std::cout << "赋值\n"; return *this; }
};

S s;            // ✅ 初始化：调用构造函数
s = 42;         // ✅ 赋值：构造函数已完成，调用 operator=
```

### ODR（One Definition Rule）

C++ 核心规则：**每个变量、函数、类在整个程序中只能有一个定义**。

```cpp
// a.cpp
int counter = 0;     // 定义

// b.cpp
int counter = 0;     // ❌ 链接错误：重复定义
```

ODR 的例外：

| 实体 | 规则 |
|------|------|
| `inline` 变量/函数 | 可在多个翻译单元出现，必须完全相同 |
| 类/模板 | 可在多个翻译单元出现，必须完全相同 |
| `static` 全局变量 | 每个翻译单元有自己的独立副本 |

## 五、`extern` 的两种角色

### 角色 1：纯声明（跨文件引用）

最常见的用法——在 b.cpp 定义，在 a.cpp 声明引用：

```cpp
// b.cpp
int global = 42;

// a.cpp
extern int global;   // 声明引用 b.cpp 中的 global
void func() {
    global = 100;
}
```

### 角色 2：`extern "C"` 改变链接约定

C++ 支持函数重载（name mangling），C 不支持。`extern "C"` 告诉编译器用 C 的链接方式，让 C++ 函数能被 C 代码调用：

```cpp
extern "C" {
    void c_function(int x);     // C 链接，不 name mangling
}
```

`extern "C"` 块内不能使用 C 不支持的语法（重载、引用参数、默认参数等）。

## 六、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `extern int x;` 是定义 | ✅ 是纯声明，不分配存储。`int x;`（无 extern）才是定义 |
| ❌ 声明和定义总是分开的 | ✅ 大多数情况声明即定义，纯声明只在跨文件引用时需要 |
| ❌ 头文件中定义变量是安全的 | ✅ 非 inline 变量在头文件中定义，多个 cpp 包含后链接时重复定义 |
| ❌ `int x = 42;` 是赋值 | ✅ 这是初始化，不是赋值。赋值发生在对象构造完成之后 |
| ❌ 不初始化的变量会自动归零 | ✅ 只有静态存储期变量会，局部变量不会，访问它是 UB |

---

## 七、一句话总结

> 声明引入名字（告诉编译器"有这么个东西"），定义分配存储，初始化给初值。默认声明即定义，纯声明（`extern`）只在跨文件引用时需要。
