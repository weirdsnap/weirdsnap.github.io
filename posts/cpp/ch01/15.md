# 隐式转换 `bool` / `operator bool`

条件判断中编译器隐式将表达式转换为 `bool`；类通过 `operator bool` 定义转换，现代 C++ 应始终用 `explicit`。

| 场景 | `explicit operator bool` 是否可用 |
|------|----------------------------------|
| `if`/`while`/`for`/`do-while` | ✅ 允许 |
| `&&`/`||`/`!` | ✅ 允许 |
| 算术比较/赋值 | ❌ 阻止 |

**原理解析 + 常见误区**

- ❌ `operator bool` 不加 `explicit` 更安全 — 错误。非 explicit 时 `bool` 可提升为 `int`，导致 `obj == 1`、`obj + 2` 等荒谬代码合法。
- ❌ `explicit operator bool` 在 `if` 中不能用 — 错误。条件语境是显式转换的例外。

**延伸点**：C++11 前使用 Safe Bool Idiom（`operator void*`）避免隐式转 `int`；`explicit operator bool` 彻底取代该惯用法。
