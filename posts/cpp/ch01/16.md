# 可调用对象：函数指针、仿函数、Lambda、std::function

C++ 中有四种方式表达"一个可调用的东西"，它们的表现力和性能各有取舍。

---

## 一、函数指针

最基础、最轻量的可调用对象——只是一个地址。

```cpp
int add(int a, int b) { return a + b; }

int (*fp)(int, int) = &add;
std::cout << fp(3, 4);   // 7
```

| 特性 | 说明 |
|------|------|
| 状态 | ❌ 无状态，只是一个指针 |
| 调用方式 | 间接调用（通过指针跳转） |
| 可内联 | ❌ 编译器通常无法内联 |
| 类型 | 精确类型 `int(*)(int,int)`，不同签名不兼容 |
| 捕获 | ❌ 不能捕获上下文变量 |

函数指针可以作为非类型模板参数：

```cpp
template<int (*F)(int)>
void apply(int x) { std::cout << F(x); }
apply<&square>(5);  // 编译期确定函数地址，可内联
```

---

## 二、仿函数（Function Object / Functor）

重载了 `operator()` 的类，可以有状态，编译器可以内联。

```cpp
struct Adder {
    int offset;
    Adder(int o) : offset(o) {}
    int operator()(int x) const { return x + offset; }
};

Adder add5(5);
std::cout << add5(10);  // 15，编译器可内联展开
```

| 特性 | 说明 |
|------|------|
| 状态 | ✅ 成员变量保存状态 |
| 调用方式 | 直接调用，可内联 |
| 泛型 | ✅ 可以是类模板，或 `operator()` 模板 |
| 语法 | 需要单独定义类，比较重 |

仿函数与函数指针的核心差异：**有无状态 + 能否内联**。

```cpp
// 函数指针——间接调用，无法内联
auto fp = &some_function;
fp(x);

// 仿函数——直接调用，编译器知道具体类型，可内联
F f;
f(x);
```

## 三、Lambda（C++11 起）

Lambda 是**匿名仿函数的语法糖**。编译器为每个 lambda 生成一个独一无二的类，重载 `operator()`。

```cpp
auto add = [](int a, int b) { return a + b; };
// 编译器生成类似：
// struct __lambda_1 {
//     int operator()(int a, int b) const { return a + b; }
// };

auto add5 = [offset = 5](int x) { return x + offset; };
// 捕获列表 = 仿函数的成员变量
```

### 捕获方式

```cpp
int x = 42;
auto by_val   = [x]() { return x; };        // 按值捕获
auto by_ref   = [&x]() { x++; };            // 按引用捕获
auto mixed    = [x, &y]() { /* ... */ };    // 混合
auto all_val  = [=]() { /* ... */ };        // 全部按值（C++14 起不推荐）
auto all_ref  = [&]() { /* ... */ };        // 全部按引用
auto move     = [ox = std::move(x)]() {};   // C++14 移动捕获
```

### Lambda 的类型

每个 lambda 的类型是**独一无二且编译器生成的**，不能直接写出它的类型名。只能用 `auto` 或 `std::function` 存储：

```cpp
auto f1 = [] { return 42; };
auto f2 = [] { return 42; };   // f1 和 f2 是不同类型！

// 想写出类型
std::function<int()> f = [] { return 42; };  // ✅ 类型擦除，有开销
```

| 特性 | 说明 |
|------|------|
| 状态 | ✅ 通过捕获列表 |
| 调用方式 | 直接调用，可内联 |
| 类型 | 唯一匿名类型，不可写出来 |
| 开销 | 无捕获：和函数指针一样；有捕获：和手写仿函数一样 |
| 便捷性 | ⭐⭐⭐⭐⭐ 最推荐 |

## 四、`std::function`（C++11 起）

类型擦除的包装器，可以存储任意可调用对象（函数指针、Lambda、仿函数）。

```cpp
#include <functional>

std::function<int(int, int)> f;

f = add;                    // 存函数指针
f = [](int a, int b) { return a * b; };  // 存 lambda
f = Adder(5);               // 存仿函数

if (f) {                    // 检查是否为空
    std::cout << f(3, 4);   // 调用
}
```

| 特性 | 说明 |
|------|------|
| 类型擦除 | ✅ 不同可调用对象签名相同即可赋值 |
| 可内联 | ❌ 多一层虚分派，无法内联 |
| 堆分配 | ⚠️ 大对象会堆分配（小对象有 SBO 优化） |
| 空调用 | 抛 `std::bad_function_call`，不是 UB |
| 适用场景 | 回调注册、事件系统、多态调用 |

## 五、四者对比

| 维度 | 函数指针 | 仿函数 | Lambda | std::function |
|------|---------|--------|--------|---------------|
| 状态 | ❌ | ✅ 成员变量 | ✅ 捕获列表 | ✅ 擦除后携带 |
| 可内联 | ❌ | ✅ | ✅ | ❌ |
| 性能 | 中等（间接跳转） | 最优（直接调用） | 同仿函数 | 最差（虚分派 + 可能堆分配） |
| 类型表达 | 精确写出 | 需定义类 | 匿名，只能用 auto | 签名统一（`std::function<int()>`） |
| 异构容器 | ❌ | ❌ | ❌ | ✅ |
| 使用便捷性 | ⭐⭐ | ⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 引入时间 | C 语言即有 | C++98 | C++11 | C++11 |

## 六、如何选择

| 场景 | 推荐方案 |
|------|---------|
| 简单的回调，无状态 | 函数指针（最轻量） |
| 需要状态，性能敏感 | Lambda 或自定义仿函数 |
| 需要泛型算法 | Lambda（STL 算法中的首选） |
| 异构容器 / 类型擦除 | `std::function` |
| 接口中存储回调 | `std::function` 或模板参数（万能引用） |
| 性能极致，不介意类型约束 | 模板 + 仿函数（编译期多态） |

```cpp
// 性能最优：模板 + Lambda
template<typename F>
void for_each(F f) { f(1); f(2); }
for_each([](int x) { /* ... */ });

// 运行时灵活：std::function
void register_callback(std::function<void()> cb);
register_callback([]{ /* ... */ });
```

---

## 七、一句话总结

> 函数指针无状态轻量，仿函数有状态可内联，Lambda 是写仿函数的语法糖，`std::function` 提供类型擦除但牺牲性能——按场景选最合适的。
