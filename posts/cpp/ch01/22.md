# `std::optional`

`std::optional<T>` 是 C++17 引入的类型安全包装器，显式表达"值可能存在也可能不存在"，替代魔数、`nullptr` 或裸指针表示缺失。

| 操作 | 行为 |
|------|------|
| `std::optional<int> o;` | 空状态（无值） |
| `o = 42;` | 持有值 |
| `if (o)` / `o.has_value()` | 检查是否存在 |
| `*o` / `o.value()` | 解引用获取值（空时 `value()` 抛异常） |
| `o.value_or(default)` | 有值返回值，无值返回默认值 |

```cpp
std::optional<int> divide(int a, int b) {
    if (b == 0) return std::nullopt;
    return a / b;
}

auto result = divide(10, 0);
if (!result) std::cout << "除零错误";
```

**原理解析 + 常见误区**

- ❌ `std::optional` 内部使用堆分配 — 错误。它是栈上的值类型，通常与 `T` 同尺寸（仅多一个字节/标志位）。
- ❌ `*o` 在空时返回默认值 — 错误。空 `optional` 解引用是 UB，应先用 `if (o)` 检查。
- ❌ `optional<T&>` 合法 — 错误。C++ 标准禁止 `optional` 持有引用，应使用 `optional<T*>` 或 `optional<std::reference_wrapper<T>>`。

**延伸点**：`std::optional` 和指针的区别：`T*` 可表示"无值"（`nullptr`），但也可指向任意位置；`optional<T>` 明确表达"这个值可能有也可能没有"，语义更清晰，且生命周期受 RAII 管理。
