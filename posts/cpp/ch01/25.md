# `enum class` vs `enum`

C++11 引入的 `enum class`（强类型枚举）将枚举值限制在自身作用域内，且禁止隐式转换为整数，避免了传统 `enum` 的命名污染和意外转换。

| 特性 | `enum`（传统） | `enum class`（强类型） |
|------|---------------|----------------------|
| 作用域 | 泄漏到外层命名空间 | 仅在枚举作用域内 |
| 隐式转 `int` | ✅ 允许 | ❌ 禁止，需显式 `static_cast` |
| 底层类型 | 实现定义 | 可显式指定（如 `: int`） |
| 前向声明 | ❌ 不可 | ✅ 可以 |

```cpp
enum Color { Red, Green, Blue };           // Red 泄漏到全局
enum class Color { Red, Green, Blue };     // 需用 Color::Red

int x = Red;        // OK（传统 enum）
int y = Color::Red; // 错误（enum class）
int z = static_cast<int>(Color::Red); // OK
```

**原理解析 + 常见误区**

- ❌ `enum class` 性能比 `enum` 差 — 错误。二者底层表示完全相同，只是编译期的类型检查更严格。
- ❌ 需要 `static_cast` 太麻烦 — 错误。这正是 `enum class` 的安全价值，防止 `if (color == 1)` 这类脆弱代码。
- ❌ `enum class` 不能指定底层类型 — 错误。`enum class Color : uint8_t { ... };` 完全合法。

**延伸点**：C++20 起 `enum class` 支持 `using enum`，可在局部作用域展开枚举值名：`using enum Color;` 后可直接写 `Red`，但仍保留强类型检查。
