# NULL、nullptr 与 void*

`NULL` 是 C 时代的宏，通常定义为 `0` 或 `(void*)0`；`nullptr` 是 C++11 引入的关键字，类型为 `std::nullptr_t`；`void*` 是通用对象指针类型，可以指向任何数据对象，但不能直接解引用。

| 特性 | `NULL` | `nullptr` | `void*` |
|------|--------|-----------|---------|
| 本质 | 宏（通常定义为 `0` 或 `(void*)0`） | C++11 关键字 | 指针类型 |
| 类型 | 实现相关，可能是 `int` 或 `void*` | `std::nullptr_t` | `void*` |
| 可转整型 | ✅ 可能（定义为 `0` 时） | ❌ 不会 | ❌ 不会 |
| 可转对象指针 | ✅ | ✅ | ⚠️ C 可隐式转，C++ 需显式转换 |
| 可解引用 | ❌ | ❌ | ❌ 不能直接解引用 |
| 重载安全 | ❌ 可能二义 | ✅ 安全 | ✅ 类型明确 |

## 示例

```cpp
void foo(int);
void foo(char*);

foo(NULL);      // ❌ 可能调用 foo(int)，产生歧义或不符合预期
foo(nullptr);   // ✅ 明确匹配 foo(char*)
foo(0);         // 调用 foo(int)

void* vp = malloc(100);   // ✅ malloc 返回 void*
char* cp = vp;            // ❌ C++ 中不允许隐式转换
char* cp2 = (char*)vp;    // ✅ 必须显式转换
```

## 原理解析

### `NULL` 的问题

`NULL` 的类型不固定。在大多数实现中：

```cpp
#define NULL 0
// 或者
#define NULL ((void*)0)
```

当 `NULL` 被定义为 `0` 时，它就是一个 `int` 字面量。如果函数同时接受整型和指针重载，编译器会优先匹配整型版本，导致意外行为。

### `nullptr` 的优势

`nullptr` 的类型是 `std::nullptr_t`，它只能转换为指针类型，不能转换为整数类型，因此不存在整型重载的歧义。

### `void*` 的特殊性

`void*` 是"无类型指针"，可以保存任何对象地址，但编译器不知道它指向什么类型：

- 不能对 `void*` 直接做 `++`、`*p` 等操作
- 在 C 中，`void*` 可以隐式赋值给任何对象指针
- 在 C++ 中，`void*` 必须显式转换为具体指针类型，这是 C++ 类型安全更严格的表现

## 常见误区

- ❌ `NULL` 和 `nullptr` 完全等价 — 错误。`NULL` 可能是整型 0，会导致重载二义。
- ❌ `nullptr` 可以赋值给 `int` — 错误。`int x = nullptr;` 编译失败。
- ❌ `foo(NULL)` 一定调用指针版本 — 错误。如果存在 `foo(int)`，可能优先匹配整型版本。
- ❌ C++ 中 `void*` 可以隐式转为 `char*` — 错误。C++ 要求显式转换。
- ❌ `void*` 可以指向函数 — 错误。`void*` 只能指向对象，函数指针需用具体函数指针类型或 `std::function`。

## 延伸点

C++11 起应始终使用 `nullptr` 表示空指针。仅在需要与 C 代码交互或维护旧代码时，才可能见到 `NULL`。`std::nullptr_t` 也可以用于函数重载，实现只接受空指针的特殊处理。`void*` 在底层编程（如内存池、序列化）中仍有价值，但现代 C++ 更推荐类型安全的方案如 `std::any` 或模板。
