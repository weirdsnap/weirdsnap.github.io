# 聚合与聚合初始化

```cpp
struct S {
    int one;
    int two;
    int three;
};

S s{1, 2};   // 只给 2 个值，3 个成员——合法吗？
```

合法。`s.one = 1`、`s.two = 2`，未显式初始化的 `s.three` 被值初始化为 `0`。

---

## 一、什么是聚合

C++ 标准 `[dcl.init.aggr]¶1` 定义了聚合的条件。

### 聚合的条件

```cpp
struct Aggregate {        // ✅ 聚合
    int x;
    double y;
};

struct NotAggregate {     // ❌ 不是聚合——有用户声明的构造函数
    int x;
    NotAggregate(int v) : x(v) {}
};
```

在 C++17 及更早版本中，一个类（或数组）是聚合，必须满足：

| 条件 | 说明 |
|------|------|
| 没有用户声明的构造函数 | 可以有 `= default` 或 `= delete` |
| 没有私有或受保护的非静态数据成员 | 可以有私有成员函数 |
| 没有基类 | C++17 起允许有 public 基类 |
| 没有虚函数 | 始终不允许 |

C++20 起聚合可以拥有用户声明的构造函数（但需满足特定条件），详见下文。

## 二、聚合初始化

聚合可以使用花括号 `{}` 进行聚合初始化：

```cpp
struct Point {
    int x;
    int y;
    int z;
};

Point p1{1, 2, 3};    // ✅ 完整初始化
Point p2{1, 2};        // ✅ 缺省初始化：p2.z = 0
Point p3{};            // ✅ 全缺省：p3.x = 0, p3.y = 0, p3.z = 0
```

### 缺省初始化器

当聚合初始化提供的值少于成员数量时，**未显式初始化的成员按声明顺序用空初始化列表 `{}` 初始化**（即值初始化，对内置类型是零初始化）：

```cpp
struct S {
    int one;
    int two;
    int three;
};

S s{1, 2};
// s.one = 1（显式）
// s.two = 2（显式）
// s.three = 0（隐式：int{} → 0）
```

如果成员有**默认成员初始化器**，则使用该初始化器代替 `{}`：

```cpp
struct S {
    int one = 10;
    int two;
    int three = 30;
};

S s{1};
// s.one = 1（显式覆盖默认值）
// s.two = 0（隐式：int{}）
// s.three = 30（默认成员初始化器）
```

## 三、为什么不是聚合就不能这样写？

如果 `S` 有自定义构造函数，就不再是聚合，不能用聚合初始化的缺省规则：

```cpp
struct S {
    int one;
    int two;
    int three;
    S(int v) : one(v) {}   // 用户声明了构造函数
};

S s{1, 2};    // ❌ 编译错误：没有匹配的构造函数
```

此时 `S` 不是聚合，`{1, 2}` 会尝试匹配构造函数，但 `S(int)` 只接受一个参数，编译失败。

## 四、C++ 各版本对聚合的扩展

| 版本 | 变化 |
|------|------|
| C++11 | 聚合不能有用户声明的构造函数、不能有私有成员、不能有基类、不能有虚函数 |
| C++14 | 聚合可以有默认成员初始化器 |
| C++17 | 聚合允许有 public 基类（聚合基类） |
| C++20 | 聚合可以拥有用户声明的构造函数（但只有满足特定条件的构造函数才允许） |
| C++20 | 聚合初始化支持指定初始化器 `Point p{.x = 1, .y = 2}` |

## 五、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `S s{1, 2}` 必须提供所有成员的初始值 | ✅ 聚合初始化允许缺省，未提供的成员用 `{}` 初始化 |
| ❌ 不完整的初始化是未定义行为 | ✅ 是定义良好的，未提供的成员被零初始化 |
| ❌ `S s{1, 2}` 会报编译警告 | ✅ 大多数编译器默认不警告，需 `-Wextra` 才警告 |
| ❌ 私有成员不影响聚合状态 | ✅ 有私有非静态数据成员的类不是聚合 |
| ❌ C++20 后所有类都是聚合 | ✅ C++20 放宽了条件，但不是所有类都能成为聚合 |

---

## 六、一句话总结

> 聚合初始化允许提供少于成员数量的初始值，未显式初始化的成员按值初始化（内置类型零初始化）——这是聚合区别于普通类的重要行为。
