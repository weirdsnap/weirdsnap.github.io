# C++ 语言基础与类型系统

## 一、类型系统的核心哲学

C++ 的类型系统建立在三个相互制衡的哲学之上：

| 哲学 | 含义 | 工程体现 |
|------|------|----------|
| **零开销抽象** | 不为不使用的特性付出代价 | `std::vector` 与手写数组性能等价 |
| **静态类型安全** | 错误尽可能在编译期捕获 | 模板编译失败、Concepts 约束 |
| **值语义与对象模型** | 明确区分对象的身份与值 | 拷贝/移动/引用语义的区别 |

**关键认知**：C++ 的类型不仅是数据的"形状描述"，更是**编译期契约**和**运行时内存布局**的双重载体。

## 二、基础类型体系

### 2.1 标量类型

```
整数家族：bool, char/short/int/long/long long（及 unsigned 版本）
浮点家族：float, double, long double
特殊标量：void, nullptr_t, std::byte
```

**工程注意**：
- `char` 的符号性由实现定义（详见 [char 的符号性](09.md)）
- `long` 在 Windows 为 32 位，在 Linux x64 为 64 位——跨平台代码应使用 `int32_t`/`int64_t`
- 字符串/数值字面量有默认类型（详见 [字面量的默认类型](20.md)、[字符串字面量](08.md)）

### 2.2 算术转换与回绕

```
long double > double > float > unsigned long long > long long > unsigned long > long > unsigned int > int
```

`unsigned int` 与 `int` 运算时，`int` 会提升为 `unsigned`，导致负数变极大正数（详见 [无符号整型回绕](01.md)）。

## 三、复合类型与类型修饰

### 3.1 类型修饰符

从右向左读声明（括号优先）：

```cpp
int* const p;           // const 指针，指向 int
const int* p;           // 指向 const int
int (*p)[10];           // 指向数组的指针
int (*f)(int, int);     // 函数指针
```

**现代建议**：优先使用 `using` 别名：
```cpp
using FuncPtr = int(*)(int, int);
```

### 3.2 引用类型

| 类型 | 绑定目标 | 可重新绑定 | 典型用途 |
|------|----------|------------|----------|
| `T&` | 左值 | 否 | 函数参数，避免拷贝 |
| `T&&` | 右值 | 否 | 移动语义，完美转发 |
| `const T&` | 左值/右值 | 否 | 只读传参 |

- `auto` 推导遵循模板按值规则，会剥离引用和顶层 const（详见 [auto 推导](16.md)）
- `decltype` 保留表达式的精确类型（详见 [decltype 推导](07.md)）
- `decltype(auto)` 保留引用和 cv（详见 [decltype(auto)](17.md)）
- `const` 引用绑定到需转换的表达式时会产生临时拷贝（详见 [const 引用与临时对象](18.md)）

### 3.3 数组类型

数组类型带有尺寸信息（属于类型的一部分），但数组名在大多数表达式中会**退化为指针**。

**现代建议**：使用 `std::array` 或 `std::vector` 替代裸数组（详见 [表达式中的类型退化](19.md)）。

## 四、用户自定义类型

### 4.1 struct / class / union

| 特性 | struct | class | union |
|------|--------|-------|-------|
| 默认访问权限 | public | private | public |
| 默认继承权限 | public | private | N/A |
| 典型用途 | POD/数据聚合 | 封装/不变式 | 类型双关 |

C++20 起 `struct` 支持**指定初始化**：`Point p = {.x = 1, .y = 2};`

**始终使用 `enum class`**，避免命名空间污染和隐式转换。

### 4.2 静态类型与动态类型

- **静态类型**：编译期已知的声明类型
- **动态类型**：运行期对象实际的类型，仅对多态类型（含虚函数）的对象通过指针/引用访问时才有意义（详见 [静态类型与动态类型](10.md)）

## 五、类型的生命周期与存储

### 5.1 存储期

| 存储期 | 声明方式 | 生命周期 |
|--------|----------|----------|
| 自动 | 局部变量 | 块作用域 |
| 静态 | 全局/static | 程序生命周期 |
| 动态 | new/malloc | 手动控制 |
| 线程 | `thread_local` | 线程生命周期 |

### 5.2 构造与析构顺序

构造：基类 → 成员（按**声明顺序**，非初始化列表顺序）→ 构造函数体；析构严格逆序（详见 [构造与析构顺序](04.md)）。

**高频陷阱**：返回局部变量的指针/引用会导致悬垂（详见 [返回局部变量指针/引用](05.md)）。

## 六、类型推导与模板

### 6.1 auto / decltype

| 关键字 | 保留引用？ | 保留 cv？ | 典型场景 |
|--------|------------|-----------|----------|
| `auto` | 否 | 否（顶层） | 局部变量 |
| `decltype(expr)` | 是 | 是 | 返回类型推导 |
| `decltype(auto)` | 是 | 是 | 完美转发返回值 |

### 6.2 模板推导三场景

1. **按值传递**：引用忽略，顶层 const 忽略
2. **按引用传递**：保留引用性和 cv
3. **转发引用 `T&&`**：通过引用折叠绑定左值或右值

### 6.3 type_traits

`<type_traits>` 提供编译期类型查询和变换，C++14 起可用 `_v` / `_t` 后缀简化：
```cpp
std::is_integral_v<T>;
std::remove_reference_t<T>;
```

## 七、现代类型特性

### 7.1 结构化绑定（C++17）

```cpp
auto [x, y] = p;       // pair/tuple/数组/struct
```

### 7.2 CTAD（C++17）

```cpp
std::vector v = {1, 2, 3};   // 推导为 vector<int>
```

### 7.3 Concepts（C++20）

将类型约束从 SFINAE 变为显式契约：
```cpp
template<Arithmetic T>
T add(T a, T b) { return a + b; }
```

### 7.4 variant / optional

```cpp
std::variant<int, float, std::string> v = 42;   // 类型安全的联合体
std::optional<int> maybe;                        // 可能不存在的值
```

（详见 [std::variant](13.md)）

## 八、底层机制

### 8.1 对齐与填充

编译器会在成员之间插入填充字节以满足对齐要求。可用 `alignof` / `alignas` 查询和指定对齐。

### 8.2 严格别名规则

通过不兼容类型的指针访问对象是 UB。合法的类型双关方式：`std::memcpy`、`union`、C++20 的 `std::bit_cast`。

### 8.3 std::launder（C++17）

在已分配存储上重新构造对象时，使用 `std::launder` 获取有效指针，确保编译器认可新对象的生命周期。

## 九、编译期元编程

### 9.1 类型作为计算媒介

模板是图灵完备的编译期语言：
```cpp
template<int N>
struct Factorial {
    static constexpr int value = N * Factorial<N - 1>::value;
};
template<>
struct Factorial<0> {
    static constexpr int value = 1;
};
```

### 9.2 CRTP

静态多态模式：基类通过模板参数知道派生类类型，避免虚函数开销。

### 9.3 SFINAE

`std::void_t` 用于检查类型是否具有某成员或某操作。C++20 Concepts 已大幅简化此类代码。

## 十、工程实践

### 10.1 类型决策树

```
需要存储数据？
├── 尺寸固定且小？
│   ├── 值语义 → 直接存储（T）
│   └── 可能不存在 → std::optional<T>
├── 尺寸动态或共享？
│   ├── 独占所有权 → std::unique_ptr<T>
│   ├── 共享所有权 → std::shared_ptr<T>
│   └── 非拥有观察 → T* 或 std::weak_ptr<T>
└── 多类型可能性？
    ├── 编译期确定 → std::variant<T1, T2, ...>
    └── 运行时多态 → 虚函数接口
```
