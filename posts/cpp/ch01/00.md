# C++ 语言基础与类型系统：从底层机制到现代演进

## 一、类型系统的核心哲学

C++ 的类型系统建立在三个相互制衡的哲学之上：

| 哲学 | 含义 | 工程体现 |
|------|------|----------|
| **零开销抽象** | 不为不使用的特性付出代价 | `std::vector` 与手写数组性能等价 |
| **静态类型安全** | 错误尽可能在编译期捕获 | 模板编译失败、Concepts 约束 |
| **值语义与对象模型** | 明确区分对象的身份（identity）与值（value） | 拷贝/移动/引用语义的区别 |

**关键认知**：C++ 的类型不仅是数据的"形状描述"，更是**编译期契约**和**运行时内存布局**的双重载体。

## 二、基础类型体系（Fundamental Types）

### 2.1 标量类型（Scalar Types）

```
整数家族：
  bool (至少1字节，通常1字节)
  char / signed char / unsigned char (至少8位)
  short / unsigned short (至少16位)
  int / unsigned int (至少16位，通常32位)
  long / unsigned long (至少32位)
  long long / unsigned long long (至少64位)
  
浮点家族：
  float (IEEE 754 单精度)
  double (IEEE 754 双精度)
  long double (扩展精度，平台相关)
  
特殊标量：
  void (不完整类型，无值无大小)
  nullptr_t (C++11，指针字面量类型)
  std::byte (C++17，底层字节访问)
```

**工程陷阱**：
- `char` 的符号性未定义（ARM 上常为 unsigned，x86 上常为 signed）
- `long` 在 Windows 为 32 位，在 Linux x64 为 64 位——跨平台代码应使用 `int32_t`/`int64_t`
- `bool` 的 `sizeof` 为 1，但向 `bool` 写入非 0/1 值是未定义行为（UB）

### 2.2 类型的算术转换层级（Usual Arithmetic Conversions）

```
long double
    ↑
double
    ↑
float
    ↑
unsigned long long
    ↑
long long
    ↑
unsigned long
    ↑
long
    ↑
unsigned int
    ↑
int
    ↑
short/char/bool (整型提升后)
```

**面试高频**：`unsigned int` 与 `int` 运算时，`int` 会提升为 `unsigned int`，导致负数变极大正数：

```cpp
unsigned u = 10;
int i = -42;
std::cout << i + u; // 输出极大的正数，而非 -32
```

## 三、复合类型与类型修饰（Compound & Modified Types）

### 3.1 类型修饰符的层级（从右向左读法则）

C++ 的声明遵循**螺旋法则**（Clockwise/Spiral Rule），但工程上更实用的是**从右向左，括号优先**：

```cpp
int* const p;           // p 是 const 的，指向 int 的指针（指针不可变，指向可变）
const int* p;           // p 指向 const int（指向不可变，指针可变）
int const* p;           // 同上（const 在类型前或后等价，但后者更一致）
int* const* p;          // p 是指向"const 指针"的指针，该指针指向 int
int (*p)[10];           // p 是指向"包含10个int的数组"的指针
int (*f)(int, int);     // f 是指向函数的指针
int (*f[10])(int);      // f 是包含10个函数指针的数组
```

**现代建议**：C++11 起优先使用 `using` 别名而非 `typedef`，可读性显著提升：

```cpp
using FuncPtr = int(*)(int, int);
using ArrayPtr = int(*)[10];
```

### 3.2 引用类型（Reference Types）

引用不是对象，而是**已存在对象的别名**（alias）。

| 类型 | 绑定目标 | 可重新绑定 | 是否可为空 | 典型用途 |
|------|----------|------------|------------|----------|
| `T&` | 左值 | 否 | 否 | 函数参数，避免拷贝 |
| `T&&` | 右值（C++11） | 否 | 否 | 移动语义，完美转发 |
| `const T&` | 左值/右值 | 否 | 否 | 万能引用（部分场景） |

**核心机制**：引用在底层通常实现为指针，但**编译器保证其非空且不可重新绑定**，这消除了大量空指针检查。

**右值引用的双重身份**：
- **作为类型**：`T&&` 绑定到右值，支持移动语义
- **在模板推导中**：`T&&` 成为**转发引用**（Forwarding Reference，曾误称万能引用），可绑定左值或右值

```cpp
template<typename T>
void foo(T&& x);  // T&& 在此处是转发引用，非右值引用

int i = 42;
foo(i);           // T 推导为 int&，x 的类型为 int&（引用折叠）
foo(42);          // T 推导为 int，x 的类型为 int&&
```

### 3.3 数组类型（Array Types）

数组是**同构元素的连续序列**，但 C++ 的数组类型带有**尺寸信息**（属于类型的一部分）：

```cpp
int a[10];          // 类型是 int[10]，不是 int*
int b[10][20];      // 类型是 int[10][20]，内存布局为 10 行 20 列

void foo(int arr[10]);  // 退化：参数实际类型为 int*，10 被忽略
void foo(int (&arr)[10]); // 正确传递数组引用，保留尺寸信息
```

**数组到指针的退化（Decay）**：
- 数组名在大多数表达式中会退化为指向首元素的指针
- 例外：`sizeof(arr)`、`&arr`、`decltype(arr)`、引用绑定时

**工程实践**：现代 C++ 中，裸数组应被 `std::array`（静态尺寸）或 `std::vector`（动态尺寸）替代，以获得边界检查和标准接口。

## 四、用户自定义类型（User-Defined Types）

### 4.1 struct vs class vs union

| 特性 | struct | class | union |
|------|--------|-------|-------|
| 默认访问权限 | public | private | public |
| 默认继承权限 | public | private | N/A |
| 可包含成员 | 变量/函数/类型 | 同左 | 同左（C++11起） |
| 内存布局 | 顺序分配（可能有填充） | 同左 | 共享内存，最大成员尺寸 |
| 典型用途 | POD/数据聚合 | 封装/不变式 | 类型双关/手动内存管理 |

**C++20 的演进**：`struct` 可用于定义**聚合体**（Aggregate），支持**指定初始化**（Designated Initializers）：

```cpp
struct Point { int x; int y; };
Point p = {.x = 1, .y = 2};  // C++20，继承自 C
```

### 4.2 枚举类型（Enum）

```cpp
enum Color { Red, Green, Blue };           // 无作用域枚举，隐式转 int
enum class Color { Red, Green, Blue };     // 强类型枚举（C++11），需显式转换
```

**工程建议**：始终使用 `enum class`，避免命名空间污染和隐式转换导致的意外。

### 4.3 POD、平凡（Trivial）与标准布局（Standard Layout）

C++11 起对类型的"简单性"做了精确分类：

| 概念 | 含义 | 检查方式 |
|------|------|----------|
| **POD** (Plain Old Data) | 兼容 C 的内存布局，可直接用于 `memcpy` | `std::is_pod` (C++20 弃用) |
| **Trivial** | 构造函数/析构/拷贝/移动均为 trivial（可由编译器生成，不执行自定义逻辑） | `std::is_trivial` |
| **Standard Layout** | 内存布局与 C 兼容，首成员地址与对象地址一致，支持 offsetof | `std::is_standard_layout` |
| **Trivially Copyable** | 可通过 `memcpy` 安全复制 | `std::is_trivially_copyable` |
| **Implicit Lifetime Type** | C++20，允许隐式对象创建 | 编译器自动判断 |

**面试追问**：为什么 `std::vector` 不能是 POD？因为它管理动态内存，需要非平凡的析构函数来释放资源。

## 五、类型的生命周期与存储

### 5.1 存储期（Storage Duration）

| 存储期 | 声明方式 | 生命周期 | 线程安全 |
|--------|----------|----------|----------|
| 自动（Automatic） | 局部变量 | 块作用域 | 栈私有，天然安全 |
| 静态（Static） | 全局/namespace/static | 程序生命周期 | 初始化顺序灾难（Static Initialization Order Fiasco） |
| 动态（Dynamic） | new/malloc | 手动控制 | 需同步机制 |
| 线程（Thread） | `thread_local` | 线程生命周期 | 线程私有 |

**关键机制**：`thread_local` 的初始化发生在**首次使用时**，而非线程创建时。

### 5.2 对象的生命周期（Lifetime）

C++ 严格定义了对象的存在区间：

1. **分配存储**（获得内存）
2. **开始生命周期**（构造函数完成）
3. **使用期**（对象有效）
4. **结束生命周期**（析构函数开始）
5. **释放存储**（内存归还）

**未定义行为（UB）的高频场景**：
- 在生命周期开始前/后访问对象（如 `delete` 后使用指针）
- 在重叠的对象上调用 `std::launder`（C++17 用于在已存在存储上重建对象）

## 六、类型推导与类型变换（Type Deduction & Transformation）

### 6.1 auto 与 decltype

| 关键字 | 推导依据 | 保留引用？ | 保留 cv？ | 典型场景 |
|--------|----------|------------|-----------|----------|
| `auto` | 初始化表达式 | 否（除非显式加 `&`/`&&`） | 否 | 局部变量，迭代器 |
| `decltype(expr)` | 表达式的精确类型 | 是 | 是 | 返回类型推导，依赖表达式 |
| `decltype(auto)` | 初始化表达式 | 是 | 是 | 函数返回类型完美转发 |

```cpp
int x = 42;
int& rx = x;

auto a = rx;            // int（剥离引用和 cv）
decltype(auto) b = rx;  // int&（保留引用）
decltype(rx) c = rx;    // int&（保留引用）
```

### 6.2 模板类型推导（Template Type Deduction）

这是 C++ 类型系统最复杂的部分之一，分为三种场景：

#### 场景一：按值传递（By Value）

```cpp
template<typename T>
void foo(T param);

int x = 42;
const int cx = x;
const int& rx = x;

foo(x);    // T = int（忽略引用，忽略顶层 const）
foo(cx);   // T = int（忽略顶层 const）
foo(rx);   // T = int（忽略引用，忽略顶层 const）
```

**规则**：引用被忽略，顶层 cv 限定符被忽略。

#### 场景二：按引用/指针传递（By Reference/Pointer）

```cpp
template<typename T>
void foo(T& param);

foo(x);    // T = int, param = int&
foo(cx);   // T = const int, param = const int&
foo(rx);   // T = const int, param = const int&

template<typename T>
void bar(T* param);
const int* px = &x;
bar(px);   // T = const int, param = const int*
```

**规则**：保留引用性和 cv 限定符，推导 `T` 为去除引用后的类型。

#### 场景三：转发引用（Forwarding Reference）

```cpp
template<typename T>
void foo(T&& param);

foo(x);    // T = int&, param = int&（引用折叠：int& + && → int&）
foo(cx);   // T = const int&, param = const int&
foo(42);   // T = int, param = int&&
foo(std::move(x)); // T = int, param = int&&
```

**核心机制**：**引用折叠**（Reference Collapsing）
- `T& &` → `T&`
- `T& &&` → `T&`
- `T&& &` → `T&`
- `T&& &&` → `T&&`

### 6.3 std::type_traits：类型系统的编译期编程

`<type_traits>` 提供了编译期类型查询和变换：

**查询类**：
```cpp
std::is_integral<T>::value          // 是否为整数类型
std::is_pointer<T>::value           // 是否为指针
std::is_reference<T>::value         // 是否为引用
std::is_same<T, U>::value           // 是否相同
std::is_base_of<Base, Derived>::value // 继承关系
```

**变换类**：
```cpp
std::remove_reference<T>::type      // 去引用
std::add_const<T>::type             // 加 const
std::decay<T>::type                 // 模拟值传递退化（去引用、去 cv、数组/函数转指针）
std::conditional<B, T, F>::type     // 编译期 if
std::enable_if<B, T>::type          // 条件启用（SFINAE）
```

**C++14/C++17 后缀**：
```cpp
std::is_integral_v<T>        // C++14，等价于 ::value
std::remove_reference_t<T>   // C++14，等价于 ::type
```

## 七、类型系统的现代演进（C++11/14/17/20/23）

### 7.1 结构化绑定（Structured Binding, C++17）

```cpp
std::pair<int, double> p = {1, 3.14};
auto [x, y] = p;  // x 为 int，y 为 double

// 绑定到数组
int arr[3] = {1, 2, 3};
auto [a, b, c] = arr;

// 绑定到 struct（需为公开非静态数据成员）
struct Point { int x; int y; };
Point pt = {1, 2};
auto [px, py] = pt;
```

**底层机制**：编译器生成匿名实体 `e`，然后绑定引用到 `e` 的成员。

### 7.2 类模板参数推导（CTAD, C++17）

```cpp
std::pair p(1, 3.14);              // 推导为 std::pair<int, double>
std::vector v = {1, 2, 3};         // 推导为 std::vector<int>
std::lock_guard lk(mtx);           // 推导互斥量类型
```

**自定义 CTAD**：
```cpp
template<typename T>
struct Wrapper {
    T value;
    Wrapper(T v) : value(v) {}
};

// 推导指引（Deduction Guide）
template<typename T>
Wrapper(T) -> Wrapper<T>;
```

### 7.3 Concepts（C++20）：类型约束的显式化

Concepts 是 C++20 对模板类型系统的革命性改进，将**类型约束**从 SFINAE 的黑暗艺术变为显式契约：

```cpp
template<typename T>
concept Arithmetic = std::is_arithmetic_v<T>;

template<typename T>
concept Addable = requires(T a, T b) {
    { a + b } -> std::same_as<T>;  // 要求支持 + 且返回类型为 T
};

template<<Arithmetic T>
T add(T a, T b) { return a + b; }

template<<Addable T>
T combine(T a, T b) { return a + b; }
```

**与 SFINAE 的对比**：
- SFINAE：错误发生在模板实例化过程中，错误信息冗长
- Concepts：约束在签名处检查，失败时提供清晰的"约束不满足"诊断

### 7.4 变体类型与可选类型（std::variant, std::optional）

```cpp
std::variant<int, float, std::string> v = 42;  // 类型安全的联合体
v = "hello";  // 切换为 std::string

std::optional<int> maybe;  // 可能不存在的值
maybe = 42;
if (maybe) {  // 显式检查存在性
    std::cout << *maybe;
}
```

**std::variant 的访问**：
```cpp
std::visit(overloaded{
    [](int arg) { std::cout << "int: " << arg; },
    [](float arg) { std::cout << "float: " << arg; },
    [](const auto& arg) { std::cout << "other"; }
}, v);
```

## 八、类型系统的底层机制

### 8.1 对象表示与对齐（Object Representation & Alignment）

```cpp
struct Example {
    char c;      // 1 byte
    // 3 bytes padding
    int i;       // 4 bytes
    double d;    // 8 bytes
};               // 总大小通常为 16 bytes（对齐到 8）
```

**对齐要求**：
- `alignof(T)`：查询类型 T 的对齐要求
- `alignas(N)`：指定对齐要求（C++11）
- `std::aligned_storage` / `std::aligned_alloc`：手动对齐分配

**面试追问**：为什么 `std::vector<bool>` 是特化的位压缩实现？因为 `bool` 的对齐要求为 1，但 `vector<bool>` 打包为位域以节省空间，这导致它不满足容器的标准序列要求。

### 8.2 严格别名规则（Strict Aliasing Rule）

C++ 规定：**通过不兼容类型的指针访问对象是未定义行为**。

```cpp
float f = 3.14f;
int* pi = reinterpret_cast<int*>(&f);  // 合法（获得地址）
int i = *pi;                           // UB！违反严格别名规则
```

**合法的类型双关（Type Punning）**：
- `std::memcpy`（逐字节复制，安全）
- `union`（C++ 标准未明确保证，但大多数编译器支持）
- `std::bit_cast`（C++20，编译期安全的类型转换）

```cpp
// C++20 安全写法
int i = std::bit_cast<int>(3.14f);
```

### 8.3 类型双关与 std::launder（C++17）

在已分配的存储上重新构造对象时，必须使用 `std::launder` 获取有效指针：

```cpp
alignas(T) char buf[sizeof(T)];
T* p = new (buf) T(42);  //  placement new
T* valid = std::launder(p);  // 确保编译器认可新对象的生命周期
```

## 九、类型系统与编译期计算（元编程）

### 9.1 类型作为计算媒介

C++ 模板是**图灵完备**的编译期语言，类型是核心数据：

```cpp
// 编译期递归：计算阶乘
template<int N>
struct Factorial {
    static constexpr int value = N * Factorial<N - 1>::value;
};

template<>
struct Factorial<<0> {
    static constexpr int value = 1;
};

static_assert(Factorial<<5>::value == 120);
```

### 9.2 CRTP（Curiously Recurring Template Pattern）

这是静态多态的经典模式，你之前询问过：

```cpp
// 基类通过模板参数知道派生类类型
template<typename Derived>
class Base {
public:
    void interface() {
        static_cast<<Derived*>(this)->implementation();
    }
};

class Derived : public Base<<Derived> {
public:
    void implementation() {
        // 实际逻辑
    }
};
```

**优势**：无虚函数表开销，编译期绑定，支持内联展开。
**代价**：代码膨胀（每个派生类实例化一份基类代码），编译时间增加。

### 9.3 SFINAE 与 std::void_t

SFINAE（Substitution Failure Is Not An Error）是 C++ 类型系统的核心元编程机制：

```cpp
// 检查类型是否有名为 foo 的成员函数
template<typename T, typename = void>
struct has_foo : std::false_type {};

template<typename T>
struct has_foo<T, std::void_t<decltype(std::declval<T>().foo())>> 
    : std::true_type {};
```

**C++20 替代**：Concepts 使此类代码大幅简化。

## 十、工程实践与面试要点总结

### 10.1 类型系统的决策树

```
需要存储数据？
├── 尺寸固定且小？
│   ├── 值语义 → 直接存储（T）
│   └── 可能不存在 → std::optional<T>
├── 尺寸动态或共享？
│   ├── 独占所有权 → std::unique_ptr<T>
│   ├── 共享所有权 → std::shared_ptr<T>
│   └── 非拥有观察 → T* 或 std::weak_ptr<T>
└── 多类型可能性？
    ├── 编译期确定 → std::variant<T1, T2, ...>
    └── 运行时多态 → 虚函数接口
```

### 10.2 高频面试追问

1. **为什么 `std::vector<T>::reference` 在 `T=bool` 时是代理类？**
   - 因为 `vector<bool>` 是位压缩，无法返回真正的 `bool&`

2. **`std::move` 是否移动对象？**
   - 否。它只是将对象转换为右值引用，实际的移动由移动构造函数/赋值运算符完成。

3. **模板推导中 `T&&` 什么时候是右值引用，什么时候是转发引用？**
   - 当 `T` 是模板参数且出现在参数列表中时，是转发引用；否则是右值引用。

4. **`decltype(auto)` 与 `auto` 的区别？**
   - `decltype(auto)` 保留引用性和 cv 限定符，`auto` 执行值退化。

5. **C++20 Concepts 与 SFINAE 的性能差异？**
   - 编译期无运行时差异；Concepts 的错误信息更友好，编译速度通常更快。
