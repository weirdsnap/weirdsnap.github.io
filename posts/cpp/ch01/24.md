# 对象切片（Object Slicing）

将派生类对象按值赋给基类对象时，派生类独有的成员和数据被"切掉"，只剩下基类子对象，导致多态行为丢失。

```cpp
class Base { public: virtual void foo() {} };
class Derived : public Base {
public:
    int extra = 42;
    void foo() override { /* 派生实现 */ }
};

Derived d;
Base b = d;        // 切片！extra 丢失，foo() 调用 Base 版本
Base& ref = d;     // OK，不切片，多态保留
```

**常见误区**

- ❌ 切片只发生在显式赋值 — 错误。按值传参、按值返回同样会切片：`void process(Base b)` 传入 `Derived` 时即切片。
- ❌ 切片是编译错误 — 错误。切片是合法的 C++ 代码，编译器不报错，但运行期行为错误。
- ❌ `Base* p = &d` 也会切片 — 错误。指针和引用不会切片，只有按值拷贝会。

**延伸点**：工程实践中，多态基类应禁用拷贝构造和拷贝赋值（`= delete`），强制使用指针/引用操作，从源头避免切片。C++11 起推荐基类析构函数设为 `virtual` 或 `protected`。
