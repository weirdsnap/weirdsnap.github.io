# 默认参数：每次调用都重新求值

C++ 函数的默认参数表达式在**每次调用时都会重新求值**，而非在定义时求值一次。

---

## 每次调用都重新求值

```cpp
int counter() {
    static int c = 0;
    return c++;
}

void foo(int x = counter()) {}

foo();  // x = 0
foo();  // x = 1
foo();  // x = 2
```

每次调用 `foo()` 时 `counter()` 都被执行一次，因此 `x` 的值依次递增。默认表达式中的函数调用、运算符、构造函数调用都在调用点发生。

---

## 默认参数是调用点的语法糖

C++ 标准规定，默认参数在**每次调用的调用点**求值。背后的机制是：编译器在调用点插入默认表达式，等价于手动传参：

```cpp
void foo(int x = counter());

// 编译器看到 foo() 时，将其处理为：
// {
//     int x = counter();  // 在调用点生成默认实参
//     foo(x);
// }
```

因此每次调用都有独立的表达式求值，互不影响。

---

## 默认参数由声明决定，不由定义决定

```cpp
// header.h
void bar(int x = 1);     // 声明：默认参数 x=1

// impl.cpp
#include "header.h"
void bar(int x = 99) {   // ❌ 重定义默认参数（编译错误）
    std::cout << x;
}
```

一个函数的默认参数**只能在一个翻译单元中被指定一次**，通常放在声明（头文件）中。如果声明和定义都指定了默认参数，编译器报错。

---

## 涉及临时对象的生命周期

因为默认表达式在调用点求值，临时对象的生命周期从默认实参创建开始，到函数调用结束为止：

```cpp
struct S {
    S() { std::cout << "构造 "; }
    ~S() { std::cout << "析构 "; }
};

void foo(const S& s = S{}) {
    std::cout << "调用 ";
}

foo();
foo();
// 输出：构造 调用 析构 构造 调用 析构
```

每次调用都构造一个新的临时 `S`，函数返回后立即析构，不跨调用复用。

---

## 常见误区

- ❌ 默认参数在程序启动时初始化一次 — 错误。每次调用都重新求值，多次调用结果可能不同。
- ❌ 默认参数在函数定义处求值 — 错误。在调用点求值，由调用点可见的声明决定。
- ❌ 多个默认参数的表达式共享同一份求值结果 — 错误。每个默认参数独立求值：

```cpp
int rand() { return 42; }
void baz(int x = rand(), int y = rand()) {}
baz();  // x=42, y=42，但两次 rand() 调用是独立的
```

- ❌ 默认参数的求值时机与非默认参数不同 — 错误。传参时无论默认参数还是显式传入的实参，求值时机和行为完全一致。

---

## 延伸点

- **虚函数的默认参数**：两个规则叠加——默认参数表达式仍是**每次调用在调用点求值**，但**用哪个默认值由静态类型决定**，而非动态类型。因此同一个虚函数调用传入的默认参数值可能和实际调用的派生类版本不匹配：

```cpp
struct Base {
    virtual void foo(int x = counter()) {
        std::cout << "Base " << x;
    }
};

struct Derived : Base {
    void foo(int x = 99) override {
        std::cout << "Derived " << x;
    }
};

Base* p = new Derived();
p->foo();  // 调用 Derived::foo，但默认参数取自 Base 的声明
           // → 仍调用一次 counter()，x = counter() 的返回值
           // 输出：Derived 0（假设 counter 从 0 开始）
```

无论虚函数怎么多态分发，默认参数表达式**仍遵守每次调用在调用点求值的规则**。详见 ch02/05.md。
- 默认参数可以是任意表达式，包括函数调用、成员访问、运算符——只要在调用点可见即可。
- 模板函数的默认参数同样每次调用求值，不受模板参数推导影响。
- 默认参数可以依赖前面的参数：`void f(int a, int b = a + 1)` 合法，因为默认表达式可见前一个参数。
