# 类型别名：`using` 与 `typedef`

类型别名（type alias）只是给已有类型起个新名字，**不会创建新类型**。C++ 里可以用 `typedef` 或 `using` 来声明，现代 C++ 更推荐 `using`。

---

## 基本语法

```cpp
// 老写法
typedef unsigned int uint;
typedef int* IntPtr;

// 现代写法
using uint = unsigned int;
using IntPtr = int*;
```

对于普通类型，两者等价。但 `using` 的语法更直观，尤其是复杂类型从右往左读比较痛苦的时候。

---

## 函数类型别名

`using` 可以给函数类型起别名，这个别名可以直接用来声明成员函数：

```cpp
using Func = int();

struct S {
    Func f;   // 等价于 int f();
};

int S::f() { return 1; }
```

这正是你贴的那道题的核心：

> 成员函数可以用函数类型的 typedef / using 别名声明，结果和直接写 `int f()` 完全一样（`[class.mfct]`）。

注意区分**函数类型**和**函数指针类型**：

```cpp
using Func    = int();    // 函数类型，可用来声明函数
using FuncPtr = int(*)(); // 函数指针类型，用来存函数地址

Func* p = &S::f;          // Func 退化为指针
FuncPtr q = &S::f;        // 直接是函数指针
```

---

## `using` 比 `typedef` 可读性更好

复杂类型尤其明显：

```cpp
// typedef 版本：从右往左读
typedef int (*FuncPtr)(int, int);
typedef std::map<std::string, std::vector<int>> MapType;

// using 版本：从左往右读，更自然
using FuncPtr = int(*)(int, int);
using MapType = std::map<std::string, std::vector<int>>;
```

---

## 数组类型别名

```cpp
using IntArray = int[10];

IntArray arr;        // 等价于 int arr[10]
static_assert(std::is_same_v<IntArray, int[10]>);
```

但要注意：数组类型作为函数参数时仍然会退化为指针，别名也不例外。

```cpp
void foo(IntArray a);   // 等价于 void foo(int* a)
```

---

## 成员指针别名

```cpp
struct S { int x; int f(); };

using MemberVar   = int S::*;      // 指向数据成员的指针
using MemberFunc  = int (S::*)();  // 指向成员函数的指针

MemberVar  mv = &S::x;
MemberFunc mf = &S::f;
```

---

## 别名模板（Alias Template）

C++11 起，`using` 可以带模板参数，`typedef` 做不到：

```cpp
template<typename T>
using Vec = std::vector<T>;

Vec<int> v;   // 等价于 std::vector<int>
```

这在泛型代码里非常常用，比如统一容器的 allocator：

```cpp
template<typename T>
using MyVec = std::vector<T, MyAllocator<T>>;
```

---

## 类型别名不创建新类型

别名只是原类型的同义词，`typeid` 和 `std::is_same` 都会把它当成原类型：

```cpp
using MyInt = int;

static_assert(std::is_same_v<MyInt, int>);
std::cout << std::boolalpha << (typeid(MyInt) == typeid(int));  // true
```

所以类型别名不能用来实现类型安全封装；要真正区分类型，用强类型/强 typedef，比如 `enum class`、自定义包装类或库如 `BOOST_STRONG_TYPEDEF`。

---

## 常见误区

- ❌ "`using Func = int();` 声明了一个函数指针" — 错误。它是函数类型，不是指针类型。
- ❌ "类型别名会生成独立类型" — 错误。`using MyInt = int;` 和 `int` 是同一个类型。
- ❌ "`typedef` 和 `using` 完全没有区别" — 不完全。语法上 `using` 可读性更好，且支持模板别名。
- ❌ "数组类型别名在参数里会保留数组大小" — 错误。和原生数组一样会退化成指针。

---

## 一句话总结

> `using`/`typedef` 只是给已有类型起别名，不创建新类型。函数类型别名可以声明成员函数，`using` 还支持模板别名，是现代 C++ 的首选写法。
