# 静态变量的默认初始化

C++ 中不同生命周期的变量，默认初始化行为不一样。静态存储期的变量会自动零初始化，而局部自动变量不会。

## 哪些变量属于静态存储期

| 变量类型 | 说明 |
|----------|------|
| 全局变量 | 定义在函数外部的变量 |
| `static` 局部变量 | 函数内部用 `static` 修饰的变量 |
| `static` 类成员 | 类的静态成员变量 |

这些变量在程序启动时会进行**零初始化（zero-initialization）**。

## 零初始化是什么

对于静态存储期变量，如果没有显式初始化，编译器会把它们置为 0（或等价于 0 的值）：

```cpp
static int a;        // 0
static double b;     // 0.0
static char c;       // '\0'
static int* p;       // nullptr
static bool flag;    // false
static int arr[5];   // 所有元素都是 0
```

## 代码验证

```cpp
#include <iostream>

static int g;           // 全局静态变量
static int* gp;         // 全局静态指针

void foo() {
    static int s;       // 局部静态变量
    std::cout << "s = " << s << "\n";
}

int main() {
    std::cout << "g = " << g << "\n";
    std::cout << "gp = " << gp << "\n";
    foo();
    return 0;
}
```

输出：

```text
g = 0
gp = 0
s = 0
```

## 与局部自动变量的对比

```cpp
void bar() {
    int x;          // ❌ 未初始化，值不确定
    static int y;   // ✅ 自动零初始化为 0
}
```

局部自动变量不会默认初始化，读取它是未定义行为。静态局部变量会默认初始化为 0。

## 常见误区

- ❌ 所有变量都会默认初始化为 0 — 错误。只有静态存储期变量会。
- ❌ 类成员变量会默认初始化为 0 — 错误。非静态成员变量不会自动初始化。
- ✅ `static int x;` 默认是 0
- ✅ `int x;`（全局）默认是 0
- ❌ `int x;`（函数内）默认不是 0

## 类成员怎么办

类中的非静态成员变量不会自动初始化，必须在构造函数中或声明时初始化：

```cpp
class Foo {
public:
    int x;          // 不会自动初始化
    int y = 0;      // ✅ C++11 起可在声明时初始化
    static int z;   // 静态成员，类外定义时零初始化
};

int Foo::z;  // 默认是 0
```

## 一句话总结

> 静态存储期变量（全局、`static` 局部、`static` 类成员）会默认零初始化；局部自动变量不会，必须显式初始化。
