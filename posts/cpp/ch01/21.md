# 右值引用与转发引用

C++11 引入 `T&&` 有两种身份：作为具体类型的右值引用（绑定右值，支持移动语义），以及在模板参数推导中的转发引用（可绑定左值或右值）。

| 语境 | `T&&` 的身份 | 绑定对象 |
|------|-------------|----------|
| `void foo(int&& x)` | 右值引用 | 只能绑定右值（如 42） |
| `template<typename T> void foo(T&& x)` | 转发引用 | 左值 → `T&`，右值 → `T&&` |

```cpp
template<typename T>
void foo(T&& x);  // 转发引用

int i = 42;
foo(i);           // T = int&, x = int&（引用折叠）
foo(42);          // T = int,  x = int&&
foo(std::move(i));// T = int,  x = int&&
```

**常见误区**

- ❌ `T&&` 总是右值引用 — 错误。模板参数推导中的 `T&&` 是转发引用，可绑定左值。
- ❌ 转发引用就是"万能引用" — 不完全。官方术语是 Forwarding Reference（转发引用），"万能引用"是旧称。
- ❌ `const T&&` 也是转发引用 — 错误。转发引用要求精确形式 `T&&`，加 `const` 后变为普通右值引用。

**延伸点**：引用折叠规则：`T& &`→`T&`、`T& &&`→`T&`、`T&& &`→`T&`、`T&& &&`→`T&&`。`std::forward<T>(x)` 结合转发引用实现完美转发。
