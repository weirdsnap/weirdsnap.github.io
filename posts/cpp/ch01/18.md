# `const` 引用绑定与临时对象

`const` 左值引用绑定到需隐式转换的表达式时，产生临时对象拷贝，引用与原对象解耦。

**场景示例：**
```cpp
double d = 3.14;
const int& r = d;  // 产生 int 临时对象，r 绑定到临时对象而非 d
d = 2.71;          // r 仍为 3
```

**原理解析 + 常见误区**

- ❌ const 引用总是直接绑定到原对象 — 错误。类型转换时产生独立临时拷贝。
- ❌ 绑定后修改原对象引用能看到变化 — 错误。临时对象与原对象完全解耦。

**延伸点**：非 const 引用禁止绑定到临时对象：`int& r = d;`（`d` 为 `double`）直接编译错误，这是 const 引用与非 const 引用的核心语义差异。
