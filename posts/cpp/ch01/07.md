# 表达式中的类型退化（Type Decay）

类型退化（decay）是指数组和函数在大多数表达式中**自动退化为指针**，同时顶层 `const`/`volatile` 在按值传递时被剥离。

---

## 一、三种退化场景

| 原始类型 | 退化后 | 触发语境 |
|----------|--------|---------|
| `T[N]` | `T*` | 大多数表达式、按值传参、`auto` 推导 |
| `T(args)` | `T(*)(args)` | 同上 |
| `const T` | `T` | 按值传递、`auto` 推导（顶层 cv 剥离） |

```cpp
void func(int);

using T1 = decltype(func);   // void(int) —— 函数类型，精确
using T2 = decltype(&func);  // void(*)(int) —— 函数指针

auto a1 = func;   // void(*)(int) —— auto 退化（函数→指针）
auto a2 = a1;     // void(*)(int) —— 函数指针不会被再退化
```

数组退化是最常见的陷阱来源：

```cpp
int arr[5] = {1, 2, 3, 4, 5};

auto a = arr;            // int*（退化）
decltype(arr) d = {};    // int[5]（decltype 不退化）
auto& ra = arr;          // int(&)[5]（引用绑定不退化）

template<typename T>
void deduce(T t) {       // T 按值推导：int*
    std::cout << sizeof(t);  // 8（指针大小，不是 20）
}

deduce(arr);             // T 推导为 int*
```

## 三、退化带来什么问题？

### 数组长度丢失

```cpp
template<typename T>
void bad(T t) {                    // T 推导为 int*
    for (size_t i = 0; i < sizeof(t) / sizeof(t[0]); ++i) { } // ❌ 不对！
}

template<typename T, size_t N>
void good(T (&arr)[N]) {            // 数组引用不退化
    for (size_t i = 0; i < N; ++i) { } // ✅ N 保留数组长度
}
```

### 完美转发时需要阻止退化

`std::forward` 内部借助引用折叠保留类型，但如果传进来的是 `auto` 推导的值，退化已经发生，信息就丢了：

```cpp
template<typename T>
decltype(auto) forward_value(T&& t) {
    return std::forward<T>(t);  // T 不退化（转发引用保留）
}

// 但如果写成下面这样，信息就丢了：
template<typename T>
auto forward_value(T t) {    // T 按值，退化
    return t;                 // 引用和 cv 都丢了
}
```

## 四、退化什么时候不会发生？

| 场景 | 数组 | 函数 |
|------|------|------|
| `sizeof(arr)` | ✅ 保留长度为数组大小 | ✅ 保留为函数类型大小（通常为 1） |
| `decltype(arr)` | ✅ 保留 `T[N]` | ✅ 保留 `T(args)` |
| `&arr` | ✅ 得到数组指针 `T(*)[N]` | ✅ 得到函数指针 |
| 引用绑定 `T&` / `T&&` | ✅ 保留数组/函数类型 | ✅ 保留 |
| `std::type_identity_t<T>` | ✅ 阻止模板推导中的退化 | ✅ |
| `auto&` / `auto&&` | ✅ 保留 | ✅ |

```cpp
int arr[3] = {};

auto a1 = arr;     // int*
auto& a2 = arr;    // int(&)[3] ✅ 引用阻止退化
auto&& a3 = arr;   // int(&)[3] ✅

std::type_identity_t<decltype(arr)> x = {}; // int[3] ✅
```

## 五、常见误区

| 误区 | 正解 |
|------|------|
| ❌ 数组名就是指针 | ✅ 数组是独立类型，`sizeof(arr)` 返回总字节数；只在表达式中退化为指针 |
| ❌ `auto` 和 `decltype` 对数组处理相同 | ✅ `auto` 退化，`decltype` 保留 |
| ❌ 模板传数组一定能拿到长度 | ✅ 只有数组引用 `T(&)[N]` 才能保留长度，按值传参退化为指针 |
| ❌ 函数类型和函数指针在模板中是等价的 | ✅ 函数类型会退化为指针，但函数引用不会 |
| ❌ `std::decay_t` 只做数组/函数退化 | ✅ 还会去掉顶层 cv 和引用，是完整的"按值传递退化" |

---

## 六、延伸点

- `std::decay_t<T>` 显式执行完整的退化转换，在模板元编程中用于统一处理实参类型。
- C++20 的 `std::type_identity` 可以在模板中强制阻止退化。
- 泛型代码中保留数组长度的惯用手法：用 `T(&)[N]` 或 `std::span`。
