# C++ 字面量与类型

C++ 字面量根据书写形式赋予默认类型。不加后缀时，编译器会按标准规则选择"最自然"的类型；加后缀则可以显式指定类型。

## 常见字面量的默认类型

| 字面量 | 默认类型 | 说明 |
|--------|----------|------|
| `42` | `int` | 十进制整型默认 int |
| `042` | `int` | 以 `0` 开头为八进制，值仍是 34 |
| `0x2A` | `int` | 以 `0x` 开头为十六进制 |
| `0b101010` | `int` | 以 `0b` 开头为二进制（C++14） |
| `3.14` | `double` | 浮点默认 double，不是 float |
| `'a'` | `char` | `sizeof('a') == 1` |
| `"abc"` | `const char[4]` | 含结尾 `\0`，表达式中退化为 `const char*` |
| `true` / `false` | `bool` | — |
| `nullptr` | `std::nullptr_t` | 可隐式转为任意指针类型 |

## 整型与浮点型后缀

后缀改变字面量的默认类型，大小写含义不同。

### 整型后缀

| 后缀 | 含义 | 示例 | 类型 |
|------|------|------|------|
| `U` / `u` | 无符号 | `42U` | `unsigned int` |
| `L` / `l` | 长整型 | `42L` | `long` |
| `LL` / `ll` | 长长整型 | `42LL` | `long long` |
| `UL` / `LU` 等 | 无符号长整型 | `42UL` | `unsigned long` |
| `ULL` / `LLU` 等 | 无符号长长整型 | `42ULL` | `unsigned long long` |

> 注意：小写 `l` 容易与数字 `1` 混淆，建议始终使用大写后缀 `L` / `LL` / `ULL`。

### 浮点后缀

| 后缀 | 含义 | 示例 | 类型 |
|------|------|------|------|
| `F` / `f` | 单精度浮点 | `3.14f` | `float` |
| `L` / `l` | 扩展精度浮点 | `3.14L` | `long double` |
| 无后缀 | 双精度浮点 | `3.14` | `double` |

> 注意：整数后缀 `L` 表示 `long`，浮点后缀 `L` 表示 `long double`，不要混淆。

## 代码示例

```cpp
auto a = 42;        // int
auto b = 42L;       // long
auto c = 42U;       // unsigned int
auto d = 42ULL;     // unsigned long long

auto e = 3.14;      // double
auto f = 3.14f;     // float
auto g = 3.14L;     // long double

auto h = "hello";   // const char[6]（包含结尾 '\0'）
auto i = 'A';       // char
auto j = true;      // bool
auto k = nullptr;   // std::nullptr_t
```

## 常见误区 / 踩坑点

- ❌ `1.1` 是 `float` — 错误。必须加 `f` 后缀才是 `float`。
- ❌ 整数后缀 `L` 和浮点后缀 `L` 含义相同 — 错误。`42L` 是 `long`，`3.14L` 是 `long double`。
- ❌ `"abc"` 是 `const char*` — 错误。真实类型是 `const char[4]`，仅在表达式中退化为指针。
- ❌ 小写 `l` 后缀没问题 — 不建议。`42l` 容易与 `421` 混淆。

## 延伸点：用户自定义字面量（UDL）

C++11 起允许程序员定义自己的字面量后缀，让 `60s`、`100ms`、`5.0_km` 这样的写法直接产生特定类型的对象，避免单位混淆。

### 标准库中的 UDL

```cpp
#include <chrono>
#include <string>

using namespace std::chrono_literals;

auto t1 = 60s;       // std::chrono::seconds
auto t2 = 100ms;     // std::chrono::milliseconds
auto s1 = "hello"s;  // std::string
```

### 自定义 UDL 示例

```cpp
struct Distance {
    double meters;
};

Distance operator"" _km(long double value) {
    return Distance{value * 1000};
}

Distance operator"" _m(long double value) {
    return Distance{value};
}

auto d1 = 5.0_km;  // Distance{5000}
auto d2 = 300.0_m; // Distance{300}
```

### 为什么用 UDL

- **类型安全**：`5.0_km` 和 `5.0_m` 是明确区分的值，不会因为单位写错导致 bug
- **可读性**：`timeout = 30s` 比 `timeout = std::chrono::seconds(30)` 更自然
- **编译期计算**：UDL 操作符在编译期调用，没有运行时开销

### UDL 注意点

- 后缀以 `_` 开头的是用户自定义，无下划线的是标准保留（如 `L`、`U`、`f`）
- 现代 C++ 中应优先使用标准库提供的 UDL（如 `std::chrono_literals`），而不是自己造轮子
