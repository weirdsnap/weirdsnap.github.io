# `char` 的符号性

`char`、`signed char`、`unsigned char` 是三种不同类型；`char` 的符号性由实现定义（ARM 默认 unsigned，x86 默认 signed）。

| 类型 | 符号性 | 用途 |
|------|--------|------|
| `char` | 实现定义 | 字符数据 |
| `signed char` | 明确有符号 | 小型有符号整数 |
| `unsigned char` | 明确无符号 | 原始字节存储 |

## 原理解析

C++ 标准把 `char`、`signed char`、`unsigned char` 视为三种**相互独立**的类型：

- `char` 的大小固定为 1 字节，但它到底是 signed 还是 unsigned 由实现决定。
- `signed char` 和 `unsigned char` 的符号性是标准明确规定的。
- 在函数重载中，这三个类型会被分别匹配：

```cpp
void f(char);
void f(signed char);
void f(unsigned char);

char c = 'A';
f(c);            // 调用 f(char)
f((signed char)c);   // 调用 f(signed char)
```

在模板推导中也是如此，传 `char` 推导出的 `T` 就是 `char`，不会因为平台 `char` 默认 signed 就变成 `signed char`。

## 常见误区

- ❌ 误以为 `char` 与 `signed char` 是同一类型 — 错误。三者是独立类型，在重载和模板推导中不可混用。
- ❌ `char` 默认有符号 — 错误。取决于平台和编译器，x86 上通常 signed，ARM 上通常 unsigned。

## 延伸点

C++17 起优先用 `std::byte` 替代 `unsigned char` 进行原始字节存储，语义更清晰。

> 相关阅读：`char` 参与算术运算时同样会先经过[整数提升](../28.md)，其提升目标是 `int` 还是 `unsigned int`，与 `char` 的符号性和平台宽度都有关。
