# 字符串字面量的特殊行为与陷阱

字符串字面量 `"abc"` 的类型是 `const char[4]`（含 `\0`），表达式中退化为 `const char*`。这个"数组退化"特性带来了很多容易踩坑的地方。

## 1. 类型：数组，不是指针

```cpp
auto s1 = "abc";           // const char*（数组退化）
decltype("abc") s2;        // const char[4]

sizeof("abc");             // 4（包含结尾 \0）
std::strlen("abc");        // 3（不包含 \0）
```

## 2. 字符串字面量位于只读数据段

字符串字面量通常存放在程序的只读数据段（RO data），修改它会产生未定义行为：

```cpp
const char* p = "hello";
// p[0] = 'H';             // ❌ 运行时崩溃或 UB
```

即使通过 `const_cast` 去掉 const 再写入，也是 UB。

## 3. `char*` 指向字符串字面量已弃用

```cpp
char* p = "abc";           // ❌ C++11 起弃用，应使用 const char*
const char* cp = "abc";    // ✅
```

旧代码中常见 `char* p = "..."`，现代 C++ 中应改为 `const char*`。

## 4. 相邻字符串字面量自动连接

```cpp
const char* s = "hello "
                "world";   // 等价于 "hello world"
```

这个特性常用于长字符串换行，但初学者可能误以为这是运行时拼接。

## 5. 原始字符串字面量（Raw String Literal）

对于包含大量反斜杠或引号的字符串，原始字符串字面量更直观：

```cpp
// 普通字符串：需要大量转义
const char* regex1 = "\\d+\\.\\d+";

// 原始字符串：R"(内容)"
const char* regex2 = R"(\d+\.\d+)";
```

如果需要字符串内容包含 `)"`，可以使用分隔符：

```cpp
const char* s = R"foo(包含 )" 的字符串)foo";
```

## 6. 宽字符与 UTF 字符串

```cpp
const wchar_t*  ws = L"中文";      // 宽字符
const char*     u8 = u8"中文";     // UTF-8（C++20 起类型为 char8_t）
const char16_t* u16 = u"中文";     // UTF-16
const char32_t* u32 = U"中文";     // UTF-32
```

> C++20 起 `u8"..."` 的类型是 `char8_t` 数组，与 `char` 不兼容，需要显式转换才能传给只接受 `char*` 的 API。

## 常见误区 / 踩坑点

- ❌ `"abc"` 是 `const char*` — 错误。真实类型是 `const char[4]`，仅在表达式中退化。
- ❌ `char* p = "abc"` 合法 — 错误。C++11 起弃用，应使用 `const char*`。
- ❌ 修改字符串字面量安全 — 错误。字符串字面量通常位于只读数据段，修改是运行时 UB。
- ❌ `sizeof("abc")` 等于 3 — 错误。等于 4，包含结尾 `\0`。
- ❌ 相邻字符串 `"a" "b"` 是运行时拼接 — 错误。编译期自动连接为 `"ab"`。

## 延伸点

`auto s = "abc"` 推导为 `const char*`（数组退化），而 `decltype("abc")` 保留 `const char[4]` — 这是 `auto` 与 `decltype` 在数组类型上的经典差异。如果需要保留数组类型，可以用 `decltype(auto)`（C++14）：

```cpp
decltype(auto) s = "abc";  // const char (&)[4]
```
