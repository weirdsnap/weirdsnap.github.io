# 数组名的类型

数组名在 C/C++ 中是一个**独立的类型**，而不是指针。理解数组名在不同语境下的行为，能解释很多初学者感到困惑的现象。

---

## 一、数组名的真实类型

```cpp
int arr[5] = {1, 2, 3, 4, 5};
```

`arr` 的类型是 `int[5]`，不是 `int*`。

```cpp
static_assert(std::is_same_v<decltype(arr), int[5]>);   // ✅ 数组类型
static_assert(!std::is_same_v<decltype(arr), int*>);    // ✅ 不是指针

sizeof(arr);   // 20 = 5 * sizeof(int)，不是 8
```

## 二、数组名退化为指针

数组名在大多数表达式中**自动退化为**指向首元素的指针：

```cpp
void func(int* p);

func(arr);        // arr 从 int[5] 退化为 int*
auto a = arr;     // int*
int* p = arr;     // ✅ 隐式转换
```

但退化不是总发生的。以下场景中数组名保留其完整类型：

| 场景 | 结果 | 示例 |
|------|------|------|
| `sizeof(arr)` | `5 * sizeof(int)` | `sizeof(arr)` → 20 |
| `decltype(arr)` | `int[5]` | 保留完整类型 |
| `&arr` | `int(*)[5]`（指向数组的指针） | 不是 `int**` |
| 引用绑定 `auto&` | `int(&)[5]` | 引用保留数组类型 |

## 三、`arr` 与 `&arr` 的区别

```cpp
int arr[5] = {};

auto a1 = arr;    // int* —— 指向首元素
auto a2 = &arr;   // int(*)[5] —— 指向整个数组

// 它们指向同一块内存地址，但类型不同，指针运算也不同
a1 + 1;    // 偏移 sizeof(int) 字节
a2 + 1;    // 偏移 sizeof(int[5]) = 20 字节
```

```cpp
int arr[5] = {};

// arr 的起始地址和 &arr 的起始地址相同
// 但它们的类型不同：
//   arr  → 指向首元素的指针（int*）
//   &arr → 指向整个数组的指针（int(*)[5]）

// 这相当于：
int*      p = arr;         // ✅ 指向 int
int(*     pa)[5] = &arr;   // ✅ 指向 int[5]
```

## 四、`arr` 与 `arr[0]` 的关系

`arr` 退化为指针后，指向 `arr[0]`，所以：

```cpp
int arr[5] = {};

arr;      // 退化为 &arr[0]
*arr;     // arr[0]
arr[0];   // 第一个元素
```

这也是为什么 `arr[i]` 等价于 `*(arr + i)`——退化为指针后再按指针偏移。

## 五、二维数组的数组名

二维数组的退化规则需要特别注意：

```cpp
int matrix[3][4] = {};

// matrix 的类型是 int[3][4]
// 退化为 int(*)[4]（指向 4 个元素的数组的指针），不是 int**
auto m = matrix;          // int(*)[4]
auto& mr = matrix;        // int(&)[3][4]

int** p = matrix;         // ❌ 编译错误！不能把 int(*)[4] 赋值给 int**
```

这意味着二维数组名退化后是一个**行指针**，不是指针的指针。

## 六、数组长度去哪儿了？

数组长度是类型的一部分，但一旦退化就丢失了：

```cpp
void func(int arr[]) {}     // 等价于 void func(int* arr) {}
```

所以函数参数里的 `int arr[]` 只是语法糖，完全等价于 `int* arr`，长度信息不会传到函数内。

保留数组长度的常见手法：

```cpp
// 1. 数组引用
template<typename T, size_t N>
void func(T (&arr)[N]) { /* N 可用 */ }

// 2. std::span（C++20）
void func(std::span<int> s) { /* s.size() 可用 */ }

// 3. std::array
void func(std::array<int, 5>& arr) { /* arr.size() 可用 */ }
```

## 七、常见误区

| 误区 | 正解 |
|------|------|
| ❌ 数组名就是指针 | ✅ 数组名类型是 `T[N]`，`sizeof` 返回完整大小 |
| ❌ `&arr` 的结果和 `arr` 完全一样 | ✅ 地址值相同，但类型不同，`&arr` 是指向数组的指针 |
| ❌ `arr + 1` 和 `&arr + 1` 偏移量相同 | ✅ 不同，`arr + 1` 偏移一个元素，`&arr + 1` 偏移整个数组 |
| ❌ 二维数组名可以赋值给 `int**` | ✅ 不能，二维数组退化后是 `int(*)[N]`，不是 `int**` |
| ❌ 函数参数 `int arr[]` 保留了数组长度 | ✅ 等价于 `int*`，长度信息已经丢失 |

## 八、一句话总结

> 数组名是 `T[N]`，不是指针；在表达式中大多退化为指针，但在 `sizeof`、`decltype`、`&` 和引用绑定时保留完整类型。
