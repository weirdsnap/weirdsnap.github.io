# `const` vs `constexpr`

`const` 承诺运行期**只读**，不保证编译期可求值；`constexpr` 要求编译期完成计算。

---

## 一、`const` 的两种语义

`const` 在不同语境下的含义不同：

```cpp
// 1. 运行期只读变量（最常用）
const int sz = std::rand();   // 运行期初始化，之后不能修改
// 编译期无法使用 sz（如 int arr[sz] 会报错）

// 2. 编译期常量（如果能）
const int n = 42;              // 编译期可求值，可作为数组大小
```

**关键认知**：`const` 只保证"不可修改"，不保证"编译期可知"。只有当初始化器是常量表达式时，它才同时是编译期常量。

---

## 二、`constexpr`：强制编译期

`constexpr` 明确要求编译期完成计算：

```cpp
constexpr int square(int x) {
    return x * x;
}

constexpr int a = square(5);      // ✅ 编译期求值
constexpr int b = std::rand();    // ❌ 编译错误！rand() 不是常量表达式
```

`constexpr` 变量必须是编译期常量，否则编译报错。`constexpr` 函数可以在编译期或运行期执行（取决于实参是否是常量）。

---

## 三、表达式

| 场景 | `const` | `constexpr` |
|------|---------|-------------|
| 编译期常量 | 不一定 | 必须是 |
| 初始化时机 | 运行期或编译期 | 必须编译期 |
| 函数修饰 | 不能 | 可以 |
| 函数调用方式 | N/A | 编译期或运行期均可 |
| 数组大小 | 仅常量表达式可 | 可以 |
| 模板参数 | 仅常量表达式可 | 可以 |

```cpp
const int a = 42;
constexpr int b = 42;

int arr1[a];       // ✅ a 是常量表达式
int arr2[b];       // ✅ b 是常量表达式

const int c = std::rand();
// int arr3[c];    // ❌ c 不是常量表达式
```

---

## 四、`constexpr` 函数的双重身份

`constexpr` 函数可以编译期或运行期调用：

```cpp
constexpr int twice(int x) { return x * 2; }

constexpr int a = twice(5);    // ✅ 编译期（实参是常量）
int n = 3;
int b = twice(n);               // ✅ 运行期（实参不是常量）

// constexpr int c = twice(n);  // ❌ 编译错误！n 不是常量
```

---

## 五、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `const` 变量一定是编译期常量 | ✅ 绝大多数不是，只有初始化器是常量表达式时才是 |
| ❌ `constexpr` 函数只能在编译期调用 | ✅ 传入运行期参数时在运行期执行 |
| ❌ `const int` 和 `constexpr int` 完全等价 | ✅ 对于字面量初始化，效果相同，但语义有区别 |
| ❌ `const` 修饰函数表示函数是常量 | ✅ `const` 修饰函数表示成员函数不修改对象，不是"常量函数" |

---

## 六、延伸点

- C++20 引入 `consteval`（强制编译期）和 `constinit`（强制编译期初始化但不隐含 `const`），与 `constexpr` 构成完整谱系，详见 [ch09/04](../09/04.md)。
- 模板参数、`static_assert` 的条件、`std::array` 的大小等语境**必须**是编译期常量，此时 `constexpr` 比 `const` 更安全（编译期检查）。
