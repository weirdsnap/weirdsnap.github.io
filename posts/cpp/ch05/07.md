# 虚继承内存布局详解

虚继承（`virtual public`）解决菱形继承的基类多份拷贝问题，代价是更复杂的**运行时偏移计算**。

---

## 一、菱形继承：没有虚继承会怎样

```cpp
struct A { int a; };
struct B : A { int b; };
struct C : A { int c; };
struct D : B, C { int d; };
```

`D` 对象中有**两份** `A` 的副本：

```
D 对象（非虚继承）：
┌────────────┐  ← this (D*)
│ A part     │  ← B 中的 A
│   int a    │
├────────────┤
│ B part     │
│   int b    │
├────────────┤
│ A part     │  ← C 中的 A（第二份）
│   int a    │
├────────────┤
│ C part     │
│   int c    │
├────────────┤
│ D part     │
│   int d    │
└────────────┘

D d;
// d.a;           // ❌ 二义性：哪个 A::a？
d.B::a = 1;        // ✅ 指定 B 路径
d.C::a = 2;        // ✅ 指定 C 路径，两者互不影响
```

---

## 二、虚继承后：共享一份虚基类

```cpp
struct A { int a; };
struct B : virtual public A { int b; };
struct C : virtual public A { int c; };
struct D : B, C { int d; };
```

`D` 对象中只有**一份** `A`：

```
D 对象（虚继承）：
┌────────────────────┐  ← this (D*)
│ B vbptr            │  ← 指向 B 的虚基类偏移表
├────────────────────┤
│ B part: int b      │
├────────────────────┤
│ C vbptr            │  ← 指向 C 的虚基类偏移表
├────────────────────┤
│ C part: int c      │
├────────────────────┤
│ D part: int d      │
├────────────────────┤
│ A part             │  ← 唯一的共享 A，在对象末尾
│   int a            │
└────────────────────┘

D d;
d.a = 42;        // ✅ 现在唯一，没有二义性
d.B::a = 1;      // ✅ 和 d.C::a 指向同一个 int
d.C::a = 2;      // d.a 现在是 2
```

**关键**：`A` 的偏移不是编译期固定的——它由**最终派生类的具体布局**决定，通过 vbptr 在运行时查找。

---

## 三、vbptr 与偏移量表

编译器为每个含虚继承的类生成一张**虚基类偏移表（vbase offset table）**：

```
B 的虚基类偏移表（当 B 作为最终派生类的子对象时）：
┌────────────────────────────┐
│ offset_to_A = 32           │ ← B 起始到 A 的字节数
└────────────────────────────┘

C 的虚基类偏移表：
┌────────────────────────────┐
│ offset_to_A = 16           │ ← C 起始到 A 的字节数
└────────────────────────────┘
```

访问 `d.a` 时，编译器生成的代码近似：

```cpp
// 通过 B* 访问 A
A& B::getA() {
    // 通过 B 的 vbptr 读取偏移量
    int offset = this->vbptr_B[0];   // 读取偏移
    return *reinterpret_cast<A*>(reinterpret_cast<char*>(this) + offset);
}
```

---

## 四、三种访问路径的偏移对比

| 访问方式 | 路径 | 偏移确定时机 |
|----------|------|-------------|
| `D::a` | 直接访问（编译期固定偏移） | 编译期 |
| `B::a` | vbptr → 偏移表 → A | 运行时 |
| `C::a` | vbptr → 偏移表 → A | 运行时 |

```cpp
D d;
D* pd = &d;
B* pb = &d;
C* pc = &d;

// 这三个指针值可能完全不同！
// pd → D 起始
// pb → B 部分起始（通常等于 D 起始）
// pc → C 部分起始（pd + sizeof(B 部分)）

// 但 pb->a, pc->a, pd->a 都指向同一个 A 的 int a
```

## 五、性能代价

| 操作 | 非虚继承 | 虚继承 |
|------|---------|--------|
| 成员变量访问 | 编译期固定偏移 | 运行时查表间接寻址 |
| `static_cast<A*>` | 固定偏移 | 运行时查表 |
| 对象大小 | 各基类独立累加 | 较紧凑（共享一份） |
| 构造顺序 | 按声明顺序 | 虚基类优先构造 |

---

## 六、常见误区

- ❌ 虚继承后所有成员访问都变慢 — ✅ 只影响虚基类的成员，非虚基类的成员访问不变。
- ❌ 虚基类由中间类构造 — ✅ 虚基类始终由**最底层派生类**构造，中间类的初始化列表中对虚基类的初始化被忽略。
- ❌ 虚继承会让所有派生类都变大 — ✅ 只有通过虚继承的派生类会增加 vbptr，不影响不相关的类。
- ❌ 虚继承和虚函数必须同时出现 — ✅ 两者无关，可以只有虚继承没有虚函数。

---

## 七、一句话总结

> 虚继承通过 vbptr + 偏移表在运行时确定虚基类的位置，换来菱形继承的唯一性；代价是一次间接寻址和更大的对象体积。
