# C++ 运行时多态概述

运行时多态（Runtime Polymorphism）是 C++ 面向对象编程的核心：**通过基类指针或引用，在运行时调用派生类的重写函数**。实现它的载体是虚函数（`virtual`），底层依赖虚函数表（vtable）。

---

## 一、什么是运行时多态

多态（Polymorphism）意为"多种形态"。在 C++ 中，它指：**同一段代码（基类指针的调用）在不同派生类对象上表现出不同的行为**。

```cpp
class Animal {
public:
    virtual void speak() { std::cout << "Animal speaks\n"; }
};

class Dog : public Animal {
public:
    void speak() override { std::cout << "Dog barks\n"; }
};

class Cat : public Animal {
public:
    void speak() override { std::cout << "Cat meows\n"; }
};

void makeSound(Animal& a) {
    a.speak();  // 运行时决定调用哪个 speak()
}

Dog dog;
Cat cat;
makeSound(dog);  // Dog barks
makeSound(cat);  // Cat meows
```

**核心**：`makeSound` 只认识 `Animal&`，但实际行为取决于传入的**动态类型**。

---

## 二、运行时多态 vs 编译期多态

| 特性 | 运行时多态（虚函数） | 编译期多态（模板/重载） |
|------|---------------------|------------------------|
| 派发时机 | 运行时（vtable 查找） | 编译期（展开/决议） |
| 类型关系 | 继承，`is-a` 关系 | 结构约束（duck typing） |
| 性能开销 | 一次间接跳转 + 无法内联 | 零开销，可内联 |
| 对象大小 | 增加一个 vptr（通常 8 字节） | 不增加 |
| 灵活性 | 运行时动态绑定，可替换 | 编译期确定，类型固定 |
| 典型场景 | 插件、框架回调、UI 事件 | 算法泛型、容器、策略选择 |

## 三、动态绑定的三个必要条件

调用虚函数触发动态绑定**必须同时满足**：

1. **通过指针或引用调用** — 值传递会触发对象切片，绑定的是静态类型
2. **调用的函数是虚函数** — 非虚函数在编译期静态决议
3. **对象的动态类型与静态类型不同** — 基类指针指向派生类对象

```cpp
Animal animal;
Dog dog;

Animal* p = &dog;
p->speak();       // ✅ 动态多态：Dog barks

Animal& r = dog;
r.speak();        // ✅ 动态多态：Dog barks

Animal a = dog;   // ❌ 切片
a.speak();        // Animal speaks（静态绑定到 Animal::speak）
```

## 四、纯虚函数与抽象基类

`= 0` 声明纯虚函数，包含纯虚函数的类称为**抽象基类**（Abstract Base Class, ABC），不可实例化：

```cpp
class Shape {
public:
    virtual double area() const = 0;    // 纯虚函数
    virtual ~Shape() = default;
};

// Shape s;              // ❌ 编译错误：不能实例化抽象类
class Circle : public Shape {
    double r;
public:
    double area() const override { return 3.14159 * r * r; }
};
```

抽象基类定义**接口契约**，派生类必须实现所有纯虚函数才能被实例化。

## 五、虚析构函数

当通过基类指针 `delete` 派生类对象时，若析构函数不是虚函数，行为是**未定义（UB）**：

```cpp
class Base {
public:
    ~Base() { std::cout << "~Base\n"; }   // ❌ 非虚
};

class Derived : public Base {
    int* data = new int[100];
public:
    ~Derived() { delete[] data; std::cout << "~Derived\n"; }
};

Base* p = new Derived();
delete p;   // ❌ 只调用了 ~Base()，data 泄漏
```

**规则**：只要类有虚函数，就应该将析构函数声明为 `virtual`（或 `= default`）。

## 六、常见误区

| 误区 | 正解 |
|------|------|
| ❌ 值传递也能实现多态 | ✅ 值传递触发切片，`virtual` 调用按静态类型决议 |
| ❌ 基类指针只能调基类方法 | ✅ 调虚函数时若有覆盖则走派生类版本 |
| ❌ `virtual` 永远加在派生类 | ✅ 派生类覆盖时 `override` 即可，`virtual` 可选（但不推荐写） |
| ❌ 虚函数一定比普通函数慢很多 | ✅ 单次间接跳转，现代分支预测下开销通常可忽略 |
| ❌ 构造函数里调虚函数也会多态 | ✅ 构造期间动态类型是当前正在构造的类，不会派发到派生类 |

---

## 七、延伸点

- 运行时多态的底层依赖**虚函数表（vtable）**与**虚指针（vptr）**，详见下一篇文章。
- `final` 关键字可阻止虚函数被进一步覆盖：`void speak() override final;`
- 关闭 RTTI（`-fno-rtti`）时，`dynamic_cast` 和 `typeid` 对多态类型不可用，但虚函数调用仍然正常（vtable 本身不依赖 RTTI）。
- 现代 C++ 中，`std::function` + `lambda` 提供了一种**非继承的运行时多态**替代方案（类型擦除），适合不需要 `is-a` 关系的场景。
