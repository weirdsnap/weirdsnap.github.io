# 无锁编程：compare_exchange 与原子操作

`std::atomic` 的 `compare_exchange`（CAS，Compare-And-Swap）是无锁编程的基础原语：**原子地比较当前值是否等于预期值，如果是则替换为新值，否则更新预期值为当前值**。

---

## 一、基本语法

```cpp
std::atomic<int> atomic_val{0};

int expected = 0;
int desired = 42;

// CAS：如果 atomic_val == expected，则设为 desired，否则 expected 更新为 atomic_val
bool success = atomic_val.compare_exchange_weak(expected, desired);
```

### `weak` vs `strong`

| 版本 | 行为 | 适用场景 |
|------|------|---------|
| `compare_exchange_weak` | 可能**虚假失败**（即使值匹配） | 循环中重试（LL/SC 架构上的优化） |
| `compare_exchange_strong` | 保证值匹配时返回 `true` | 循环次数少或不适合重试的场景 |

```cpp
// weak 版本需要在循环中使用
int expected = 0;
int desired = 42;
while (!atomic_val.compare_exchange_weak(expected, desired)) {
    expected = 0;  // 重置 expected（CAS 失败时 expected 会被更新）
}

// strong 版本不需要循环（如果只是简单替换）
bool ok = atomic_val.compare_exchange_strong(expected, desired);
```

## 二、实现自旋锁

```cpp
class SpinLock {
    std::atomic<bool> locked_{false};
public:
    void lock() {
        bool expected = false;
        while (!locked_.compare_exchange_weak(expected, true)) {
            expected = false;        // CAS 失败，expected 被修改，要重置
            // 可选：CPU pause 提示
            #if defined(__x86_64__)
                __builtin_ia32_pause();
            #endif
        }
    }

    void unlock() {
        locked_.store(false, std::memory_order_release);
    }
};
```

## 三、实现无锁栈

```cpp
template<typename T>
class LockFreeStack {
    struct Node {
        T data;
        Node* next;
    };
    std::atomic<Node*> head_{nullptr};
public:
    void push(T val) {
        Node* new_node = new Node{std::move(val), nullptr};
        new_node->next = head_.load(std::memory_order_relaxed);
        while (!head_.compare_exchange_weak(
            new_node->next, new_node,
            std::memory_order_release, std::memory_order_relaxed)) {
            // CAS 失败时 new_node->next 已被更新为 head 的最新值
        }
    }

    bool pop(T& val) {
        Node* old_head = head_.load(std::memory_order_relaxed);
        while (old_head && !head_.compare_exchange_weak(
            old_head, old_head->next,
            std::memory_order_acquire, std::memory_order_relaxed)) {
            // CAS 失败，继续重试（可能 head 已被其他线程修改）
        }
        if (!old_head) return false;
        val = std::move(old_head->data);
        delete old_head;  // ⚠️ 注意：简单实现有 ABA 问题
        return true;
    }
};
```

## 四、ABA 问题

```cpp
// 场景：线程 1 读取 head → A
//       线程 1 被调度出去
//       线程 2 pop A, 线程 3 push A（A 的地址被回收后再分配）
//       线程 1 恢复，CAS(A) 成功——但 A 已经不是原来的 A 了
```

**解决方案**：
- 带标记的指针（tagged pointer）：在指针中嵌入版本号
- 引用计数 / 风险指针（hazard pointer）
- 不用 `delete`，只在确实安全时释放

## 五、常见误区

- ❌ CAS 保证无锁编程总是更快 — ✅ 争用激烈时自旋锁比互斥锁更差，CAS 循环消耗 CPU。
- ❌ `compare_exchange_weak` 比 `strong` 慢 — ✅ 在 x86 上两者差不多；在 ARM LL/SC 上 weak 更快。
- ❌ CAS 成功后不需要考虑内存序 — ✅ 错误，push 需要 `release`，pop 需要 `acquire`。
- ❌ 无锁数据结构不会死锁 — ✅ 不会死锁，但可能活锁（livelock，循环重试无法进展）。
- ❌ ABA 问题只影响无锁栈 — ✅ 影响任何基于指针 CAS 的无锁数据结构。

---

## 六、一句话总结

> `compare_exchange_weak` 在循环中重试是大多数无锁数据结构的核心模式，但要警惕 ABA 问题和内存序正确性。
