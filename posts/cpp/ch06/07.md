# 数据竞争（基础）

C++11 起，多线程访问同一内存位置且至少一个为写操作、无 happens-before 同步关系时，即构成数据竞争，后果是 UB。

| 场景 | 是否构成数据竞争 |
|------|------------------|
| 多线程只读同一内存 | ❌ 否 |
| 一写一读，无同步 | ✅ 是 |
| 多写，无同步 | ✅ 是 |
| 使用 `std::atomic` / `mutex` | ❌ 否（同步保护） |

**原理解析 + 常见误区**

- ❌ "多线程写才会引发数据竞争" — 不完整。一读一写无同步同样构成数据竞争。
- ❌ "同一类的成员变量就是共享数据" — 错误。只有同一对象实例的成员变量被多线程访问才是共享。
- ❌ `volatile` 可替代锁/原子 — 错误。`volatile` 只保证内存可见性，完全不提供原子性或同步语义。

**延伸点**：引用计数操作（`shared_ptr` 的 `+1/-1`）是线程安全的原子操作，但 `shared_ptr` 对象本身的读写（拷贝/赋值/reset）非线程安全，需外部同步。
