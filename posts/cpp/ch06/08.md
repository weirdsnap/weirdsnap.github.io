# std::thread 与 std::jthread

线程是 C++ 并发编程的起点。C++11 提供了 `std::thread`，C++20 新增了 `std::jthread`（joining thread），后者会自动 join，并支持可取消的协作式中断。

---

## 一、`std::thread` 基础

```cpp
#include <thread>
#include <iostream>

void hello() {
    std::cout << "Hello from thread\n";
}

int main() {
    std::thread t(hello);
    t.join();   // 等待线程结束
}
```

### join / detach

```cpp
std::thread t(f);
t.join();    // ✅ 等待 t 完成
// t.detach();  // ✅ 分离，t 独自运行

// 必须 join 或 detach 其中一个，否则析构时调用 std::terminate
```

**关键规则**：
- 必须调用 `join()` 或 `detach()`，否则析构时触发 `std::terminate`
- `join()` 只能调用一次，第二次会抛 `std::system_error`
- 可先检查 `t.joinable()` 再 join

---

## 二、传参

```cpp
void work(int id, const std::string& msg) {
    std::cout << id << ": " << msg << "\n";
}

int main() {
    // 参数直接跟在函数后面
    std::thread t(work, 42, "hello");

    // 注意：引用参数需要显式用 std::ref
    std::string s = "world";
    std::thread t2(work, 43, std::cref(s));

    t.join();
    t2.join();
}
```

**陷阱**：如果参数是引用类型而没有用 `std::ref`，会按值传递并拷贝，修改不会反映到原变量。

## 三、`std::jthread`（C++20）

`std::jthread` 是对 `std::thread` 的改进：

```cpp
#include <thread>

void f() {
    // 模拟一些工作
    for (int i = 0; i < 1000000; ++i);
}

int main() {
    std::jthread jt(f);
    // 不需要显式调用 jt.join()，析构时自动 join
}  // jt 离开作用域，析构自动 join
```

### 两个核心改进

| 特性 | `std::thread` | `std::jthread` |
|------|---------------|----------------|
| 析构行为 | `std::terminate`（如果未 join/detach） | 自动 `join()` |
| 协作式中断 | 不支持 | 支持 `request_stop()` / `stop_token` |

### 协作式中断

```cpp
#include <thread>
#include <chrono>

void loop(const std::stop_token& st) {
    while (!st.stop_requested()) {
        std::cout << "Working...\n";
        std::this_thread::sleep_for(std::chrono::seconds(1));
    }
    std::cout << "Stopped!\n";
}

int main() {
    std::jthread jt(loop);
    std::this_thread::sleep_for(std::chrono::seconds(3));
    jt.request_stop();  // 通知线程停止
    // jt 析构自动 join
}
```

## 四、常见误区

| 误区 | 正解 |
|------|------|
| ❌ 不 join/detach 也能正常结束 | ✅ 析构时如果还 joinable，直接 `std::terminate` |
| ❌ `jthread` 会自动 join 所以不需要关心生命周期 | ✅ 自动 join 在析构时，如果期间抛异常，对象生命周期提前结束，仍可能有问题 |
| ❌ `request_stop()` 强制停止线程 | ✅ 只是请求，线程需要主动检查 `stop_token` 才能响应 |
| ❌ 传引用参数不需要 `std::ref` | ✅ 默认按值传递，引用必须用 `std::ref` 或 `std::cref` |

---

## 五、一句话总结

> `std::thread` 创建线程必须手动 join/detach；`std::jthread` 自动 join 且支持协作式中断，是现代 C++ 的首选。
