# std::async / std::future / std::promise

C++ 提供了三种机制将异步任务的结果**从工作线程传递回主线程**：`std::async`（最简）、`std::packaged_task`（中等）、`std::promise`（最灵活）。

---

## 一、`std::async`：一键异步

```cpp
#include <future>
#include <iostream>

int slow_square(int x) {
    std::this_thread::sleep_for(std::chrono::seconds(1));
    return x * x;
}

int main() {
    // 异步启动，返回 future 句柄
    std::future<int> result = std::async(slow_square, 5);

    std::cout << "Doing other work...\n";

    // 获取结果（阻塞直到就绪）
    std::cout << result.get();   // 输出 25
}
```

### 启动策略

```cpp
// 默认策略：由编译器决定是否惰性执行
auto f1 = std::async(slow_square, 5);

// 强制在新线程执行
auto f2 = std::async(std::launch::async, slow_square, 5);

// 惰性求值：只在 get() 调用时执行（当前线程）
auto f3 = std::async(std::launch::deferred, slow_square, 5);
```

| 策略 | 执行时机 | 执行线程 |
|------|---------|---------|
| `launch::async` | 立即 | 新线程 |
| `launch::deferred` | `get()` 时 | 调用 `get()` 的线程 |
| 默认（两者都有） | 由编译器选择 | 不确定 |

## 二、`std::packaged_task`：可调用的异步任务

```cpp
#include <future>

int add(int a, int b) { return a + b; }

int main() {
    // 把函数包装成可异步调用的任务
    std::packaged_task<int(int, int)> task(add);
    std::future<int> result = task.get_future();

    // 在另一个线程执行
    std::thread t(std::move(task), 3, 4);
    t.join();

    std::cout << result.get();  // 7
}
```

`packaged_task` 适合把任务**丢给线程池**或**队列**管理的场景。

---

## 三、`std::promise`：手动设置值/异常

当你想在任意线程**手动触发**一个值时，用 `std::promise`：

```cpp
#include <future>

void worker(std::promise<int> p) {
    try {
        int result = 42;  // 模拟计算
        p.set_value(result);
    } catch (...) {
        p.set_exception(std::current_exception());
    }
}

int main() {
    std::promise<int> prom;
    std::future<int> fut = prom.get_future();

    std::thread t(worker, std::move(prom));
    t.detach();

    std::cout << fut.get();  // 42，阻塞直到 set_value 被调用
}
```

典型场景：需要在线程执行过程中**多次**传递状态（用 `shared_future`）或从**回调**中触发.

---

## 四、`shared_future`：多线程共享结果

```cpp
std::promise<int> prom;
std::shared_future<int> sf = prom.get_future().share();

// 多个线程可以同时读取结果
auto reader1 = std::async([sf]() { return sf.get(); });
auto reader2 = std::async([sf]() { return sf.get(); });
```

`std::future` 只能调用一次 `get()`，`shared_future` 可以多次，且线程安全。

---

## 五、三者的选择

| 工具 | 适用场景 | 是否需要手动管理线程 |
|------|---------|-------------------|
| `std::async` | 简单的异步调用，想要结果 | ❌ 自动 |
| `std::packaged_task` | 需要把任务交给线程池/队列 | ✅ 需要控制执行线程 |
| `std::promise` | 需要从回调或任意位置触发结果 | ✅ 完全手动 |

---

## 六、常见误区

- ❌ `future::get()` 可以多次调用 — ✅ 只能调用一次，第二次调用抛异常（`shared_future` 除外）。
- ❌ `std::async` 总是创建新线程 — ✅ 默认策略可能惰性执行（`deferred`），需要明确指定 `launch::async`。
- ❌ `promise` 和 `future` 是跨线程通信的唯一方式 — ✅ 还可以用 `condition_variable` + 共享变量。
- ❌ 异步任务抛异常时 `future::get()` 会返回错误码 — ✅ 会抛出原始异常（在调用 `get()` 的线程）。

---

## 七、一句话总结

> `std::async` 是傻瓜式异步，`std::packaged_task` 是带返回值的可调用对象，`std::promise` 是最底层的"设值"原语——三者都通过 `std::future` 获取结果。
