# 模板静态成员 vs 非模板静态成员

模板静态成员遵循按需实例化（ODR-use 驱动），非模板静态成员在程序启动前（`main` 之前）完成初始化。

| 维度 | 模板静态成员 | 非模板静态成员 |
|------|-------------|---------------|
| 初始化时机 | 首次 ODR-use 时隐式实例化 | 程序启动前（`main` 之前） |
| 与使用关系 | 不用不生成，不初始化 | 只要链接进程序，无论是否使用 |
| 定义位置 | 通常头文件 | 通常单个 cpp |

**常见误区**

- ❌ 模板静态成员在模板定义时就生成 — 错误。模板静态成员在模板定义时仅有声明，直到某处代码 ODR-use 时才触发隐式实例化。
- ❌ 未使用的非模板静态成员不会初始化 — 错误。只要目标文件被链接进最终可执行文件，其静态成员就会在 `main` 前初始化。

**案例解析**

```cpp
#include <iostream>

struct Printer {
    Printer() { std::cout << 1; }
};

template <typename T>
struct C {
    C(T i) { std::cout << i; }
    static inline Printer printer{};
};

int main() {
    C c(2);  // C++17 CTAD：推导为 C<int>
}
// 输出：2
```

输出只有 `2`，`Printer` 的构造函数从未执行。原因是 `C<int>::printer` 虽然声明了，但从未被 ODR-use——`main` 中只构造了 `C<int>` 对象，没有以任何方式引用 `printer`。根据 `[temp.inst]¶4`，静态数据成员的初始化不会发生，除非该成员本身被使用且需要其定义存在。

**每个模板实例独立拥有静态成员**

类模板的静态成员是按模板参数分实例独立存储的：

```cpp
C<int> a(1);   // 实例化 C<int>::printer（如果被使用）
C<int> b(2);   // 复用 C<int>::printer，同一个对象
C<char> c(3);  // 实例化 C<char>::printer，与 C<int>::printer 完全独立
```

`C<int>` 和 `C<char>` 即使共享同一个类模板定义，它们的 `printer` 也是两个完全不同的静态对象。

**延伸点**

- C++17 `inline` 变量允许非模板静态成员在头文件中定义且跨翻译单元合并，打破了传统"声明在头、定义在 cpp"的分离编译限制。
- CTAD（C++17 类模板参数推导）通过 `C c(2)` 自动推导为 `C<int>`，使得代码简洁但容易忽略 `printer` 的存在。
- `[temp.inst]¶4` 明确指出：静态数据成员的初始化仅在被 ODR-use 时发生，不会因所在类被实例化而连带初始化。
