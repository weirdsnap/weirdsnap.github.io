# 七、标准库与工程实践（91–94）

---

### 91. `std::cout` 默认输出类型

`std::cout` 通过 `operator<<` 重载支持内置算术类型、指针、C 字符串及部分标准库类型。

| 类型 | 默认输出形式 | 陷阱 |
|------|-------------|------|
| `bool` | `0` / `1` | 不是 `true`/`false`；需 `std::boolalpha` |
| `char` | **单个字符** | 不是 ASCII 数值；需显式转 `int` |
| `char*` | C 字符串 | 想输出地址必须显式转 `void*` |
| `std::nullptr_t` | 未定义/编译错误 | 应转 `const void*` |

**原理解析 + 常见误区**

- ❌ `char` 输出 ASCII 码 — 错误。`std::cout << 'a'` 输出字符 `a`。
- ❌ `std::cout << nullptr` 输出地址 — 错误。`std::nullptr_t` 没有标准 `operator<<` 重载。

**延伸点**：自定义 `operator<<` 的标准签名：`std::ostream& operator<<(std::ostream& os, const T& obj)`，返回 `ostream&` 支持链式调用。

### 92. `std::function` 类型擦除

`std::function` 通过虚函数机制擦除具体可调用对象的类型，统一存储函数指针、Lambda、仿函数等。

| 特性 | 直接模板调用 | `std::function` |
|------|-------------|----------------|
| 类型信息 | 编译期确定，内联友好 | 运行期擦除，难以内联 |
| 内存分配 | 栈上直接存储 | 大对象堆分配（SBO 除外） |
| 异构容器 | ❌ 无法放入同一容器 | ✅ 同一类型可存不同可调用对象 |

**原理解析 + 常见误区**

- ❌ `std::function` 和函数指针一样快 — 错误。`std::function` 多一层虚分派，且无法内联。
- ❌ 空 `std::function` 调用是 UB — 错误。调用空 `std::function` 会抛 `std::bad_function_call`。

**延伸点**：C++23 `std::move_only_function` 支持移动语义避免拷贝分配；`llvm::function_ref` 是不拥有可调用对象的零拷贝视图。

### 93. `argv[argc]` 的明确定义

C++ 标准明确定义 `argv[argc] == 0`（空指针），访问它不是数组越界。

**场景示例：**
```cpp
int main(int argc, char* argv[]) {
    std::cout << (argv[argc] == nullptr);  // 输出 1，明确定义
}
```

**原理解析 + 常见误区**

- ❌ `argv[argc]` 是越界 UB — 错误。标准对此有明确例外规定。
- ❌ 所有 C 风格数组末尾后一位都保证为 0 — 错误。仅 `main` 的 `argv` 有此保证。

**延伸点**：可利用 `argv[argc] == nullptr` 作为遍历终止条件，比显式比较 `i < argc` 更符合 C 风格字符串数组的惯用写法。

### 94. 常见的未定义行为（UB）

| UB 场景 | 典型示例 | 常见后果 |
|---------|---------|---------|
| Use-after-free | `delete p; *p;` | 堆损坏、信息泄漏 |
| Double-free | `delete p; delete p;` | 堆管理器元数据破坏 |
| 缓冲区溢出 | `arr[n]` 其中 `n >= size` | 栈/堆金丝雀破坏 |
| 未初始化使用 | `int x; cout << x;` | 垃圾值、信息泄漏 |
| `this == nullptr` 调用成员 | `((T*)nullptr)->f();` | 若 `f` 未访问成员可能"正常"，但标准定义为 UB |

**原理解析 + 常见误区**

- ❌ UB 一定会崩溃 — 错误。UB 最常见的表现是"看似正常"，导致 bug 隐蔽且难以复现。
- ❌ 调试模式能捕获所有 UB — 错误。use-after-free 和未初始化读取在调试模式下同样可能静默通过。

**延伸点**：AddressSanitizer (ASan) 和 MemorySanitizer (MSan) 是检测 UB 的主要工具，但只能检测运行时触发的路径。
