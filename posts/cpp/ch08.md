# 八、异常安全与未定义行为（95–98）

---

### 95. `noexcept`

`noexcept` 是函数对调用者的编译期契约，承诺不抛出异常；若违反则调用 `std::terminate`。

| 特性 | `noexcept` | 无标记 |
|------|-----------|--------|
| 异常传播 | 触发 `std::terminate` | 正常传播 |
| 栈展开保证 | ❌ 不保证 | ✅ 保证 |
| 容器移动（vector 扩容） | ✅ 优先使用 | ❌ 回退拷贝 |

**原理解析 + 常见误区**

- ❌ `noexcept` 只是编译器优化提示，违反也没事 — 错误。违反即终止程序，不保证对象析构。
- ❌ 所有函数都该标 `noexcept` — 错误。仅对真正不抛的函数标记，尤其是移动构造、移动赋值、析构、交换函数。

**延伸点**：`noexcept` 运算符（编译期检查）配合条件 `noexcept` 可实现模板函数的异常安全转发。

---

### 96. `noexcept` 工程实践

`noexcept` 的正确使用遵循"契约即承诺"原则：仅对真正不抛的函数标记，移动语义必须标记以激活标准库优化。

| 场景 | 标记策略 |
|------|---------|
| 移动构造/赋值 | ✅ **必须标记** |
| 析构函数 | 通常不标（C++11 起隐式 `noexcept`） |
| 可能分配内存 | ❌ **不标** |
| 泛型转发函数 | 条件 `noexcept(...)` |

**延伸点**：移动构造忘记 `noexcept` 的代价不是慢一点，而是算法复杂度从 O(1) 退化为 O(n)（`std::vector` 扩容时回退到拷贝构造）。

---

### 97. 析构函数抛出异常与栈展开

析构函数默认隐式 `noexcept(true)`，若在栈展开期间抛出异常且未被捕获，或因违反 `noexcept` 契约，均触发 `std::terminate`。

| 场景 | 结果 |
|------|------|
| 默认 `noexcept` 析构抛异常 | 立即 `terminate`（违反契约） |
| 栈展开期间析构抛异常 | 立即 `terminate`（双异常规则） |
| 正常调用，显式 `noexcept(false)` 析构抛异常 | 异常传播（极危险，不推荐） |

**原理解析 + 常见误区**

- ❌ 析构函数可以抛异常，外面 catch 住就行 — 错误。默认 `noexcept` 的析构函数抛出异常不会进入任何 catch 块。
- ❌ `terminate` 前会完成其他对象的析构 — 错误。`std::terminate` 是立即终止，不保证调用其他析构函数。

**延伸点**：析构函数中绝对禁止抛出异常；若内部操作可能抛，应在析构函数内用 `try-catch` 就地吞掉异常并记录日志。

### 98. `return` 语句析构异常与返回对象销毁

C++23 规定：在 `return` 语句触发局部变量销毁期间若抛出异常，已构造的返回对象也必须在栈展开中被销毁。

**场景示例：**
```cpp
A f() {
    try {
        A a('a');
        Y y;  // ~Y() noexcept(false)，抛出异常
        A b('b');
        return {'c'};  // 返回对象构造完成
    } catch (...) {}
    return {'d'};
}
// 标准保证输出：bcad
// GCC 实际：bacd；Clang/MSVC 实际：bad
```

**原理解析 + 常见误区**

- ❌ 返回对象只在函数正常结束时销毁 — 错误。标准明确要求异常路径中返回对象也要被销毁。
- ❌ 所有编译器都遵循标准输出 bcad — 错误。GCC 输出 `bacd`，Clang/MSVC 输出 `bad`。

**延伸点**：此规则揭示了返回值生命周期的边界情况：即使 RVO/NRVO 将返回对象构造在调用者栈帧，若 `return` 语句触发局部变量销毁时异常抛出，返回对象仍被视为"已构造的自动对象"而需销毁。
