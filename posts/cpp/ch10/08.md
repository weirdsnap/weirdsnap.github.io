# std::filesystem 入门：路径操作与文件遍历

C++17 `std::filesystem` 提供了跨平台的文件系统操作，替代了 C 风格的 `stat`/`opendir` 和 POSIX 的路径字符串拼接。

---

## 一、路径操作

### 构建与分解路径

```cpp
#include <filesystem>
namespace fs = std::filesystem;

fs::path p = "/home/user/docs/file.txt";

std::cout << p.root_name();      // ""（Windows 上可能是 "C:"）
std::cout << p.root_directory(); // "/"
std::cout << p.parent_path();    // "/home/user/docs"
std::cout << p.filename();       // "file.txt"
std::cout << p.stem();           // "file"（无扩展名）
std::cout << p.extension();      // ".txt"

// 拼接路径（自动处理分隔符）
fs::path dir = "/home/user";
fs::path full = dir / "docs" / "file.txt";  // ✅ 用 / 运算符
```

### 路径检查

```cpp
fs::path p = "test.txt";

bool exists    = fs::exists(p);       // 文件或目录是否存在
bool is_reg    = fs::is_regular_file(p); // 是普通文件
bool is_dir    = fs::is_directory(p);  // 是目录
bool is_sym    = fs::is_symlink(p);    // 是符号链接
auto file_size = fs::file_size(p);     // 文件大小（字节）
auto last_wr   = fs::last_write_time(p); // 最后修改时间
```

## 二、目录遍历

```cpp
// 1. 非递归遍历当前目录
for (const auto& entry : fs::directory_iterator(".")) {
    std::cout << entry.path() << "\n";
}

// 2. 递归遍历所有子目录（深度优先）
for (const auto& entry : fs::recursive_directory_iterator(".")) {
    if (entry.is_regular_file()) {
        std::cout << entry.path() << " (" << entry.file_size() << " bytes)\n";
    }
}

// 3. 控制递归深度
fs::recursive_directory_iterator it(".", fs::directory_options::skip_permission_denied);
it.disable_recursion_pending();  // 跳过当前目录的子目录
```

## 三、文件操作

```cpp
fs::path src = "source.txt";
fs::path dst = "backup/source.txt";

fs::copy(src, dst);                              // 复制文件
fs::copy(src, dst, fs::copy_options::overwrite_existing);  // 覆盖
fs::rename(src, dst);                            // 重命名/移动
fs::remove("old.txt");                           // 删除文件
fs::remove_all("temp_dir");                      // 递归删除目录
fs::create_directory("new_dir");                 // 创建单层目录
fs::create_directories("a/b/c/d");               // 递归创建多层目录
```

## 四、异常处理

大多数 filesystem 操作在失败时抛 `fs::filesystem_error`，也可以使用不抛异常的重载：

```cpp
std::error_code ec;

// 有异常版本
if (fs::exists("maybe_not_there.txt")) { }

// 无异常版本（返回错误码）
bool ok = fs::exists("maybe_not_there.txt", ec);
if (ec) {
    std::cout << ec.message() << "\n";
}
```

## 五、常见误区

| 误区 | 正解 |
|------|------|
| ❌ 路径字符串用 `/` 拼接 | ✅ 用 `/` 运算符，自动处理跨平台分隔符 |
| ❌ `exists` 返回 false 说明文件不存在 | ✅ 也可能是权限不足，检查 `ec` 更准确 |
| ❌ `directory_iterator` 会遍历子目录 | ✅ 不会，需要 `recursive_directory_iterator` |
| ❌ 删除非空目录用 `remove` | ✅ `remove` 只能删空目录或文件，非空用 `remove_all` |

---

## 六、一句话总结

> `std::filesystem` 是 C++17 的跨平台文件系统 API，用 `/` 运算符拼接路径、`directory_iterator` 遍历目录、`copy`/`remove`/`create_directory` 操作文件，配合 `error_code` 避免异常中断。
