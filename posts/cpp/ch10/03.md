# doctest：轻量级 C++ 测试框架入门

doctest 是一个单头文件（single-header）的 C++ 测试框架，接口与 Catch2 类似，但体积更小、编译更快，非常适合给示例代码、算法题或小型项目加测试。

## 1. 引入方式

从 [doctest GitHub](https://github.com/doctest/doctest) 下载 `doctest.h`，放到项目里：

```text
my_project/
├── doctest.h
└── main.cpp
```

一个测试文件只需两行：

```cpp
#define DOCTEST_CONFIG_IMPLEMENT_WITH_MAIN
#include "doctest.h"
```

- `DOCTEST_CONFIG_IMPLEMENT_WITH_MAIN` 让 doctest 自己提供 `main()` 函数；
- 多个 `.cpp` 文件时，**只在一个文件里定义这个宏**，其余文件只 `#include "doctest.h"`。

## 2. 基本语法

### 2.1 测试用例 `TEST_CASE`

```cpp
TEST_CASE("简单加法") {
    int a = 2;
    int b = 3;
    CHECK(a + b == 5);
}
```

### 2.2 常用断言

| 断言 | 说明 |
|------|------|
| `CHECK(expr)` | 失败继续执行 |
| `REQUIRE(expr)` | 失败立即退出当前测试用例 |
| `CHECK_EQ(a, b)` | 等价 `CHECK(a == b)`，失败时输出更友好 |
| `CHECK_NE(a, b)` | `a != b` |
| `CHECK_GT(a, b)` | `a > b` |
| `CHECK_GE(a, b)` | `a >= b` |
| `CHECK_LT(a, b)` | `a < b` |
| `CHECK_LE(a, b)` | `a <= b` |
| `CHECK_MESSAGE(expr, msg)` | 失败时附带自定义信息 |

```cpp
TEST_CASE("常用断言") {
    CHECK_EQ(2 + 2, 4);
    CHECK_NE(2 + 2, 5);
    CHECK_GT(10, 3);
    CHECK_MESSAGE(std::vector{1, 2}.size() == 2, "容器大小不对");
}
```

### 2.3 异常测试

```cpp
#include <stdexcept>

int divide(int a, int b) {
    if (b == 0) throw std::invalid_argument("b == 0");
    return a / b;
}

TEST_CASE("异常测试") {
    CHECK_THROWS(divide(10, 0));                       // 只要抛异常就通过
    CHECK_THROWS_AS(divide(10, 0), std::invalid_argument); // 指定异常类型
    CHECK_THROWS_WITH(divide(10, 0), "b == 0");        // 指定异常信息
    CHECK_NOTHROW(divide(10, 2));                      // 不抛异常才通过
}
```

### 2.4 子测试 `SUBCASE`

```cpp
TEST_CASE("vector 操作") {
    std::vector<int> v;

    SUBCASE("push_back") {
        v.push_back(42);
        CHECK_EQ(v.size(), 1);
        CHECK_EQ(v[0], 42);
    }

    SUBCASE("pop_back") {
        v.push_back(1);
        v.push_back(2);
        v.pop_back();
        CHECK_EQ(v.size(), 1);
    }
}
```

每个 `SUBCASE` 都会独立执行，互不影响。

## 3. 浮点数比较

直接用 `==` 比较浮点数容易失败，用 `doctest::Approx`：

```cpp
TEST_CASE("浮点比较") {
    double x = 0.1 + 0.2;
    CHECK_EQ(x, doctest::Approx(0.3));
    CHECK_EQ(x, doctest::Approx(0.3).epsilon(1e-10));
}
```

## 4. 多文件项目结构

```text
src/
├── add.cpp
├── add.h
└── main.cpp
tests/
├── test_add.cpp
└── doctest.h
```

**main.cpp**
```cpp
#define DOCTEST_CONFIG_IMPLEMENT_WITH_MAIN
#include "doctest.h"
```

**test_add.cpp**
```cpp
#include "doctest.h"
#include "add.h"

TEST_CASE("add") {
    CHECK_EQ(add(2, 3), 5);
}
```

编译：
```bash
g++ -std=c++17 -I src -I tests src/add.cpp tests/test_add.cpp tests/main.cpp -o test_runner
./test_runner
```

## 5. 命令行过滤

doctest 生成的可执行文件支持命令行参数：

```bash
./test_runner          # 运行全部测试
./test_runner -tc="add"    # 只跑名字含 add 的用例
./test_runner -ts="math"   # 只跑某个测试套件
./test_runner -l         # 列出所有测试用例
```

## 6. 注意事项

- doctest 是**编译期**测试框架，不能用来跑有未定义行为（UB）的代码，例如有符号整数溢出、返回局部变量引用后访问等；
- 不要把会崩溃的 UB 代码放进 `CHECK`/`REQUIRE`；这类代码更适合作为"演示 + 文字说明"；
- 测试异常时，如果异常类型不匹配或异常信息不匹配，用例会失败。

## 7. 完整示例

```cpp
#define DOCTEST_CONFIG_IMPLEMENT_WITH_MAIN
#include "doctest.h"

#include <stdexcept>
#include <vector>

int factorial(int n) {
    if (n < 0) throw std::invalid_argument("n must be >= 0");
    return n <= 1 ? 1 : n * factorial(n - 1);
}

TEST_CASE("factorial") {
    CHECK_EQ(factorial(0), 1);
    CHECK_EQ(factorial(1), 1);
    CHECK_EQ(factorial(5), 120);
    CHECK_THROWS_AS(factorial(-1), std::invalid_argument);
}
```

编译运行：

```bash
g++ -std=c++17 -Wall -Wextra factorial_test.cpp -o factorial_test
./factorial_test
```
