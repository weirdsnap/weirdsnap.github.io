# std::string_view 与 std::span：零拷贝视图

C++17 引入的两个"视图"类型：`std::string_view` 是字符串的只读零拷贝视图，`std::span` 是任意连续序列的零拷贝视图。

---

## 一、`std::string_view`：不拥有的字符串引用

```cpp
#include <string_view>

void process(std::string_view sv) {
    std::cout << sv.size() << ": " << sv.substr(0, 3) << "\n";
}

process("Hello, World!");      // ✅ 直接传字符串字面量，无拷贝
std::string s = "Hello";
process(s);                     // ✅ 传 std::string，隐式转换
process(s.substr(0, 3));        // ✅ 传子串，无需担心生命周期（substr 返回新 string）
```

### 为什么用 string_view？

```cpp
// ❌ 旧写法：按值传 string，每次调用都有拷贝
void old(const std::string& s);

// ✅ 新写法：string_view，传什么都能用且零拷贝
void modern(std::string_view sv) noexcept;
```

| 特性 | `std::string` | `std::string_view` |
|------|--------------|-------------------|
| 所有权 | 拥有数据 | 不拥有（指针+长度） |
| 修改 | 可修改 | 只读 |
| 传参开销 | 拷贝/引用 | 两个指针（8~16 字节） |
| 生命周期管理 | 自动 | 需要确保原数据存活 |
| 子串操作 | 返回新 string（有拷贝） | `sv.substr()` 返回视图（零拷贝） |

## 二、`std::span`：任意连续序列的视图

C++20 泛化了 `string_view` 的概念到任何连续容器：

```cpp
#include <span>
#include <vector>

// 接收任何连续的序列：数组、vector、原生数组
void process(std::span<int> sp) {
    for (int& x : sp) x *= 2;   // span 可读写（取决于参数类型）
}

int arr[] = {1, 2, 3};
std::vector<int> vec = {4, 5, 6};
std::array<int, 3> std_arr = {7, 8, 9};

process(arr);       // ✅
process(vec);       // ✅
process(std_arr);   // ✅
```

### `span` 的类型

| 签名 | 可读 | 可写 | 动态/静态长度 |
|------|------|------|-------------|
| `span<int>` | ✅ | ✅ | 动态长度 |
| `span<const int>` | ✅ | ❌ | 动态长度 |
| `span<int, 3>` | ✅ | ✅ | 编译期固定长度 |

## 三、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `string_view` 指向的字符串以 `\0` 结尾 | ✅ `string_view` 不保证空字符结尾，`sv.data()` 可能不是 C 字符串 |
| ❌ `string_view` 返回值安全 | ✅ **不安全！** `sv.substr()` 返回的视图指向原数据；若原数据销毁则悬垂 |
| ❌ `span` 只能用于修改数据 | ✅ `span<const T>` 是只读的 |
| ❌ `span` 可以用于 `std::list` | ✅ 不能，`list` 不是连续存储，`span` 需要连续内存 |

---

## 四、一句话总结

> `std::string_view` 和 `std::span` 是零拷贝的"窗口"——它们不持有数据，只是让你看一眼存在的连续内存。
