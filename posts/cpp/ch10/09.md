# std::chrono 深入：duration / time_point / clock

C++11 引入的 `<chrono>` 库提供了**类型安全的时间处理**，彻底告别 C 风格的 `time_t` 和 `struct timeval` 混用陷阱。

---

## 一、核心概念

`<chrono>` 由三个核心组件构成：

| 组件 | 用途 | 示例 |
|------|------|------|
| **`duration`** | 时间长度（"多久"） | 5 秒、100 毫秒 |
| **`time_point`** | 时间点（"什么时候"） | 2024-01-01 00:00:00 |
| **`clock`** | 时钟源（"从哪获取当前时间"） | `steady_clock`、`system_clock` |

---

## 二、`duration`：时间长度

```cpp
#include <chrono>
using namespace std::chrono_literals;  // 使用字面量后缀

// 常用时间单位
auto ms = 100ms;         // 100 毫秒
auto sec = 2s;           // 2 秒
auto min = 3min;         // 3 分钟
auto hour = 4h;          // 4 小时

// 各单位的底层类型
std::chrono::milliseconds a(100);      // 100 毫秒
std::chrono::seconds      b(2);        // 2 秒
std::chrono::minutes      c(3);        // 3 分钟
std::chrono::hours        d(4);        // 4 小时
std::chrono::microseconds e(1000);     // 1000 微秒
std::chrono::nanoseconds  f(1000000);  // 1000000 纳秒
```

### duration 运算

```cpp
auto total = 1h + 30min + 15s;    // 类型安全的混合运算
auto half = total / 2;            // 除法

// 转换为不同单位
auto in_sec = std::chrono::duration_cast<std::chrono::seconds>(total);
auto in_ms  = std::chrono::duration_cast<std::chrono::milliseconds>(total);

// 比较
if (total > 1h) { }
```

**为什么不用 `int` 表示时间？** 因为 `100ms + 2s` 如果都是用 `int`，你会忘掉单位换算。`chrono::duration` 编译期检查单位，杜绝这类错误。

## 三、`time_point`：时间点

```cpp
using Clock = std::chrono::system_clock;
auto now = Clock::now();                         // 当前时间点
auto later = now + 1h;                           // 1 小时后

// 时间点之间的差值就是 duration
auto diff = later - now;                         // 1h（duration）

// 时间点运算
auto earlier = now - 30min;

// 取整到整秒
auto floored = std::chrono::floor<std::chrono::seconds>(now);
```

## 四、三种时钟

| 时钟 | C++ 版本 | 是否单调 | 是否与系统时间关联 | 用途 |
|------|---------|---------|-----------------|------|
| `system_clock` | C++11 | ❌ 可能被调整 | ✅ 是 | 获取当前日期时间 |
| `steady_clock` | C++11 | ✅ 单调递增 | ❌ 否 | 测量时间间隔、性能计时 |
| `high_resolution_clock` | C++11 | 实现定义 | 实现定义 | 最高精度时钟（通常是 steady 的别名） |
| `utc_clock` | C++20 | ✅ | ✅ | UTC 时间 |
| `tai_clock` / `gps_clock` | C++20 | ✅ | ✅ | TAI / GPS 时间 |

```cpp
// 用 steady_clock 测量耗时
auto start = std::chrono::steady_clock::now();
do_something();
auto end = std::chrono::steady_clock::now();
auto elapsed_ms = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);
std::cout << elapsed_ms.count() << "ms\n";
```

## 五、与 C 风格 time_t 互转

```cpp
// system_clock → time_t → 可读字符串
auto now = std::chrono::system_clock::now();
std::time_t tt = std::chrono::system_clock::to_time_t(now);
std::cout << std::ctime(&tt);   // "Thu Jun 25 15:00:00 2026"

// time_t → system_clock::time_point
auto tp = std::chrono::system_clock::from_time_t(tt);
```

## 六、常见误区

| 误区 | 正解 |
|------|------|
| ❌ `steady_clock` 可以获取当前日期时间 | ✅ `steady_clock` 不关联系统时间，只能计算间隔 |
| ❌ `duration.count()` 返回的值单位是秒 | ✅ `count()` 返回**底层 tick 数**，单位取决于 duration 类型 |
| ❌ 不同时钟的 `time_point` 可以互相减 | ✅ 不同类型不能直接运算，需要先转换 |
| ❌ `high_resolution_clock` 精度一定最高 | ✅ 在大多数实现中它只是 `steady_clock` 的别名 |

---

## 七、一句话总结

> `std::chrono` 用类型安全的方式区分"多久"（duration）和"什么时候"（time_point），`steady_clock` 计时、`system_clock` 取日历时间。
