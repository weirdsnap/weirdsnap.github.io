# C++ 中的睡眠：从 sleep 到高精度等待

很多初学者会困惑：C++ 里怎么让程序暂停一小段时间？`sleep(1)` 只能睡整秒吗？微秒、毫秒怎么睡？高精度场景又该用什么？

## 1. C 风格的睡眠函数

### `sleep`：秒级

```cpp
#include <unistd.h>

sleep(1);      // 睡 1 秒
sleep(0.5);    // 错误！参数是 unsigned int
```

`sleep` 的参数是 `unsigned int`，所以只能表示整秒。想睡 0.5 秒是不行的。

### `usleep`：微秒级（已废弃）

```cpp
#include <unistd.h>

usleep(100000);   // 睡 100 毫秒 = 100,000 微秒
usleep(500000);   // 睡 500 毫秒
```

`usleep` 可以精确到微秒，但 POSIX.1-2008 已把它标记为**废弃**，新代码不建议用。

### `nanosleep`：纳秒级

```cpp
#include <time.h>

struct timespec ts;
ts.tv_sec = 0;
ts.tv_nsec = 100000000L;  // 100 毫秒 = 100,000,000 纳秒
nanosleep(&ts, nullptr);
```

`nanosleep` 是 POSIX 标准的高精度睡眠函数，精度可以达到纳秒级别，但实际受系统调度影响。

---

## 2. 现代 C++ 推荐：`std::this_thread::sleep_for

C++11 开始，标准库提供了跨平台的线程睡眠：

```cpp
#include <thread>
#include <chrono>

// 100 毫秒
std::this_thread::sleep_for(std::chrono::milliseconds(100));

// 500 微秒
std::this_thread::sleep_for(std::chrono::microseconds(500));

// 1 纳秒（只是表达意图，实际精度取决于系统）
std::this_thread::sleep_for(std::chrono::nanoseconds(1));
```

### 常用时间单位

| 单位 | 类型 | 示例 |
|------|------|------|
| 小时 | `std::chrono::hours` | `hours(1)` |
| 分钟 | `std::chrono::minutes` | `minutes(2)` |
| 秒 | `std::chrono::seconds` | `seconds(1)` |
| 毫秒 | `std::chrono::milliseconds` | `milliseconds(100)` |
| 微秒 | `std::chrono::microseconds` | `microseconds(500)` |
| 纳秒 | `std::chrono::nanoseconds` | `nanoseconds(1000)` |

### 为什么比 C 风格好

- **类型安全**：不会把秒和毫秒搞混
- **跨平台**：Windows、Linux、macOS 都能用
- **可读性强**：`milliseconds(100)` 一眼看懂

---

## 3. 绝对时间的睡眠：`sleep_until

`sleep_for` 是"睡多久"，而 `sleep_until` 是"睡到什么时候"，适合需要固定时间点的场景：

```cpp
#include <thread>
#include <chrono>

auto start = std::chrono::steady_clock::now();
auto next = start + std::chrono::milliseconds(100);

// 睡到 next 这个时间点
std::this_thread::sleep_until(next);
```

在周期性任务里，`sleep_until` 比 `sleep_for` 更准，因为它不会因为任务执行时间抖动而累积误差。

---

## 4. 实际精度有多高？

虽然 `std::chrono::nanoseconds` 可以表示 1 纳秒，但**实际睡眠精度**取决于操作系统：

| 平台 | 典型精度 |
|------|---------|
| Linux | 通常 1~10 毫秒，配置实时内核可达微秒级 |
| Windows | 通常 15 毫秒左右 |
| macOS | 通常 1~10 毫秒 |

原因是 sleep 会让出 CPU，操作系统调度器不会立刻把你唤醒。想要真正的高精度，需要考虑自旋（busy-wait）。

---

## 5. 高精度场景：自旋等待

如果你需要微秒甚至亚微秒级别的精度，sleep 类函数都不行，要用自旋：

```cpp
#include <chrono>

auto start = std::chrono::high_resolution_clock::now();
auto target = start + std::chrono::microseconds(100);

while (std::chrono::high_resolution_clock::now() < target) {
    // 空转，CPU 占用 100%
}
```

### 带提示的自旋

现代 CPU 支持 `pause` 指令，可以让自旋更省电：

```cpp
#if defined(__x86_64__) || defined(__i386__)
    __builtin_ia32_pause();  // x86
#elif defined(__aarch64__)
    __builtin_arm_yield();   // ARM
#endif
```

### 混合策略

工程上常用"sleep + 自旋"的组合：

```cpp
auto target = std::chrono::steady_clock::now() + std::chrono::microseconds(500);

// 先 sleep 到目标前 1 毫秒
std::this_thread::sleep_until(target - std::chrono::microseconds(1000));

// 最后 1 毫秒自旋，保证精度
while (std::chrono::steady_clock::now() < target) {
    // 可选 pause
}
```

这样既能避免长时间占用 CPU，又能保证最后时刻的精度。

---

## 6. 常见误区

- ❌ `sleep(0.5)` 能睡 0.5 秒 — 错误，`sleep` 参数是整秒
- ❌ `std::this_thread::sleep_for(nanoseconds(1))` 真的能睡 1 纳秒 — 错误，只是表达意图，实际精度由系统决定
- ❌ 高精度场景直接用 `usleep` — 已废弃，且精度不够
- ✅ 一般延迟用 `std::this_thread::sleep_for(milliseconds)`
- ✅ 周期性任务用 `sleep_until`
- ✅ 微秒级高精度用自旋或 sleep + 自旋混合

---

## 7. 一句话总结

| 需求 | 推荐方案 |
|------|---------|
| 秒级延迟 | `std::this_thread::sleep_for(seconds(n))` |
| 毫秒/微秒延迟 | `std::this_thread::sleep_for(microseconds(n))` |
| 周期性定时 | `std::this_thread::sleep_until(time_point)` |
| 微秒级高精度 | 自旋或 sleep + 自旋混合 |
| 纳秒级精度 | 基本不可能稳定达到，需专用实时硬件 |
