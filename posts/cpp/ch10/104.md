# 基础共享内存实现

## 原理解释

IPC也就是进程间通信，有很多种手段，比如通过管道，通过socket等
共享内存也是一种常用的IPC手段，而且相比于其他的方式，它的优势在于通过多个程序共享同一块内存空间，减少了内存拷贝的次数，

## 核心函数

要使用共享内存，我们得通过三个步骤/函数来实现我们的目标

| 函数 | 作用 |
|------|------|
| `shm_open()` | 创建/打开共享内存对象|
| `ftruncate()` | 设置共享内存对象的大小 |
| `mmap()` | 将对象映射到进程虚拟地址空间 |

### shm_open

来自头文件 fcntl.h 

**作用：** 创建或者打开一个共享内存对象。

```cpp
int fd = shm_open(const char* name, int oflag, mode_t mode);
```

**参数：**
- `name`：建议以 / 开头（POSIX 标准要求；Linux 实现不强制，但其他系统可能拒绝）。
- `oflag`：标志位（可以用 `|` 组合多个）
  - `O_CREAT`：不存在则创建
  - `O_RDWR`：读写打开
  - `O_RDONLY`：只读打开
- `mode`：权限（八进制，如 `0666`、`0600`）

**返回值：**
- 成功：返回映射区的起始地址
- 失败：返回 `MAP_FAILED`

**注意：** `mode` 只在**创建时**生效，对象已存在则被忽略。

### 2.3 ftruncate

来自头文件 unistd.h

```cpp
int ftruncate(int fd, off_t length);
```

**作用：** 设置共享内存对象的大小（字节）。

**参数：**
- `fd`：共享内存对象的描述符
- `length`：共享内存对象的大小

**返回值：**
- 成功：返回映射区的起始地址
- 失败：返回 `MAP_FAILED`

**注意：** 新创建的共享内存对象初始大小为 0，**必须调 ftruncate 后才能 mmap**，否则访问会触发 **SIGBUS**。

### 2.4 mmap

```cpp
void* mmap(void* addr, size_t length, int prot, int flags, int fd, off_t offset);
```

**参数（按逻辑记忆）：**

| 参数 | 含义 | 常用值 |
|------|------|--------|
| `addr` | 映射起始地址（让内核选）| `nullptr` 或 `NULL` |
| `length` | 映射长度（字节）| 和 `ftruncate` 一致 |
| `prot` | 内存保护权限 | `PROT_READ`、`PROT_WRITE` |
| `flags` | 映射特性 | `MAP_SHARED`、`MAP_PRIVATE` |
| `fd` | 文件描述符 | `shm_open` 返回值 |
| `offset` | 文件内偏移 | `0`（从头开始）|

**返回值：**
- 成功：返回映射区的起始地址
- 失败：返回 `MAP_FAILED`

**mmap 成功后可以立即 `close(fd)`**，映射不依赖 fd 保持打开。

### 2.5 辅助函数

| 函数 | 作用 | 谁调用 |
|------|------|--------|
| `munmap(addr, size)` | 解除当前进程的虚拟地址映射 | **每个进程** |
| `close(fd)` | 关闭文件描述符 | **每个进程** |
| `shm_unlink(name)` | 删除共享内存对象的**名字** | **创建者** |

**⚠️ `shm_unlink` 的关键语义：**
- 删除 `/dev/shm/my_shm` 这个名字
- 但**已经 `mmap` 的进程不受影响**，继续正常读写
- 当**所有进程**都 `munmap` + `close` 后，内核才真正回收物理内存

## 三、完整流程

### 3.1 写入端流程

```cpp

    // 创建共享内存对象
    constexpr const char* name = "my_shm";
    int fd = shm_open(name, O_CREAT | O_RDWR, 0666);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    // 设置共享内存对象的大小
    if (ftruncate(fd, 4096) == -1) {
        perror("ftruncate");
        close(fd);
        return 1;
    }

    // 完成共享内存对象到当前进程虚拟内存地址的映射
    void* addr = mmap(NULL, 4096, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        close(fd);
        return 1;
    }

    // 此时通过内存地址访问，已经可以释放共享内存对象的描述符
    close(fd);

    // 写入数据
    const char* msg = "Hello, SHM!";
    std::memcpy(addr, msg, std::strlen(msg) + 1);

    std::cout << "[Writer] 已写入，按回车键退出..." << std::endl;
    std::cin.get();

    // 清理和释放资源
    munmap(addr, 4096);
    shm_unlink(name);
```

### 3.2 读取端流程

```
shm_open(O_RDONLY)  →  mmap(PROT_READ, MAP_SHARED)
  →  读取 addr  →  munmap
  （不要 ftruncate，不要 shm_unlink）
```

### 3.3 读写端对比

| | 写入端 | 读取端 |
|---|---|---|
| `shm_open` 标志 | `O_CREAT \| O_RDWR` | `O_RDONLY`（无 `O_CREAT`）|
| `ftruncate` | ✅ 必须 | ❌ 不需要 |
| `mmap` 权限 | `PROT_READ \| PROT_WRITE` | `PROT_READ` |
| `shm_unlink` | ✅ 负责清理 | ❌ 不调 |
| `munmap` | ✅ 退出时调 | ✅ 退出时调 |
