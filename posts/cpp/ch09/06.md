# SFINAE / `enable_if` / `requires`：模板约束的演进

模板约束在 C++ 中经历了三个阶段的发展，从"故意让模板编译失败"的 SFINAE，到显式 `enable_if`，再到 C++20 的 Concepts。

---

## 一、SFINAE（Substitution Failure Is Not An Error）

SFINAE 是 C++98 就有的核心规则：**模板参数替换失败不算编译错误，只是把这个候选模板从重载集合中移除**。

```cpp
// 检查一个类型是否有 .size() 成员函数
template<typename T>
decltype(std::declval<T>().size(), void(), true) has_size_impl(int) {
    return true;
}

template<typename T>
bool has_size_impl(...) {
    return false;
}

bool b = has_size_impl<std::vector<int>>(0);   // true
bool b2 = has_size_impl<int>(0);                // false
```

`int` 没有 `.size()`，所以第一个模板替换失败，SFINAE 让它静默移除，只保留第二个。

## 二、`std::enable_if`

C++11 提供了 `std::enable_if`，更干净的语法：

```cpp
// 只为整数类型启用
template<typename T>
std::enable_if_t<std::is_integral_v<T>, T>
    square(T x) {
    return x * x;
}

// 只为浮点类型启用
template<typename T>
std::enable_if_t<std::is_floating_point_v<T>, T>
    square(T x) {
    return x * x;
}

square(5);     // ✅ 调用整数版本
square(3.14);  // ✅ 调用浮点版本
// square("hello");  // ❌ 编译错误，两个版本都被 SFINAE 排除
```

### 常见写法

```cpp
// 1. 返回值 SFINAE
template<typename T>
auto foo(T t) -> std::enable_if_t<condition<T>, decltype(...)>;

// 2. 默认模板参数 SFINAE
template<typename T, std::enable_if_t<condition<T>, int> = 0>
void foo(T t);

// 3. 函数参数 SFINAE
template<typename T>
void foo(T t, std::enable_if_t<condition<T>, int> = 0);
```

## 三、`std::void_t` 检测任意表达式

C++17 的 `std::void_t` 让 SFINAE 检测更加简洁：

```cpp
// 检测是否有 .begin() 成员
template<typename, typename = void>
struct has_begin : std::false_type {};

template<typename T>
struct has_begin<T, std::void_t<decltype(std::declval<T>().begin())>>
    : std::true_type {};
```

---

## 四、C++20 Concepts：模板约束的终极形态

Concepts 让约束从"通过编译错误反向选择"变成**正面声明**：

```cpp
// 定义 concept
template<typename T>
concept Integral = std::is_integral_v<T>;

template<typename T>
concept FloatingPoint = std::is_floating_point_v<T>;

// 使用 concept 约束模板
template<Integral T>
T square(T x) { return x * x; }

template<FloatingPoint T>
T square(T x) { return x * x; }
```

### 三种用法

```cpp
// 1. template 前缀
template<Integral T>
void f(T t);

// 2. auto 缩写
void f(Integral auto t);

// 3. requires 子句
template<typename T>
requires Integral<T>
void f(T t);
```

### `requires` 表达式直接检测

```cpp
template<typename T>
concept HasSize = requires(T t) {
    { t.size() } -> std::convertible_to<std::size_t>;
};

template<typename T>
concept Streamable = requires(std::ostream& os, T t) {
    os << t;  // 可以 << 到 ostream
};
```

## 五、三阶段对比

| 特性 | SFINAE (C++98) | `enable_if` (C++11) | Concepts (C++20) |
|------|----------------|---------------------|------------------|
| 可读性 | ❌ 极差 | 🟡 一般 | ✅ 直观 |
| 错误信息 | ❌ 冗长难懂 | ❌ 略好但依然难懂 | ✅ 清晰 |
| 表达能力 | ✅ 可以检测任意表达式 | ✅ 同上 | ✅ 同上，更简洁 |
| 编译速度 | 🟡 依赖模板展开 | 🟡 同左 | ✅ 通常更快 |
| 是否强制检查 | ❌ 被动（替换失败才排除） | ❌ 被动 | ✅ 主动约束 |
| 适用于变量模板 | ❌ | ✅ | ✅ |
| 复用性 | ❌ 每次重写 | ❌ 每次写 enable_if | ✅ 可定义可复用 concept |

---

## 六、常见误区

- ❌ SFINAE 只发生在函数模板中 — ✅ 也发生在类模板的部分特化中。
- ❌ `enable_if` 的默认模板参数写法不如返回值 SFINAE — ✅ 三种写法等价，选可读性最好的。
- ❌ `requires` 表达式是运行时判断 — ✅ `requires` 完全是编译期检查。
- ❌ Concepts 能完全取代 SFINAE — ✅ 基本可以，但在需要"检测某个表达式是否合法"的泛型库代码中，`void_t` 技巧仍有用武之地。

---

## 七、一句话总结

> 从 SFINAE 到 `enable_if` 再到 Concepts，C++ 的模板约束从"撞墙式排除"变成了"正面声明约束"，可读性和易用性大幅提升。
