# constexpr vs consteval vs constinit

C++20 引入了两个新关键字 `consteval` 和 `constinit`，与已有的 `constexpr` 构成了完整的**编译期执行谱系**。三者的区别可以用一个表格说清：

| 关键字 | 引入版本 | 修饰对象 | 语义 |
|--------|---------|---------|------|
| `constexpr` | C++11（变量）/C++14（函数） | 变量/函数 | 可能编译期求值，也可能运行期 |
| `consteval` | C++20 | 函数 | **必须**编译期求值（立即函数） |
| `constinit` | C++20 | 变量 | 保证编译期初始化，但不隐含 `const` |

---

## 一、`constexpr`：可能编译期

```cpp
constexpr int square(int x) {
    return x * x;
}

constexpr int a = square(5);   // ✅ 编译期求值
int x = 3;
int b = square(x);             // ✅ 运行期求值（不要求 constexpr 实参）
```

`constexpr` 不强制编译期执行——它只是**允许**。当实参是常量表达式时就编译期执行，否则退化为运行期调用。

---

## 二、`consteval`：立即函数，强制编译期

```cpp
consteval int cube(int x) {
    return x * x * x;
}

constexpr int a = cube(3);     // ✅ 编译期求值
int x = 3;
// int b = cube(x);            // ❌ 编译错误！consteval 要求实参也是常量
```

`consteval` 函数只能在编译期求值。它的所有实参必须也是常量表达式。如果你需要"运行时调用也允许"，用 `constexpr`；如果你需要"绝对不能在运行时调用"，用 `consteval`。

典型用途：编译期哈希、编译期正则、编译期代码生成。

---

## 三、`constinit`：编译期初始化，但运行期可修改

`constexpr` 变量隐式是 `const`，不能修改。`constinit` 则只保证**初始化在编译期完成**，允许后续修改。

```cpp
constexpr std::array<int, 3> arr1 = {1, 2, 3};   // ✅ const，不可改
// arr1[0] = 10;                                  // ❌ 编译错误

constinit std::array<int, 3> arr2 = {1, 2, 3};    // ✅ 非 const，可改
arr2[0] = 10;                                      // ✅ 运行期修改

// constinit int x;                                // ❌ 必须编译期初始化
```

`constinit` 主要用于**静态存储期**的变量（全局、`static`、`thread_local`），防止"静态初始化顺序惨案"（Static Initialization Order Fiasco）。

---

## 四、对比表格

| 特性 | `constexpr` (变量) | `constexpr` (函数) | `consteval` | `constinit` |
|------|-------------------|-------------------|-------------|-------------|
| 编译期求值 | 必须 | 允许，但非强制 | 必须 | 初始化必须 |
| 运行时可用 | ✅ 可读（如果 `const`） | ✅ 允许 | ❌ 禁止 | ✅ 可读可写 |
| 是否隐含 `const` | ✅ 是 | N/A | N/A | ❌ 否 |
| 用于静态存储期变量 | ✅ | N/A | N/A | ✅ 防初始化顺序问题 |
| 函数修饰 | ❌ 变量关键字 | ✅ | ✅ | ❌ 变量关键字 |

---

## 五、常见误区

- ❌ `consteval` 函数在调试 `-O0` 下也会编译期执行 — ✅ 标准强制要求，与优化级别无关。
- ❌ `constexpr` 变量就是编译期常量 — ✅ 不一定，`constexpr int x = rand();` 会在运行期执行。
- ❌ `constinit` 和 `constexpr` 变量一样是 `const` — ✅ 错误，`constinit` 不隐含 `const`。
- ❌ `consteval` 是 `constexpr` 的超集 — ✅ 语义不同，`consteval` 强制编译期，`constexpr` 允许退化为运行期。

---

## 六、工程建议

```cpp
// ✅ 需要编译期常量 → constexpr 变量
constexpr double PI = 3.14159265358979;

// ✅ 需要编译期计算且必须编译期完成 → consteval 函数
consteval unsigned hash(const char* str) { /* ... */ }

// ✅ 全局变量需要编译期初始化但运行期可改 → constinit
constinit std::mutex global_mutex;  // constexpr 不行：std::mutex 非字面类型
```

---

## 七、一句话总结

> `constexpr` 是"尽力编译期"，`consteval` 是"必须编译期"，`constinit` 是"编译期初始化但之后可改"。
