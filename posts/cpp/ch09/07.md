# Type Traits 入门：`is_same` / `conditional` / `remove_reference`

`<type_traits>` 是 C++ 模板元编程的标准工具箱，提供了一组**编译期类型查询和变换**的工具。C++14 起所有 traits 都有 `_v` / `_t` 后缀简化写法。

---

## 一、类型查询：以 `is_same` 为例

```cpp
#include <type_traits>

static_assert(std::is_same_v<int, int>);          // ✅ true
static_assert(!std::is_same_v<int, const int>);   // ✅ false（顶层 const 算不同）

// 常见的检查
static_assert(std::is_integral_v<int>);           // true
static_assert(std::is_floating_point_v<double>);  // true
static_assert(std::is_class_v<std::string>);      // true
static_assert(std::is_pointer_v<int*>);           // true
```

### 常见查询 traits

| Trait | 用途 | 示例 |
|-------|------|------|
| `is_integral_v<T>` | 是否为整数类型 | `int`→true, `float`→false |
| `is_floating_point_v<T>` | 是否为浮点类型 | `double`→true |
| `is_pointer_v<T>` | 是否为指针 | `int*`→true |
| `is_reference_v<T>` | 是否为引用 | `int&`→true |
| `is_class_v<T>` | 是否为类/结构体 | `std::string`→true |
| `is_enum_v<T>` | 是否为枚举 | `enum E`→true |
| `is_const_v<T>` | 是否带 const | `const int`→true |
| `is_volatile_v<T>` | 是否带 volatile | |
| `is_same_v<T, U>` | T 和 U 是否相同 | |
| `is_base_of_v<B, D>` | B 是否是 D 的基类 | |

## 二、类型变换：以 `remove_reference` 为例

```cpp
#include <type_traits>

using T1 = std::remove_reference_t<int&>;    // int
using T2 = std::remove_reference_t<int&&>;   // int
using T3 = std::remove_reference_t<int>;     // int（不变）

static_assert(std::is_same_v<T1, int>);
static_assert(std::is_same_v<T2, int>);
```

### 常见变换 traits

| Trait | 功能 |
|-------|------|
| `remove_reference_t<T>` | 去掉引用（`int&` → `int`） |
| `add_reference_t<T>` | 加左值引用（`int` → `int&`） |
| `remove_const_t<T>` | 去掉顶层 const |
| `add_const_t<T>` | 加 const |
| `remove_pointer_t<T>` | 去掉指针（`int*` → `int`） |
| `decay_t<T>` | 类型退化（数组→指针、函数→指针、去掉 cv/引用） |
| `common_type_t<T, U>` | T 和 U 的公共类型（如 `int`+`double`→`double`） |
| `conditional_t<bool, T, F>` | 编译期三元（`true ? T : F`） |

## 三、`conditional`：编译期三元

```cpp
template<typename T>
using BestType = std::conditional_t<
    (sizeof(T) <= sizeof(int)),  // 如果 T 大小 ≤ int
    int,                          // 用 int
    T                              // 否则保持原类型
>;

static_assert(std::is_same_v<BestType<char>, int>);     // char 小，提升为 int
static_assert(std::is_same_v<BestType<double>, double>); // double 大，保留
```

## 四、组合使用：完整的 remove_cvref

C++20 标准库提供了 `std::remove_cvref_t`，它等价于：

```cpp
template<typename T>
struct remove_cvref {
    using type = std::remove_cv_t<std::remove_reference_t<T>>;
};

// C++14 起用 _t 后缀
template<typename T>
using remove_cvref_t = typename remove_cvref<T>::type;

static_assert(std::is_same_v<remove_cvref_t<const int&>, int>);  // ✅
```

## 五、实现原理简介

一个 type trait 的本质是一个**类模板 + 特化**：

```cpp
// 主模板：默认 false
template<typename T, typename U>
struct is_same : std::false_type {};

// 特化：只有 T 和 U 完全相同时才匹配
template<typename T>
struct is_same<T, T> : std::true_type {};
```

`std::true_type` 和 `std::false_type` 是继承自 `std::integral_constant` 的编译期常量包装，`_v` 后缀其实就是 `::value` 的简写：

```cpp
// _v 后缀的底层实现
template<typename T, typename U>
inline constexpr bool is_same_v = is_same<T, U>::value;
```

---

## 六、常见误区

- ❌ `is_same<T, T>` 的 `T` 在不同模板参数下不匹配 — ✅ 只有当两个 T 完全相同时特化才匹配。
- ❌ `remove_reference` 也能去掉 const — ✅ 只能去掉引用，const 需要 `remove_const` 或用 `remove_cvref`（C++20）。
- ❌ `conditional` 在运行时工作 — ✅ 完全是编译期选择，不会生成两个分支的代码。
- ❌ type traits 是运行时检查 — ✅ 所有操作都是编译期完成的，零运行时开销。

---

## 七、一句话总结

> Type traits 是 C++ 编译期反射和类型变换的标准 API——`_v` 查类型、`_t` 改类型、`conditional` 做编译期选择。
