# 数组引用 vs 指针重载优先级

`int[2]` 匹配 `int (&)[2]` 是引用绑定（identity，精确匹配），而 `int[2]` 退化为 `int*` 属于 array-to-pointer 转换。在现行 C++ 标准中，**这两种转换序列的等级都是精确匹配（exact match）**，因此当两个重载同时存在时，调用会产生二义性，而非引用版本自动胜出。

```cpp
void f(int*);              // #1
void f(int (&)[2]);        // #2

int arr[2] = {1, 2};
f(arr);                    // 错误：ambiguous
```

| 候选 | 形参 | 对 `int[2]` 的 ICS | 等级 | 结果 |
|------|------|-------------------|------|------|
| #1 | `int*` | array-to-pointer | 精确匹配 | 无法区分 |
| #2 | `int (&)[2]` | 引用绑定（同类型） | 精确匹配 | 无法区分 |

**原理解析 + 常见误区**

- ❌ 引用版本因为"更直接"所以优先 — 错误。两种转换序列都是 exact match，没有谁比谁更优。
- ✅ 真正让数组引用"胜出"的场景，是当指针版本需要额外的转换时。例如 `void*` 版本需要 array-to-pointer + pointer conversion，等级降为标准转换，此时 `int (&)[N]` 的 exact match 才会被选中。
- ✅ 数组引用的核心价值在于**阻止数组退化**，在函数体内保留元素类型和数组大小，而不是在重载优先级上碾压指针版本。

**延伸点**

标准库 `std::begin` 的设计并不依赖"数组引用打败指针"这一规则，因为它的重载集合中根本没有裸指针版本的竞争对手：数组走 `T (&)[N]` 重载，容器走 `C&` 重载。真正需要避免的是同时提供 `f(T*)` 和 `f(T (&)[N])` 这种会产生二义性的重载组合。

CWG 1789 曾讨论过让引用绑定优于 array-to-pointer 转换，但截至目前该提案尚未进入正式标准。
