# explicit 转换函数

C++11 引入 `explicit` 修饰转换函数，阻止**隐式转换**，但允许**显式转换**（`static_cast`）和**特定上下文转换**（如 `if` 条件、`while` 条件、`for` 循环条件、`?:` 条件、`static_assert`、`noexcept` 等）。

| 场景 | `operator bool()` | `explicit operator bool()` |
|------|-------------------|---------------------------|
| `if (obj)` | ✅ 隐式转换 | ✅ 上下文转换 |
| `bool b = obj;` | ✅ | ❌ 编译失败 |
| `int i = obj;` | ❌（歧义或窄化） | ❌ |
| `static_cast<bool>(obj)` | ✅ | ✅ |

```cpp
#include <iostream>

struct SafeBool {
    int value;
    explicit operator bool() const {  // 显式转换
        return value != 0;
    }
};

int main() {
    SafeBool sb{42};
    
    if (sb) {                         // ✅ 上下文转换
        std::cout << "true\n";
    }
    
    // bool b = sb;                   // ❌ 编译错误：不允许隐式
    bool b = static_cast<bool>(sb);  // ✅ 显式转换
}
```

**原理解析 + 常见误区**

- ❌ `explicit` 阻止所有转换 — 错误。条件语境和显式语法仍可用。
- ❌ `T(obj)` 是隐式转换 — 错误。函数风格转换属于显式类型转换。
- ✅ 现代标准库类型（`std::optional`、`std::unique_ptr`）的 `operator bool` 均为 `explicit`。

**延伸点**：C++20 的 `explicit(bool)` 允许条件显式：`explicit(true) operator bool()` 等价于 `explicit operator bool()`。
