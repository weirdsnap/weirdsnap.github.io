# operator void() 不合法

转换函数的 *conversion-type-id* 必须是**对象类型**或**对象类型的引用**。`void` 不是对象类型，因此 `operator void()` 是非法的，无法通过编译。

| 转换目标 | 合法性 | 原因 |
|----------|--------|------|
| `operator int()` | ✅ | int 是对象类型 |
| `operator int&()` | ✅ | 引用类型合法 |
| `operator void()` | ❌ | void 不是对象类型 |
| `operator void*()` | ✅ | `void*` 是指针对象类型 |

```cpp
struct Bad {
    // 编译错误：void is not a class-type or cv-qualified class-type
    // operator void() { }
    
    // 合法：返回 void* 指针
    operator void*() { return nullptr; }
};

int main() {
    Bad b;
    void* p = b;  // ✅ 隐式转换为 void*
}
```

**原理解析 + 常见误区**

- ❌ `operator void()` 可以用来「丢弃对象」— 错误。`void` 不是对象类型，不能作为转换目标。
- ✅ `operator void*()` 合法，因为 `void*` 是对象指针类型。
- ❌ 编译器会忽略 `operator void()` — 错误。这是编译错误，不是警告。

**延伸点**：试图「转换到 void」没有语义价值，因为 `void` 表示「无类型」，不能用于初始化变量或参与表达式求值。若需要「丢弃对象」的语义，应使用普通成员函数而非转换函数。
