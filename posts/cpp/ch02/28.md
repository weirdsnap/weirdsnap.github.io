# 构造与析构顺序

构造：基类 → 成员（按声明顺序）→ 构造函数体；析构严格逆序。

**场景示例：**
```cpp
class D : public B {
    Member1 m1;  // 声明顺序第一
    Member2 m2;  // 声明顺序第二
public:
    D() : m2(...), m1(...) {}  // 初始化列表顺序无效！
};
```

**原理解析 + 常见误区**

- ❌ 初始化列表顺序决定构造顺序 — 错误。成员始终按类内声明顺序构造，与初始化列表无关。
- ❌ 析构顺序与初始化列表逆序 — 错误。析构只与声明顺序逆序相关。
- ✅ 栈展开（stack unwinding）保证异常抛出时已构造的自动对象被析构，这是 RAII 的根基。

**延伸点**：析构函数中绝不可再抛异常（标记 `noexcept`），否则栈展开期间二次抛异常直接触发 `std::terminate`。
