# 转换函数的 `const` 正确性

转换函数若**不修改对象状态**，应标记为 `const`。否则临时对象（如 `Foo()` 或 `const` 对象）无法调用该转换函数，导致编译失败。

| 调用方 | `operator int()` | `operator int() const` |
|--------|-----------------|----------------------|
| `Foo f; int a = f;` | ✅ | ✅ |
| `const Foo f; int a = f;` | ❌ | ✅ |
| `int a = Foo();`（临时对象） | ❌ | ✅ |

```cpp
#include <iostream>

struct Foo {
    int x = 42;
    // 忘记 const：临时对象无法调用
    operator int() { return x; }  // 非 const
    
    // 正确做法
    // operator int() const { return x; }
};

int main() {
    // int a = Foo();  // ❌ 编译错误：临时对象无法绑定到非 const 成员函数
    // 修正后：int a = Foo();  // ✅ 输出 42
}
```

**原理解析 + 常见误区**

- ❌ 转换函数默认是 `const` — 错误。和普通成员函数一样，默认非 `const`。
- ❌ 只有 `const` 对象才需要 `const` 转换函数 — 错误。临时对象也只能调用 `const` 成员函数。
- ✅ 不修改状态的转换函数应始终标记 `const`。

**延伸点**：临时对象（prvalue）只能调用 `const` 成员函数。如果同时提供 `const` 和非 `const` 版本，非 `const` 左值优先匹配非 `const` 版本，临时对象和 `const` 左值匹配 `const` 版本。与 `std::vector` 的 `begin()` / `begin() const` 设计模式一致。
