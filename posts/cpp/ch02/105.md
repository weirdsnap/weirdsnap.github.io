# 转换函数重载的二义性

若类定义了多个转换函数到不同类型，在需要进一步标准转换的上下文中可能产生**二义性**。编译器不会通过「哪个转换更优」来裁决，而是直接报错。

| 定义 | 调用上下文 | 结果 | 原因 |
|------|-----------|------|------|
| `operator int()` + `operator double()` | `long a = obj;` | ❌ 二义性 | int→long 和 double→long 都是标准转换 |
| `operator int()` + `operator long()` | `long a = obj;` | ✅ 选 `long` | 精确匹配优于标准转换 |

```cpp
struct Ambiguous {
    operator int() const { return 1; }
    operator double() const { return 2.0; }
};

void take_long(long) {}

int main() {
    Ambiguous a;
    // long x = a;        // ❌ 二义性：int→long vs double→long
    
    // 解决方式 1：显式指定
    long x = static_cast<int>(a);
    
    // 解决方式 2：删除不必要的转换
    // 解决方式 3：使用 explicit 阻止隐式
}
```

**原理解析 + 常见误区**

- ❌ 编译器会选「更接近」的类型 — 错误。两个标准转换序列同级时直接报错。
- ✅ 精确匹配优于标准转换，因此 `operator long()` 不会与 `operator int()` 在目标为 `long` 时产生歧义。
- ❌ 二义性只在隐式转换时出现 — 错误。显式 `static_cast` 指定目标类型即可消除。

**延伸点**：二义性规则同样适用于**用户定义转换 + 内置转换**的序列。例如 `operator int()` 和 `operator short()` 在传入 `int` 时，`short→int` 需要提升，而 `int` 精确匹配，因此选 `int` 无歧义。但若目标是 `long`，两者都需要标准转换，则歧义。设计时应遵循「单一职责」：一个类通常只应提供一个有意义的隐式转换目标。
