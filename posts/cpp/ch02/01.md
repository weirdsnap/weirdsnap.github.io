# 虚函数的底层支持（vtable / vptr）

含虚函数的类有虚表（vtable）存储虚函数地址，每个对象有虚表指针（vptr）指向虚表；vptr 位置是 ABI 实现细节。

| 层级 | 实体 | 数量 |
|------|------|------|
| Per-class | vtable | 每个含虚函数的类一个 |
| Per-object | vptr | 单继承通常 1 个，多重继承多个 |

**原理解析 + 常见误区**

- ❌ vptr 一定在对象首部 — 错误。这是 ABI 实现细节，非标准保证。
- ❌ vtable 只存函数指针 — 错误。典型实现还包含 RTTI 指针和偏移修正信息。
- ❌ `memset(this, 0, sizeof(*this))` 可以安全地"重置"对象 — 错误。对非 POD 类会覆盖 vptr 等编译器生成的元数据，导致调用虚函数时崩溃或产生未定义行为。

### 为什么 `memset(this, 0, sizeof(*this))` 危险

`memset` 会把对象所在内存按字节全部填 0，它不了解对象的内部结构。对于**POD（Plain Old Data）**类型（如普通结构体、基本类型数组），这样做通常没有问题；但对于含虚函数的类，会覆盖 **vptr**：

```cpp
class Base {
public:
    virtual void foo() {}
    int x;
};

class Derived : public Base {
public:
    void reset() {
        memset(this, 0, sizeof(*this));  // ❌ UB：vptr 被清零
    }
};
```

对象内存布局大致如下：

```
Derived 对象：
├─ [0:7]   vptr → 指向 Derived 的虚函数表
└─ [8:11]  int x
```

vptr 被清零后，对象找不到虚函数表，后续调用 `foo()` 可能崩溃或跳转到错误地址。

**正确做法**：显式重置每个成员，或在严格控制下使用 placement new 重新构造。

**延伸点**：构造/析构时 vptr 按继承链动态切换，这是基类构造/析构中调用虚函数无法到达派生类实现的底层原因。
