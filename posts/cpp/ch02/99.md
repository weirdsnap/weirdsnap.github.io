# 类型转换函数基础语法

转换函数**没有显式返回类型声明**，其名称 `operator T` 本身即定义了返回类型。必须是**非静态成员函数**，参数列表为空，不能指定返回类型（连 `void` 都不能写）。

| 特性 | 普通成员函数 | 转换函数 |
|------|-------------|---------|
| 返回类型声明 | `int foo()` | **禁止**（由 `operator T` 隐含） |
| 函数名语法 | `foo` | `operator T` |
| 参数 | 可有 | **必须为空** |
| 静态成员 | 可以 | **不可以** |
| 虚函数 | 可以 | 可以 |

```cpp
struct Foo {
    int x;
    // 正确：没有返回类型，operator int 即声明返回 int
    operator int() const { return x; }
    
    // 错误：编译失败，不能写返回类型
    // int operator double() { return x; }
};

int main() {
    Foo f{42};
    int a = f;          // 隐式调用 operator int()
    int b = static_cast<int>(f);  // 显式调用
}
```

**原理解析 + 常见误区**

- ❌ 转换函数可以写成 `T operator T()` — 错误。任何形式的返回类型声明都非法。
- ❌ 转换函数可以带参数 — 错误。参数列表必须为空，否则不是转换函数。
- ❌ 转换函数可以是静态的 — 错误。必须是对象上的非静态成员函数。

**延伸点**：转换函数可以被 `const` / `volatile` / `ref-qualifier`（`&` / `&&`）修饰，也可以标记 `noexcept`。C++11 起支持 `explicit`，C++20 起支持 `explicit(bool)` 条件显式。
