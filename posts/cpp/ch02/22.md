# `operator auto()` 返回类型推导 (C++23)

C++23 允许**普通转换函数**使用 `operator auto()`，编译器从函数体内的 `return` 语句推导出 `T`。但标准明确禁止**转换函数模板**使用推导返回类型。

| 形式 | 是否允许 | 推导机制 | 标准依据 |
|------|---------|---------|---------|
| `operator auto()` | ✅ 允许 | 从 return 推导 `T` | [class.conv.fct]¶3 |
| `template<> operator auto()` | ❌ 禁止 | N/A | [class.conv.fct]¶9 |
| `template<typename T> operator T()` | ✅ 允许 | 由模板参数 `T` 决定 | 正常模板规则 |

```cpp
#include <iostream>

struct Foo {
    operator auto() {      // C++23: 推导为 operator int()
        std::cout << "A";
        return 1;
    }
};

int main() {
    int a = Foo();         // 创建临时对象，隐式转换
    std::cout << a;       // 输出: A1
}
```

**原理解析 + 常见误区**

- ❌ `operator auto()` 在 C++20 可用 — 错误。这是 C++23 才加入的特性。
- ❌ 转换函数模板也能用 `auto` — 错误。模板参数已提供类型，叠加 `auto` 推导会造成双重推导歧义。
- ✅ `operator auto()` 的推导规则与普通函数 `auto foo()` 一致，遵循 [dcl.spec.auto]。

**延伸点**：标准 [class.conv.fct]¶3 指出转换函数的类型是「…returning *conversion-type-id*」，其中 *conversion-type-id* 就是 `T`。`auto` 占位符在此处遵循返回类型推导规则。
