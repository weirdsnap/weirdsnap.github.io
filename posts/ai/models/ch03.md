# 位置编码与长上下文窗口

Transformer 的核心是自注意力机制，但 Attention 本身有一个特点：**它对输入 token 没有顺序概念**。

也就是说，如果你把一句话的词序打乱，Attention 计算出来的结果形式上是一样的——它只看 token 之间的相似度，不关心谁在前谁在后。

为了让模型理解顺序，需要引入**位置编码（Positional Encoding）**。这一章我们讲位置编码的演进，以及它如何支撑今天的大模型长上下文能力。

---

## 1. 为什么需要位置编码？

比如输入：

```
["我", "喜欢", "猫"]
```

Attention 计算时，每个 token 会和其他所有 token 计算相似度。如果没有位置信息，模型无法区分：

```
["我", "喜欢", "猫"] 和 ["猫", "喜欢", "我"]
```

这两句话语义完全不同，但 Attention 的输入矩阵形状完全一样。

位置编码的作用就是给每个位置一个唯一的"身份证"，让模型知道：

```
"我" 在位置 0
"喜欢" 在位置 1
"猫" 在位置 2
```

---

## 2. 绝对位置编码：Sinusoidal

原始 Transformer 论文提出的方案是**正弦位置编码**：

```
PE(pos, 2i)   = sin(pos / 10000^(2i / d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i / d_model))
```

其中：
- `pos`：token 在序列中的位置
- `i`：维度索引
- `d_model`：模型维度（为简洁有时也写作 `d`）

### 2.1 设计思想

每个位置得到的位置编码是一个 `d` 维向量。不同维度用不同频率的正弦/余弦函数：

- 低维度：频率慢，反映长距离位置差异
- 高维度：频率快，反映短距离位置差异

这样模型可以从不同尺度感知位置关系。

### 2.2 优点

- **无需学习参数**：直接按公式计算
- **可以外推**：即使训练时最大长度是 2048，测试时也可以处理更长的位置（虽然效果不一定好）

### 2.3 缺点

- 每个位置是绝对独立的，没有显式的相对位置信息
- 模型需要自己学习"位置 5 和位置 6 很接近"这种关系

## 3. 可学习位置编码

BERT 和早期 GPT 用的是**可学习位置编码**：

```python
max_len = 512
pos_embed = nn.Parameter(torch.randn(max_len, d_model))

# 输入 x 的 shape: [batch, seq_len, d_model]
x = x + pos_embed[:seq_len]
```

每个位置对应一个可学习的向量，模型自己学会怎么用。

### 3.1 优点

- 灵活：模型可以学习最适合自己的位置表示
- 简单：直接加在输入 embedding 上

### 3.2 缺点

- **长度固定**：训练时最大长度 512，测试时就不能超过 512
- 外推能力弱：超过训练长度的位置没有对应的 embedding

这也是为什么早期 BERT/GPT 都有 512 token 的限制。

---

## 4. 旋转位置编码（RoPE）

RoPE（Rotary Position Embedding）是目前主流大模型（Llama、Qwen、ChatGLM、Baichuan 等）广泛采用的位置编码方案。

### 4.1 核心思想

RoPE 不是把位置编码加到输入上，而是**在 Attention 计算时，把相对位置信息融入 query 和 key 的旋转操作中**。

具体做法：把 Q 和 K 向量每两个维度分成一对，然后按位置角度旋转。

```
原始 q = [q0, q1, q2, q3, ...]

按位置 m 旋转后：
q'_0 = q0 * cos(mθ) - q1 * sin(mθ)
q'_1 = q0 * sin(mθ) + q1 * cos(mθ)
q'_2 = q2 * cos(mθ) - q3 * sin(mθ)
q'_3 = q2 * sin(mθ) + q3 * cos(mθ)
```

其中 `θ` 是预设的角度基数，通常 `θ_i = 10000^(-2i/d_model)`。

### 4.2 为什么 RoPE 好用

RoPE 有一个很好的性质：**旋转后的 Q 和 K 做点积，结果只和它们的相对位置有关**。

```
RoPE(q, m) · RoPE(k, n) = f(q, k, m - n)
```

这意味着模型天然能学到相对位置关系，而不需要像正弦编码那样自己去学。

### 4.3 RoPE 的超参数：base

RoPE 里有一个重要超参数 `base`，默认 10000。它决定了不同维度旋转角度的频率。

```
base = 10000  → 标准设置，适合 2k~4k 上下文
base = 1000000 → 长上下文模型常用，降低高频衰减
```

后面讲长上下文外推时会再提到 base 的作用。

---

## 5. RoPE 的外推问题

RoPE 虽然比可学习位置编码外推能力强，但也不是无限外推。

当序列长度超过训练时的最大长度时，RoPE 会出现**注意力分数退化**：远距离位置的注意力权重变得很小，模型开始"看不到"远处的 token。

直观理解：训练时模型见过位置 0~4095 的旋转角度，测试时突然要处理位置 8192，对应的角度模型没见过，注意力分布会出问题。

---

## 6. 位置插值（Positional Interpolation）

位置插值是延长 RoPE 上下文窗口最简单的方法。

### 6.1 思想

训练时最大长度是 `L`，想扩展到 `L'`。位置插值把所有位置坐标按比例压缩：

```
新位置 = 原位置 × (L / L')
```

比如训练长度 4096，想扩展到 8192：

```
原位置 8191 → 新位置 8191 × (4096 / 8192) = 4095.5
```

这样所有新位置都落在训练时见过的角度范围内，模型不会"懵"。

### 6.2 优点和缺点

优点：
- 简单，只需要微调少量步数
- 不需要改模型结构

缺点：
- 位置被压缩后，相邻 token 的角度差异变小，短距离区分度下降
- 通常需要继续训练来恢复性能

## 7. ALiBi：另一种思路

ALiBi（Attention with Linear Biases）是另一种位置编码方案，被 MPT、BLOOM 等模型采用。

### 7.1 核心思想

ALiBi 不用旋转，也不加位置 embedding。它直接在 Attention 分数上加上一个**基于距离的负偏置**：

```
Attention_Score(i, j) = Q_i · K_j + m × (i - j)
```

其中 `m < 0` 是一个与 head 相关的斜率。距离越远（即 `i - j` 越小，因为 `j` 是历史位置），注意力分数被压得越低。也可以写成带绝对值的形式 `m × |i - j|`，此时 `m > 0` 表示惩罚。

### 7.2 优点

- **实现简单**：不需要修改 Q/K 的计算方式
- **外推能力强**：因为偏置是线性的，超过训练长度时行为可预测

### 7.3 缺点

- 远距离注意力被强制压低，可能不适合需要强长距离依赖的任务
- 主流模型用得不如 RoPE 多

## 8. 长上下文技术概览

除了位置插值，还有很多延长上下文窗口的技术：

### 8.1 NTK-aware 缩放

NTK（Neural Tangent Kernel）缩放改的是 RoPE 的 base 和频率分布。它不像位置插值那样均匀压缩所有频率，而是**保留高频信息，只压缩低频**，这样短距离区分度不会被破坏。

### 8.2 YaRN

YaRN（Yet another RoPE extensioN method）结合了位置插值和 NTK 的优点，并且重新调整了 Attention Temperature。实验上 YaRN 可以把 Llama 2 的上下文从 4k 扩展到 128k。

### 8.3 LongRoPE

LongRoPE 进一步优化了位置插值，通过搜索最优的非均匀缩放策略，减少微调需求。

### 8.4 Ring Attention / StreamingLLM

这些不是改位置编码，而是从计算层面解决长序列问题：

- **Ring Attention**：把长序列分块，多块 GPU 并行计算 Attention
- **StreamingLLM**：发现 Attention Sink 现象，固定保留开头的几个 token，可以无限延长生成长度

---

## 9. 对推理部署的影响

上下文窗口变长，直接影响推理成本：

| 方面 | 影响 |
|------|------|
| **KV Cache** | 长度线性增长。32k 上下文的 KV Cache 是 4k 的 8 倍 |
| **Prefill 阶段** | 计算量和序列长度平方成正比（Attention 的 O(N²)） |
| **Decode 阶段** | 每步要加载更长的 KV Cache，显存带宽压力增大 |
| **量化方向** | 长上下文下 KV Cache 量化比权重量化更紧迫 |

这也是为什么很多模型标称"支持 128k 上下文"，但实际部署时往往限制在 4k~32k。

---

## 10. 一句话总结

> 位置编码让 Transformer 拥有了"顺序感"，RoPE 因其优雅的相对位置性质成为主流。而长上下文技术的本质，是在模型训练后通过位置插值、NTK 缩放、YaRN 等手段，让位置编码能覆盖更长的序列——但代价是显存和计算量都会显著增加。
