# Transformer 详解：attention 到底在注意什么？

上一章聊了从 N-gram 到 GPT 的演进史，其中 2017 年的 Transformer 是一个分水岭。它彻底改变了自然语言处理，也成为了今天几乎所有大语言模型的基础骨架。

这一章我们就把 Transformer 拆开，看看它到底为什么强，以及 attention 机制到底在"注意"什么。

---

## 1. 为什么需要 Transformer？

### 1.1 RNN 的困局

在 Transformer 之前，序列建模主要靠 RNN 及其变体 LSTM、GRU。

RNN 的处理方式像一个流水线：

```
词 1 → 隐藏状态 1 → 词 2 → 隐藏状态 2 → 词 3 → 隐藏状态 3 → ...
```

每个词只能依赖前面的隐藏状态，信息要一步步传递。这就带来两个问题：

**第一，长距离依赖困难。**

如果第 100 个词要依赖第 1 个词的信息，它必须经过 99 步传递。信息会被稀释、遗忘。

**第二，难以并行。**

RNN 必须按顺序计算，无法像 CNN 那样一次性处理整个序列。在 GPU 上，这种串行特性非常吃亏。

### 1.2 CNN 的尝试

有人用 CNN 做序列建模，把一维卷积滑过句子。CNN 可以并行，但捕捉长距离关系需要很多层卷积，或者很大的感受野。

本质上，CNN 更适合局部特征，不太适合语言这种全局依赖很强的数据。

### 1.3 Transformer 的突破

Transformer 的核心思想非常大胆：

> **完全抛弃循环和卷积，直接用注意力机制捕捉序列中任意两个位置的关系。**

在 Transformer 里，第 100 个词要看第 1 个词，不需要经过 99 步传递，而是**直接建立连接**。

同时，整个序列可以**并行处理**，非常适合 GPU 加速。

## 2. 自注意力：Transformer 的心脏

### 2.1 直觉理解

看这句话：

> "那只猫坐在垫子上，因为它很舒服。"

最后一个"它"指的是什么？根据常见理解，可能是"垫子"，但语义上"猫"也说得通，这个句子本身有歧义。

自注意力机制会让模型计算"它"和前面每个词的关联程度。一种可能的权重分布是：

```
它 → 猫：0.3
它 → 坐在：0.05
它 → 垫子：0.6
它 → 上：0.05
```

"垫子"得到最高分，模型就判断"它"最可能指代"垫子"。注意这反映的是模型的概率偏好，不一定是唯一正确答案。

### 2.2 Query、Key、Value

自注意力的计算需要把每个词变成三个向量：

- **Query（Q）**：我要查询什么？
- **Key（K）**：我有什么信息可以提供？
- **Value（V）**：我真正携带的内容是什么？

对于输入序列中的每个词，模型分别用三个不同的权重矩阵得到 Q、K、V：

```
Q_i = X_i × W_Q
K_i = X_i × W_K
V_i = X_i × W_V
```

其中 `X_i` 是词 i 的输入向量，`W_Q`、`W_K`、`W_V` 是可学习的参数矩阵。

### 2.3 注意力分数计算

对于序列中的每个位置 i，它要关注所有位置 j，关注程度用注意力分数表示：

```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
```

拆解步骤：

1. **QK^T**：计算每个 Query 和每个 Key 的相似度
2. **除以 √d_k**：缩放，防止点积结果过大导致 softmax 梯度消失
3. **softmax**：把分数变成概率分布，所有位置加起来为 1
4. **乘 V**：用概率分布对 Value 做加权求和

最终，每个位置得到一个新的向量，这个新向量融合了它"注意"到的所有位置的信息。

### 2.4 为什么是 Q、K、V 三套？

这是个好问题。为什么不能只用一套向量？

因为 Q、K、V 分工不同：

- **Q 决定"我要什么"**：代表当前位置的查询意图
- **K 决定"我能提供什么"**：代表每个位置可以被匹配的特征
- **V 决定"我给什么"**：代表每个位置真正要传递的内容

这种分离让模型更灵活。比如同一个词，作为"被查询者"和"查询者"时，可以表现出不同的语义角色。

## 3. 多头注意力：多个视角看世界

### 3.1 单一注意力的局限

一个注意力层只能学到一种"关注模式"。但语言中需要关注的关系有很多种：

- 主谓关系："猫"和"坐"
- 指代关系："它"和"垫子"
- 修饰关系："红色"和"苹果"

只用一个注意力头，很难同时学会这么多模式。

### 3.2 多头机制

Transformer 的做法是：**把 Q、K、V 投影到多个低维子空间，分别做注意力，最后拼接起来。**

以 8 个头为例：

```
head_1 = Attention(XW_Q1, XW_K1, XW_V1)
head_2 = Attention(XW_Q2, XW_K2, XW_V2)
...
head_8 = Attention(XW_Q8, XW_K8, XW_V8)

MultiHeadOutput = Concat(head_1, ..., head_8) × W_O
```

每个头可以关注不同的关系。有的头关注语法结构，有的头关注语义指代，有的头关注位置邻近关系。

研究发现，不同注意力头确实会自发学到不同的功能。比如某些头专门处理指代消解，某些头专门处理数字和日期。

## 4. 位置编码：给序列加上顺序

### 4.1 为什么需要位置编码？

Transformer 的自注意力是**位置无关**的。也就是说：

```
"我爱你" 和 "你爱我"
```

如果不加位置信息，Transformer 会认为这两个句子完全一样，因为它们的词集合相同。

显然不对。所以需要给每个词注入位置信息。

### 4.2 正弦余弦位置编码

原始 Transformer 论文使用的是固定位置编码：

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

其中 `pos` 是位置，`i` 是维度索引，`d_model` 是模型维度。

这种编码有两个优点：

1. **唯一性**：每个位置都有独特的编码
2. **相对位置可推断**：通过三角函数的性质，模型可以学到"相距 k 个位置"的关系

### 4.3 可学习位置编码

后来很多模型（如 BERT、GPT）使用**可学习的位置编码**：

> 直接给每个位置分配一个向量，和词向量一起训练。

实践发现，可学习位置编码效果通常更好，但泛化到比训练时更长的序列时会有问题。

### 4.4 RoPE 和 ALiBi

现代 LLM 常用更先进的位置编码：

- **RoPE（Rotary Position Embedding）**：通过旋转矩阵注入位置信息，目前最流行，Llama、Qwen、ChatGLM 都在用
- **ALiBi（Attention with Linear Biases）**：在注意力分数上加上一个基于距离的偏置，外推性很好

这些改进主要是为了解决**长上下文泛化**问题：模型在训练时见过的序列长度有限，但推理时要处理更长的文本。

## 5. 前馈网络：增加非线性表达能力

每个 Transformer 层除了注意力子层，还有一个**前馈神经网络（Feed-Forward Network，FFN）**：

```
FFN(x) = max(0, xW_1 + b_1) W_2 + b_2
```

也就是：

```
线性变换 → ReLU 激活 → 线性变换
```

现代变体常用 **GELU** 替代 ReLU，或者用 **SwiGLU**（Llama、Qwen 等在用）。

FFN 的中间维度通常比模型维度大 4 倍。比如 d_model = 4096，FFN 中间维度就是 16384。

### FFN 的作用

注意力负责**全局信息融合**，FFN 负责**局部特征变换和非线性增强**。

可以粗略理解为：

- 注意力："看看别人在说什么"
- FFN："基于看到的信息，做个复杂的判断"

两者交替工作，一层一层地提炼语义表示。

## 6. Layer Normalization 和残差连接

### 6.1 残差连接

Transformer 很深，比如 GPT-3 有 96 层。深层网络容易出现**梯度消失或爆炸**，训练困难。

残差连接的做法是：把子层的输入直接加到输出上。

```
Output = x + Sublayer(x)
```

这样梯度可以直接沿着短路路径回传，避免梯度消失。注意这里只是残差连接本身，LayerNorm 的放置位置我们在后面 6.3 节讨论。

### 6.2 Layer Normalization

Layer Norm 对每个样本的所有特征做归一化：

```
LayerNorm(x) = γ × (x - μ) / √(σ² + ε) + β
```

它让每一层的输出保持稳定，训练更平滑。

注意 Transformer 原始论文是 **Post-LN**（残差连接后再 LayerNorm），但现代模型常用 **Pre-LN**（先 LayerNorm 再进子层），训练更稳定。

## 7. 编码器、解码器与 Decoder-Only

### 7.1 编码器（Encoder）

编码器由多个相同层堆叠，每层包含：

1. 多头自注意力（双向，可以看到整个输入）
2. 前馈网络
3. 残差连接和 LayerNorm

代表模型：**BERT**。它适合理解任务，如文本分类、命名实体识别、问答。

### 7.2 解码器（Decoder）

解码器也由多个相同层堆叠，每层包含：

1. **掩码多头自注意力**（只能看到当前位置及之前的位置，防止偷看未来）
2. 交叉注意力（可选，关注编码器的输出）
3. 前馈网络
4. 残差连接和 LayerNorm

代表模型：**GPT**。它适合生成任务，因为只能根据前面的词生成后面的词。

### 7.3 原始 Transformer：编码器-解码器

机器翻译时代的 Transformer 同时包含编码器和解码器：

```
源语言句子 → 编码器 → 上下文表示 → 解码器 → 目标语言句子
```

编码器理解源语言，解码器生成目标语言。

### 7.4 为什么现代 LLM 大多用 Decoder-Only？

GPT 系列、Llama、Qwen、Mistral、Claude 等都是 Decoder-Only。原因有几个：

1. **生成任务天然匹配**：语言模型就是预测下一个词，解码器的自回归方式完美契合
2. **结构更简单**：只有一半，工程实现和优化更方便
3. **扩展性更好**：Decoder-Only 模型在规模扩大时表现优异
4. **预训练效率更高**：不需要成对的输入输出数据，任意文本都能训练

BERT 开创了预训练 + 微调范式，但 GPT 把"生成式预训练"做到了极致，最终引爆了大模型时代。

## 8. Transformer 为什么如此强大？

| 特性 | Transformer | RNN | CNN |
|------|------------|-----|-----|
| 长距离依赖 | ✅ 任意距离一步可达 | ❌ 容易遗忘 | ⚠️ 需要大感受野 |
| 并行计算 | ✅ 整个序列同时处理 | ❌ 必须顺序 | ✅ 可以并行 |
| 可解释性 | ✅ 注意力权重直观 | ❌ 隐藏状态难解释 | ⚠️ 卷积核较抽象 |
| 训练稳定性 | ⚠️ 需要小心调参 | ✅ 相对稳定 | ✅ 稳定 |
| 对数据量需求 | 大 | 中 | 中 |

Transformer 的强大不是某一个组件的功劳，而是整个架构设计的协同：

1. **自注意力**提供了强大的全局建模能力
2. **多头机制**让模型同时学习多种关系
3. **位置编码**注入了不可或缺的顺序信息
4. **FFN**增强了非线性表达能力
5. **残差连接和 LayerNorm**让深层训练成为可能
6. **并行计算**让大规模训练成为现实

当这些组件组合在一起，再遇上海量数据和超强算力，量变就引发了质变。

---

## 9. 小结

Transformer 的核心可以概括为一句话：

> **用自注意力机制，让每个词都能直接关注整个上下文中的任何其他词，同时保持高度的并行性。**

关键组件回顾：

| 组件 | 作用 |
|------|------|
| 自注意力 | 计算词与词之间的关联权重 |
| Q/K/V | 分别表示查询、索引、内容 |
| 多头注意力 | 多个子空间并行学习不同关系 |
| 位置编码 | 注入序列顺序信息 |
| FFN | 非线性特征变换 |
| 残差连接 | 缓解梯度消失，支持深层网络 |
| LayerNorm | 稳定每层输出分布 |
| 掩码注意力 | 防止生成时偷看未来 |

理解了 Transformer，你就理解了 GPT、Llama、Qwen 等现代大语言模型的共同骨架。
