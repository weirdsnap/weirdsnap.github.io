# FFN、激活函数与归一化

Transformer 除了 Attention，还有另一个重要组成部分：**前馈神经网络（Feed-Forward Network, FFN）**。再加上**激活函数**和**归一化层**，三者共同构成了 Transformer 块的主体。

这一章我们讲清楚：

- FFN 在 Transformer 里到底起什么作用？
- 为什么从 ReLU 发展到 GELU、SwiGLU？
- LayerNorm 和 RMSNorm 有什么区别？
- 这些设计对推理部署有什么影响？

---

## 1. 为什么 Transformer 需要 FFN？

Attention 的作用是**全局信息聚合**：每个 token 都能和其他 token 交互。

但 Attention 本身是**线性变换**（QK^T 和 softmax 后的加权平均）。如果没有 FFN，Transformer 的多层叠加就只是在做不同形式的线性组合，表达能力有限。

FFN 给模型引入了**非线性**和**维度变换**：

```
Attention 输出：[batch, seq_len, d_model]
FFN：先升维到 4*d_model，再降维回 d_model
```

这个"先放大再压缩"的过程，让模型能在高维空间中对每个位置进行复杂的非线性变换。

可以把 Attention 理解为"交流"，FFN 理解为"思考"。Attention 让 token 之间交换信息，FFN 让每个 token 基于交换后的信息做独立变换。

## 2. 原始 Transformer 的 FFN

原始 Transformer 论文里的 FFN 非常简单：

```python
FFN(x) = max(0, xW1 + b1)W2 + b2
```

用 PyTorch 表示：

```python
class FFN(nn.Module):
    def __init__(self, d_model, d_ff):
        super().__init__()
        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)
        self.relu = nn.ReLU()

    def forward(self, x):
        x = self.linear1(x)   # [batch, seq, d_model] -> [batch, seq, d_ff]
        x = self.relu(x)
        x = self.linear2(x)   # [batch, seq, d_ff] -> [batch, seq, d_model]
        return x
```

其中 `d_ff` 通常是 `4 * d_model`。比如 d_model=4096 时，d_ff=16384。

参数量：

```
W1: d_model × d_ff = 4096 × 16384 = 67,108,864
W2: d_ff × d_model = 16384 × 4096 = 67,108,864
总参数量：约 1.34 亿
```

FFN 的参数量占 Transformer 总参数量的很大一部分。

## 3. 从 ReLU 到 GELU

### 3.1 ReLU 的问题

ReLU 函数：

```
ReLU(x) = max(0, x)
```

优点：简单、计算快。

缺点：
- **神经元死亡**：输入为负时梯度为 0，这个神经元可能永远不再更新
- **输出不是 0 均值**：正数输出会让下一层输入均值偏移

### 3.2 GELU

GELU（Gaussian Error Linear Unit）是 BERT、GPT-2 等模型开始采用的激活函数：

```
GELU(x) = x × Φ(x)
```

其中 `Φ(x)` 是标准正态分布的累积分布函数。

直观理解：GELU 对输入做"概率门控"，输入越大越可能被保留，输入越小越可能被置零，但过渡比 ReLU 平滑。

优点：
- 没有"硬截断"，梯度不会突然为 0
- 输出更接近 0 均值
- 实验上比 ReLU 效果好

### 3.3 近似实现

实际计算中，GELU 常用 tanh 近似：

```python
def gelu_approx(x):
    return 0.5 * x * (1 + torch.tanh(
        torch.sqrt(torch.tensor(2.0 / 3.141592653589793)) *
        (x + 0.044715 * torch.pow(x, 3))
    ))
```

PyTorch 有内置实现：

```python
x = torch.nn.functional.gelu(x)
```

## 4. SwiGLU 和门控 FFN

### 4.1 GLU 结构

GLU（Gated Linear Unit）最早用于 NLP，结构是：

```
GLU(x) = (xW + b) ⊗ σ(xV + c)
```

其中：
- `xW + b` 是主分支
- `σ(xV + c)` 是门控分支（用 sigmoid）
- `⊗` 是逐元素乘法

门控分支控制主分支哪些信息可以通过。

### 4.2 SwiGLU

SwiGLU 把 sigmoid 换成 Swish（也叫 SiLU）：

```
Swish(x) = x × σ(x)
SwiGLU(x) = (xW + b) ⊗ Swish(xV + c)
```

Swish 比 sigmoid 更平滑，训练更稳定。

### 4.3 现代 LLM 的 FFN

Llama、Qwen、PaLM 等模型用的是 **SwiGLU 变体**：

```python
class SwiGLU(nn.Module):
    def __init__(self, d_model, d_ff):
        super().__init__()
        self.gate_proj = nn.Linear(d_model, d_ff, bias=False)
        self.up_proj = nn.Linear(d_model, d_ff, bias=False)
        self.down_proj = nn.Linear(d_ff, d_model, bias=False)

    def forward(self, x):
        gate = torch.silu(self.gate_proj(x))
        up = self.up_proj(x)
        x = self.down_proj(gate * up)
        return x
```

注意这里用了三个线性层，而不是两个。

为了控制参数量，`d_ff` 通常取 `2/3 * 4 * d_model`，即原始 FFN 隐藏层大小的 2/3。这样三个矩阵的总参数量和原始两个矩阵差不多。

## 5. 激活函数对比

| 激活函数 | 公式 | 优点 | 缺点 | 代表模型 |
|---------|------|------|------|---------|
| ReLU | `max(0, x)` | 简单快速 | 神经元死亡 | 原始 Transformer |
| GELU | `x × Φ(x)` | 平滑、效果好 | 计算稍复杂 | BERT、GPT-2 |
| Swish/SiLU | `x × σ(x)` | 自门控、平滑 | - | MobileNet、EfficientNet |
| SwiGLU | 门控结构 | 效果最佳 | 三个矩阵 | Llama、Qwen、PaLM |

从实验结果看，SwiGLU 在大多数 LLM 任务上表现最好，所以现在主流模型基本都用 SwiGLU。

## 6. 归一化：LayerNorm

### 6.1 为什么需要归一化

神经网络训练时，每一层的输入分布会不断变化，一种常见的解释是**内部协变量偏移（Internal Covariate Shift）**，即每一层输入的分布不断变化，归一化可以缓解这个问题。

归一化的作用是让每一层的输入保持稳定的分布，加速训练、提高稳定性。

### 6.2 LayerNorm

LayerNorm 对每个样本的每个位置单独做归一化：

```python
def layer_norm(x, gamma, beta, eps=1e-6):
    mean = x.mean(dim=-1, keepdim=True)
    var = x.var(dim=-1, keepdim=True, unbiased=False)
    x_norm = (x - mean) / torch.sqrt(var + eps)
    return gamma * x_norm + beta
```

其中 `gamma` 和 `beta` 是可学习的缩放和平移参数。

### 6.3 预归一化 vs 后归一化

Transformer 有两种归一化放置方式：

**后归一化（Post-LN，原始 Transformer）**：

```
x = LayerNorm(x + Attention(x))
x = LayerNorm(x + FFN(x))
```

**预归一化（Pre-LN，现代 LLM）**：

```
x = x + Attention(LayerNorm(x))
x = x + FFN(LayerNorm(x))
# 最后可能再加一个 LayerNorm
```

区别在于 LayerNorm 是在残差连接**内部**还是**外部**：Post-LN 先加后归一化，Pre-LN 先归一化再加。

Post-LN：
```
x = LayerNorm(x + Attention(x))
x = LayerNorm(x + FFN(x))
```

Pre-LN：
```
x = x + Attention(LayerNorm(x))
x = x + FFN(LayerNorm(x))
# 最后可能再加一个 LayerNorm
```

现代 LLM 几乎都用 Pre-LN，因为它训练更稳定，更容易扩展到深层网络。

## 7. RMSNorm

### 7.1 从 LayerNorm 到 RMSNorm

RMSNorm（Root Mean Square Normalization）是 LayerNorm 的简化版。它去掉了均值计算，只对 RMS 做归一化：

```python
def rms_norm(x, gamma, eps=1e-6):
    rms = torch.sqrt(torch.mean(x ** 2, dim=-1, keepdim=True) + eps)
    x_norm = x / rms
    return gamma * x_norm
```

### 7.2 为什么 RMSNorm 更受欢迎

1. **计算量更小**：不用算均值
2. **效果相当**：实验发现去掉均值影响不大
3. **训练稳定**：和 Pre-LN 配合效果很好

Llama、Qwen、Mistral 等模型都使用 RMSNorm。

### 7.3 对推理的影响

RMSNorm 比 LayerNorm 省一点点计算。对于超大模型和超长序列，这点节省会累积起来。

更重要的是：RMSNorm 没有 `beta` 参数，模型文件里少了一部分数据，加载和存储都稍微轻量一点。

## 8. 对推理部署的影响

### 8.1 FFN 是参数大户

FFN 通常占模型总参数的 2/3 左右。所以：

- 量化 FFN 权重能显著减小模型体积
- FFN 的计算量和参数量大，是推理时的主要开销之一

### 8.2 激活函数影响推理速度

不同激活函数的计算成本不同：

| 激活函数 | 相对速度 |
|---------|---------|
| ReLU | 最快 |
| GELU | 中等 |
| SwiGLU | 较慢（但现代 GPU 优化得很好） |

不过在实际 LLM 推理中，激活函数通常不是瓶颈，瓶颈在 Attention 和矩阵乘法。

### 8.3 归一化影响训练到推理的转换

一些推理引擎对 LayerNorm/RMSNorm 做了专门优化：

- 把 gamma 和 beta 合并到前面的线性层权重里（kernel fusion）
- 把 RMSNorm 和后面的矩阵乘法融合成一个 kernel

这些优化能减少显存读写，提高推理吞吐。

---

## 9. 一句话总结

> FFN 给 Transformer 引入非线性变换，SwiGLU 是目前主流的 FFN 设计；激活函数从 ReLU 到 GELU 再到 SwiGLU，趋势是越来越平滑、越来越像门控；归一化从 LayerNorm 到 RMSNorm，计算越来越简洁。这些设计选择不仅影响训练效果，也决定了模型推理时的参数量、计算量和可优化空间。
