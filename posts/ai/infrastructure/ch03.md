# GPU 互联拓扑：NVLink、NVSwitch 与 PCIe

前面两章聊了多机之间的高速通信：RDMA、InfiniBand、RoCEv2。但在考虑"机器之间怎么传数据"之前，还有一个更基础的问题：

**单机内部的多张 GPU，是怎么连起来的？**

这个问题很关键。因为单机内的 GPU 互联带宽，往往比跨机网络高一个数量级。理解它，才能理解为什么分布式训练里要区分 **Tensor Parallelism（TP，张量并行）** 和 **Data Parallelism（DP，数据并行）**，以及为什么 TP 通常优先放在单机内。

---

## 1. PCIe：通用但不够快的总线

### 1.1 什么是 PCIe

PCIe（Peripheral Component Interconnect Express）是服务器里最通用的外设互联总线。CPU、GPU、网卡、SSD 都挂在 PCIe 上。

```
CPU ── PCIe Root Complex ── PCIe Switch ── GPU 1
                                      ├─ GPU 2
                                      ├─ GPU 3
                                      └─ GPU 4
```

### 1.2 PCIe 的带宽

PCIe 每一代的单 lane 带宽如下：

| 版本 | 单 lane 速率 | x16 双向带宽 |
|------|------------|-------------|
| PCIe 3.0 | 8 GT/s | ~32 GB/s |
| PCIe 4.0 | 16 GT/s | ~64 GB/s |
| PCIe 5.0 | 32 GT/s | ~128 GB/s |

注意这是**双向带宽**，实际有效带宽还要考虑协议开销，通常按 80% 估算。

### 1.3 PCIe 的问题

对于 AI 训练来说，PCIe 有两个明显瓶颈：

1. **带宽不够**：一张 H100 的显存带宽是 3 TB/s，而 PCIe 5.0 x16 只有 ~128 GB/s，差了一个数量级
2. **需要经过 CPU**：GPU 之间通过 PCIe 通信，数据往往要经过 CPU 的 Root Complex 或 PCIe Switch，增加了延迟

所以 PCIe 适合连接 CPU 和外设，但不适合作为 GPU 之间高速互联的首选。

## 2. NVLink：专为 GPU 设计的高速互联

### 2.1 为什么需要 NVLink

NVIDIA 在 2016 年推出 NVLink，目的就是解决 GPU 之间通过 PCIe 通信太慢的问题。

NVLink 的设计目标：

- **高带宽**：远超 PCIe
- **低延迟**：绕过 CPU，GPU 直连
- **支持缓存一致性**：允许多 GPU 共享内存地址空间

### 2.2 NVLink 带宽演进

| 版本 | 单链路带宽 | 典型配置 |
|------|-----------|---------|
| NVLink 1.0 | 20 GB/s | P100 |
| NVLink 2.0 | 25 GB/s | V100 |
| NVLink 3.0 | 50 GB/s | A100 |
| NVLink 4.0 | 50 GB/s | H100 |

注意：NVLink 4.0 单链路速率没有提升，但链路数量增加，整体带宽更高。

以 H100 为例，单卡支持 **18 条 NVLink 4.0 链路**，总双向带宽达到 **900 GB/s**，是 PCIe 5.0 x16 的 7 倍左右。

### 2.3 NVLink 的典型连接方式

早期 NVLink 是 GPU 之间点对点连接：

```
GPU 1 ←────→ GPU 2
  ↑      ↗
GPU 3 ←────→ GPU 4
```

但这种方式有个问题：当 GPU 数量增加时，每个 GPU 的链路数有限，无法做到任意两卡之间全连接。比如 4 张 GPU 两两互联需要 6 条链路，8 张 GPU 需要 28 条，单机 GPU 数量再多就难以实现。

这就需要 **NVSwitch**。

---

## 3. NVSwitch：让 8 卡甚至更多卡全连接

### 3.1 NVSwitch 是什么

NVSwitch 是 NVIDIA 推出的专用交换芯片，用于实现多个 GPU 之间的全连接（full mesh）。

它本质上是一个**专门为 NVLink 设计的交换机**。

### 3.2 DGX-1 / DGX-A100 / DGX-H100 的拓扑

#### DGX-1（V100 时代）

- 8 张 V100
- 6 个 NVSwitch 芯片
- 任意两张 GPU 之间通过 NVSwitch 连接，形成一个全连接网络
- 任意两卡之间带宽：~150 GB/s

#### DGX-A100

- 8 张 A100
- 6 个第三代 NVSwitch
- 任意两卡之间 NVLink 带宽：~600 GB/s
- 支持 **NVLink Switch System**，可以跨服务器连接多张 A100

#### DGX-H100

- 8 张 H100
- 4 个第四代 NVSwitch
- 任意两卡之间 NVLink 带宽：~900 GB/s
- 支持 **NVLink Switch System 2.0**，最多可连接 256 张 H100 形成一个大型计算域

### 3.3 NVSwitch 的意义

```
        NVSwitch
       /   |   \
    GPU1  GPU2  GPU3 ... GPU8
       \   |   /
        NVSwitch
```

在 NVSwitch 出现之前，8 卡之间的通信需要经过多跳；有了 NVSwitch 之后，任意两卡之间都是**一跳直达**，带宽和延迟都更稳定。

这对 **Tensor Parallelism** 至关重要。TP 需要在每张前向/反向传播中频繁同步中间激活值，如果通信延迟不稳定，训练效率会大幅下降。

---

## 4. 三种互联方式对比

| 维度 | PCIe | NVLink | NVSwitch |
|------|------|--------|----------|
| **用途** | 通用外设互联 | GPU 直连 | 多 GPU 全连接 |
| **带宽** | ~32-128 GB/s | ~150-900 GB/s | 同 NVLink |
| **是否经过 CPU** | 是 | 否 | 否 |
| **缓存一致性** | 否 | 是 | 是 |
| **拓扑** | 树形 | 点对点 | 全连接 |
| **典型场景** | CPU-GPU、GPU-网卡 | 双 GPU、早期多 GPU | 8 GPU 服务器、跨机 NVLink |

---

## 5. 对分布式训练的影响

### 5.1 为什么 TP 通常放在单机内

Tensor Parallelism 需要在每次前向和反向传播中同步激活值，通信非常频繁。

- 单机内 NVSwitch 提供 **600–900 GB/s 级** 稳定带宽
- 跨机即使使用 InfiniBand NDR，也只有 **400 Gbps = 50 GB/s** 左右

两者差了一个数量级。所以 TP 通常只放在单机内的 8 卡之间，不会跨机。

### 5.2 DP / PP 可以跨机

- **Data Parallelism**：每个 GPU 独立计算梯度，只在反向传播结束后 AllReduce 梯度，通信频率低，可以跨机
- **Pipeline Parallelism**：按层切分模型，通信的是激活值，频率介于 TP 和 DP 之间，可以跨机但需要考虑流水线气泡

```
单节点 8 卡：        多节点集群：
┌─────────────┐      ┌─────┐  ┌─────┐  ┌─────┐
│ GPU0 ... 7  │      │Node1│  │Node2│  │Node3│
│  NVSwitch   │      │TP×8 │  │TP×8 │  │TP×8 │
└─────────────┘      └──┬──┘  └──┬──┘  └──┬──┘
                        └────┬───┘        │
                          IB/RoCE          │
                             └─────────────┘
                              DP/PP AllReduce
```

### 5.3 网络拓扑设计原则

基于以上分析，一个常见的大型训练集群设计是：

- **单节点内**：8 张 GPU 通过 NVSwitch 全连接，跑 TP
- **节点间**：通过 InfiniBand 或 RoCEv2 连接，跑 DP / PP
- **模型特别大时**：使用 3D 并行（TP + PP + DP）组合

---

## 6. 小结

- **PCIe** 是通用总线，带宽有限，通常用于 CPU-GPU 和外设连接
- **NVLink** 是 NVIDIA GPU 专用高速互联，绕过 CPU，带宽远高于 PCIe
- **NVSwitch** 让单机内 8 卡甚至更多卡实现全连接，是 Tensor Parallelism 的物理基础
- 分布式训练的策略选择，很大程度上取决于**单机内和跨机的带宽差距**

下一章我们可以聊聊 **NCCL** —— NVIDIA 的集合通信库。它屏蔽了底层是 NVLink、PCIe 还是 IB/RoCE 的差异，让上层代码只需要调用 `ncclAllReduce` 这样的 API。
