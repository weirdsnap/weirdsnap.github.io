# 从模型到推理：一次推理调用的拆解

LLM 推理时的显存主要由两大消耗者组成：模型权重和 KV Cache。

有读者可能会问：KV Cache 存的是什么？推理前向到底怎么跑的？这些问题在 [AI Model 基础系列](./blog.html?post=ai/models/ch01.md) 中有详细讲解（Transformer 结构、自注意力机制等），这里只做一个快速衔接，聚焦"从模型到推理"的运行时视角转换。

---

## 1. 训练 vs 推理：两种完全不同的视角

学习完 [models 系列](./blog.html?post=ai/models/ch01.md) 后，你已经知道：

- 大模型 ≈ Transformer 层的堆叠
- 每个 Transformer 层 = Self-Attention + FFN
- Self-Attention 的公式：Attention(Q, K, V) = softmax(QKᵀ/√d)V

但这些知识是**训练视角**的。切换到**推理视角**时，有三个关键差异：

**视角切换**：

| | 训练 | 推理 |
|---|---|---|
| **输入** | 整个序列一次喂入，并行算 loss | 一个 token 一个 token 往外出 |
| **batch** | 大量样本同时训练 | 单个或少量的用户请求 |
| **目标** | 更新参数、降低 loss | 从概率分布采样生成文本 |
| **瓶颈** | 算力（matrix multiply） | 显存带宽（memory bandwidth） |

训练看的是"一批数据能不能并行算完"，推理看的是"用户请求能不能实时响应"。

---

## 2. 推理的两阶段：Prefill 和 Decode

一次完整的推理调用，内部其实分成两个阶段：

### 2.1 Prefill（预填充）

```
用户输入 "请介绍一下你自己"
       ↓
分词 → [1024, 305, 208, ...]   ← 一串 token ID
       ↓
一次性通过所有 Transformer 层   ← 可以并行，和训练时一样
       ↓
生成第一个 token + 构建 KV Cache ← 这就是 Prefill 阶段产生的主要内存开销
```

特点：**计算密集型**，和训练类似，适合 GPU。

### 2.2 Decode（解码）

```
已有：[请介绍一下你自己] 我
       ↓
输入最后一个 token "我"
       ↓
通过 Transformer 层（复用 KV Cache，不需要重新算前面的）
       ↓
生成下一个 token "是"
       ↓
继续...
```

特点：**显存密集型**，每次只算一个 token，GPU 利用率很低。

这也是为什么推理优化的大部分精力都花在"如何让 decode 阶段更快"——我们后面几章会大量回到这个话题。

## 3. KV Cache 到底是什么？

我们在推理部署 ch01 从**内存角度**算过一笔账：KV Cache 占多少 GB。

从**计算角度**来理解更直观：

> KV Cache = Self-Attention 中算过的 Key 和 Value 矩阵的缓存

每次生成新 token 时，Self-Attention 需要计算当前 token 和所有历史 token 的注意力分数。如果没有缓存，每步都要重新算一遍前面所有 token 的 K 和 V——代价大到不可接受。

解决方案就是：**算过的 K 和 V 存下来，下次直接读**。

```
第一次（Prefill）：
  "请" → K₀, V₀  (缓存)
  "介" → K₁, V₁  (缓存)
  "绍" → K₂, V₂  (缓存)
  ...

第二次（Decode step 1）：
  输入 "我"
  从缓存读取 K₀~Kₙ, V₀~Vₙ  ← 省掉了重新计算的成本
  计算 attention(Q("我"), K₀~Kₙ, V₀~Vₙ)
  生成 "是"，缓存 K("我"), V("我")

第三次（Decode step 2）：
  输入 "是"
  从缓存读取 K₀~Kₙ₊₁, V₀~Vₙ₊₁
  ...
```

这个缓存就是前文里占据大量显存的 **KV Cache**。

## 4. 整条链路速览

用一个流程图总结一次推理请求的完整路径：

```
用户输入 prompt
       ↓
① Tokenizer: "请介绍一下你自己" → [1024, 305, 208, ...]
       ↓
② Transformer 层 × N (Prefill)
   ┌─────────────────────────────┐
   │  Self-Attention: QKV 计算    │
   │  → softmax(QKᵀ/√d)V         │
   │  → 构建并缓存 K、V           │
   ├─────────────────────────────┤
   │  FFN: 非线性变换              │
   ├─────────────────────────────┤
   │  LayerNorm + Residual        │
   └─────────────────────────────┘
       ↓
③ LM Head: hidden → vocab 概率分布
       ↓
④ Sampler: 采样/选最大概率 → 第一个 token
       ↓
⑤ 进入 Decode 循环（直到遇到 EOS 或达到 max_tokens）
   复用 KV Cache → Transformer 层 × N (一次一个 token)
   → LM Head → Sampler → 下一个 token
       ↓
⑥ 输出完整的生成文本
```
