# GQA：从模型架构缩减 KV Cache

Flash Attention 让 Attention 计算更快，但 Attention 计算的对象——KV Cache——本身的大小也值得关注。

**GQA（Grouped Query Attention）** 从模型架构层面入手，在训练时就减少了 KV Cache 的总量。

---

## 1. 从 MHA 到 MQA 到 GQA

### 1.1 MHA：原始 Transformer 的多头注意力

在标准 Transformer 的 Multi-Head Attention 中，每个注意力头都独立维护自己的 Key、Query、Value：

```
MHA（多头注意力）：
  Head 0:    Q₀    K₀    V₀
  Head 1:    Q₁    K₁    V₁
  Head 2:    Q₂    K₂    V₂
  ...（假设 32 个头）

→ 每个头的 KV 都要独立缓存
→ KV Cache = 2 × num_heads × head_dim × seq_len
→ 32 个头时，KV Cache 是 1×（基准线）
```

KV Cache 的大小和**注意力头数成正比**。一个很自然的想法：能否减少 K 和 V 的头数，从而缩减 KV Cache？

### 1.2 MQA：所有查询头共享一组 KV

**MQA（Multi-Query Attention）** 由谷歌在 2019 年提出，是第一个尝试：

```
MQA：所有 Query 头共享 1 组 K、V
  Head 0:    Q₀ ──┐
  Head 1:    Q₁ ──┤── 所有 Query 头共享 1 组 K、V
  Head 2:    Q₂ ──┘
                  K₀  V₀     （只有 1 组 Key/Value）

→ KV Cache = 2 × 1 × head_dim × seq_len
→ KV Cache 缩小到原来的 1/32！
```

MQA 的 KV Cache 降低了一个数量级，但代价是所有 query 头被迫使用相同的 key/value 表示。实验表明，MQA 在部分任务上有质量损失——信息多样性不够。

### 1.3 GQA：折中方案

**GQA（Grouped Query Attention）** 由谷歌在 2023 年提出，是 MHA 和 MQA 的折中：

```
GQA（分组查询注意力）：
                  Query 头      Key/Value 头
  Group 0:        Q₀  Q₁       K₀  V₀
  Group 1:        Q₂  Q₃       K₁  V₁
  
→ KV Cache = 2 × num_kv_heads × head_dim × seq_len
→ 如果 num_kv_heads = 2（32 个 query 头分为 2 组）
→ KV Cache 缩小到原来的 2/32 = 1/16
```

### 1.4 三种注意力机制的对比

| 注意力机制 | KV 头数 | 相对 KV Cache | 表达能力 |
|-----------|--------|:------------:|:--------:|
| **MHA** | 等于 query 头数（如 32） | 1×（基准线） | 最强 |
| **MQA** | 1 | 1/32 | 最弱 |
| **GQA** | 2~8 | 1/16 ~ 1/4 | 接近 MHA |

GQA 是"免费午餐"式的优化：KV Cache 降到 1/4 ~ 1/8，但质量损失极小。

## 2. GQA 的实际应用

### 2.1 哪些模型在用

| 模型 | 注意力机制 | KV 头数 | KV Cache 缩减比例 |
|------|-----------|:-------:|:----------------:|
| Llama 1 | MHA | 32 | 1×（基准） |
| Llama 2 7B/13B | MHA | 32 | 1× |
| **Llama 2 70B** | **GQA-8** | **8** | **1/4** |
| **Llama 3 / 3.1 / 3.2 全系列** | **GQA-8** | **8** | **1/4** |
| **Qwen2.5 7B** | **GQA-4** | **4** | **1/7** |
| **Qwen2.5 72B** | **GQA-8** | **8** | **1/4** |
| **Qwen2.5 32B** | **GQA-8** | **8** | **1/4** |
| Mistral 7B / Mixtral 8×7B | GQA-8 | 8 | 1/4 |
| **Gemma 2 9B/27B** | **GQA-8** | **8** | **1/4** |
| Yi 6B/34B | GQA-4 | 4 | 1/8 |
| **DeepSeek-V2/V3/R1** | **MLA** | — | 极致压缩 |
| **DeepSeek-V4** | **Hybrid (SWA + HCA + CSA)** | — | 按层自适应 |
| **Grok-1** | **GQA-8** | **8** | **1/4** |

从 Llama 2 70B 开始，几乎所有新发布的模型都采用了 GQA。**Llama 3 系列将 GQA 下放到全尺寸模型**，即使 8B 也使用 GQA-8，而非像 Llama 2 那样只有 70B 大模型才用。

### 2.2 为什么 GQA 几乎不损失质量？

直觉理解：对于 Attention 计算，**query 头之间的多样性更多体现在 Q 上**。K 和 V 相对更"通用"——同一个 key 服务于多个 query 头时，这些 query 头仍然可以从不同角度去匹配这个 key。

实验也验证了这一点：GQA-8（32 个 query 头分为 8 组，每组共享一组 KV）在几乎全部 benchmark 上持平 MHA。

## 3. GQA 与后续优化的配合

### 3.1 经典三件套

| 技术 | 层面 | 解决的问题 |
|------|------|-----------|
| **GQA** | 模型架构 | 从源头缩减 KV Cache 总量 |
| **PagedAttention** | 内存管理 | KV Cache 的分页分配与共享 |
| **Flash Attention** | 计算优化 | 减少 Attention 的 HBM 带宽消耗 |

三者是**正交且互补**的：

- GQA 让 KV Cache 需求降到 1/4
- PagedAttention 把剩下的 KV Cache 用得更高效（消除碎片、COW 共享）
- Flash Attention 让 Attention 计算不再受限于 HBM 带宽

### 3.2 KV Cache 量化

GQA 缩减了 KV Cache 的**头数**，KV Cache 量化则在**精度**上进一步压缩：

- **KV4-KV8 量化**：将 KV Cache 的浮点值量化到 4-bit 或 8-bit，可将内存占用再压缩 2~4 倍
- **KIVI**（2024）：第一个系统化的非对称 KV Cache 量化方案
- GQA + KV Cache 量化可叠加：GQA-8 将头数降到 1/4，再配合 4-bit 量化降到 1/4，总压缩倍率可达 16×
- 主流推理框架（vLLM、TensorRT-LLM、SGLang）均已集成

### 3.3 跨层 KV 共享（Cross-Layer KV Cache Sharing）

更进一步：相邻层的 KV 高度相似，是否可以直接共享？

- **CLA（Cross-Layer Attention，跨层注意力）**：某些层之间共享 KV投影矩阵，不每层独立存储。多层共用一个 KV Cache slot。Llama 4 系列已部分采用
- **YoCo（Y-shaped Cross-layer KV Cache）**：每 N 层才存一份 KV Cache，其余层复用
- **效果**：KV Cache 再降低 2~4 倍
- 已从研究方向逐步进入工程实践，部分推理框架开始支持

## 4. 延伸：DeepSeek 的注意力架构演进

### 4.1 MLA（V2/V3）：潜空间压缩

DeepSeek-V2/V3/R1 提出了一种比 GQA 更激进的 KV Cache 压缩方案——**MLA（Multi-head Latent Attention，多查询潜注意力）**。

MLA 的核心思想：不直接缓存 K 和 V 的全部维度，而是将 K 和 V **压缩到一个低维的潜空间（latent space）** 中缓存，计算时再解压回来。

```
标准 MHA：缓存完整 K、V → 维度 = head_dim × num_kv_heads
MLA：    缓存压缩后的 K、V → 维度 = d_compress（远小于原始维度）
         计算时用 LoRA 风格的矩阵解压
```

MLA 将 KV Cache 降低到标准 MHA 的 **~1/10** 甚至更低，且声称质量持平或优于 MHA。

### 4.2 V4：混合注意力架构

DeepSeek-V4 放弃了 MLA，改用**混合注意力（Hybrid Attention）架构**。每层在三种注意力模式中选择一种，由**每层压缩比**（per-layer compression ratio）控制：

| 注意力模式 | 缩写 | 含义 | 适用场景 |
|-----------|:----:|:----|:--------|
| **滑窗注意力** | SWA | 固定窗口大小的局部注意力 | 局部上下文捕获 |
| **重压缩注意力** | HCA | 对 KV 做高强度压缩（Heavily Compressed Attention） | 长程语义压缩 |
| **压缩稀疏注意力** | CSA | 学习选择哪些 key 参与 attention（Compressed Sparse Attention） | 关键信息精准定位 |

V4 不再基于 MLA 的 latent-KV，而是用三种不同的注意力机制按层分工。V4 的新增关键组件：

- **Sink-softmax**：传统 softmax 的改进变体，优化了注意力分布的长尾问题
- **流形约束超连接（mHC）**：替代传统残差连接，用 Sinkhorn/Birkhoff 双随机矩阵做多路混合，每 token 动态调整各路径权重
- **FP4/FP8 量化权重**：权重压缩到 4-bit/8-bit，推理框架直接加载量化权重

V4 的混合注意力本质上是**用架构灵活性换取推理效率**：靠近输入的低层用滑窗捕捉局部模式，中层用 HCA 压缩语义，高层用 CSA 精准选择关键信息。这使得 V4 在 1M token 上下文窗口下仍保持可控的 KV Cache 开销。

---

## 5. 一句话总结

> GQA 在模型训练阶段就决定让多个 Query 头共享同一组 Key/Value，推理时 KV Cache 直接按比例缩小——压缩发生在模型架构层面，运行时无需额外处理。

**当前趋势**：GQA 已成为事实标准，所有新发布的 LLM 几乎都默认采用；在其之上，KV Cache 量化和跨层 KV 共享进一步叠加压缩比例，MLA 和 V4 的混合注意力则在极低 KV Cache 路线上持续演进。
