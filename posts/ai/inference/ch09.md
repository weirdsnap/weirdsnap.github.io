# 模型量化：用 INT4/INT8 让 LLM 瘦身

当你去 Hugging Face 或 ModelScope 下载一个大模型时，经常会看到文件名里带着一堆后缀：

```
Meta-Llama-3-8B-Instruct-Q4_K_M.gguf
Qwen2.5-7B-Instruct-AWQ
Llama-2-7b-chat-int8
```

这些 `Q4_K_M`、`AWQ`、`int8` 都是**量化（Quantization）**的标记。它们的意思是把模型权重从高精度浮点数压缩成低精度整数，让模型体积更小、加载更快、推理时占的显存更少。

这一章我们讲清楚量化到底是怎么工作的，以及下载模型时这些名字到底在说什么。

---

## 1. 为什么需要量化？

一个 70B 参数的模型，如果用 FP16（16 位浮点数）存储：

```
70B 参数 × 2 字节 = 140 GB
```

单张消费级显卡（比如 24GB 显存）根本放不下。即使能放下，推理时还有 KV Cache、激活值等额外开销。

量化的核心思路很简单：**权重不需要那么高的精度**。

实验发现，把 FP16 权重映射到 INT8 甚至 INT4，模型在很多任务上的表现下降非常小，但体积和显存占用可以降到原来的 1/2 或 1/4。

| 精度 | 每参数字节 | 7B 模型大小 | 70B 模型大小 |
|------|-----------|------------|-------------|
| FP32 | 4 | 28 GB | 280 GB |
| FP16 | 2 | 14 GB | 140 GB |
| INT8 | 1 | 7 GB | 70 GB |
| INT4 | 0.5 | 3.5 GB | 35 GB |

> 注意：这只是权重占用的估算，实际运行时还有 KV Cache、激活值、优化器状态等额外开销。

---

## 2. 量化到底是什么？

量化就是把连续的浮点数值映射到离散的整数数值。

假设有一组 FP16 权重，取值范围在 `[-1.0, 1.0]` 之间。我们想用 INT8 表示它们，INT8 能表示的范围是 `[-128, 127]`。

最简单的方式是**线性映射**。假设权重关于 0 对称，取值范围在 `[-1.0, 1.0]` 之间，用 INT8 对称量化：

```
scale = max(|原始值|) / 127 = 1.0 / 127 ≈ 0.00787

量化值 = round(原始值 / scale)
原始值 ≈ 量化值 × scale
```

举个例子：

```
原始权重：0.5
scale：0.00787
量化值：round(0.5 / 0.00787) = round(63.53) = 64
反量化：64 × 0.00787 = 0.5037
误差：0.5037 - 0.5 = 0.0037
```

这个误差很小，而且大量权重量化后的误差会互相抵消，所以模型整体表现不会明显下降。

---

## 3. 对称量化 vs 非对称量化

上面的例子假设权重分布关于 0 对称，这种方式叫**对称量化（Symmetric Quantization）**。

```
量化值 = round(原始值 / scale)
scale = max(|原始值|) / 127
```

对称量化的好处是计算简单，恢复时只需要乘一个 scale。

如果权重分布不关于 0 对称（比如激活值经过 ReLU 后都是正数），就需要**非对称量化（Asymmetric Quantization）**，多引入一个 `zero point`：

```
scale = (max - min) / 255
zero_point = round(-min / scale)

量化值 = round(原始值 / scale) + zero_point
原始值 ≈ scale × (量化值 - zero_point)
```

对于 LLM 权重来说，通常分布近似对称，所以对称量化更常用。但激活值的量化往往需要用非对称量化。

---

## 4. 训练后量化 vs 量化感知训练

量化有两种主要路线：

### 训练后量化（PTQ, Post-Training Quantization）

模型已经训练好了，直接对权重做量化。优点是简单、快速、不需要重新训练。缺点是精度损失可能比 QAT 大一些。

大多数 LLM 量化工具（GPTQ、AWQ、GGUF）都是 PTQ。

### 量化感知训练（QAT, Quantization-Aware Training）

在训练过程中模拟量化误差，让模型学会适应低精度表示。优点是精度高，缺点是需要重新训练，成本高。

对于超大模型来说，QAT 基本不可行，所以 PTQ 是主流。

---

## 5. INT8 量化

INT8 量化是最直观的量化方式，把 FP16 权重映射到 8 位整数。

```
FP16 权重范围：[-1.0, 1.0]
INT8 表示范围：[-128, 127]
scale = 1.0 / 127 ≈ 0.00787

量化：int8_weight = round(fp16_weight / scale)
反量化：fp16_weight ≈ int8_weight × scale
```

INT8 能把模型体积减半，精度损失通常很小（<1%）。很多推理框架（TensorRT、ONNX Runtime）都原生支持 INT8 推理。

但 INT8 对激活值也做量化时，需要校准（calibration）：用一批真实输入数据跑模型，找出激活值的动态范围，再确定 scale。

---

## 6. INT4 与分组量化

INT4 只能表示 16 个值：

```
有符号 INT4：[-8, 7]
无符号 INT4：[0, 15]
```

精度比 INT8 低很多，直接把整个层的权重按一个 scale 量化会损失很大。

解决方法是**分组量化（Group-wise Quantization）**：把权重分成很多小组，每组单独计算 scale。比如每 128 个权重一组，每组有自己的 scale。

```
原始权重：[0.1, -0.5, 0.8, ..., 0.3]  共 1024 个
分组：每 128 个一组，共 8 组
每组分别计算 scale 和量化值
```

分组越细，精度越高，但存储的 scale 也越多，会占用额外空间。

GGUF 格式里的 `Q4_K_M`、`Q5_K_S` 等，本质上就是不同的分组策略和位分配方案。

---

## 7. GPTQ：用二阶信息减少误差

INT4 直接量化误差较大，因为每次四舍五入都会丢失一些信息。

GPTQ 的核心思想是：**量化当前权重时，把误差传递给还没量化的权重，让它们去补偿**。

更具体地说，GPTQ 用 Hessian 矩阵（二阶导数信息）来计算每个权重对损失函数的影响。重要的权重尽量保留精度，不重要的权重承担更多误差。

```
朴素 INT4：每个权重独立量化，误差各自为政
GPTQ：量化第 i 个权重时，把误差分配到后面还没量化的权重上
```

GPTQ 通常以层为单位逐层进行，不需要反向传播，所以仍然是 PTQ，速度比 QAT 快很多。

---

## 8. AWQ：保护对激活值敏感的权重

AWQ 观察到：**不是所有权重都同等重要**。

有些权重连接的是激活值中幅度很大的神经元，这些权重被量化后误差会被放大；而有些权重连接的激活值很小，量化误差影响不大。

AWQ 的做法是：

1. 分析每一层的激活值分布
2. 找出对激活值敏感的权重（salient weights）
3. 给这些权重更大的"保护"，让它们保留更多精度
4. 其他权重可以量化得更激进

实验发现，只需要保护大约 1% 的权重，就能显著降低量化带来的精度损失。

---

## 9. FP8 量化

INT8/INT4 是整数量化，而 FP8 是 8 位浮点数量化。

FP8 有两种主要格式：

```
E4M3：4 位指数，3 位尾数，适合权重
E5M2：5 位指数，2 位尾数，适合梯度
```

FP8 的优势：
- 比 INT8 更容易训练（梯度也可以 FP8）
- NVIDIA Hopper 和 Ada 架构有原生 FP8 Tensor Core 支持
- 不需要复杂的 scale 搜索，直接按范围缩放即可

FP8 目前主要用于训练和高端推理卡（H100/H200）。RTX 4090 等 Ada 架构消费级卡也支持 FP8 Tensor Core，但显存和带宽仍是瓶颈，实际收益不如 Hopper 明显。

---

## 10. 量化对推理的实际影响

| 方面 | 影响 |
|------|------|
| **显存占用** | 显著降低。INT4 权重约为 FP16 的 1/4 |
| **推理速度** | 取决于硬件支持。有 INT4/INT8 专用单元时更快；纯 CPU 或没有专用单元时可能更慢 |
| **精度** | INT8 通常损失很小；INT4 损失更明显，但 GPTQ/AWQ 能控制在可接受范围 |
| **KV Cache** | 量化权重不直接减少 KV Cache。KV Cache 需要单独的 KV Cache 量化 |
| **下载体积** | 明显减少，方便本地部署 |

---

## 11. 下载模型时那些名字到底什么意思？

### GGUF 格式命名

llama.cpp 使用的 GGUF 格式常见命名：

```
Q4_0：INT4，每组 32 个权重一个 scale，最简单
Q4_K_M：INT4，K-quant 混合方案，M 表示 medium，平衡速度和精度
Q4_K_S：INT4，K-quant 小模型优化版
Q5_K_M：INT5，精度更高
Q8_0：INT8，精度最高但体积大
```

一般推荐：
- 追求最小体积：Q4_K_S
- 平衡选择：Q4_K_M
- 追求精度：Q5_K_M 或 Q8_0

### AWQ / GPTQ

```
model-AWQ：用 AWQ 算法量化为 INT4，通常需要 GPU 推理
model-GPTQ：用 GPTQ 算法量化为 INT4
model-int8：简单的 INT8 量化
```

AWQ 和 GPTQ 都是 INT4，但算法不同：
- AWQ 通常精度更好，但需要按激活值校准
- GPTQ 通用性更强，不需要激活值数据

### 实际选择建议

| 场景 | 推荐 |
|------|------|
| CPU / 笔记本本地跑 | GGUF Q4_K_M |
| 消费级显卡（12-24GB）| AWQ 或 GPTQ INT4 |
| 服务器显卡（A100/H100）| FP8 或 INT8 |
| 对精度要求极高 | Q8_0 或原始 FP16 |

---

## 12. 一句话总结

> 量化是用更低的数值精度表示模型权重，在几乎不损失效果的前提下，把 LLM 的体积和显存占用砍掉一半甚至四分之三。理解 `scale`、`分组`、`GPTQ`、`AWQ` 这几个关键点，就能看懂大多数量化模型的命名和选择了。
