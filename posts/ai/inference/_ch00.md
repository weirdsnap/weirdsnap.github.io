## 二、当前结构的优点

1. **顺序合理**  
   内存 → 推理流程 → 分布式 → 服务调度 → Attention 底层优化 → KV Cache 压缩 → 系统级技巧，符合"先懂问题，再看解法"的认知路径。

2. **软硬结合**  
   既有算法层面的 Flash Attention / GQA，也有系统层面的 PagedAttention / Continuous Batching，还有硬件通信层面的 RDMA。

3. **小步快跑**  
   每篇 4-10 分钟阅读，适合碎片化阅读。

## 七、写作风格约定

1. **不要写死相邻文章链接**  
   相邻导航由 `blog.js` 从 `index.json` 自动生成。正文里只引用**非相邻的特定文章**，用 `?post=...` 形式。

2. **代码块若含 `{{` 要处理**  
   GitHub Pages 已禁用 Jekyll（`.nojekyll`），但为防万一，含大量 `{{` 的代码块用 `{% raw %}{% endraw %}` 包裹。

3. **新增文章后必须重新生成索引**  
   ```bash
   python3 scripts/build_index.py
   ```

4. **内部规划文件命名**  
   以 `_` 开头（如此文件），不进入索引。
