# 边权赋值方案数（3558 / 3559）

## 3558：怎么想到答案的

这道题给一棵树，每条边只能赋 1 或 2，要求从根到每个叶子的路径上，所有前缀和互不相同。问有多少种方案。

我一开始想的是枚举，但边数一多显然不行。然后试着从小例子找规律。

### 从小例子开始

一条路径，只有 1 条边：
- 赋 1 或 2 都可以，前缀和就是 {1} 或 {2}，都唯一
- 方案数：2

2 条边：
- 第一条边赋 1，第二条可以 1 或 2 → 前缀和 {1, 2} 或 {1, 3}
- 第一条边赋 2，第二条可以 1 或 2 → 前缀和 {2, 3} 或 {2, 4}
- 全部合法，方案数：4 = 2²

3 条边：
- 枚举一下，发现也全部合法
- 方案数：8 = 2³

等等，是不是任意赋都合法？那答案就是 2^(边数)？

但题目说"根到每个叶子的路径上所有边权值和互不相同"，不是任意两条子路径和相等，而是**前缀和**互不相同。前缀和天然递增（边权都是正的），所以不可能重复。

那岂不是答案就是 2^(n-1)？

不对，重新读题。题意是"根到每个叶子的路径上，所有边的权值和互不相同"——不是前缀和，是**任意两条子路径的和**都不能相等。

这就复杂了。继续试：

2 条边，路径和：
- 边权 (1,1)：子路径和 = {1, 1, 2}，两个 1 重复，不合法
- 边权 (1,2)：子路径和 = {1, 2, 3}，全部不同，合法
- 边权 (2,1)：子路径和 = {2, 1, 3}，全部不同，合法
- 边权 (2,2)：子路径和 = {2, 2, 4}，两个 2 重复，不合法

所以 2 条边时，方案数是 2，不是 4。

3 条边，枚举所有 8 种：
- (1,1,1)：子路径和 {1,1,1,2,2,3}，大量重复，不合法
- (1,1,2)：子路径和 {1,1,2,2,3,4}，1 和 2 都重复，不合法
- (1,2,1)：子路径和 {1,2,1,3,2,4}，1 和 2 重复，不合法
- (1,2,2)：子路径和 {1,2,2,3,4,5}，2 重复，不合法
- (2,1,1)：子路径和 {2,1,1,3,2,4}，1 和 2 重复，不合法
- (2,1,2)：子路径和 {2,1,2,3,3,5}，2 和 3 重复，不合法
- (2,2,1)：子路径和 {2,2,1,4,3,5}，2 重复，不合法
- (2,2,2)：子路径和 {2,2,2,4,4,6}，大量重复，不合法

等等，全部不合法？那 3 条边时方案数是 0？

但题目给的样例应该有答案。我重新理解题意——可能我理解错了"子路径"的范围。

再读题："从根节点 1 出发到每个叶子的路径上，所有边的权值和互不相同"。

哦，是**根到每个叶子的路径**上的所有边权值和互不相同。不是任意子路径，是**不同叶子之间的路径和**互不相同？

不对，再仔细看。如果是不同叶子之间的路径和互不相同，那和边权赋值有什么关系？

重新理解：一条从根到叶子的路径，上面有 k 条边。要求这条路径上的**所有前缀和**互不相同。因为边权都是正的，前缀和天然递增，所以自动满足。

那又回到最初的结论：任意赋都合法？

但 3558 的样例输出不是 2^(n-1)。说明我理解还是有问题。

### 重新理解题意

看了讨论区才明白：题目要求的是**从根到每个叶子的路径上，任意两条不相交的子路径的和不能相等**。

更准确地说，是路径上任意两个**连续子数组**的和不能相等。

对于一条 k 条边的路径，有 k 个单条边（和 = 边权），k-1 个两条边的连续子路径，...，1 个 k 条边的完整路径。总共 k(k+1)/2 个子路径和，要求全部不同。

这个限制就强多了。

### 继续找规律

1 条边：子路径和 = {1} 或 {2}，都唯一。方案数 2 = 2^1。

2 条边：
- (1,1)：子路径和 {1, 1, 2}，1 重复，不合法
- (1,2)：{1, 2, 3}，唯一，合法
- (2,1)：{2, 1, 3}，唯一，合法
- (2,2)：{2, 2, 4}，2 重复，不合法

方案数 2 = 2^1。

3 条边：
- (1,2,1)：单条边 {1,2,1}，1 重复，不合法
- (1,2,2)：单条边 {1,2,2}，2 重复，不合法
- (2,1,1)：单条边 {2,1,1}，1 重复，不合法
- (2,1,2)：单条边 {2,1,2}，2 重复，不合法
- (2,2,1)：单条边 {2,2,1}，2 重复，不合法
- (2,2,2)：单条边 {2,2,2}，2 重复，不合法
- (1,1,2)：单条边 {1,1,2}，1 重复，不合法
- (2,1,2)：已经看过了

等等，我漏了 (1,2,1) 之外还有 (2,1,2) 已经看了。那有没有单条边不重复的？

单条边不重复意味着三个边权互不相同，但边权只能是 1 或 2，三个数不可能互不相同。所以 3 条边时方案数确实是 0？

但这样和 3558 的样例对不上。再仔细看题——哦，题目说的是"从根到每个叶子的路径上，所有边的权值和互不相同"，可能是指**不同叶子之间的路径和**互不相同？

也就是说，树有多个叶子，每个叶子有一条从根到它的路径，要求这些路径的**总长度**（所有边权和）互不相同。

这就合理了。对于一棵树，根到每个叶子的路径总长度必须不同。边权只能是 1 或 2。

### 正确的理解

根到每个叶子的路径总长度必须互不相同。边权 1 或 2。

对于一条有 k 条边的路径，总长度在 [k, 2k] 范围内。不同叶子路径长度必须不同。

但题目问的是"有多少种赋值方案"，不是判断是否存在。那对于一棵树，方案数取决于什么？

关键观察：限制最紧的是**最长路径**。如果最长路径有 d 条边，它的总长度可以是 d 到 2d 之间的任意整数。其他较短路径的总长度范围更小，自动是 [d, 2d] 的子集。

但不同叶子路径长度必须不同，这意味着什么？

实际上，对于一条 k 条边的路径，总长度 = k + (赋 2 的边数)。所以总长度范围 [k, 2k]，共 k+1 种可能。

如果树有 m 个叶子，需要 m 个不同的总长度。如果 m > k+1，不可能。但题目保证有解。

方案数怎么算？对于最长路径（d 条边），总长度可以是 d 到 2d 中的任意值，对应赋 2 的边数从 0 到 d。方案数就是 C(d, 0) + C(d, 1) + ... + C(d, d) = 2^d？

不对，总长度确定后，哪些边赋 2 的方案数不是任意的，因为边权只有 1 和 2，总长度 = d + (赋 2 的边数)。所以总长度确定等价于赋 2 的边数确定，方案数是 C(d, 赋2边数)。

但题目要求的是所有叶子的路径长度都不同。最长路径确定了总长度，其他路径的总长度必须和它不同，且彼此不同。

这变得很复杂。我换个思路：直接看答案的规律。

看了样例和讨论，发现答案只和树的**最大深度**有关，和树的形状无关。最大深度为 d 时，答案 = 2^(d-1)。

为什么？

因为最长路径有 d 条边，总长度范围 [d, 2d]。其他路径更短，总长度范围更小，自动落在 [d, 2d] 内。只要最长路径的总长度确定了，其他路径只要总长度和它不同就行——但边权是 1 或 2，短路径的总长度天然比长路径小（如果边权分配"合理"）。

实际上，关键观察是：**对于一条 k 条边的路径，合法方案数恰好是 2^(k-1)**。这里的"合法"指的是路径上所有前缀和互不相同——等等，又回到前缀和了。

我搞混了。直接接受结论：对于一条 k 条边的路径，方案数是 2^(k-1)。整棵树的方案数取决于最大深度 d，答案是 2^(d-1)。

### 代码

```cpp
class Solution {
public:
    int assignEdgeWeights(vector<vector<int>>& edges) {
        constexpr int MOD = 1e9 + 7;
        const int n = edges.size() + 1;
        
        vector<vector<int>> adj(n + 1);
        for (const auto& e : edges) {
            adj[e[0]].push_back(e[1]);
            adj[e[1]].push_back(e[0]);
        }
        
        vector<bool> visited(n + 1, false);
        queue<int> q;
        q.push(1);
        visited[1] = true;
        int max_depth = -1;
        
        while (!q.empty()) {
            int size = q.size();
            max_depth++;
            while (size--) {
                int u = q.front(); q.pop();
                for (int v : adj[u]) {
                    if (!visited[v]) {
                        visited[v] = true;
                        q.push(v);
                    }
                }
            }
        }
        
        if (max_depth == 0) return 0;
        
        // 快速幂 2^(max_depth-1) % MOD
        long long ans = 1, base = 2, exp = max_depth - 1;
        while (exp > 0) {
            if (exp & 1) ans = (ans * base) % MOD;
            base = (base * base) % MOD;
            exp >>= 1;
        }
        return ans;
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | BFS 求最大深度 + 快速幂 |
| 空间 | O(n) | 邻接表和访问标记 |

## 3559：带查询的版本

3559 是 3558 的扩展，给定查询 `(u, v)`，问 u 到 v 路径上的方案数。

直接用 LCA 求路径长度，然后答案就是 2^(路径边数 - 1)。

```cpp
class Solution {
public:
    vector<int> assignEdgeWeights(vector<vector<int>>& edges, vector<vector<int>>& queries) {
        const int n = edges.size() + 1;
        constexpr const int LOG = 17; 
        constexpr const int MOD = 1e9 + 7;

        vector<vector<int>> tree(n + 1);
        for (const auto & e: edges) {
            tree[e[0]].push_back(e[1]);
            tree[e[1]].push_back(e[0]);
        }

        vector<int> depths(n + 1, -1);
        vector<vector<int>> parent(LOG, vector<int>(n + 1, -1));
        queue<int> q;
        q.push(1);
        depths[1] = 0;
        
        while (!q.empty()) {
            int cur = q.front();
            q.pop();
            for (const auto & node : tree[cur]) {
                if (depths[node] == -1) {
                    depths[node] = depths[cur] + 1;
                    parent[0][node] = cur;
                    q.push(node);
                }
            }
        }

        for (int j = 1; j < LOG; j++) {
            for (int k = 1; k <= n; k++) {
                if (parent[j-1][k] != -1) {
                    parent[j][k] = parent[j-1][parent[j-1][k]];
                }
            }
        }

        vector<long long> pow2(n);
        pow2[0] = 1;
        for (int i = 1; i < n; i++) {
            pow2[i] = (pow2[i-1] * 2) % MOD;
        }

        auto getLCA = [&](int u, int v) {
            if (depths[u] < depths[v]) swap(u, v);
            int diff = depths[u] - depths[v];
            for (int j = 0; j < LOG; j++) {
                if (diff & (1 << j)) {
                    u = parent[j][u];
                }
            }
            if (u == v) return u;
            for (int j = LOG - 1; j >= 0; j--) {
                if (parent[j][u] != parent[j][v]) {
                    u = parent[j][u];
                    v = parent[j][v];
                }
            }
            return parent[0][u];
        };

        vector<int> result;
        for (const auto& query : queries) {
            int u = query[0], v = query[1];
            if (u == v) {
                result.push_back(0);
                continue;
            }
            int w = getLCA(u, v);
            int path_edges = depths[u] + depths[v] - 2 * depths[w];
            result.push_back(pow2[path_edges - 1]);
        }
        
        return result;
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O((n + q) log n) | LCA 预处理 O(n log n)，每次查询 O(log n) |
| 空间 | O(n log n) | 倍增表 |

## 踩坑点

- ❌ 3558 中 `max_depth` 的初始值和边界处理容易出错。如果树只有一个节点（没有边），`max_depth` 为 0，应返回 0。
- ❌ 3559 中 `LOG` 取值要足够大。`n <= 10^4` 时 `LOG = 17` 足够，但通用写法应根据 `n` 动态计算。
- ❌ 3559 中 `u == v` 时路径长度为 0，没有边，方案数为 0（不是 `2^(-1)`）。
- ❌ 题意理解容易偏差，建议先看样例再推导。
