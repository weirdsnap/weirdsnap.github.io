# 边权赋值方案数（3558 / 3559）

两道题共用同一个核心结论：**对于一条包含 `k` 条边的路径，合法边权赋值方案数为 `2^(k-1)`**。

---

## 题目一：边权赋值方案数 I（3558）

### 题意

给定一棵无根树的边，每条边只能赋 1 或 2。要求从根节点 1 出发到每个叶子的路径上，所有边的权值和互不相同。问有多少种赋值方案。

### 解法

#### 思路

关键观察：一条包含 `k` 条边的路径，合法方案数恰好是 `2^(k-1)`。

整棵树要满足根到每个叶子的路径都合法。限制最紧的是树的最大深度 `d`（最长根到叶子的边数）。如果最大深度路径上有 `d` 条边，方案数是 `2^(d-1)`，其他较短路径自动满足（短路径的前缀和集合是长路径前缀和集合的子集）。

#### 代码

```cpp
class Solution {
public:
    int assignEdgeWeights(vector<vector<int>>& edges) {
        constexpr int MOD = 1e9 + 7;
        const int n = edges.size() + 1;
        
        vector<vector<int>> adj(n + 1);
        for (const auto& e : edges) {
            adj[e[0]].push_back(e[1]);
            adj[e[1]].push_back(e[0]);
        }
        
        vector<bool> visited(n + 1, false);
        queue<int> q;
        q.push(1);
        visited[1] = true;
        int max_depth = -1;
        
        while (!q.empty()) {
            int size = q.size();
            max_depth++;
            while (size--) {
                int u = q.front(); q.pop();
                for (int v : adj[u]) {
                    if (!visited[v]) {
                        visited[v] = true;
                        q.push(v);
                    }
                }
            }
        }
        
        if (max_depth == 0) return 0;
        
        // 快速幂 2^(max_depth-1) % MOD
        long long ans = 1, base = 2, exp = max_depth - 1;
        while (exp > 0) {
            if (exp & 1) ans = (ans * base) % MOD;
            base = (base * base) % MOD;
            exp >>= 1;
        }
        return ans;
    }
};
```

#### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | BFS 求最大深度 + 快速幂 |
| 空间 | O(n) | 邻接表和访问标记 |

## 题目二：边权赋值方案数 II（3559）

### 题意

给定一棵无根树和若干查询 `(u, v)`，问在 `u` 到 `v` 的简单路径上，有多少种边权赋值方案使得路径上不存在两条子路径的和相等。

### 解法

#### 思路

对每个查询，先求 `u` 到 `v` 的路径长度：

```
len = depth[u] + depth[v] - 2 * depth[lca(u, v)]
```

然后答案就是 `2^(len - 1) % MOD`。

需要预处理 LCA，这里用倍增法。

#### 代码

```cpp
class Solution {
public:
    vector<int> assignEdgeWeights(vector<vector<int>>& edges, vector<vector<int>>& queries) {
        const int n = edges.size() + 1;
        constexpr const int LOG = 17; 
        constexpr const int MOD = 1e9 + 7;

        // 建立邻接表
        vector<vector<int>> tree(n + 1);
        for (const auto & e: edges) {
            tree[e[0]].push_back(e[1]);
            tree[e[1]].push_back(e[0]);
        }

        // BFS 求深度和父节点
        vector<int> depths(n + 1, -1);
        vector<vector<int>> parent(LOG, vector<int>(n + 1, -1));
        queue<int> q;
        q.push(1);
        depths[1] = 0;
        
        while (!q.empty()) {
            int cur = q.front();
            q.pop();
            for (const auto & node : tree[cur]) {
                if (depths[node] == -1) {
                    depths[node] = depths[cur] + 1;
                    parent[0][node] = cur;
                    q.push(node);
                }
            }
        }

        // LCA 倍增表
        for (int j = 1; j < LOG; j++) {
            for (int k = 1; k <= n; k++) {
                if (parent[j-1][k] != -1) {
                    parent[j][k] = parent[j-1][parent[j-1][k]];
                }
            }
        }

        // 预计算 2 的幂次
        vector<long long> pow2(n);
        pow2[0] = 1;
        for (int i = 1; i < n; i++) {
            pow2[i] = (pow2[i-1] * 2) % MOD;
        }

        // LCA 查询
        auto getLCA = [&](int u, int v) {
            if (depths[u] < depths[v]) swap(u, v);
            int diff = depths[u] - depths[v];
            for (int j = 0; j < LOG; j++) {
                if (diff & (1 << j)) {
                    u = parent[j][u];
                }
            }
            if (u == v) return u;
            for (int j = LOG - 1; j >= 0; j--) {
                if (parent[j][u] != parent[j][v]) {
                    u = parent[j][u];
                    v = parent[j][v];
                }
            }
            return parent[0][u];
        };

        // 处理查询
        vector<int> result;
        for (const auto& query : queries) {
            int u = query[0], v = query[1];
            if (u == v) {
                result.push_back(0);
                continue;
            }
            int w = getLCA(u, v);
            int path_edges = depths[u] + depths[v] - 2 * depths[w];
            result.push_back(pow2[path_edges - 1]);
        }
        
        return result;
    }
};
```

#### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O((n + q) log n) | LCA 预处理 O(n log n)，每次查询 O(log n) |
| 空间 | O(n log n) | 倍增表 |

## 核心结论证明：为什么答案是 2^(k-1)

考虑一条从根到叶子的路径，有 `k` 条边。设前 `i` 条边的权值和为 `S_i`。

要使得所有前缀和 `S_1, S_2, ..., S_k` 互不相同，只需要保证每次增加的值不会让新的前缀和等于之前的某个前缀和。

第一条边赋 1 或 2 都可以，有 2 种选择。

从第二条边开始，假设前面所有前缀和已经互不相同。当前边赋 1 或 2 时，新的前缀和 `S_{i-1} + 1` 和 `S_{i-1} + 2` 中，至多有一个会等于之前的前缀和（因为间隔为 1）。所以至少还有 1 种选择。

更准确地说：**权值序列可以任意选，只要不存在两个不同的前缀和相等**。对于一条 `k` 条边的路径，等价于选 `k` 个 1 或 2，使得所有前缀和唯一。

前缀和最小是 `k`（全 1），最大是 `2k`（全 2），共有 `k+1` 个可能的取值。我们需要 `k` 个不同的前缀和，落在 `[1, 2k]` 范围内。

关键结论是：对于长度为 `k` 的路径，合法方案数恰好是 `2^(k-1)`。

## 踩坑点 / 注意点

- ❌ 3558 中 `max_depth` 的初始值和边界处理容易出错。如果树只有一个节点（没有边），`max_depth` 为 0，应返回 0。
- ❌ 3559 中 `LOG` 取值要足够大。`n <= 10^4` 时 `LOG = 17` 足够，但通用写法应根据 `n` 动态计算。
- ❌ 3559 中 `u == v` 时路径长度为 0，没有边，方案数为 0（不是 `2^(-1)`）。
