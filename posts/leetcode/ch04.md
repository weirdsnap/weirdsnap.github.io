# 分发糖果（135. Candy）

## 题意

`n` 个孩子站成一排，每个孩子有一个评分 `ratings[i]`。按照以下要求分发糖果：

- 每个孩子至少分到 1 个糖果
- 评分更高的孩子必须比相邻位置左右两个孩子获得更多的糖果

求最少需要准备多少糖果。

## 解法一：两次扫描

### 思路

贪心 + 两次线性扫描：

1. **从左到右**：如果 `ratings[i] > ratings[i-1]`，则 `i` 的糖果数要比左边多 1
2. **从右到左**：如果 `ratings[i] > ratings[i+1]`，则 `i` 的糖果数要同时满足比右边多 1，取两次扫描的最大值

这样每个位置同时满足左右两个方向的约束。

### 代码

```cpp
class Solution {
public:
    int candy(vector<int>& ratings) {
        const int N = ratings.size();

        vector<int> candy(N, 0);
        candy[0] = 0;
        for (int i = 1; i < N; i++) {
            if (ratings[i] > ratings[i - 1]) {
                candy[i] = candy[i - 1] + 1;
            }
        }

        for (int i = N - 2; i >= 0; i--) {
            if (ratings[i] > ratings[i + 1]) {
                candy[i] = max(candy[i + 1] + 1, candy[i]);
            }
        }

        return std::accumulate(candy.begin(), candy.end(), 0) + N;
    }
};
```

### 代码解读

- `candy[i]` 初始为 0，表示这个孩子比基准多拿几颗糖
- 基准是每个孩子都先拿 1 颗，所以最后结果要 `+ N`
- 例如 `candy = [0, 1, 0]` 表示实际糖果数为 `[1, 2, 1]`，总和为 `accumulate(candy) + N = 1 + 3 = 4`

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 两次线性扫描 + 一次累加 |
| 空间 | O(n) | 辅助数组 `candy` |

## 解法二：单次遍历 O(1) 空间

### 思路

把评分序列看成一座座"山峰"：
- 连续上升段（up）：每个孩子比前一个孩子多 1 颗糖
- 连续下降段（down）：下降段里的孩子要反向递增，如果下降段长度超过山顶高度，还要给山顶补 1 颗糖
- 平地（相等）：直接给 1 颗糖，重置所有计数

### 代码

```cpp
class Solution {
public:
    int candy(vector<int>& ratings) {
        const int N = ratings.size();
        if (N == 0) return 0;

        int candies = 1;   // 第一个孩子先给 1 颗
        int up = 0;        // 当前连续上升长度
        int down = 0;      // 当前连续下降长度
        int peak = 0;      // 上一个山顶的高度

        for (int i = 1; i < N; i++) {
            if (ratings[i] > ratings[i - 1]) {
                // 上升段
                up++;
                down = 0;
                peak = up;
                candies += 1 + up;  // 当前孩子比前一个孩子多 1
            } else if (ratings[i] < ratings[i - 1]) {
                // 下降段
                down++;
                up = 0;
                candies += down;    // 给下降段新增的这个孩子分配糖果
                if (down > peak) {
                    candies++;      // 下降段比山顶还长，山顶需要补 1 颗
                }
            } else {
                // 平地
                up = 0;
                down = 0;
                peak = 0;
                candies += 1;
            }
        }

        return candies;
    }
};
```

### 为什么下降段超过 peak 时要补发？

假设评分为 `[1, 3, 4, 5, 2, 1]`：
- 上升段 `1 < 3 < 4 < 5`：糖果分配为 `[1, 2, 3, 4]`
- 山顶是 5，peak = 3（从 1 到 5 上升了 3 步）
- 下降段 `5 > 2 > 1`：长度 down = 2，没有超过 peak，所以 5 保持 4 颗即可

但如果评分为 `[1, 5, 4, 3, 2, 1]`：
- 上升段只有 1 步（1 → 5），peak = 1
- 下降段有 4 步，down 会达到 4
- 如果山顶只给 2 颗糖，无法满足下降段 `[4,3,2,1]` 需要 `[4,3,2,1]` 的分配
- 所以山顶必须补到 5 颗，整体变成 `[1, 5, 4, 3, 2, 1]`

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 单次遍历 |
| 空间 | O(1) | 只使用几个变量 |

## 解法对比

| 解法 | 时间 | 空间 | 特点 |
|------|------|------|------|
| 两次扫描 | O(n) | O(n) | 思路直观，容易理解 |
| 单次遍历 | O(n) | O(1) | 空间更优，但需要处理山峰补发 |

## 踩坑点 / 注意点

- ❌ 只做一次从左到右的扫描 — 错误。必须同时满足左右两个方向的约束。
- ❌ 第二次扫描直接用 `candy[i] = candy[i+1] + 1` — 错误。可能覆盖掉第一次扫描更大的值，应取 `max`。
- ❌ 忘记最后 `+ N` — 错误。两次扫描解法里 `candy[i]` 存的是"额外糖果数"，基准 1 颗需要补上。
- ❌ 单次遍历中平地处理不正确 — 平地时应该重置 `up`、`down`、`peak`，并给当前孩子 1 颗糖。
