# 合并区间（56. Merge Intervals）

## 题意

给定一个区间数组 `intervals`，其中每个区间表示为 `[start, end]`。合并所有重叠的区间，返回不重叠的区间数组。

```
输入: intervals = [[1,3],[2,6],[8,10],[15,18]]
输出: [[1,6],[8,10],[15,18]]
解释: [1,3] 和 [2,6] 重叠，合并成 [1,6]
```

---

## 思路

### 关键观察

如果区间按**左端点排序**，那么重叠的区间一定是**相邻**的。排序后只需要线性扫描一遍，维护当前合并区间即可。

### 步骤

1. 按左端点升序排序
2. 遍历每个区间：
   - 如果和当前合并区间重叠（`cur[1] >= interval[0]`），扩展右端点
   - 如果不重叠，把当前区间加入结果，新区间成为新的合并起点

### 代码

```cpp
class Solution {
public:
    vector<vector<int>> merge(vector<vector<int>>& intervals) {
        // 按左端点升序排序，O(n log n)
        sort(intervals.begin(), intervals.end(), [](const vector<int>& a, const vector<int>& b){
            return a[0] < b[0];
        });
        vector<vector<int>> result;
        
        for (const auto& interval : intervals) {
            if (result.empty()) {
                result.push_back(interval);
            } else {
                auto& cur_interval = result.back();
                if (cur_interval[1] >= interval[0]) {
                    // 有重叠，扩展当前合并区间的右端点
                    cur_interval[1] = max(cur_interval[1], interval[1]);
                } else {
                    // 无重叠，当前区间成为新的合并起点
                    result.push_back(interval);
                }
            }
        }
        return result;
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N log N) | 排序 dominates |
| 空间 | O(N) | 结果数组（或 O(1) 如果原地修改） |

---

## 核心理解

排序是关键。如果不排序，重叠的区间可能分散在数组各处，需要 O(N²) 两两比较。排序后问题变成线性扫描，贪心即可。

`result.back()` 取最后一个合并区间，和当前区间比较。重叠就扩展，不重叠就新开一个。

---

## 踩坑点

- ❌ 忘记排序，直接线性扫描 — 会漏掉不相邻的重叠区间
- ❌ 比较时只判断 `cur[1] == interval[0]` — 应该用 `>=`，包含端点相接的情况
- ❌ 扩展右端点时用 `interval[1]` 而不是 `max(cur[1], interval[1])` — 当前区间可能完全包含在合并区间里
- ✅ 排序比较器用 `a[0] < b[0]`，不要用 `<=`（严格弱序要求）
