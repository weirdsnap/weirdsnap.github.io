# 网格安全路径（2812. Find the Safest Path in a Grid）

## 题意

给定一个 `N × M` 的 0-1 网格 `grid`，其中 `1` 表示格子中有**小偷**，`0` 是空位。定义每个格子的**安全因子**为它到最近小偷的曼哈顿距离。路径的安全因子是路径上所有格子的安全因子的**最小值**。求从左上角 `(0,0)` 到右下角 `(N-1, M-1)` 的最大安全因子路径。

如果起点或终点本身就有小偷，安全因子为 0。

## 解法：多源 BFS + 最大堆 Dijkstra

### 思路

核心分两步：

1. **预处理**：先用多源 BFS 计算每个格子到最近的 `1` 的距离（安全因子）
2. **找最优路径**：用最大堆从起点走到终点，每次选择安全因子最大的方向扩展，路径的安全因子 = 路径上各格子安全因子的最小值

把最大安全因子转化为**路径瓶颈最大值**问题——类似"在图中找一条路径，使路径上最小边权最大"。

### 代码

```cpp
class Solution {
public:
    int maximumSafenessFactor(vector<vector<int>>& grid) {
        // 准备常量和方向数组
        const int N = grid.size();
        const int M = grid[0].size();
        int dir[4][2] = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};

        // 如果起点或终点有贼，安全因子为 0
        if (grid[0][0] == 1 || grid[N - 1][M - 1] == 1) {
            return 0;
        }

        // 第一步：多源 BFS，计算每个格子到最近 1 的距离
        vector<vector<int>> dist(N, vector<int>(M, -1));
        queue<pair<int, int>> q;
        for (int i = 0; i < N; i++) {
            for (int j = 0; j < M; j++) {
                if (grid[i][j] == 1 && dist[i][j] == -1) {
                    q.push({i, j});
                    dist[i][j] = 0;
                }
            }
        }

        while (!q.empty()) {
            auto [x, y] = q.front(); q.pop();
            int cur = dist[x][y];
            for (int d = 0; d < 4; d++) {
                int nx = x + dir[d][0];
                int ny = y + dir[d][1];
                if (nx < 0 || nx >= N || ny < 0 || ny >= M ||
                    dist[nx][ny] != -1) {
                    continue;
                }
                dist[nx][ny] = cur + 1;
                q.push({nx, ny});
            }
        }

        // 第二步：最大堆找最优路径（Dijkstra 风格松弛）
        // best[x][y] 记录到达 (x,y) 时的最大安全因子
        priority_queue<tuple<int, int, int>> pq;
        vector<vector<int>> best(N, vector<int>(M, -1));
        best[0][0] = dist[0][0];
        pq.push({dist[0][0], 0, 0});

        while (!pq.empty()) {
            auto [safe, x, y] = pq.top(); pq.pop();

            // 如果堆顶安全因子比已记录的最优值还小，跳过
            if (safe < best[x][y]) continue;

            if (x == N - 1 && y == M - 1) {
                return safe;
            }

            for (int d = 0; d < 4; d++) {
                int nx = x + dir[d][0];
                int ny = y + dir[d][1];
                if (nx < 0 || nx >= N || ny < 0 || ny >= M) continue;

                int nSafe = min(safe, dist[nx][ny]);
                if (nSafe > best[nx][ny]) {
                    best[nx][ny] = nSafe;
                    pq.push({nSafe, nx, ny});
                }
            }
        }

        return 0;
    }
};
```

### 代码演化

**BFS 扩展：从手写四个 if 到方向数组**

最常见的 BFS 写法是手写四个方向的判断：

```cpp
if (x > 0 && dist[x - 1][y] == -1) {
    dist[x - 1][y] = cur + 1; q.push({x - 1, y});
}
if (x < N - 1 && dist[x + 1][y] == -1) {
    dist[x + 1][y] = cur + 1; q.push({x + 1, y});
}
if (y > 0 && dist[x][y - 1] == -1) {
    dist[x][y - 1] = cur + 1; q.push({x, y - 1});
}
if (y < M - 1 && dist[x][y + 1] == -1) {
    dist[x][y + 1] = cur + 1; q.push({x, y + 1});
}
```

方向数组版本用一个循环统一：

```cpp
int dir[4][2] = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};
for (int d = 0; d < 4; d++) {
    int nx = x + dir[d][0], ny = y + dir[d][1];
    if (nx < 0 || nx >= N || ny < 0 || ny >= M || dist[nx][ny] != -1) continue;
    dist[nx][ny] = cur + 1;
    q.push({nx, ny});
}
```

两种写法等价。方向数组的好处是**逻辑集中**——边界检查和 dist 检查写一次即可，不易遗漏。而且 BFS 和第二步堆搜索可以**复用**同一个 `dir` 数组。性能上没有区别（编译器会优化成一样的指令），属于代码风格的选择。

**第二步搜索：visited 布尔矩阵 vs best 松弛矩阵**

`visited` 方案的核心逻辑是"每个格子只入堆一次"：

```cpp
if (visited[x][y]) continue;
visited[x][y] = true;
for (int d = 0; d < 4; d++) {
    if (!visited[nx][ny]) {
        pq.push({min(safe, dist[nx][ny]), nx, ny});
    }
}
```

`best` 松弛方案的核心逻辑是"格子可以重复入堆，只保留更优解"：

```cpp
if (safe < best[x][y]) continue;       // 跳过过时的堆顶
// ...
if (nSafe > best[nx][ny]) {            // 只有更优才更新
    best[nx][ny] = nSafe;
    pq.push({nSafe, nx, ny});          // 可能覆盖之前的较差路径
}
```

两种方案的区别：

| | `visited` | `best` 松弛 |
|--|-----------|-------------|
| 每个格子的处理次数 | 最多一次 | 可能多次（每次找到更优解） |
| 堆中冗余元素 | 无 | 可能有（需要 `if (safe < best[x][y])` 跳过） |
| 内存开销 | `bool` 矩阵 | `int` 矩阵 |
| 正确性保证 | 第一次到达即最优（最大堆性质） | 松弛到无法再提升时停止 |

两种写法对于这个具体问题都是正确的。`visited` 方案更紧凑，`best` 松弛更符合 Dijkstra 泛型模板的思维习惯——你习惯用哪个都可以。

### 算法步骤

**Step 1：多源 BFS（方向数组简化）**
- 从所有 `1` 的位置同时出发，用方向数组 `dir[4][2]` 统一四个方向扩展
- 距离矩阵 `dist[i][j]` 就是该格子的安全因子（到最近小偷的曼哈顿距离）
- 时间复杂度 O(N×M)

**Step 2：最大堆松弛（Dijkstra 风格）**
- `best[x][y]` 记录当前从起点到达 `(x,y)` 的最大安全因子（初始化为 -1）
- 堆按安全因子降序排列，每次取出当前最优的路径尝试扩展
- `nSafe = min(safe, dist[nx][ny])` 是经过新格子后的路径安全因子
- 松弛条件：如果 `nSafe > best[nx][ny]`，说明找到了一条 **更安全** 的路径到达 `(nx, ny)`，更新并入堆
- 当堆顶正好是终点时，`safe` 就是全局最大安全因子

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N×M log(N×M)) | BFS O(NM) + 堆扩展 O(NM log NM) |
| 空间 | O(N×M) | dist 矩阵 + best 矩阵 + 堆 |

### 为什么是最大堆而不是最小堆？

这个问题的核心是**最大化路径上的最小值**。如果用 Dijkstra 的思路：

- 最小堆 Dijkstra → 求最短路径（路径总和最小），每次取最小值扩展
- **最大堆** → 求路径瓶颈最大值（路径上最小值最大），每次取最大值扩展

这里 `best[x][y]` 存的是"当前能到达该格子的最大安全因子"，堆按降序排列保证优先探索安全因子大的路径。当第一次从堆中弹出终点时，取到的就是**所有路径中最小安全因子的最大值**。

### 踩坑点 / 注意点

- ❌ 起点或终点有 1 要提前返回 0 — 否则 BFS 得到的 `dist[0][0] = 0` 也能走，但实际应该直接返回
- ❌ BFS 入队时同时检查 `dist[nx][ny] != -1` — 防止重复入队，保证每个格子只被最近的源更新
- ❌ 松弛条件用 `if (nSafe > best[nx][ny])` 而非 `if (nSafe != best[nx][ny])` — 只有更优的解才更新，避免无效入堆
- ❌ 方向数组 `dir` 统一扩展逻辑 — 比手写四个 `if` 更简洁，但注意边界检查的顺序（先判越界再判 dist）
- ❌ `priority_queue<tuple<int,int,int>>` 默认最大堆，tuple 按第一个元素（安全因子）降序排列，天然适合此题
