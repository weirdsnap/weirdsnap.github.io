# 买卖股票的最佳时机（121. Best Time to Buy and Sell Stock）

## 题意

给定一个数组 `prices`，其中 `prices[i]` 是某只股票第 `i` 天的价格。你只能选择**某一天买入**，然后在**未来的某一天卖出**，求能获得的最大利润。如果无法盈利，返回 `0`。

```
输入: prices = [7,1,5,3,6,4]
输出: 5
解释: 第 2 天买入（价格 1），第 5 天卖出（价格 6），利润 6 - 1 = 5
```

---

## 解法：一次遍历

### 思路

假设第 `i` 天卖出，那买入日一定是第 `0` 到 `i-1` 天中价格最低的那天。所以遍历过程中维护两个变量：

- `min_p`：到目前为止看到的最低价格
- `result`：到目前为止能获得的最大利润

每天做两件事：

1. 更新最低价格：`min_p = min(min_p, price)`
2. 更新最大利润：`result = max(result, price - min_p)`

### 代码

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int min_p = INT_MAX;
        int result = 0;
        for (const auto& price : prices) {
            min_p = min(min_p, price);
            result = max(result, price - min_p);
        }
        return result;
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N) | 只遍历一次数组 |
| 空间 | O(1) | 只用到两个变量 |

---

## 为什么这样是对的

关键观察：**卖出日确定后，最优买入日一定是它之前的最低价那天**。

所以不用枚举所有买卖组合（O(N²)），只需要在遍历到第 `i` 天时，知道第 `0` 到 `i-1` 天的最低价，就能算出第 `i` 天卖出的最大利润。顺便维护全局最大利润即可。

---

## 踩坑点

- ❌ 初始 `min_p` 不要设为 `0`，否则 `price - min_p` 会算错。用 `INT_MAX` 或者 `prices[0]`
- ❌ 不要返回负数。题目要求无法盈利时返回 `0`，`result` 初始化为 `0` 正好满足
- ❌ 不要枚举所有 `i < j` 的组合，会超时
- ✅ `INT_MAX` 需要 `#include <climits>`，或者改用 `min_p = prices[0]` 并从下标 `1` 开始遍历
