> **标签**：动态规划
# 最小路径和（64. Minimum Path Sum）

## 题意

给定一个 `M × N` 的非负整数网格 `grid`，找到一条从左上角 `(0,0)` 到右下角 `(M-1, N-1)` 的路径，使得路径上的数字之和最小。每次只能向右或向下移动一步。

```
输入：grid = [[1,3,1],
              [1,5,1],
              [4,2,1]]
输出：7
路径：1 → 3 → 1 → 1 → 1
```

---

## 解法：一维 DP（逐行递推）

### 思路

定义 `dp[j]` 为从起点到达当前行第 `j` 列的最小路径和。逐行更新。

**状态转移**：

```
dp[j] = min(左, 上) + grid[i][j]
```

- `dp[j-1]` 是当前行左列（已更新为当前行值）
- `dp[j]` 是上一行同列（尚未更新，仍为上一行的值）

**边界**：

- 第一行：`dp[j] = dp[j-1] + grid[0][j]`（只能从左向右走）
- 每行首列：`dp[0] = dp[0] + grid[i][0]`（只能从上向下走）

### 代码

```cpp
class Solution {
public:
    int minPathSum(vector<vector<int>>& grid) {
        const int M = grid.size();
        const int N = grid[0].size();

        // 第一行：只能从左向右累加
        vector<int> dp(N, 0);
        dp[0] = grid[0][0];
        for (int j = 1; j < N; j++) {
            dp[j] = dp[j - 1] + grid[0][j];
        }

        // 从第二行开始逐行递推
        for (int i = 1; i < M; i++) {
            dp[0] = dp[0] + grid[i][0];          // 首列：只能从上边来
            for (int j = 1; j < N; j++) {
                dp[j] = min(dp[j - 1], dp[j]) + grid[i][j];
            }
        }

        return dp[N - 1];
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(M×N) | 遍历整个网格一次 |
| 空间 | O(N) | 一维 DP 数组 |

## 另一种写法：原地修改

如果允许修改输入数组，可以直接在 `grid` 上做 DP，空间 O(1)：

```cpp
class Solution {
public:
    int minPathSum(vector<vector<int>>& grid) {
        const int M = grid.size();
        const int N = grid[0].size();

        // 第一行
        for (int j = 1; j < N; j++) {
            grid[0][j] += grid[0][j - 1];
        }
        // 第一列
        for (int i = 1; i < M; i++) {
            grid[i][0] += grid[i - 1][0];
        }
        // 中间
        for (int i = 1; i < M; i++) {
            for (int j = 1; j < N; j++) {
                grid[i][j] += min(grid[i - 1][j], grid[i][j - 1]);
            }
        }

        return grid[M - 1][N - 1];
    }
};
```

## 与 120. Triangle 的联系

两道题都是**网格/三角形最小路径和**，DP 思路一致：

| 题号 | 形状 | 移动方向 | DP 数组初始化 |
|------|------|---------|-------------|
| 64 | 矩形 M×N | 右、下 | 先初始化第一行 |
| 120 | 三角形 N×(N+1)/2 | 下、右下 | 逐行处理边界 |

核心思想相同：`dp[j] = min(左, 上) + 当前值`，一维滚动数组节省空间。

## 踩坑点 / 注意点

- ❌ 忘记初始化第一行或第一列 — `dp[j]` 在第一行需要从左向右累加，每行首列也要单独处理
- ❌ 原地修改版本中 `min(grid[i-1][j], grid[i][j-1])` 的索引不要写反
- ❌ `dp[0] = dp[0] + grid[i][0]` 必须在每行的 `j` 循环之前更新
- ❌ 网格中元素非负（>=0），所以路径和单调递增，不存在"绕路减少和"的可能
