# 三角形最小路径和（120. Triangle）

## 题意

给定一个三角形 `triangle`，求从顶部到底部的最小路径和。每一步可以移动到下一行的相邻元素上——即下标 `i` 处的元素可以移动到下一行的 `i` 或 `i + 1`。

```
   2
  3 4
 6 5 7
4 1 8 3
```

最小路径和为 `2 + 3 + 5 + 1 = 11`。

---

## 解法一：自上而下 DP

### 思路

定义 `dp[j]` 为从顶部到达当前行第 `j` 个元素的最小路径和。逐行递推：

```
dp[0] = triangle[0][0]

对于第 i 行：
  dp[i]   = dp[i-1] + triangle[i][i]         // 最右元素，只能从上方来
  dp[j]   = min(dp[j], dp[j-1]) + triangle[i][j]  // 中间元素，从上方或左上方来
  dp[0]   = dp[0] + triangle[i][0]            // 最左元素，只能从左上方的来
```

### 代码

```cpp
class Solution {
public:
    int minimumTotal(vector<vector<int>>& triangle) {
        const int N = triangle.size();
        vector<int> dp(N, 0);
        dp[0] = triangle[0][0];
        
        for (int i = 1; i < N; i++) {
            // 最右元素（j = i）：只能从上一行的 i-1 走下来
            dp[i] = dp[i - 1] + triangle[i][i];
            
            // 中间元素（j = 1..i-1）：min(右上, 正上) + 当前值
            // 从右向左遍历，确保 dp[j-1] 是上一行的值（未被更新覆盖）
            for (int j = i - 1; j > 0; j--) {
                dp[j] = min(dp[j], dp[j - 1]) + triangle[i][j];
            }
            
            // 最左元素（j = 0）：只能从上一行的 0 走下来
            dp[0] = dp[0] + triangle[i][0];
        }
        
        // 遍历最后一行，取最小值
        int result = INT_MAX;
        for (int x : dp) {
            result = min(result, x);
        }
        return result;
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N²) | 双层循环遍历三角形 |
| 空间 | O(N) | 一维 DP 数组 |

---

## 解法二：自底向上 DP（空间优化）

### 思路

自上而下需要处理每行的边界情况（最左和最右），最后还要遍历到取最小值。**自底向上**更优雅：

- 从倒数第二行开始往上递推
- `dp[j] = min(dp[j], dp[j+1]) + triangle[i][j]`
- 递推到顶部时，`dp[0]` 就是全局最小路径和

```
           dp[0]            ← 最终结果
         dp[0] dp[1]
       dp[0] dp[1] dp[2]
     dp[0] dp[1] dp[2] dp[3]   ← 初始化为最后一行
```

### 代码

```cpp
class Solution {
public:
    int minimumTotal(vector<vector<int>>& triangle) {
        const int N = triangle.size();
        // 用最后一行初始化 dp 数组
        vector<int> dp = triangle[N - 1];
        
        // 从倒数第二行开始，向上递推
        for (int i = N - 2; i >= 0; i--) {
            for (int j = 0; j <= i; j++) {
                dp[j] = min(dp[j], dp[j + 1]) + triangle[i][j];
            }
        }
        
        // dp[0] 即为全局最小路径和
        return dp[0];
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N²) | 双层循环 |
| 空间 | O(N) | 一维 DP 数组，无需处理边界特判 |

### 为什么自底向上更简洁？

自上而下需要考虑**三个边界情况**：最左（只能从正上方来）、最右（只能从左上角来）、中间（取 min 两种走法）。自底向上则**统一为一种转移**——每个位置都取正下方和右下方的较小值——边界情况自然消失，且最终结果直接聚到 `dp[0]`，无需遍历取最小值。

## 解法对比

| 解法 | 思路 | 边界处理 | 返回 |
|------|------|---------|------|
| 自上而下 | 从顶向下递推 | 需单独处理最左和最右 | 遍历最后一行取 min |
| 自底向上 | 从底向上递推 | 无需 | 直接 `dp[0]` |

## 踩坑点 / 注意点

- ❌ 自上而下遍历时，内层循环**必须从右向左**遍历 `j`，否则 `dp[j-1]` 会被当前行的新值覆盖，导致计算错误
- ❌ 自上而下最后需要遍历 DP 数组取最小值，容易忘记
- ❌ `INT_MAX` 初始化 `result` 需要 `#include <climits>`，可改用 `result = dp[0]` 跳过该头文件
- ❌ 空三角形（`triangle` 为空）需要单独处理，但题目保证至少有一行
