# 买卖股票的最佳时机 V（3573. Best Time to Buy and Sell Stock V）

## 题意

给定一个数组 `prices`，`prices[i]` 是第 `i` 天的股票价格。你可以进行最多 `k` 笔交易，每笔交易可以是：

- **做多**：先买入，后卖出（正常交易）
- **做空**：先卖出，后买入（反向交易）

每笔交易完成后，**必须隔至少一天**才能进行下一笔交易。求能获得的最大利润。

---

## 思路

对于买卖股票 III 和 IV，我们已经写了 2 次和 k 次的状态 DP 解法。

定义 `dp[i][j][0]` 为第 i 天完成了 j 次交易、当前状态 0（卖出状态）的最大收益；`dp[i][j][1]` 为状态 1（买入状态）的最大收益。

状态转移：

```
dp[i][j][0] = max(dp[i-1][j][0], dp[i][j-1][1] + prices[i])
```

今天可以不操作，取昨天的 j 次交易后的卖出状态；也可以卖出一次，取交易完成次数少一次的持有状态 + 今天的价格。两者取大。

买入状态类似：

```
dp[i][j][1] = max(dp[i-1][j][1], dp[i][j][0] - prices[i])
```

这里用 j 次交易减去今天买入花掉的钱，只买入不算完成交易，所以还是用 j。

那这道题有什么变化？

**两个点**：

1. 除了低买高卖，还允许做空操作，也就是先高价卖出，再低价买入。
2. 一天之内不许交易两次，也就是只能买或者卖。

---

### 做空怎么做

额外加一个持有状态 `dp[i][j][2]` 来表示做空。再"买入"（高价卖出）的时候先 `+prices[i]`，等到"卖出"（低价买入）的时候再 `-prices[i]`。这样就把做空也统一成买入和卖出的逻辑。

### 一天只能一次操作

`dp[i]` 的状态转移只能从 `dp[i-1]` 来计算。之前我们是同时用了 `dp[i-1]` 和 `dp[i]` 来更新的，因为可以今天卖出之后又买入持有。现在不行了，今天卖了明天才能买。

---

## 解题过程

先完整地按照 `dp[i][j][0/1/2]` 去实现，但很快观察到其实 dp 只跟 `dp[i-1]` 有关系，没必要完整存储 i 这个维度的所有数据，只需要前一天的就好了。

这样压缩到 `pre` 和 `cur` 两天轮换往后走。

但再仔细观察，发现其实只要限制今天卖了之后不许买入就可以了，今天买入之后又卖出其实没有意义。那怎么做到？**先更新买入的状态，再更新卖出的状态**。这样就可以在一个数组上操作了。

定义三个数组，长度为 `k+1`（代表 0 次到 k 次交易完成）：

- `sell[j]`：完成 j 笔交易、空仓
- `buy1[j]`：做多持有（先买后卖）
- `buy2[j]`：做空持有（先卖后买）

转移：

```cpp
buy1[j] = max(buy1[j], sell[j] - pi);   // 用昨天空仓的钱买入
buy2[j] = max(buy2[j], sell[j] + pi);   // 做空：先卖出，承诺未来买入
sell[j] = max(sell[j], max(buy1[j-1] + pi, buy2[j-1] - pi));  // 今天卖出
```

**注意更新顺序**：buy 必须先更新，以确保 sell 是昨天的值，这样才能算隔一天买入。sell 可以后更新，因为当天买入卖出本来就没意义。

## 代码

```cpp
class Solution {
public:
    long long maximumProfit(vector<int>& prices, int k) {
        vector<long long> sell(k + 1, 0);
        vector<long long> buy1(k + 1);  // 普通
        vector<long long> buy2(k + 1);  // 做空

        // init
        for (int i = 0; i <= k; i++) {
            buy1[i] = -prices[0];
            buy2[i] = prices[0];
        }

        // dp
        for (int i = 1; i < prices.size(); i++) {
            long long pi = prices[i];
            buy1[0] = max(buy1[0], -pi);
            buy2[0] = max(buy2[0], pi);
            for (int j = 1; j <= k; j++) {
                // 注意更新顺序，buy 必须先更新以确保 sell 是昨天的，这样才能算是隔一天买入
                buy1[j] = max(buy1[j], sell[j] - pi);
                buy2[j] = max(buy2[j], sell[j] + pi);
                // sell 可以后更新，因为当天买入卖出本来就没有意义
                sell[j] = max(
                    sell[j], 
                    max(
                        buy1[j - 1] + pi,
                        buy2[j - 1] - pi
                    )
                );
            }
        }

        return sell[k];
    }
};
```

## 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N × K) | 每天遍历 k 笔交易 |
| 空间 | O(K) | 三个长度为 k+1 的数组 |

---

## 退化优化：k 很大时

和 188 类似，当 `k` 很大时限制形同虚设。但 3573 有"隔一天"限制，做空 + 间隔让贪心条件复杂，写贪心不比 DP 简单。

所以 3573 的 `k` 退化优化，**实际意义不大**，不像 188 那样 `k >= n // 2` 时直接变成一行贪心。面试中直接写 O(N×K) 的 DP 即可。

---

## 核心对比：188 vs 3573

| | 188. 买卖股票 IV | 3573. 买卖股票的最佳时机 V |
|---|---|---|
| 交易方向 | 只能做多 | 做多 + 做空 |
| 交易间隔 | 无限制 | 必须隔一天 |
| 状态数 | `buy[k]`, `sell[k]` | `buy1[k]`, `buy2[k]`, `sell[k]` |
| 更新顺序 | 任意（同一天买卖无意义） | **必须先 buy 后 sell** |
| 退化条件 | `k >= n // 2` 退化成贪心 | 无简单退化 |

---

## 踩坑点

- ❌ **更新顺序搞反**：如果先 `sell` 后 `buy`，`buy` 会用到今天的 `sell`，违反隔一天限制
- ❌ `buy2` 初始化为 `prices[0]` 而不是 `-prices[0]` — 做空是先卖出，第一天做空相当于"借一股卖出"，收益是 `prices[0]`
- ❌ 用 `int` 存价格 — `prices` 和利润可能很大，用 `long long`
- ✅ `buy1[0]` 和 `buy2[0]` 单独处理 — 第 0 笔交易没有前置 `sell` 状态
- ✅ `sell[j]` 取 `max(buy1[j-1] + pi, buy2[j-1] - pi)` — 上一笔交易可以是做多或做空
