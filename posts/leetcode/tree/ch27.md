# 二叉树的中序遍历（94. Binary Tree Inorder Traversal）

## 题意

给定一棵二叉树的根节点 `root`，返回它的**中序遍历**结果（左-根-右）。

```
输入: [1,null,2,3]
输出: [1,3,2]

  1
   \
    2
   /
  3
```

---

## 思路

中序遍历顺序：**左子树 → 根节点 → 右子树**。

### 解法一：递归（最直观）

直接按照定义递归：先遍历左子树，再访问根，再遍历右子树。

### 解法二：迭代（用栈模拟递归）

递归的本质是系统栈。用显式栈模拟：

1. **一路向左**，把沿途节点全部压栈
2. **左子树到底**，弹出栈顶访问（此时该节点的左子树已处理完）
3. **转向右子树**，重复上述过程

核心：栈里保存的是"左子树还没处理完"的节点，弹出时意味着左子树已处理完，可以访问根了。

---

## 代码

### 递归

```cpp
class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> result;
        inorderHelper(root, result);
        return result;
    }
    
private:
    void inorderHelper(TreeNode* node, vector<int>& result) {
        if (node == nullptr) {
            return;
        }
        inorderHelper(node->left, result);   // 左
        result.push_back(node->val);          // 根
        inorderHelper(node->right, result);  // 右
    }
};
```

### 迭代

```cpp
class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> result;
        stack<TreeNode*> st;
        TreeNode* cur = root;
        
        while (cur || !st.empty()) {
            // 一路向左，全部入栈
            while (cur) {
                st.push(cur);
                cur = cur->left;
            }
            // 左子树到底，弹出访问，再转向右子树
            cur = st.top(); st.pop();
            result.push_back(cur->val);
            cur = cur->right;
        }
        return result;
    }
};
```

---

## 关键细节

### 迭代：为什么先一路向左入栈？

中序遍历要先访问左子树。把沿途节点压栈，是为了**记住回去的路**。当左子树遍历完，弹出栈顶就是当前子树的根，访问它，然后处理右子树。

```
    1
   / \
  2   3
 /
4

栈的变化：
1. push 1 → push 2 → push 4 → cur = null
2. pop 4, visit, cur = null（4 无右子树）
3. pop 2, visit, cur = 2->right = null
4. pop 1, visit, cur = 1->right = 3
5. push 3 → cur = null
6. pop 3, visit, cur = null

结果：[4, 2, 1, 3]
```

### 循环条件 `cur || !st.empty()`

- `cur != nullptr`：还有未处理的节点（正在向左走）
- `!st.empty()`：栈里还有节点待回溯

两者都为假时才结束。

### 递归 vs 迭代

| | 递归 | 迭代 |
|--|------|------|
| 代码量 | 少 | 稍多 |
| 空间 | 系统栈 O(h) | 显式栈 O(h) |
| 风险 | 树太深可能栈溢出 | 可控 |
| 面试 | 写递归即可 | 常考迭代 |

### 解法三：Morris 遍历（O(1) 空间）

利用**叶子节点的空右指针**作为临时线索，指向中序后继节点，遍历完再恢复。不需要栈。

**核心思想**：
- 对于节点 `cur`，找其左子树的最右节点 `pre`（即 `cur` 的中序前驱）
- 如果 `pre->right == nullptr`，说明第一次访问 `cur`，架桥 `pre->right = cur`，然后向左走
- 如果 `pre->right == cur`，说明左子树已遍历完，拆桥恢复 `pre->right = nullptr`，访问 `cur`，然后向右走

```cpp
class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> result;
        TreeNode* cur = root;
        
        while (cur != nullptr) {
            // 情况1：没有左子树，直接访问，向右走
            if (cur->left == nullptr) {
                result.push_back(cur->val);
                cur = cur->right;
            }
            // 情况2：有左子树，找中序前驱
            else {
                TreeNode* pre = cur->left;
                while (pre->right != nullptr && pre->right != cur) {
                    pre = pre->right;
                }
                
                if (pre->right == nullptr) {
                    // 第一次访问 cur，架桥，向左走
                    pre->right = cur;
                    cur = cur->left;
                } else {
                    // 第二次访问 cur（从左子树回来），拆桥，访问 cur，向右走
                    pre->right = nullptr;
                    result.push_back(cur->val);
                    cur = cur->right;
                }
            }
        }
        
        return result;
    }
};
```

### 复杂度对比

| | 递归 | 迭代（栈） | Morris |
|--|------|-----------|--------|
| 时间 | O(n) | O(n) | O(n) |
| 空间 | O(h) | O(h) | **O(1)** |
| 修改树结构 | 否 | 否 | **是（临时，会恢复）** |

## 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个节点访问一次 |
| 空间 | O(h) | 递归/栈空间；Morris O(1) |

---

## 踩坑点 / 注意点

- ❌ **迭代时 `cur = cur->right` 后忘记继续循环**。`cur` 指向右子树后，外层循环会再次进入"一路向左"阶段。
- ❌ **栈空时访问 `st.top()`**。确保只在 `!st.empty()` 时弹出。
- ❌ **只判断 `while (!st.empty())`**。如果 `cur` 还有节点但栈已空，会漏掉右子树的处理。
- ❌ **递归时返回局部 `vector`**。用引用参数 `result` 避免多次拷贝。
- ❌ **Morris 遍历忘记拆桥**。`pre->right = nullptr` 必须恢复，否则树结构被永久破坏。
- ❌ **Morris 找前驱时 `pre->right != cur` 条件漏写**。没有这个条件会死循环。
