# 二叉树的右视图（199. Binary Tree Right Side View）

## 题意

给定一棵二叉树的根节点 `root`，想象自己站在树的右侧，按照从顶部到底部的顺序，返回你能看到的节点值。

```
输入: [1,2,3,null,5,null,4]
输出: [1,3,4]

     1            <--- 看到 1
   /   \
  2     3         <--- 看到 3
   \     \
    5     4       <--- 看到 4
```

---

## 思路

### 关键观察

从右侧能看到的节点，恰好是**每一层最右边的节点**。问题转化为：层序遍历，每层取最后一个。

### BFS 层序遍历

用队列实现 BFS，核心技巧是**按层处理**：

1. 遍历当前层之前，先记录队列大小 `size`
2. 只处理这 `size` 个节点（就是当前层的全部节点）
3. 处理过程中把下一层节点入队
4. 每层最后一个节点的值加入结果

这样不需要额外的 `temp` 数组，空间更优。

---

## 代码

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    vector<int> rightSideView(TreeNode* root) {
        vector<int> result;
        queue<TreeNode*> q;
        if (root == nullptr) {
            return result;
        }
        q.push(root);
        
        while (!q.empty()) {
            // 当前层最右边的节点，就是队列尾部
            result.push_back(q.back()->val);
            
            // 按层处理：只处理当前层的 size 个节点
            int size = q.size();
            for (int i = 0; i < size; i++) {
                auto c = q.front();
                if (c->left != nullptr)
                    q.push(c->left);
                if (c->right != nullptr)
                    q.push(c->right);
                q.pop();
            }
        }
        return result;
    }
};
```

---

## 关键细节

### 1. 取队尾 vs 取最后一个出队

两种写法都可以：

- **取队尾**（本代码）：`result.push_back(q.back()->val)`，在进入内层循环前取，此时队列里全是当前层节点，队尾就是最右。
- **取最后一个出队**：在内层循环中判断 `i == size - 1`，最后一个出队的节点就是最右。

取队尾更简洁，少一个条件判断。

### 2. 为什么先取队尾再处理当前层？

如果先处理当前层（`pop` + `push` 子节点），队列里会混入下一层节点，队尾就不再是当前层最右了。所以必须在**任何 `pop` 之前**取 `q.back()`。

### 3. `size` 必须在循环前记录

```cpp
int size = q.size();  // 当前层节点数
for (int i = 0; i < size; i++)  // 只处理这么多
```

如果在 `for` 条件里写 `i < q.size()`，`q.size()` 会随着 `push` 子节点而变大，导致循环处理到下一层节点。

## 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个节点访问一次 |
| 空间 | O(w) | 队列空间，w 为树的最大宽度 |

---

## 踩坑点 / 注意点

- ❌ **空树处理**。`root == nullptr` 时直接返回空数组，不要入队。
- ❌ **`q.back()` 的时机**。必须在 `for` 循环之前取，否则队列里会有下一层节点。
- ❌ **`q.size()` 在循环条件里动态变化**。必须先记录到局部变量 `size`。
- ❌ **子节点为空也要处理**。只把非空子节点入队，避免队列里堆积 `nullptr`。
