# 最长递增子序列（300. Longest Increasing Subsequence）

## 题意

给定一个整数数组 `nums`，找到其中**最长严格递增子序列**的长度。

```
输入: nums = [10,9,2,5,3,7,101,18]
输出: 4
解释: 最长递增子序列是 [2,3,7,101]，长度 4
```

注意：子序列不要求连续，只要相对顺序不变。

---

## 思路

### 和 53 的对比

53 最大子数组和要求**连续**，所以 `dp[i]` 必须以 `i` 结尾，只能从 `i-1` 转移。

300 最长递增子序列不要求连续，`dp[i]` 表示**以 i 结尾**的最长递增子序列长度，但可以从任意 `j < i` 转移：

```
if nums[i] > nums[j]:
    dp[i] = max(dp[i], dp[j] + 1)
```

### 代码

```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        vector<int> dp(nums.size(), 1);

        for (int i = 1; i < nums.size(); i++) {
            // 遍历所有 j < i，找能接在 nums[j] 后面的最长子序列
            for (int j = 0; j < i; j++) {
                if (nums[i] > nums[j]) {
                    dp[i] = max(dp[i], dp[j] + 1);
                }
            }
        }

        int result = 0;
        for (const auto& d : dp) {
            result = max(result, d);
        }
        return result;
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N²) | 双重循环 |
| 空间 | O(N) | dp 数组 |

---

## 核心理解

`dp[i]` 的定义和 53 类似，都是"以 i 结尾"。但转移范围不同：

| | 53 最大子数组和 | 300 最长递增子序列 |
|---|---|---|
| 连续性 | **连续** | **不连续** |
| 转移来源 | 只能 `i-1` | 任意 `j < i` |
| 转移条件 | 无（必须连续） | `nums[i] > nums[j]` |
| 时间复杂度 | O(N) | O(N²) |

53 的 `dp[i]` 必须包含 `nums[i]`，且必须连续，所以只能从 `i-1` 转移。300 的 `dp[i]` 也必须包含 `nums[i]`，但前面可以从任意位置跳过来，只要值更小。

---

## 优化：二分查找 O(N log N)

维护一个数组 `tails`，`tails[k]` 表示长度为 `k+1` 的递增子序列的最小末尾元素。

遍历 `nums` 时，用二分查找找到 `tails` 中第一个大于等于 `nums[i]` 的位置，替换它。如果 `nums[i]` 比所有元素都大，就追加到末尾。

### 用 `lower_bound`

```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        vector<int> tails;
        
        for (int num : nums) {
            auto it = lower_bound(tails.begin(), tails.end(), num);
            if (it == tails.end()) {
                tails.push_back(num);
            } else {
                *it = num;
            }
        }
        
        return tails.size();
    }
};
```

### 手写二分

```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        vector<int> tail;
        tail.push_back(nums[0]);
        
        for (int i = 1; i < nums.size(); i++) {
            if (nums[i] > tail.back()) {
                tail.push_back(nums[i]);
                continue;
            }
            
            // 手写二分：找第一个 >= nums[i] 的位置
            int left = 0;
            int right = tail.size();
            while (left < right) {
                int mid = (left + right) / 2;
                if (tail[mid] < nums[i]) {
                    left = mid + 1;
                } else {
                    right = mid;
                }
            }
            tail[left] = nums[i];
        }
        
        return tail.size();
    }
};
```

### 为什么正确

`tails` 维护的是"长度为 k 的子序列的最小末尾"。末尾越小，后面越容易接更大的数。二分替换保证了 `tails` 始终有序，且长度就是最长递增子序列的长度。

注意手写二分的边界：`left < right`，`right = mid`，找的是第一个 `>= nums[i]` 的位置。这和 `lower_bound` 的行为一致。

---

## 踩坑点

- ❌ 和 53 搞混，只比较相邻元素 `nums[i] > nums[i-1]`，漏掉非连续的情况
- ❌ `dp` 初始化为 0 而不是 1，每个元素自身就是一个长度为 1 的子序列
- ❌ 二分优化时把 `lower_bound` 写成 `upper_bound`，导致严格递增变非严格
- ✅ 子序列不要求连续，所以必须从所有 `j < i` 中找转移
