# 最大利润（3573. Maximum Profit）

## 题意

给定一个数组 `prices`，`prices[i]` 是第 `i` 天的股票价格。你可以进行最多 `k` 笔交易，每笔交易可以是：

- **做多**：先买入，后卖出（正常交易）
- **做空**：先卖出，后买入（反向交易）

每笔交易完成后，**必须隔至少一天**才能进行下一笔交易。求能获得的最大利润。

---

## 思路：极限压缩的状态机

### 退化优化：k 很大时

和 188 类似，当 `k` 很大时限制形同虚设。但 3573 有"隔一天"限制，每笔交易至少占用 3 天（买、卖、间隔），所以最多能做 `n // 3` 笔完整交易。

当 `k >= n // 3` 时，可以直接贪心：遍历所有可能的做多/做空机会，只要有利润就做。但做空和间隔让贪心条件比 188 复杂，面试中可以先写 O(N×K) 的 DP，再提到这个退化。

### 状态定义

这道题是 188 的升级版，多了两个限制：

1. **可以做空** — 每笔交易可以是做多或做空
2. **必须隔一天** — 完成一笔交易后，不能立即开始下一笔

"隔一天"这个限制意味着：如果今天卖出，明天才能买入。所以今天的 `buy` 状态，必须基于**昨天**的 `sell` 状态。

三个状态数组：

| 状态 | 含义 |
|------|------|
| `buy1[j]` | 第 `j` 笔做多买入后的最大收益（持有一股） |
| `buy2[j]` | 第 `j` 笔做空买入后的最大收益（持有负一股，即先卖后买） |
| `sell[j]` | 完成 `j` 笔交易后的最大收益（空仓） |

### 关键观察：更新顺序

```cpp
buy1[j] = max(buy1[j], sell[j] - pi);      // 用昨天的 sell 买入
buy2[j] = max(buy2[j], sell[j] + pi);      // 用昨天的 sell 做空
sell[j] = max(sell[j], max(buy1[j-1] + pi, buy2[j-1] - pi));  // 今天卖出
```

**必须先更新 buy，再更新 sell**。

原因：`buy1[j]` 和 `buy2[j]` 用的是 `sell[j]`，这个 `sell[j]` 必须是**昨天**的值（因为隔一天才能买入）。如果先更新 `sell[j]`，那 `buy1[j]` 就会用到今天的 `sell[j]`，相当于当天卖出当天买入，违反了"隔一天"的限制。

而 `sell[j]` 后更新没问题，因为 `sell[j]` 依赖的是 `buy1[j-1]` 和 `buy2[j-1]`，这两个是**上一笔交易**的状态，和当前笔不冲突。而且当天买入当天卖出本来就没意义（利润为 0，还浪费一次交易机会）。

### 代码

```cpp
class Solution {
public:
    long long maximumProfit(vector<int>& prices, int k) {
        vector<long long> sell(k + 1, 0);
        vector<long long> buy1(k + 1);  // 做多
        vector<long long> buy2(k + 1);  // 做空

        // 初始化：第一天买入/做空后的状态
        for (int i = 0; i <= k; i++) {
            buy1[i] = -prices[0];
            buy2[i] = prices[0];
        }

        for (int i = 1; i < prices.size(); i++) {
            long long pi = prices[i];
            
            // 第 0 笔交易：直接买入/做空
            buy1[0] = max(buy1[0], -pi);
            buy2[0] = max(buy2[0], pi);
            
            for (int j = 1; j <= k; j++) {
                // 必须先更新 buy，确保 sell 是昨天的值
                buy1[j] = max(buy1[j], sell[j] - pi);   // 做多：用昨天空仓的钱买入
                buy2[j] = max(buy2[j], sell[j] + pi);   // 做空：先卖出，承诺未来买入
                
                // sell 后更新，依赖的是 buy1[j-1]/buy2[j-1]（上一笔交易）
                sell[j] = max(
                    sell[j],
                    max(
                        buy1[j - 1] + pi,   // 做多卖出
                        buy2[j - 1] - pi    // 做空买入（平仓）
                    )
                );
            }
        }

        return sell[k];
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N × K) | 每天遍历 k 笔交易 |
| 空间 | O(K) | 三个长度为 k+1 的数组 |

## 核心对比：188 vs 3573

| | 188. 买卖股票 IV | 3573. 最大利润 |
|---|---|---|
| 交易方向 | 只能做多 | 做多 + 做空 |
| 交易间隔 | 无限制 | 必须隔一天 |
| 状态数 | `buy[k]`, `sell[k]` | `buy1[k]`, `buy2[k]`, `sell[k]` |
| 更新顺序 | 任意（同一天买卖无意义） | **必须先 buy 后 sell** |
| 退化条件 | `k >= n // 2` 退化成贪心 | 无简单退化 |

---

## 踩坑点

- ❌ **更新顺序搞反**：如果先 `sell` 后 `buy`，`buy` 会用到今天的 `sell`，违反隔一天限制
- ❌ `buy2` 初始化为 `prices[0]` 而不是 `-prices[0]` — 做空是先卖出，第一天做空相当于"借一股卖出"，收益是 `prices[0]`
- ❌ 用 `int` 存价格 — `prices` 和利润可能很大，用 `long long`
- ✅ `buy1[0]` 和 `buy2[0]` 单独处理 — 第 0 笔交易没有前置 `sell` 状态
- ✅ `sell[j]` 取 `max(buy1[j-1] + pi, buy2[j-1] - pi)` — 上一笔交易可以是做多或做空
