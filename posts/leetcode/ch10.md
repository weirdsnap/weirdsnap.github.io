# 买卖股票的最佳时机 IV（188. Best Time to Buy and Sell Stock IV）

## 题意

给定一个数组 `prices` 和一个整数 `k`，其中第 `i` 个元素是某只股票在第 `i` 天的价格。设计一个算法来计算你所能获取的最大利润。你**最多可以完成 k 笔交易**（一次买入 + 一次卖出 = 一笔交易）。

注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

---

## 思路：从 2 次交易推广到 k 次

在 [ch09 买卖股票的最佳时机 III](./blog.html?post=leetcode/ch09.md) 中，我们用一个有限状态机处理了最多两次交易的情况——四个状态：`buy1/sell1/buy2/sell2`。

对于 k 次交易，不可能写 k 对 buy/sell 变量。需要用一个通用的 DP 框架：把"第几天"、"交易次数"、"当前是否持有"三个维度组织起来。

---

## DP 定义

```
dp[i][j][0] = 第 i 天结束时，已经完成了 j 笔交易，当前不持有股票，的最大利润
dp[i][j][1] = 第 i 天结束时，已经完成了 j 笔交易，当前持有股票，的最大利润
```

其中交易次数的定义：**每完成一次买入+卖出算一笔交易**。`dp[i][j][1]` 的 j 表示"已经完成 j 笔交易，当前正在持有第 j+1 笔的买入"。

---

## 状态转移

```
持有状态 dp[i][j][1]：
  前一天就已经持有                max(dp[i-1][j][1], 
  前一天未持有，今天买入（不增加交易次数，只是开启一笔新交易）    dp[i-1][j][0] - prices[i])

卖出状态 dp[i][j][0]：
  前一天就已经不持有              max(dp[i-1][j][0],
  前一天持有，今天卖出（完成第 j 笔交易，j 递增）   dp[i-1][j-1][1] + prices[i])
```

关键区别：
- **买入**不增加交易次数（`j` 不变），只是把"未持有"变成"持有"
- **卖出**完成一笔交易，交易次数 `j` 增加（从 `j-1` 变到 `j`）

---

## 代码实现

```python
class Solution:
    def maxProfit(self, k: int, prices: List[int]) -> int:
        if not prices or k == 0:
            return 0

        n = len(prices)

        # 当 k 足够大时退化到无限次交易
        if k >= n // 2:
            profit = 0
            for i in range(1, n):
                if prices[i] > prices[i-1]:
                    profit += prices[i] - prices[i-1]
            return profit

        # dp[i][j][0]：不持有，dp[i][j][1]：持有
        dp = [[[0, 0] for _ in range(k + 1)] for _ in range(n)]

        # 初始化第 0 天
        for j in range(k + 1):
            dp[0][j][1] = -prices[0]    # 第 0 天买入，成本 prices[0]

        # 动态规划
        for i in range(1, n):
            # 处理 0 次交易的情况：只能持有不能卖出
            dp[i][0][1] = max(dp[i-1][0][1], -prices[i])

            for j in range(1, k + 1):
                # 持有：今天买入或继续持有
                dp[i][j][1] = max(dp[i-1][j][1], dp[i-1][j][0] - prices[i])
                # 卖出：今天卖出或继续空仓
                dp[i][j][0] = max(dp[i-1][j][0], dp[i-1][j-1][1] + prices[i])

        return dp[n-1][k][0]
```

## 与两次交易（123）的对比

| 维度 | 123（两次交易） | 188（k 次交易） |
|------|---------------|----------------|
| 状态变量 | 4 个固定变量 | 三维 DP 表 |
| 交易次数 | 硬编码为 2 | 参数 k 控制 |
| 代码量 | 短，O(1) 空间 | 稍长，O(n×k) 空间 |
| 可扩展性 | 不易扩展 | k 任意 |
| k 很大时 | N/A | 退化到无限次交易 |

两种写法的状态转移本质相同：

```
// 123 中的 buy2 → sell1 - price
// 188 中的 dp[i][j][1] → dp[i-1][j][0] - price

// 123 中的 sell2 → buy2 + price
// 188 中的 dp[i][j][0] → dp[i-1][j-1][1] + price
```

188 是 123 的通用化版本。

## 复杂度优化

上面的 DP 是 O(n×k) 空间。可以压缩到 O(k)：

```python
class Solution:
    def maxProfit(self, k: int, prices: List[int]) -> int:
        if not prices or k == 0:
            return 0
        n = len(prices)
        if k >= n // 2:
            return sum(max(0, prices[i+1] - prices[i]) for i in range(n-1))

        # 压缩到 O(k) 空间
        hold = [float('-inf')] * (k + 1)    # hold[j]：已完成 j 笔交易，当前持有股票
        sold = [0] * (k + 1)                # sold[j]：已完成 j 笔交易，当前空仓

        for price in prices:
            for j in range(1, k + 1):
                hold[j] = max(hold[j], sold[j-1] - price)
                sold[j] = max(sold[j], hold[j] + price)

        return sold[k]
```

注意内层循环中 `hold[j]` 更新后，下一行 `sold[j]` 使用的 `hold[j]` 已经是当天的值了——这恰好对应"同一天先卖再买"的逻辑，和 123 题中 `buy1 → sell1 → buy2 → sell2` 的顺序一致性同理。

---

## 踩坑点

- ❌ **k 很大时不处理会导致超时/爆内存**：当 `k >= n//2` 时，相当于不限交易次数，直接用贪心（相邻涨价就交易）即可，O(n)。
- ❌ **买入/卖出和交易次数的关系搞混**：买入不增加交易次数，只有卖出才增加。`hold[j]` 中的 j 表示"已经完成了 j 笔交易"，当前持有的是第 j+1 笔的买入。
- ❌ **初始化时所有 j 的 `hold[j]` 都设为 `-price[0]`**：因为第 0 天可以认为已经完成了 j 笔交易（都是 0 利润），然后以 `-prices[0]` 持有下一笔。
