# 买卖股票的最佳时机 IV（188. Best Time to Buy and Sell Stock IV）

## 题意

给定一个数组 `prices` 和一个整数 `k`，其中第 `i` 个元素是某只股票在第 `i` 天的价格。设计一个算法来计算你所能获取的最大利润。你**最多可以完成 k 笔交易**（一次买入 + 一次卖出 = 一笔交易）。

注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

---

## 思路：从 2 次交易推广到 k 次

在 [ch09 买卖股票的最佳时机 III](./blog.html?post=leetcode/ch09.md) 中，我们用一个有限状态机处理了最多两次交易的情况——四个状态：`buy1/sell1/buy2/sell2`。

对于 k 次交易，不可能写 k 对 buy/sell 变量。需要用通用的 DP。

---

## DP 定义

状态定义的关键点：**`dp[i][j]` 表示第 i 天结束时的状态**。

```
dp[i][j][0] = 第 i 天结束时，已经完成了 j 笔交易，当前持有股票的最大利润
dp[i][j][1] = 第 i 天结束时，已经完成了 j 笔交易，当前不持有股票的最大利润
```

| 维度 | 含义 |
|------|------|
| `i` | 第几天（0-indexed） |
| `j` | 已经完成的交易次数（0 ~ k） |
| `0` | 当前**持有**股票（买入后还没卖） |
| `1` | 当前**不持有**股票（空仓） |

注意：买入不增加交易次数 `j`，只有卖出才增加。所以 `dp[i][j][0]` 中持有的是第 `j` 笔交易之后的下一笔买入，实际已完成 `j` 笔。

---

## 状态转移

```
dp[i][j][0] = 第 i 天结束时持有股票：
  max(dp[i-1][j][0],                  // 前一天就持有，今天什么也没做
      dp[i-1][j][1] - prices[i])     // 前一天不持有，今天买入（j 不变）

dp[i][j][1] = 第 i 天结束时不持有股票：
  max(dp[i-1][j][1],                  // 前一天就不持有，今天什么也没做
      dp[i-1][j-1][0] + prices[i])   // 前一天持有，今天卖出（完成第 j 笔交易）
```

关键规则：
- **买入**：`j` 不变，只是从不持有变成持有
- **卖出**：`j` 增加，从持有变成不持有，完成一笔交易

---

## 代码

```cpp
class Solution {
public:
    int maxProfit(int k, vector<int>& prices) {
        const int n = prices.size();
        if (n == 0) return 0;

        // 当 k 足够大时退化到无限次交易
        if (k >= n / 2) {
            int result = 0;
            for (int i = 1; i < n; i++) {
                if (prices[i] > prices[i - 1]) {
                    result += prices[i] - prices[i - 1];
                }
            }
            return result;
        }

        // dp[i][j][0]：持有，dp[i][j][1]：空仓
        vector<vector<vector<int>>> dp(n,
            vector<vector<int>>(k + 1,
                vector<int>(2, 0)));

        // 第 0 天初始化：在任何 j 下买入股票，成本 prices[0]
        for (int j = 0; j <= k; j++) {
            dp[0][j][0] = -prices[0];   // 买入，持有
            // dp[0][j][1] = 0（默认），第 0 天不持有 = 没交易
        }

        for (int i = 1; i < n; i++) {
            // 0 次交易的情况：只能买入不能卖出
            dp[i][0][0] = max(dp[i-1][0][0], -prices[i]);

            for (int j = 1; j <= k; j++) {
                // 持有：前一天持有，或前一天空仓且今天买入
                dp[i][j][0] = max(dp[i-1][j][0],
                                  dp[i-1][j][1] - prices[i]);
                // 空仓：前一天空仓，或前一天持有且今天卖出
                dp[i][j][1] = max(dp[i-1][j][1],
                                  dp[i-1][j-1][0] + prices[i]);
            }
        }

        return dp[n-1][k][1];   // 完成 k 笔交易后空仓
    }
};
```

## 与两次交易（123）的对比

两次交易那题中，我们用 4 个变量 `buy1/sell1/buy2/sell2` 分别表示第 1/2 次持有的状态。它们和三维 DP 的对应关系：

```
buy1  = dp[i][0][0]     // 已完成 0 笔，当前持有
sell1 = dp[i][1][1]     // 已完成 1 笔，当前空仓
buy2  = dp[i][1][0]     // 已完成 1 笔，当前持有
sell2 = dp[i][2][1]     // 已完成 2 笔，当前空仓
```

状态转移完全一致：

```
// 123 的 buy1 → max(buy1, -price)
// 188 的 dp[i][0][0] → max(dp[i-1][0][0], -prices[i])

// 123 的 sell1 → max(sell1, buy1 + price)
// 188 的 dp[i][1][1] → max(dp[i-1][1][1], dp[i-1][0][0] + prices[i])
```

188 是 123 的通用化：`dp[i][j][0/1]` 把 4 个固定变量展开成了 j × 2 个。

| 维度 | 123（两次交易） | 188（k 次交易） |
|------|---------------|----------------|
| 状态 | 4 个变量，j 硬编码为 0/1 | 三维 DP，j = 0..k |
| 空间 | O(1) | O(n × k) |
| 可扩展 | k 固定为 2 | 任意 k |
| k 很大时 | N/A | 退化到无限次交易 |

## 复杂度优化

### 为什么能压缩？

观察 DP 的递推关系：

```
dp[i][j][0] 只依赖 dp[i-1][j][0] 和 dp[i-1][j][1]
dp[i][j][1] 只依赖 dp[i-1][j][1] 和 dp[i-1][j-1][0]
```

所有 `dp[i]` 的计算都只用到 `dp[i-1]`（前一天），**跟 i-2、i-3 等更早的天数无关**。所以我们不需要保留整个 n×k 的表格，只需要两行（或滚动覆盖）即可。

具体做法：去掉 `i` 维度，用两个一维数组 `hold[j]`（持有）和 `sold[j]`（空仓）滚动更新。第 i 天的结果直接覆盖第 i-1 天的结果，不会影响后续计算。

```
第 i-1 天 → 第 i 天 → 第 i+1 天
  hold[j]    hold[j]（覆盖）  hold[j]（覆盖）
  sold[j]    sold[j]（覆盖）  sold[j]（覆盖）
```

这样空间从 O(n×k) 降到了 O(k)。

### 压缩后的代码

```cpp
class Solution {
public:
    int maxProfit(int k, vector<int>& prices) {
        const int n = prices.size();
        if (n == 0) return 0;
        if (k >= n / 2) {
            int result = 0;
            for (int i = 1; i < n; i++) {
                if (prices[i] > prices[i-1])
                    result += prices[i] - prices[i-1];
            }
            return result;
        }

        // hold[j]：已完成 j 笔交易，当前持有股票
        // sold[j]：已完成 j 笔交易，当前空仓
        vector<int> hold(k + 1, INT_MIN / 2);
        vector<int> sold(k + 1, 0);

        for (int price : prices) {
            hold[0] = max(hold[0], -price);
            for (int j = 1; j <= k; j++) {
                hold[j] = max(hold[j], sold[j-1] - price);
                sold[j] = max(sold[j], hold[j] + price);
            }
        }

        return sold[k];
    }
};
```

注意内层循环中 `hold[j]` 更新后，下一行 `sold[j]` 使用的 `hold[j]` 已经是当天的值了——这恰好对应"同一天先卖再买"的逻辑。

---

## 状态定义的补充说明

这里的 `dp[i][j][0/1]` 定义的是**第 i 天结束时**的状态。这和"当天中的状态"有区别：

```
当天开始时 → 当天操作 → 当天结束时
  (i-1 天结束时)     (i 天结束时)

dp[i][j][0] = max(dp[i-1][j][0],       // 什么都不做
                  dp[i-1][j][1] - p)   // 今天买入
```

如果我们定义"当天中的状态"（允许当天先卖再买），那买入和卖出的顺序会影响结果。但"当天结束"的定义已经把操作压缩到一天结束时的结果，天然包含了"当天先卖再买"的可能——因为 `dp[i][j][1]` 先算出（当天可能卖出），然后 `dp[i][j][0]` 基于它再买入。

---

## 踩坑点

- ❌ **k 很大时不处理会导致超时/爆内存**：`k >= n/2` 时相当于不限交易次数，用贪心 O(n) 解决。
- ❌ **`[0]` 和 `[1]` 的含义搞反**：`[0]` = 持有（买入后），`[1]` = 空仓（卖出后）。注意和 ch09 的命名可能不同。
- ❌ **买入/卖出和交易次数的关系**：买入不增加 `j`，卖出才增加。`dp[i][j][0]` 的 `j` 表示"已经完成 j 笔交易，当前持有的是第 j+1 笔的买入"。
