# 零钱兑换（322. Coin Change）

## 题意

给定不同面额的硬币 `coins` 和一个总金额 `amount`，求**凑成总金额所需的最少硬币个数**。如果无法凑出，返回 `-1`。

```
输入: coins = [1,2,5], amount = 11
输出: 3
解释: 11 = 5 + 5 + 1
```

---

## 思路

### 暴力想

枚举所有硬币组合，求和等于 amount 的里面硬币数最少的。但组合数量爆炸，不可行。

### DP

`dp[i]` 表示凑成金额 `i` 所需的最少硬币数。

状态转移：对于每个金额 `i`，遍历所有硬币 `c`，如果 `i >= c`，则：

```
dp[i] = min(dp[i], dp[i - c] + 1)
```

`dp[i - c]` 是凑成 `i - c` 的最少硬币数，再加一个硬币 `c` 就是 `i`。

### 初始化

`dp[0] = 0`（凑成 0 元需要 0 个硬币）。

其他 `dp[i]` 初始化为一个不可能的大值。这里用 `amount + 1`，因为最多用 `amount` 个 1 元硬币（假设有 1 元），所以 `amount + 1` 一定表示"不可达"。

### 代码

```cpp
class Solution {
public:
    int coinChange(vector<int>& coins, int amount) {
        if (amount == 0) {
            return 0;
        }
        
        // 用 amount+1 作为不可达标记，避免 INT_MAX+1 溢出问题
        vector<int> dp(amount + 1, amount + 1);
        dp[0] = 0; // 基础情况：凑成金额0需要0个硬币
        
        for (int i = 1; i <= amount; i++) {
            for (const auto& c : coins) {
                if (i - c >= 0) {
                    dp[i] = min(dp[i], dp[i - c] + 1);
                }
            }
        }
        
        // 若仍为初始值，说明无法凑成该金额
        return dp[amount] > amount ? -1 : dp[amount];
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(amount × n) | amount 是目标金额，n 是硬币种类数 |
| 空间 | O(amount) | dp 数组 |

---

## 核心理解

这是**完全背包**问题的变形：每种硬币可以无限使用，求凑成指定金额的最少物品数。

外层循环是金额 `i`，内层循环是硬币 `c`。对于每个金额，尝试用每种硬币去更新。

`amount + 1` 作为不可达标记比 `INT_MAX` 更安全，因为 `INT_MAX + 1` 会溢出。

---

## 踩坑点

- ❌ 用 `INT_MAX` 初始化，然后 `dp[i - c] + 1` 溢出 — 用 `amount + 1` 更安全
- ❌ 忘记处理 `amount == 0` — 直接返回 0
- ❌ 返回 `dp[amount]` 而不判断是否可达 — 可能返回初始化的不可达值
- ❌ 硬币不能凑成时返回 0 而不是 -1
- ✅ 外层循环金额，内层循环硬币，顺序无所谓（完全背包）
