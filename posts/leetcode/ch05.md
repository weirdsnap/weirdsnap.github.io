# 用特殊操作处理字符串 I（3612. Process String with Special Operations I）

## 题意

给定一个字符串 `s`，由小写英文字母和特殊字符 `*`、`#`、`%` 组成。从左到右处理每个字符，构造结果字符串：

- 普通字母：追加到结果末尾
- `*`：删除结果末尾的最后一个字符（如果存在）
- `#`：将当前结果复制一份追加到自身后面
- `%`：反转当前结果

返回最终字符串。

## 解法一：直接模拟

### 思路

用一个可变字符串（`std::string`）维护当前结果，按顺序处理每个字符：

- 字母：`push_back`
- `*`：`pop_back`（非空时）
- `#`：`result += result`
- `%`：`reverse`

### 代码

```cpp
class Solution {
public:
    string processStr(string s) {
        string result;

        for (const auto& c : s) {
            if (c == '*') {
                if (!result.empty()) {
                    result.pop_back();
                }
                continue;
            }
            if (c == '#') {
                result += result;
                continue;
            }
            if (c == '%') {
                reverse(result.begin(), result.end());
                continue;
            }
            result.push_back(c);
        }

        return result;
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N × 2^N) 最坏 | 连续 `#` 会让字符串长度指数增长；每次 `#` 需要 O(L) 复制 |
| 空间 | O(2^N) | 结果字符串最长可达 `2^N` |

> 注：如果题目数据保证结果长度可控（例如测试用例中 `#` 数量很少），实际运行时间与上述上界相差很远。

## 解法二：用 `std::vector<char>` 或 `std::deque` 优化反转

### 思路

如果 `#` 和 `%` 操作频繁交替出现，`std::string` 的 `reverse` 和 `+=` 都会触发大量数据搬移。可以用 `std::deque` 或双向链表维护字符，配合一个"逻辑反转标志"来避免物理反转：

- 维护一个布尔标志 `reversed`
- `%` 操作时只需要翻转 `reversed` 标志
- 在 `reversed` 为 true 时，字母追加到头部，`*` 删除头部
- `#` 复制时需要实际复制整个容器

这种优化对连续 `#` 很多的情况没有改善（复制不可避免），但对 `%` 频繁出现的场景有效。

### 代码

```cpp
#include <deque>
#include <algorithm>

class Solution {
public:
    string processStr(string s) {
        std::deque<char> dq;
        bool reversed = false;

        for (char c : s) {
            if (c == '*') {
                if (!dq.empty()) {
                    if (reversed) dq.pop_front();
                    else dq.pop_back();
                }
            } else if (c == '#') {
                std::deque<char> copy = dq;
                if (reversed) {
                    // 逻辑反转时，追加反向副本
                    dq.insert(dq.end(), copy.rbegin(), copy.rend());
                } else {
                    dq.insert(dq.end(), copy.begin(), copy.end());
                }
            } else if (c == '%') {
                reversed = !reversed;
            } else {
                if (reversed) dq.push_front(c);
                else dq.push_back(c);
            }
        }

        string result;
        if (reversed) {
            result.assign(dq.rbegin(), dq.rend());
        } else {
            result.assign(dq.begin(), dq.end());
        }
        return result;
    }
};
```

### 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N × 2^N) 最坏 | `#` 的复制仍是瓶颈 |
| 空间 | O(2^N) | 结果长度上限不变 |

## 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 直接模拟 | 代码简洁，易于理解 | `%` 频繁时多次物理反转 | 普通测试用例 |
| 逻辑反转标志 | 避免 `%` 的 O(L) 反转 | 实现复杂，索引逻辑易错 | `%` 出现频繁 |

## 踩坑点 / 注意点

- ❌ `*` 在结果为空时仍调用 `pop_back` — 错误。必须先检查非空。
- ❌ `isalpha(c)` 直接用于 `char` — 危险。标准 `isalpha` 接受 `int`，若 `char` 为负值（如 UTF-8 扩展字符）会 UB。本题限定小写字母，所以直接比较字符即可。
- ❌ 忽略 `#` 的指数增长风险 — 错误。连续 `N` 个 `#` 会让长度达到 `2^N`，需要注意时间和空间限制。
- ❌ `result += result` 在自引用时出错 — 注意。`std::string::operator+=` 对同一对象自增是安全的，因为标准库处理了自引用。
