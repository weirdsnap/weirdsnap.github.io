# 俄罗斯套娃信封问题（354. Russian Doll Envelopes）

## 题意

给定一些信封，每个信封有宽度和高度 `(w, h)`。如果信封 A 的宽度和高度都比信封 B 大，那么 B 可以放进 A 里。问最多能套多少层（一个套一个）。

```
输入: envelopes = [[5,4],[6,4],[6,7],[2,3]]
输出: 3
解释: 最多信封的个数为 3, 组合为: [2,3] => [5,4] => [6,7]
```

---

## 思路

### 从二维降到一维

直接二维 LIS 太复杂。先排序：

- **宽度升序**：这样排完序后，后面的信封宽度一定不小于前面的
- **宽度相同则高度降序**：防止宽度相同的信封被误认为可以套（比如 `[3,3]` 和 `[3,4]`，宽度相同不能套）

排完序后，问题变成：**在高度序列上求最长严格递增子序列**。

因为宽度已经按升序排好了，如果后面的高度也比前面大，那就可以套。

### 为什么宽度相同要高度降序

如果宽度相同高度升序：

```
[3, 3], [3, 4], [3, 5]
```

高度序列是 `3, 4, 5`，LIS 会算出 3，但这三个信封宽度相同，根本不能互相套！

如果高度降序：

```
[3, 5], [3, 4], [3, 3]
```

高度序列是 `5, 4, 3`，LIS 算出来是 1（严格递增只能选一个），正确。

### 在高度上求 LIS

用二分优化的 LIS，O(N log N)：

```cpp
vector<int> tail;  // tail[k] 表示长度为 k+1 的递增子序列的最小末尾高度

for (auto& e : envelopes) {
    int h = e[1];
    auto it = lower_bound(tail.begin(), tail.end(), h);
    if (it == tail.end()) {
        tail.push_back(h);  // 可以延长
    } else {
        *it = h;  // 替换，让同样长度的序列末尾更小
    }
}
```

---

## 代码

```cpp
class Solution {
public:
    int maxEnvelopes(vector<vector<int>>& envelopes) {
        // 排序：宽度升序，宽度相同则高度降序
        // 目的：将二维问题降为一维，后续只需关注高度
        sort(envelopes.begin(), envelopes.end(), [](const vector<int>& a, const vector<int>& b) {
            return a[0] < b[0] || (a[0] == b[0] && a[1] > b[1]);
        });

        // LIS：在高度序列上求最长严格递增子序列
        // tail[k] 表示长度为 k+1 的递增子序列的最小末尾高度
        vector<int> tail;
        
        for (auto& e : envelopes) {
            int h = e[1];
            
            // 找第一个 >= h 的位置
            // 用 >= 而非 >，保证严格递增（相等时会被替换，不会连续选入）
            auto it = lower_bound(tail.begin(), tail.end(), h);
            
            if (it == tail.end()) {
                // h 比所有现有末尾都大，可以延长序列
                tail.push_back(h);
            } else {
                // 替换：同样长度下，用更小的末尾，留给后面更大的扩展空间
                *it = h;
            }
        }
        
        // tail 的长度即为 LIS 长度，也就是最多能套的信封数
        return tail.size();
    }
};
```

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(N log N) | 排序 O(N log N) + LIS O(N log N) |
| 空间 | O(N) | tail 数组 |

---

## 核心理解

这道题的关键是**降维**：

1. 先按宽度排序，把二维变成一维（只关注高度）
2. 宽度相同高度降序，避免误选同一宽度的多个信封
3. 然后在高度上跑 LIS

`lower_bound` 找 `>= h` 的位置，保证严格递增。如果找 `> h`（即 `upper_bound`），就变成非严格递增，宽度相同高度也相同的信封会被算进去。

---

## 踩坑点

- ❌ 宽度相同高度升序排序 — 导致同一宽度的信封被误认为可以套
- ❌ 用 `upper_bound` 代替 `lower_bound` — 变成非严格递增，相等高度会被连续选入
- ❌ 忘记排序直接求 LIS — 二维问题没法直接做
- ✅ 排序比较器要写对：`a[0] < b[0] || (a[0] == b[0] && a[1] > b[1])`
