# 买卖股票的最佳时机 III（123. Best Time to Buy and Sell Stock III）

> **标签**：状态压缩 DP、有限状态机、买卖股票系列

## 题意

给定一个数组 `prices`，其中第 `i` 个元素是某只股票在第 `i` 天的价格。设计一个算法来计算你所能获取的最大利润。你**最多可以完成两笔交易**（一次买入 + 一次卖出 = 一笔交易）。

注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

---

## 思路一：左右分割（O(n) 空间）

将整个交易区间分成两部分：`[0, i]` 和 `[i, n-1]`，分别计算在 i 天之前完成一笔交易的最大利润，和在 i 天之后完成一笔交易的最大利润，然后加起来就是两笔交易的总利润。

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        n = len(prices)
        
        # left[i]：在第 i 天之前完成一笔交易的最大利润
        left = []
        min_p = 100001
        for price in prices:
            min_p = min(min_p, price)
            left.append(max(price - min_p, left[-1] if left else 0))
        
        # right[i]：在第 i 天之后完成一笔交易的最大利润
        right = []
        max_p = 0
        for price in prices[::-1]:
            max_p = max(max_p, price)
            right.append(max(max_p - price, right[-1] if right else 0))
        right = right[::-1]
        
        # 枚举分割点，取最大和
        result = 0
        for i in range(n):
            result = max(left[i] + right[i], result)
        return result
```

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n) |
| 空间复杂度 | O(n) |

## 思路二：状态压缩（O(1) 空间）

用**有限状态机**的视角，将两笔交易分解为四个状态，顺序更新：

| 状态 | 含义 | 初始值 |
|------|------|--------|
| `buy1` | 第一次买入后的最大收益 | `-prices[0]`（持有一股） |
| `sell1` | 第一次卖出后的最大收益 | `0`（空仓） |
| `buy2` | 第二次买入后的最大收益 | `-prices[0]`（再持有一股） |
| `sell2` | 第二次卖出后的最大收益 | `0`（最终空仓） |

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        buy1 = -prices[0]
        sell1 = 0
        buy2 = -prices[0]
        sell2 = 0
        
        for i in range(1, len(prices)):
            price = prices[i]
            # 按顺序更新，确保依赖关系正确
            buy1 = max(buy1, -price)                     # 要么保持之前买入，要么今天买入
            sell1 = max(sell1, buy1 + price)              # 要么保持之前卖出，要么今天卖出第一次
            buy2 = max(buy2, sell1 - price)               # 要么保持之前第二次买入，要么用第一次收益今天买入
            sell2 = max(sell2, buy2 + price)              # 要么保持之前第二次卖出，要么今天卖出第二次
            
        return sell2
```

### C++ 写法

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        // 第一次买入、第一次卖出、第二次买入、第二次卖出
        int buy_1 = -prices[0];
        int sell_1 = 0;
        int buy_2 = -prices[0];
        int sell_2 = 0;

        for (int i = 1; i < prices.size(); i++) {
            buy_1 = max(buy_1, -prices[i]);
            sell_1 = max(sell_1, prices[i] + buy_1);
            buy_2 = max(buy_2, sell_1 - prices[i]);
            sell_2 = max(sell_2, buy_2 + prices[i]);
        }

        return sell_2;
    }
};
```

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n) |
| 空间复杂度 | O(1) |

---

## 状态转移的理解

四个变量的更新顺序非常重要——必须按 `buy1 → sell1 → buy2 → sell2` 依次更新，不能用之前的状态覆盖当前天的决策。

```python
# 错误顺序：交叉依赖会出错
buy1 = max(buy1, -price)
buy2 = max(buy2, sell1 - price)  # 这里 sell1 可能是当天卖出后的值，逻辑混乱
sell1 = max(sell1, buy1 + price)
sell2 = max(sell2, buy2 + price)
```

正确顺序下，第 i 天的 `sell1` 使用第 i 天的 `buy1`（如果当天卖出），`buy2` 使用当天卖出后的 `sell1`，`sell2` 使用当天第二次买入后的 `buy2`——每个状态都基于**同一天可能发生的最新状态**。

---

## 两版对比

| 维度 | 左右分割法 | 状态压缩法 |
|------|-----------|-----------|
| 空间 | O(n) | O(1) |
| 思路 | 枚举分割点，两段独立求最大利润 | 有限状态机，状态递推 |
| 理解门槛 | 低，容易想到 | 高，需要理解状态转移 |
| 扩展性 | 不易推广到 k 次交易 | 容易推广到任意 k 次交易（开 k 个状态） |

---

## 延伸：推广到 k 次交易（188. Best Time to Buy and Sell Stock IV）

状态压缩法可以自然地扩展到**最多 k 笔交易**。当 `k` 很大时（`k >= n // 2`），相当于无限次交易，退化成 122 题的贪心解法。

### Python

```python
class Solution:
    def maxProfit(self, k: int, prices: List[int]) -> int:
        n = len(prices)
        if k >= n // 2:
            return sum(max(0, prices[i+1] - prices[i]) for i in range(n-1))
        
        buy = [float('-inf')] * (k + 1)
        sell = [0] * (k + 1)
        
        for price in prices:
            for j in range(1, k + 1):
                buy[j] = max(buy[j], sell[j-1] - price)
                sell[j] = max(sell[j], buy[j] + price)
        
        return sell[k]
```

### C++

```cpp
class Solution {
public:
    int maxProfit(int k, vector<int>& prices) {
        int n = prices.size();
        // 当 k 很大时，相当于无限次交易，退化成贪心
        if (k >= n / 2) {
            int result = 0;
            for (int i = 1; i < n; i++) {
                if (prices[i] > prices[i - 1]) {
                    result += prices[i] - prices[i - 1];
                }
            }
            return result;
        }

        vector<int> buy(k + 1);
        vector<int> sell(k + 1, 0);
        for (int j = 0; j <= k; j++) {
            buy[j] = -prices[0];
        }
        //  time     0     ...  k
        //  buy    -p[0]       Null
        //  sell    Null      result

        for (int i = 1; i < n; i++) {
            buy[0] = max(buy[0], -prices[i]);
            sell[0] = max(sell[0], prices[i] + buy[0]);
            for (int j = 1; j <= k; j++) {
                buy[j] = max(buy[j], sell[j] - prices[i]);
                sell[j] = max(sell[j], buy[j - 1] + prices[i]);
            }
        }
        return sell[k];
    }
};
```

### 为什么 `k >= n // 2` 可以退化

每天最多买卖一次，`n` 天最多做 `n // 2` 笔交易（买一次卖一次算一笔）。如果 `k` 比这个上限还大，限制就形同虚设，直接变成 122 题的无限次交易。

### 状态转移的理解

- `buy[j]`：第 `j` 次买入后的最大收益（手中持股）
- `sell[j]`：第 `j` 次卖出后的最大收益（手中无股）

每天对每个 `j`：
1. `buy[j] = max(buy[j], sell[j] - prices[i])` — 要么保持之前买入，要么用第 `j` 次卖出收益今天买入
2. `sell[j] = max(sell[j], buy[j-1] + prices[i])` — 要么保持之前卖出，要么用第 `j-1` 次买入后的持股今天卖出

注意 `buy[0]` 和 `sell[0]` 单独处理，表示第一次交易。

## 买卖股票系列总结

| 题号 | 限制 | 核心思路 |
|------|------|---------|
| 121 | 最多 1 次 | 一次遍历，维护最小买入价和最大利润 |
| 122 | 无限次 | 贪心，累加所有上涨段 |
| 123 | 最多 2 次 | 4 个状态：buy1/sell1/buy2/sell2 |
| 188 | 最多 k 次 | k 对 buy/sell 状态，k 大时退化到贪心 |

状态压缩法是这个系列最通用的框架，从 2 次扩展到 k 次只是数组长度变化。

---

## 踩坑点

- ❌ **更新顺序搞反**：四个状态必须 `buy1 → sell1 → buy2 → sell2`，否则同一天内的交易会被重复计算。
- ❌ **`sell1 - price` 误解**：`sell1` 是第一次卖出后手中的现金，再减去 `price` 相当于用这笔现金第二次买入，理解成"第二次买入的成本来自第一次的利润"。
- ❌ **两次交易允许同一天**：可以在同一天先卖再买（卖完立刻买），代码中的更新顺序天然支持这一点。
- ❌ **一次交易和零次交易的边界**：如果只能获得一次最优交易，`sell2` 会继承 `sell1` 的结果，不会强制做两笔亏本的交易。
