# 网络延迟时间（743. Network Delay Time）

## 题意

给定一个有向带权图，节点编号 `1` 到 `n`，边集 `times[i] = [u, v, w]` 表示从 `u` 到 `v` 需要 `w` 时间。从节点 `k` 发出一个信号，问所有节点都收到信号需要多少时间。如果有节点收不到，返回 `-1`。

这就是个单源最短路模板题，标准解法是 Dijkstra。

---

## 我的第一版写法：基于边

一开始思路跑歪了，没把状态放在节点上，而是把边塞进了堆里：

```cpp
class Solution {
public:
    int networkDelayTime(vector<vector<int>>& times, int n, int k) {
        map<int, map<int, int>> m;
        for (const auto& time : times) {
            m[time[0]][time[1]] = time[2];
        }

        vector<int> best(n + 1, -1);
        best[k] = 0;
        // time, u, v
        priority_queue<tuple<int, int, int>> q;
        for (const auto& [v, time]: m[k]) {
            q.push({-time, k, v});
        }

        while (!q.empty()) {
            auto [time, u, v] = q.top();
            q.pop();
            if (best[v] != -1) {
                if (best[u] - time > best[v]) {
                    continue;
                }
                best[v] = min(best[v], best[u] - time);
            } else {
                best[v] = best[u] - time;
            }
            for (const auto& [next_v, next_time]: m[v]) {
                if(best[next_v] == -1 || best[v] + next_time < best[next_v])
                    q.push({-next_time, v, next_v});
            }

        }

        int min_time = -1;
        for (int i = 1; i < n + 1; i++) {
            if (best[i] == -1) {
                return -1;
            }
            min_time = max(min_time, best[i]);
        }
        return min_time;
    }
};
```

能过，但是堆里塞的是边，导致很多过期边被反复 pop 出来，性能差不少。而且代码写得也绕，自己看着都烦。

## 正确写法：基于点

Dijkstra 的状态天然就是节点：`dist[u]` 表示从起点 `k` 到 `u` 的当前最短路。堆里应该放 `(距离, 节点)`，每次取距离最小的节点出来松弛邻居。

```cpp
class Solution {
public:
    int networkDelayTime(vector<vector<int>>& times, int n, int k) {
        vector<vector<pair<int, int>>> adj(n + 1);
        for (const auto& t : times) {
            adj[t[0]].push_back({t[1], t[2]});
        }

        int dist[101];
        memset(dist, 0x3f, sizeof(dist));
        dist[k] = 0;

        priority_queue<pair<int, int>, vector<pair<int, int>>, greater<>> pq;
        pq.push({0, k});

        while (!pq.empty()) {
            auto [d, u] = pq.top();
            pq.pop();

            if (d > dist[u]) continue;

            for (auto& [v, w] : adj[u]) {
                int newDist = dist[u] + w;
                if (newDist < dist[v]) {
                    dist[v] = newDist;
                    pq.push({newDist, v});
                }
            }
        }

        int ans = 0;
        for (int i = 1; i <= n; i++) {
            if (dist[i] == 0x3f3f3f3f) return -1;
            ans = max(ans, dist[i]);
        }
        return ans;
    }
};
```

简洁多了。

## 另一种实现：显式 `greater<pair<int, int>>`

如果不喜欢 `greater<>` 这种依赖模板参数推导的写法，也可以把类型写全。这里 `dist` 用 `-1` 表示未访问，和前面 `0x3f` 的写法等价，看个人习惯。

```cpp
class Solution {
public:
    int networkDelayTime(vector<vector<int>>& times, int n, int k) {
        // times 转邻接表（vector 存储）
        vector<vector<pair<int, int>>> g(n + 1); // 预留 n+1 个槽位，节点编号 1 到 n
        for (const auto& time : times) {
            g[time[0]].emplace_back(time[1], time[2]);
        }

        // 建立优先队列（最小堆）
        priority_queue<
            pair<int, int>,                    // 元素类型
            vector<pair<int, int>>,            // 底层容器
            greater<pair<int, int>>            // 比较器
        > pq;

        vector<int> dist(n + 1, -1);
        pq.push({0, k});

        // 出堆扩展
        while (!pq.empty()) {
            auto [d, u] = pq.top();
            pq.pop();

            // 已经处理过更优距离，跳过
            if (dist[u] != -1 && dist[u] < d) {
                continue;
            }

            dist[u] = d;
            for (const auto& [v, w] : g[u]) {
                if (dist[v] == -1 || dist[v] > dist[u] + w) {
                    pq.push({dist[u] + w, v});
                    dist[v] = dist[u] + w;
                }
            }
        }

        int result = 0;
        for (int i = 1; i < n + 1; i++) {
            int d = dist[i];
            if (d == -1) {
                return -1;
            }
            result = max(result, d);
        }

        return result;
    }
};
```

这种写法和前面的区别主要是：

- `greater<pair<int, int>>` 把类型显式写出来，对 `priority_queue` 的三个模板参数更清楚
- `dist` 初始化为 `-1`，用 `-1` 表示还没找到最短路
- 入堆前先更新 `dist[v]`，和前面"先 push 再 lazy 跳过"略有不同，但效果一样

---

## 关键比较条件含义

这几道题里出现了很多看起来很像但含义略有不同的 `if` 判断，整理一下：

### 1. 跳过过期堆顶：`if (d > dist[u]) continue`

小根堆会优先弹出距离最小的节点，但同一个节点可能被多次 push。比如先以距离 `10` 把 `u` 入堆，后来又找到一条距离 `6` 到 `u` 的路径再次入堆。`6` 会先被弹出并处理，`dist[u]` 变成 `6`；等 `10` 再被弹出时，`d > dist[u]` 成立，直接跳过。

含义：**堆顶这个距离已经不是当前已知最短路了，不用处理**。

### 2. 跳过已处理节点：`if (dist[u] != -1 && dist[u] < d)`

`-1` 版本和 `0x3f` 版本本质一样。`dist[u] != -1` 表示已经找到过到 `u` 的最短路，`dist[u] < d` 表示当前堆顶距离更大。两者同时满足就跳过。

含义：**这个节点已经处理过了，而且当时处理的距离更优**。

### 3. 基于边版本的剪枝：`if (best[u] - time > best[v])`

这里 `time` 是负数（因为用大根堆模拟小根堆），`best[u] - time` 实际是 `best[u] + |time|`，也就是通过这条边到达 `v` 的距离。如果它大于已经记录的 `best[v]`，说明这条边不能更优。

含义：**走这条边到 v 不会比已知路径更短，跳过**。

### 4. 松弛条件：`if (newDist < dist[v])`

或者展开写：

```cpp
if (dist[v] == -1 || dist[v] > dist[u] + w)
```

含义：**从 u 经权重 w 的边到 v，比当前已知的到 v 的最短路更短，更新 dist[v]**。

这是 Dijkstra 的核心操作，因为边权非负，每次从堆顶取出的节点都已经确定了最短路，所以可以放心用它来松弛邻居。

### 5. 不可达判断：`if (dist[i] == 0x3f3f3f3f)` 或 `if (d == -1)`

遍历完所有节点后，如果还有节点没被松弛过，说明从起点 `k` 到不了这个节点，返回 `-1`。

含义：**这个节点从未被访问过，图不连通**。

## 两种写法对比

| | 基于点 | 基于边 |
|---|---|---|
| 堆元素 | `(距离, 节点)` | `(距离, 起点, 终点)` |
| 过期数据 | 只有节点旧距离 | 同一终点的多条边都可能过期 |
| 剪枝 | `if (d > dist[u]) continue` 一行搞定 | 要判断 `best[v]` 是否存在、是否更优 |
| 代码量 | 少 | 多 |
| 性能 | 快 | 慢 |

其实核心就一句话：**Dijkstra 维护的是"到每个点的最短路"，不是"每条边的状态"**。把状态放在点上，代码和思路都会顺很多。

---

## 踩坑点

- ❌ 基于边写法会堆很多过期边，数据量大容易超时
- ❌ 用 `map<int, map<int, int>>` 存图比邻接表慢，且代码冗余
- ❌ 用大根堆存负数模拟小根堆可以，但不如直接用 `greater<>` 直观
- ✅ `dist` 数组初始化为 `0x3f`，最后判断 `0x3f3f3f3f` 表示不可达
- ✅ `if (d > dist[u]) continue` 是关键剪枝，去掉会多处理很多过期状态
