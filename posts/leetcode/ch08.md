# 最长公共前缀（14. Longest Common Prefix）

## 题意

编写一个函数来查找字符串数组中的最长公共前缀。

如果不存在公共前缀，返回空字符串 `""`。

## 第一版：能跑，但不够稳

### 思路

以第一个字符串为基准，逐列比较后面每个字符串的对应字符。如果所有字符串在第 `l` 位都相同，就把这个字符加入结果；一旦发现不同，就停止。

### 代码

```cpp
class Solution {
public:
    string longestCommonPrefix(vector<string>& strs) {
      int len;
      string result = "";
      len = strs.size();
      if (len == 0) {
        return result;
      } else if (len == 1) {
        return strs[0];
      } else {
        bool p = true;
        int l = 0;
        while (p) {
          char temp = strs[0][l];
          if (temp != '\0')
          for (int i = 1; i < len; i++) {
            if (strs[i][l] != temp) {
              p = false;
            }
          }
          if (p) {
            result.append(1,temp);
          }
          l++;
        }
        return result;
      }
  }
};
```

### 问题

- **`strs[0][l]` 可能越界**：如果第一个字符串不是最短的，访问到 `size()` 之后的位置是未定义行为。代码靠 `temp != '\0'` 兜底，但 `std::string` 的 `operator[]` 并不保证以 `\0` 结尾（虽然主流实现通常有），依赖这个很脆弱。
- **`p` 的语义不清**：布尔变量命名成 `p`，别人读起来费劲。
- **空数组和单字符串的特殊处理**：虽然逻辑上没错，但和主循环分得太开，可以更早统一处理。
- **发现不匹配后没有立即退出**：当某个字符串已经 shorter 时，`strs[i][l]` 越界访问没有被提前拦截。

## 第二版：显式边界检查，结构更清晰

### 思路

不再依赖字符串的 `\0`，而是显式用 `.size()` 判断每个字符串是否还有第 `i` 个字符。只要有一个字符串越界或字符不同，就停止。

### 代码

```cpp
class Solution {
public:
    string longestCommonPrefix(vector<string>& strs) {

        string result = "";

        int i = 0;
        while (true) {
            bool flag = true;
            if (strs[0].size() <= i) {
                break;
            }
            auto first = strs[0][i];
            for (const auto& str : strs) {
                if (i >= str.size() || first != str[i]) {
                    flag = false;
                    break;
                }
            }
            if (!flag) {
                break;
            }
            result.push_back(first);
            i++;
        }

        return result;   
    }
};
```

### 改进

- ✅ **显式长度检查**：用 `i >= str.size()` 判断是否越界，不再依赖 `\0`。
- ✅ **语义更清楚**：`flag` 表示"当前字符是否仍然匹配"，比 `p` 好读。
- ✅ **边界情况自然处理**：只要 `strs` 非空，循环内部统一处理第一个字符串先结束、某个字符串更短、字符不一致等情况。

### 还存在的问题

- 没有处理 `strs.empty()`，如果传入空数组，`strs[0]` 会直接崩溃。
- `flag` 变量其实可以省掉，发现不匹配时直接 `return result` 更干脆。
- `while (true)` 加多处 `break` 虽然可读，但可以改成更直接的循环条件。

## 第三版：最简洁的直接返回写法

### 思路

先处理空数组。之后按第一个字符串的长度作为上限逐列比较，一旦发现不匹配就立即返回当前累积的前缀，不必再用 `flag` 绕一圈。

### 代码

```cpp
class Solution {
public:
    string longestCommonPrefix(vector<string>& strs) {
        if (strs.empty()) return "";
        
        string result = "";
        int i = 0;
        
        while (i < strs[0].size()) {
            char current_char = strs[0][i];
            
            for (int j = 1; j < strs.size(); j++) {
                if (i >= strs[j].size() || strs[j][i] != current_char) {
                    return result;
                }
            }
            
            result += current_char;
            i++;
        }
        
        return result;
    }
};
```

### 改进

- ✅ **空数组保护**：开头 `if (strs.empty()) return ""` 直接兜底。
- ✅ **循环条件即边界**：`while (i < strs[0].size())` 天然保证不会越界访问基准字符串。
- ✅ **发现不匹配立即返回**：不再需要 `flag`，逻辑更直截了当。
- ✅ **变量名可读**：`current_char` 比 `temp`、`first` 都更明确。

## 复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(S) | S 为所有字符串字符总数之和，最坏情况下需要比较每个字符一次 |
| 空间 | O(1) | 只使用结果字符串和少量变量（不计返回结果本身） |

## 三版对比

| 版本 | 主要问题 | 亮点 |
|------|----------|------|
| 第一版 | 依赖 `\0`、可能越界、命名差 | 思路雏形 |
| 第二版 | 未处理空数组、`flag` 可省 | 显式长度检查、结构清晰 |
| 第三版 | 无明显问题 | 简洁、安全、直接返回 |

## 踩坑点

- ❌ **用 `\0` 判断字符串结束**：C 风格字符串可以，但 `std::string` 的 `operator[]` 访问 `size()` 之后是未定义行为。
- ❌ **忘记处理空数组**：`strs[0]` 在数组为空时会崩溃。
- ❌ **忽略某个字符串更短的情况**：比较时必须检查 `i >= str.size()`，不能只看第一个字符串。
- ❌ **索引变量命名随意**：`l`、`p`、`temp` 这种命名在算法题里虽然常见，但改写成 `i`、`current_char` 会更容易回顾。

## 延伸

- 面试中有时会要求不逐个字符比较，可以用**排序后比较首尾**的思路，时间复杂度相同但代码更短。
- 也可以把前缀看作节点，用 **Trie（前缀树）** 解决，适合大量查询的场景。
