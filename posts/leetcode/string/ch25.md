# 计算器（面试题 16.26. Calculator LCCI）

## 题意

给定一个包含正整数、`+`、`-`、`*`、`/` 和空格的字符串 `s`，计算表达式的值。整数除法只保留整数部分。

```
输入: "3+2*2"
输出: 7

输入: " 3/2 "
输出: 1

输入: " 3+5 / 2 "
输出: 5
```

---

## 思路

### 关键观察

`*` 和 `/` 的优先级高于 `+` 和 `-`。如果直接从左到右计算，`3+2*2` 会变成 `(3+2)*2=10`，但正确答案是 `3+(2*2)=7`。

### 核心技巧：栈保存中间结果

遇到 `+`/`-` 时，无法确定后面是否有更高优先级的运算，所以先把当前数字（带符号）**压栈**。

遇到 `*`/`/` 时，可以直接和栈顶元素运算（因为栈顶就是上一个数字），**结果替换栈顶**。

最后栈里只剩加减法，全部累加即可。

### 为什么最后只剩加法？

- `+`：`push(+num)`
- `-`：`push(-num)`
- `*`：`stack.back() *= num`
- `/`：`stack.back() /= num`

所有减法都转成了加负数，乘除当场结算。最后遍历栈求和，等价于从左到右做加法。

## 代码

```cpp
class Solution {
public:
    int calculate(string s) {
        vector<int> stack;
        int number = 0;
        char op = '+';
        
        for (int i = 0; i <= s.size(); i++) {
            // 末尾哨兵：强制结算最后一个数
            char c = (i < s.size()) ? s[i] : '+';
            
            if (c == ' ') {
                continue;
            }
            
            if (isdigit(c)) {
                number = number * 10 + (c - '0');
            } else {
                // 遇到运算符，结算前一个数字
                if (op == '+') {
                    stack.push_back(number);
                } else if (op == '-') {
                    stack.push_back(-number);
                } else if (op == '*') {
                    stack.back() *= number;
                } else if (op == '/') {
                    stack.back() /= number;
                }
                op = c;      // 更新运算符
                number = 0;  // 重置数字
            }
        }
        
        int result = 0;
        for (int num : stack) {
            result += num;
        }
        return result;
    }
};
```

---

## 关键细节

### 1. 末尾哨兵

循环条件 `i <= s.size()`，在 `i == s.size()` 时设置 `c = '+'`。这样最后一个数字会被强制结算，不需要循环结束后再处理一次。

```cpp
char c = (i < s.size()) ? s[i] : '+';  // 哨兵
```

### 2. 空格跳过

`if (c == ' ') continue;` 放在数字判断之前，确保空格不影响数字拼接。

### 3. 运算符滞后

`op` 记录的是**上一个**运算符，不是当前运算符。当前运算符要等到下一个数字读完才生效。

```
"3+5*2"
 ^ ^ ^
 3 5 2
   ↑
   op='+' 时结算 3
     ↑
     op='*' 时结算 +5
       ↑
       哨兵 '+' 结算 *2
```

### 4. 整数除法

C++ 的整数除法向零取整，和题目要求一致。`-3/2 == -1`，`3/2 == 1`。

|------|--------|------|
| 时间 | O(n) | 一次遍历 |
| 空间 | O(n) | 栈空间，最坏全是 `+`/`-` |

---

## 踩坑点 / 注意点

- ❌ **忘记处理末尾数字**。没有哨兵的话，循环结束后还要单独结算最后一个 `number`。
- ❌ `isdigit(c)` 在 `c` 为 `char` 时，如果 `char` 是有符号的且字符值大于 127，行为未定义。但本题输入只有 ASCII 数字和运算符，安全。
- ❌ 运算符和数字的对应关系搞混。记住：`op` 是**上一个**运算符，用于结算**当前已经读完的** `number`。
- ❌ `number` 重置位置放错。必须在结算后立即 `number = 0`，否则下一个数字会累加。
